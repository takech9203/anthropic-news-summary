Claude Managed Agents パブリックベータと ant CLI のローンチ

2026-04-08

概要

🤖

Managed Agents パブリックベータ

Claude を自律的なエージェントとして実行するためのフルマネージドハーネス。セキュアなサンドボックス環境、組み込みツール、SSE ストリーミングを備え、開発者がインフラを自前で構築することなくエージェントを運用できます。

🖥️

ant CLI のリリース

Claude API を操作するためのコマンドラインクライアント。リソースの操作、レスポンスのフィルタリング、YAML ファイルによる API リソースのバージョン管理、Claude Code とのネイティブ統合を提供します。

🔧

組み込みツール群

Bash、ファイル操作、Web 検索 / フェッチ、MCP サーバーを標準搭載。エージェントに必要なツールがすぐに利用可能です。

📡

SSE ストリーミング

Server-Sent Events によるリアルタイムのイベントストリーミングでエージェントの実行状況をリアルタイムに監視できます。

🔗

相互補完的な 2 つの機能

ant CLI で Managed Agents のリソースを YAML ファイルとして定義・管理し、バージョン管理システムで追跡。エージェント開発のワークフロー全体をカバーします。

Before / After

Before

  • 自前でエージェントループを構築
  • 手動でサンドボックス環境を用意
  • ツール実行・コンテキスト管理を自力で実装
  • SDK 経由でのみ API を操作
  • デバッグにコード記述が必要
  • API リソースのバージョン管理手段なし

After

  • Managed Agents がエージェント実行基盤を提供
  • クラウドコンテナで自動サンドボックス化
  • 組み込みツールとセッション管理を標準搭載
  • ant CLI でコマンドラインから直接操作
  • --debug フラグで即座に HTTP 検査
  • YAML ファイルでリソースをバージョン管理

主な機能

🤖 Managed Agents

🤖 Agent

エージェントの定義を保持するリソース。モデル、システムプロンプト、ツール設定などを構成します。

📦 Environment

エージェントが実行されるクラウドコンテナ環境。ファイルシステム、ネットワークアクセス、MCP サーバー接続などを定義します。

🔗 Session

Agent と Environment の組み合わせで作成されるステートフルな実行セッション。会話の状態が維持されます。

📡 Events

セッションに送信されるユーザーメッセージやシステムイベント。SSE ストリーミングでレスポンスを受信します。

🚀 パフォーマンス最適化

組み込みのプロンプトキャッシングとコンパクション機能で、長時間実行タスクを効率的に処理します。

🔬 リサーチプレビュー

outcomes、multiagent、memory などの先進機能を提供。別途アクセス申請が必要です。

🖥️ ant CLI

💻 マルチプラットフォーム

Homebrew (macOS)、curl (Linux/WSL)、go install に対応。お好みの環境でインストール可能です。

📊 7 種類の出力フォーマット

  • auto / json / jsonl
  • yaml / pretty / raw
  • explore (TUI)

🔍 レスポンスフィルタリング

--transform フラグで GJSON パスによるフィルタリング / 整形。必要なデータだけを抽出できます。

📁 ファイル参照とバージョン管理

@path 構文でファイル内容をインライン展開。YAML ファイルでエージェント、環境、デプロイメントを管理します。

🔌 Claude Code 統合

Claude Code が ant CLI をネイティブに理解。beta: プレフィックスでベータヘッダーを自動送信します。

🐛 デバッグ機能

--debug フラグで HTTP リクエスト / レスポンスを検査。シェル補完は bash、zsh、fish、PowerShell に対応します。

⚠️ ブランディングガイドライン: 「Claude Agent」は許可されていますが、「Claude Code Agent」は不許可です。

コードサンプル

Python Managed Agents: エージェントの作成と実行 
import anthropic

client = anthropic.Anthropic()

# 1. Agent を作成
agent = client.beta.managed_agents.agents.create(
    model="claude-opus-4-6-20260205",
    name="research-agent",
    instructions="You are a research assistant. Use web search and file operations to gather and organize information.",
    tools=[
        {"type": "bash"},
        {"type": "file_operations"},
        {"type": "web_search"},
        {"type": "web_fetch"},
    ],
    betas=["managed-agents-2026-04-01"],
)
print(f"Agent created: {agent.id}")

# 2. Environment を作成
environment = client.beta.managed_agents.environments.create(
    agent_id=agent.id,
    betas=["managed-agents-2026-04-01"],
)
print(f"Environment created: {environment.id}")

# 3. Session を開始
session = client.beta.managed_agents.sessions.create(
    agent_id=agent.id,
    environment_id=environment.id,
    betas=["managed-agents-2026-04-01"],
)
print(f"Session started: {session.id}")

# 4. Events を送信して SSE でレスポンスを受信
with client.beta.managed_agents.sessions.events.stream(
    session_id=session.id,
    agent_id=agent.id,
    messages=[{"role": "user", "content": "Search for the latest AI safety research papers and summarize the key findings."}],
    betas=["managed-agents-2026-04-01"],
) as stream:
    for event in stream:
        if hasattr(event, "type"):
            print(f"[{event.type}] ", end="")
        if hasattr(event, "text"):
            print(event.text, end="", flush=True)
Bash ant CLI: インストールと基本操作 
# macOS - Homebrew でインストール
brew install anthropic/tap/ant

# Linux/WSL - curl でインストール
curl -fsSL https://cli.anthropic.com/install.sh | sh

# Go でインストール
go install github.com/anthropics/ant-cli@latest

# 認証設定
export ANTHROPIC_API_KEY="sk-ant-..."

# シェル補完の有効化 (zsh の例)
ant completion zsh > "${fpath[1]}/_ant"

# モデル一覧を取得
ant models list

# メッセージ送信
ant messages create --model claude-opus-4-6-20260205 --max-tokens 1024 \
  --messages '[{"role": "user", "content": "Hello, Claude"}]'

# JSON 出力で特定フィールドを抽出
ant messages create --model claude-opus-4-6-20260205 --max-tokens 1024 \
  --messages '[{"role": "user", "content": "Hello"}]' \
  --output json --transform "content.0.text"

# ファイル内容をインライン展開して送信
ant messages create --model claude-opus-4-6-20260205 --max-tokens 4096 \
  --messages '[{"role": "user", "content": "Review this code: @src/main.py"}]'

# ベータリソース (Managed Agents) の操作
ant beta:managed-agents:agents list
ant beta:managed-agents:agents create --from agent.yaml

# デバッグモード (HTTP リクエスト/レスポンスを表示)
ant messages create --model claude-opus-4-6-20260205 --max-tokens 256 \
  --messages '[{"role": "user", "content": "Test"}]' --debug

  - type: web_fetch

ユースケース

🔬 自律型リサーチエージェント

Managed Agents の Web 検索、ファイル操作、Bash ツールを組み合わせて、情報収集から分析・レポート作成までを自動化。セッション管理により複数のステップにまたがるリサーチタスクを継続的に実行できます。

🔄 CI/CD パイプラインへの統合

ant CLI と YAML ファイルによるリソース定義で、エージェントの構成をコードとして管理。Git でバージョン管理し、CI/CD パイプラインから ant CLI でデプロイメントを自動化できます。

💻 開発ワークフローの効率化

ant CLI の explore モード (TUI) でレスポンスをインタラクティブに閲覧、--transform でフィルタリング、--debug で HTTP 通信を検査。Claude Code との統合により、開発中の API 操作をシームレスに行えます。

移行ガイド

1

ant CLI をインストール

まず ant CLI を導入し、API リソースの操作に慣れましょう。シェル補完を有効化して生産性を向上させます。

2

小規模なエージェントタスクで試行

小規模なエージェントタスクで Managed Agents を試行し、既存実装との性能・挙動を比較します。

3

本番ワークロードの移行計画を策定

検証結果を踏まえ、本番ワークロードの移行計画を策定。Messages API を直接使用する既存のステートレスなアプリケーションは変更不要です。
