MCP サーバー alwaysLoad オプション

MCP サーバー設定に alwaysLoad: true を指定すると、そのサーバーの全ツールが遅延読み込みをスキップし、セッション開始時から常に利用可能になります。頻繁に使用するツールへの即座のアクセスを確保できます。

🧹

claude plugin prune コマンド

🧠

画像処理時の数 GB 規模の RSS 増加、/usage コマンドの約 2GB メモリリーク、長時間実行ツールのメモリリークの 3 件が修正され、長時間セッションの安定性が大幅に改善されました。

🔄

MCP サーバーの信頼性向上

起動時の一時的なエラーに対する自動リトライ (最大 3 回)、Claude.ai コネクタの重複排除、認証エラーによるコネクタ消失の防止など、MCP 接続の安定性が包括的に強化されました。

⚠️ 重大メモリリーク修正 - 3 件

CRITICAL

🖼️ 画像処理時のメモリ無制限増加修正

多数の画像を処理するセッションで RSS が数 GB に膨れ上がるメモリ無制限増加問題が修正されました。画像データのバッファが適切に解放されず、長時間のコードレビューや画像を含むドキュメント処理で特に顕著でした。

CRITICAL

📊 /usage のメモリリーク修正

大量のトランスクリプト履歴を持つマシンで /usage コマンドが最大約 2GB のメモリをリークする問題が修正されました。履歴データの処理中にメモリが解放されない問題でした。

CRITICAL

⏳ 長時間実行ツールのメモリリーク修正

進捗イベントを正しく送出しない長時間実行ツール (例: 大規模な検索やビルド処理) で、内部バッファが蓄積し続ける問題が修正されました。

🔧 技術的背景

これらの修正により、長時間セッションでのメモリ使用量が大幅に安定しました。特に画像を多用するワークフローやトランスクリプト履歴の多い環境での安定性が向上しています。
