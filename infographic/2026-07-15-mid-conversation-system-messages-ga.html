Mid-conversation system messages が GA

2026-07-15

概要

🎉

GA リリース

ベータヘッダー不要に。anthropic-beta ヘッダーなしで即座に利用可能

🤖

対応モデル

Claude Fable 5、Claude Mythos 5、Claude Opus 4.8 で利用可能

🌐

全プラットフォーム対応

Claude API、Amazon Bedrock、Google Cloud Vertex AI の全プラットフォームで使用可能

💾

プロンプトキャッシュ統合

キャッシュ無効化なしでシステム指示を追加。コストとレイテンシを最適化

🔒

ZDR 対応

Zero Data Retention ポリシー対象。データの保持なしで安全に利用可能

Before / After

Before

  • システムプロンプトの変更でキャッシュ全体が無効化
  • コスト・レイテンシが増加
  • ユーザーメッセージに指示を埋め込み
  • オペレーターレベル権限なし

After

  • 会話途中にシステムメッセージ挿入
  • キャッシュ保持でコスト最適化
  • system ロールで正式に指示
  • オペレーターレベル権限あり

主な機能

⚡ プロンプトキャッシュ統合

キャッシュされたプレフィックスを保持したまま、会話の途中でシステム指示を追加可能

🔄 動的ポリシー変更

セッション中にアシスタントの振る舞いを切り替え。ペルソナやルールの即時変更に対応

🤖 エージェントループ制御

エージェントのループを中断せずにユーザー入力をリレー。処理の連続性を維持

📊 優先順位制御

後のシステムメッセージが優先。トップレベル system フィールドより上位で適用

🛡️ オペレーターレベル権限

ユーザーメッセージより高い権限で指示を付与。セキュリティとコントロールを強化

コードサンプル

Python Mid-conversation system message の使用例 
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-8",
    max_tokens=1024,
    cache_control={"type": "ephemeral"},
    system="You are a code review assistant. Be concise.",
    messages=[
        {"role": "user", "content": "Review process() in utils.py for performance issues."},
        {"role": "assistant", "content": "The list comprehension is fine for small inputs..."},
        {"role": "user", "content": "Now review the calling code that invokes process()."},
        {"role": "system", "content": "From now on, every suggestion must include explicit type annotations."},
    ],
)

ユースケース

🔧 セッション中のポリシー変更

会話の途中でアシスタントの振る舞いを切り替える。例: 型アノテーション必須、出力フォーマットの変更など

🤖 エージェントループでのユーザー入力リレー

エージェントの動作を中断せずにユーザーの追加指示を反映。処理の連続性を保ちながら方向性を修正

🔐 モード切り替えの永続的権限付与

特定の操作モードを有効化する指示をオペレーターレベルで付与。ユーザーによる上書きを防止

開発者への影響

  • 🗑️ ベータヘッダーの削除が可能。anthropic-beta ヘッダーからフラグを除去してコードを簡素化
  • 🤖 対応モデル: Claude Fable 5、Mythos 5、Opus 4.8 (Claude Sonnet 5 は非対応)
  • 💰 プロンプトキャッシュとの併用でコスト最適化。キャッシュ無効化を回避し、レイテンシとコストを削減
  • ⚠️ 信頼できないコンテンツには使用禁止。システムメッセージはオペレーターレベルの権限を持つため注意が必要
