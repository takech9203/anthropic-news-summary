キーポイント

🔌

プラグイン依存関係の強制管理

disable/enable 操作時にトランジティブな依存関係を自動検証・解決。依存するプラグインの動作不能を防止

💾

バックグラウンドセッションの設定永続化

/bg やデタッチ操作で --mcp-config、--fallback-model、--settings などの設定がフォーク時に保持されるように改善

🛡️

安全機構の強化

ストップフックの無限ループ防止、ワークツリーの安全なクリーンアップ、認証情報バリデーション改善

💻

クロスプラットフォーム改善

Windows PowerShell のデフォルト有効化、macOS TCC 権限継承修正、Windows Terminal での操作不具合修正

Before / After

Before

  • ❌ /bg フォーク時に MCP 設定が失われる
  • ❌ プラグイン無効化で依存先が壊れる
  • ❌ ストップフックが無限ループする
  • ❌ macOS で ~/Documents にアクセス不可
  • ❌ ワークツリー削除で rm -rf にフォールバック

After

  • ✅ 全設定がフォーク/リスポーン時に保持
  • ✅ 依存関係を自動検証しブロック
  • ✅ 8 回でターン終了する安全キャップ
  • ✅ TCC 権限が正しく継承される
  • ✅ git worktree remove のみ使用

新機能ハイライト

🔗 プラグイン依存関係

disable 時に依存チェーンを検証し、コピーペースト可能な一括無効化コマンドを提示。enable 時は推移的依存を自動有効化

💰 コンテキストコスト表示

/plugin マーケットプレイスにターンあたり・呼び出しあたりのトークン推定コストを表示

🌳 ワークツリーバイパス

worktree.bgIsolation: "none" 設定で EnterWorktree なしの直接編集が可能に

⚡ PowerShell 強化

-ExecutionPolicy Bypass 自動付与。Bedrock/Vertex/Foundry ユーザーでデフォルト有効化

🔄 設定永続化

/bg フォーク・デタッチ時に --mcp-config、--fallback-model、--settings が保持される

🎛️ agents フラグ拡張

claude agents に --add-dir、--settings、--mcp-config、--permission-mode など 8 フラグ追加

重要なバグ修正

🔑
認証ハング修正: .credentials.json の scopes が非配列値の場合に CLI がハングする問題を解消
♾️
ストップフック無限ループ修正: 8 回連続ブロックでターンを強制終了。CLAUDE_CODE_STOP_HOOK_BLOCK_CAP で調整可能
🍎
macOS Full Disk Access 修正: バックグラウンドジョブが ~/Documents、~/Desktop、~/Downloads にアクセスできない問題を解消
😴
macOS App Nap 対策: ホストスリープ後のワーカーストール検出の偽陽性ストームを防止
🪟
Windows Terminal 修正: 右クリックペースト、ストリーミング中のフリーズ、スクロール時のレンダリング問題を修正
🌐
バックグラウンドセッション修正: IDE ファイル参照のサイレントキャプチャ、permissions.defaultMode 無視、デーモンスポーン失敗を修正

新しい環境変数・設定

環境変数 / 設定 説明
CLAUDE_CODE_STOP_HOOK_BLOCK_CAP ストップフックの連続ブロック上限 (デフォルト: 8)
CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY 1 に設定で ExecutionPolicy バイパスを無効化
CLAUDE_CODE_USE_POWERSHELL_TOOL 0 に設定で PowerShell ツールを無効化
worktree.bgIsolation: "none" バックグラウンドセッションのワークツリー分離を無効化

コードサンプル

Bash プラグイン依存関係管理 
# プラグイン B がプラグイン A に依存している場合
claude plugin disable plugin-a
# エラー: "plugin-b" depends on "plugin-a"
# Hint: claude plugin disable plugin-b && claude plugin disable plugin-a

# 依存チェーンを含めて無効化
claude plugin disable plugin-b && claude plugin disable plugin-a

# プラグインを有効化 (推移的依存を自動有効化)
claude plugin enable plugin-b
# "plugin-a" was automatically enabled as a dependency of "plugin-b"
JSON ワークツリーバイパス設定 
// .claude/settings.json
{
  "worktree": {
    "bgIsolation": "none"
  }
}
Bash /bg での設定永続化 
# MCP 設定とフォールバックモデルを指定して起動
claude --mcp-config ./my-mcp.json --fallback-model sonnet-4-6

# /bg でバックグラウンドに移行 (設定が保持される)
/bg テスト実行を続けて

# バックグラウンドセッションは以下を保持:
# - --mcp-config ./my-mcp.json
# - --fallback-model sonnet-4-6
# - --settings, --add-dir, --plugin-dir
Bash claude agents のデフォルト設定 
# モデルとパーミッションモードを指定してエージェントダッシュボードを起動
claude agents \
  --model opus-4-7 \
  --effort high \
  --permission-mode plan \
  --mcp-config ./team-mcp.json \
  --add-dir /shared/libs

# ダッシュボードからディスパッチされるすべてのセッションに
# これらの設定がデフォルトとして適用される

ユースケース

🏢 チームでのエージェント運用

claude agents に --mcp-config と --permission-mode を指定してダッシュボードを起動。ディスパッチされる全セッションが統一設定で動作し、/bg でバックグラウンドに移行しても設定が維持される。チーム全体で一貫したエージェント動作を保証

🔧 プラグインエコシステムの安全な管理

複数のプラグインが連携するワークフローで、依存関係を持つプラグインを安全に管理。無効化時に依存チェーンが自動検証され、意図しないプラグインの動作不能を防止。マーケットプレイスでコンテキストコストを事前確認してインストール判断が可能に

🏗️ 大規模モノレポでの開発

ワークツリーの作成に時間がかかる大規模リポジトリで worktree.bgIsolation: "none" を設定し、バックグラウンドセッションが直接作業コピーを編集。サブモジュールやシンボリックリンクが多用される環境で特に有効

必要なアクション

⚡

Windows ユーザー

PowerShell ツールがデフォルト有効化。不要な場合は CLAUDE_CODE_USE_POWERSHELL_TOOL=0 を設定

🍎

macOS ユーザー

~/Documents、~/Desktop、~/Downloads へのアクセス問題が修正。速やかなアップデートを推奨

🔌

プラグイン開発者

依存関係が正しく宣言されているか確認。disable 時の挙動が変更されている

⏱️

ストップフック利用者

8 回連続ブロックでターン終了する新安全機構を認識。CLAUDE_CODE_STOP_HOOK_BLOCK_CAP で調整可能
