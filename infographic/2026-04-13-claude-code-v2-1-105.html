概要

プラグインバックグラウンドモニター

プラグインマニフェストの monitors キーで、セッション開始時に自動アームされるバックグラウンド監視機能を定義可能に

🔧

PreCompact フックで圧縮制御

フックが終了コード 2 または {"decision":"block"} を返すことで、会話コンテキストのコンパクションをブロック可能に

🌊

API ストリームのストール検出

5 分間データなしのストリームを自動中断し、ノンストリーミングでリトライ。無限ハングを防止

🔍

WebFetch コンテンツ最適化

<style> / <script> タグの除去で、CSS 過多ページでも本文テキストを優先取得

📂

EnterWorktree path パラメータ

既存ワークツリーへの直接切り替えが可能に。ワークツリー間の移動が効率化

✨ 新機能 4 件

📂 EnterWorktree path

既存のワークツリーに直接切り替えるための path パラメータを EnterWorktree ツールに追加。新規作成だけでなく既存への切り替えも可能に

🔧 PreCompact フック

コンパクション実行前にフックを呼び出し、終了コード 2 または JSON レスポンスでブロック可能。重要なコンテキストの保持を保証

📡 バックグラウンドモニター

プラグインマニフェストにトップレベル monitors キーを追加。セッション開始時に自動アームされ、継続的な監視タスクを実行

🔄 /proactive エイリアス

/proactive が /loop のエイリアスとして追加。定期実行コマンドをより直感的な名前で呼び出し可能に

Before / After

Before

  • API ストリームがデータ停止で無限ハング
  • 接続エラー時にサイレントスピナーが回り続ける
  • WebFetch が CSS/JS でバジェット枯渇
  • MCP 不正出力でセッション全体がハング
  • 429 エラーが生の JSON ダンプで表示

After

  • 5 分のストール検出で自動リトライ
  • リトライメッセージが即座に表示
  • style/script 除去で本文を優先取得
  • MCP フェイルファストで即座にエラー通知
  • クリーンなエラーメッセージを表示

📈 改善 9 件

🌊

API ストリームのストール検出

5 分間データ受信なしのストリームを自動中断。ノンストリーミングモードでリトライし、無限ハングを防止

🌐

ネットワークエラーメッセージ改善

接続エラー時にサイレントスピナーではなく、リトライメッセージを即座に表示

📄

ファイル書き込み表示改善

長い 1 行書き込み (圧縮 JSON 等) が UI でトランケーション。多数ページのスクロールが不要に

🩺

/doctor レイアウト改善

ステータスアイコン追加。f キーで報告された問題を Claude に修正させることが可能に

⚙️

/config ラベル・説明改善

設定画面のラベルと説明文がより明確に

📝

スキル説明文の上限引き上げ

リスティング文字数上限を 250 → 1,536 文字に拡大。トランケーション時には起動時警告を表示

🔍

WebFetch コンテンツ最適化

<style> / <script> タグ内容を除去。CSS 過多ページのコンテンツバジェット問題を解消

🧹

古いワークツリーのクリーンアップ改善

PR がスカッシュマージされたワークツリーを適切にクリーンアップ。無期限保持を防止

📋

MCP 大容量出力のトランケーション改善

フォーマット別レシピを提供: JSON は jq、テキストは計算された Read チャンクサイズ

🐛 バグ修正 24 件

🖥️ UI/UX 7

  • キューメッセージに添付された画像がドロップされる問題
  • 長い会話でプロンプト入力が 2 行目に折り返される際の画面空白
  • フルスクリーンモードでの先頭空白コピーの問題
  • アシスタントメッセージの先頭空白トリミングで ASCII アート崩壊
  • Python rich/loguru ロギングによる Bash 出力の文字化け
  • alt+enter / Ctrl+J の改行挿入が機能しない問題 (v2.1.100 リグレッション)
  • 短いターミナル高さで /help のタブバーとフッターが非表示

📂 ワークツリー・セッション 3

  • EnterWorktree/ExitWorktree の "Creating worktree" テキスト重複
  • フォーカスモードでキューメッセージが消失
  • /resume, --worktree, /branch 後に終了ヒントが非表示

🔌 MCP・プラグイン 5

  • stdio MCP サーバーの不正出力でセッションがハング → フェイルファスト化
  • ヘッドレスセッション初回ターンで MCP ツールが利用不可
  • マーケットプレイスプラグインの依存関係が自動インストールされない
  • マーケットプレイス自動更新でプラグインが壊れた状態に
  • keybindings.json の不正エントリがサイレントにロードされる

🔑 API・認証 4

  • 429 レートリミットエラーが生の JSON ダンプで表示
  • Bedrock 非 US リージョンで無効な us.* モデル ID が settings.json に保存
  • 不正テキストブロックを含むセッション再開時のクラッシュ
  • 1 プロジェクトの CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC が全プロジェクトに影響

📨 スケジュール・通知 2

  • ワンショットスケジュールタスクが繰り返し再実行
  • Team/Enterprise チャネル通知が最初のメッセージ以降ドロップ

🔧 その他 3

  • フィードバック調査ショートカットキーがプロンプト末尾で誤発火
  • SSH/mosh 環境の 16 色パレットが色あせ (Ghostty, Kitty, Alacritty, WezTerm 等)
  • Bash ツールが acceptEdits 権限モードを不適切に提案

コードサンプル

JSON PreCompact フックの設定 
{
  "hooks": {
    "PreCompact": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "bash /path/to/pre-compact-check.sh"
          }
        ]
      }
    ]
  }
}
Bash PreCompact チェックスクリプト 
#!/bin/bash
# pre-compact-check.sh
# コンパクション前に重要なコンテキストの有無をチェック

if [ -f ".no-compact" ]; then
  echo '{"decision":"block"}'
  exit 0
fi

# コンパクションを許可
exit 0
JSON プラグインバックグラウンドモニター定義 
{
  "name": "my-plugin",
  "monitors": {
    "my-monitor": {
      "description": "バックグラウンドで状態を監視"
    }
  }
}

開発者への影響

🔌 プラグイン開発者

バックグラウンドモニターサポートにより、セッション中の継続的な監視機能をプラグインに組み込み可能に。マーケットプレイスプラグインの依存関係インストールと自動更新の信頼性も向上

📂 ワークツリー活用ユーザー

EnterWorktree の path パラメータで既存ワークツリーへの切り替えが効率化。スカッシュマージ済み PR のワークツリーも適切にクリーンアップ

🔧 フック・自動化ユーザー

PreCompact フックでコンパクションの実行をプログラム的に制御。重要なコンテキストの保持を保証できるように

🔌 MCP サーバー利用者

stdio MCP サーバーの不正出力対応、ヘッドレスセッションのツール可用性、大容量出力のトランケーションレシピなど、MCP 接続の信頼性が大幅に向上

☁️ AWS Bedrock ユーザー

非 US リージョンでの /model ピッカーのモデル ID 保存問題が修正。429 レートリミットエラーもクリーンなメッセージで表示

👥 Team/Enterprise ユーザー

インバウンドチャネル通知が最初のメッセージ以降もドロップされなくなり、通知の信頼性が向上

🖥️ SSH/mosh ユーザー

Ghostty、Kitty、Alacritty、WezTerm、foot、rio、Contour でのカラーパレット修正。SSH/mosh 経由でも正しい色表示に
