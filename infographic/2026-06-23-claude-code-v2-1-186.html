概要: 3 つの注目ポイント

🔐

MCP CLI 認証

claude mcp login/logout コマンドにより、インタラクティブメニューを開かずに CLI から直接 MCP サーバーへ認証可能に。--no-browser で SSH 環境にも対応する。

💻

Bash 自動応答

! で実行したシェルコマンドの出力に対して Claude が自動的に応答。対話的なデバッグフローが大幅に効率化される。

🔒

バックグラウンドサブエージェント権限

従来の自動拒否方式からメインセッションへのプロンプト表示方式に変更。どのエージェントが権限を要求しているか表示され、Esc で個別拒否が可能。

Before / After: Bash コマンド実行

Before (v2.1.185 以前)

! コマンドの出力はコンテキストに追加されるのみ。開発者が自分で結果を読み取る必要があった。

! git status
=> (出力がコンテキストに追加されるだけ)

After (v2.1.186)

Claude が出力を解析し、自動的にフィードバックを返す。対話的なワークフローに。

! git status
=> Claude: "3 つのファイルが変更されています"

Before / After: バックグラウンド権限

Before

バックグラウンドサブエージェントの権限要求が自動拒否され、権限が必要なツールは使用不可。

SubAgent -> Permission? -> Auto-Deny

After

権限プロンプトがメインセッションに表示。ユーザーが明示的に Allow/Deny を判断可能。

SubAgent -> Permission? -> Main Session -> User Decision

改善 & 変更

💡 MCP タイポ候補

claude mcp get/remove でサーバー名のタイポ時に最も近い名前を提案。長いリストも適切に切り詰め。

🧠 メモリコンパクト化

MEMORY.md がサイズ制限に近づくとエージェントにコンパクト化を促すリマインダーを表示。

📝 スキル柔軟キー形式

フロントマターキーが kebab-case、snake_case、camelCase いずれも受付可能に。不正 YAML も安全にハンドリング。

🔄 リトライ上限変更

CLAUDE_CODE_MAX_RETRIES の上限が 15 に。無人セッションには RETRY_WATCHDOG を推奨。

コードサンプル

Bash MCP CLI 認証の使用例 
# MCP サーバーへの CLI 認証
claude mcp login my-github-server
# ブラウザが開き OAuth フローを完了

# SSH 環境での認証 (ブラウザなし)
claude mcp login my-server --no-browser
# 認証 URL が表示されるので、別端末でアクセスしてトークンを入力

# MCP サーバーからのログアウト
claude mcp logout my-github-server
Bash Bash 自動応答の動作例 
# ! を使うと Claude が結果を解析して応答
! git status
# => Claude: "3 つのファイルが変更されています。main.py の差分を確認しましょうか?"

# MCP サーバー名のタイポ時の候補表示
claude mcp get githbu-server
# => Error: Server "githbu-server" not found.
#    Did you mean "github-server"?
JSON settings.json 設定例 
{
  "respondToBashCommands": false,
  "teammateMode": "iterm2"
}

ユースケース

🌐 リモート開発環境での MCP 接続

SSH でリモートサーバーに接続し、claude mcp login --no-browser で OAuth 認証を完了。ブラウザが使えない環境でも MCP サーバーのツールをフル活用できる。

🐛 対話的デバッグの高速化

! git log --oneline -5 や ! npm test の結果に Claude が即座にフィードバック。「テストが 2 件失敗しています。エラーを見てみましょう」のような提案で次のアクションがすぐに分かる。

🤖 バックグラウンドエージェントの実用化

バックグラウンドで動作するエージェントがファイル書き込みや外部 API アクセスの権限を必要とする場合、メインセッションにプロンプトが表示される。開発者は作業を中断せずに権限を判断できる。
