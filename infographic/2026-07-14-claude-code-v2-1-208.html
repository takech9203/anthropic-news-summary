Claude Code v2.1.208 — スクリーンリーダーモード、メモリリーク修正、パフォーマンス大幅改善

2026-07-14

概要

♿

アクセシビリティ強化

スクリーンリーダーモード追加、vim カスタムキーマッピング対応により、多様なユーザーのワークフローをサポート。

🧠

メモリリーク包括的修正

MCP stderr 64MB 上限、LSP ドキュメント LRU 50 件、編集キャッシュ 16MB の制限を導入し、長時間セッションの安定性を大幅に向上。

⚡

パフォーマンス大幅改善

ツール呼び出し最大 7 倍高速化、トランスクリプト 79 倍削減により、レスポンスとリソース効率が飛躍的に向上。

新機能

📖 スクリーンリーダーモード

claude --ax-screen-reader または環境変数 CLAUDE_AX_SCREEN_READER=1、settings で "axScreenReader": true を設定してプレーンテキスト出力を有効化。

⌨️ vimInsertModeRemaps

jj → Escape などのカスタムキーマッピングを設定可能に。vim モードユーザーの操作効率を大幅に向上。

🔗 CLAUDE_CODE_PROCESS_WRAPPER

企業ランチャー経由のプロセス起動に対応。エンタープライズ環境でのセキュリティポリシー適用を容易に。

🖱️ マウスクリックサポート

フルスクリーンモードのマルチセレクトメニューでマウスクリック操作に対応。キーボード以外の操作手段を提供。

Before / After

Before

  • 長時間セッションでメモリが無制限に蓄積
  • MCP stderr: 上限なし
  • LSP ドキュメント: 無制限
  • ツール呼び出しが遅い
  • トランスクリプトが膨大

After

  • メモリ制限の導入で安定稼働
  • MCP stderr: 64MB/サーバー
  • LSP: LRU 50 件上限
  • ツールプールキャッシュで 7 倍高速化
  • 79 倍のサイズ削減

バグ修正ハイライト

🚨 Bedrock SSO 認証失敗修正 (v2.1.207 リグレッション)
🌐 HTTP/2 GOAWAY によるセッションクラッシュ修正
💬 バックグラウンドエージェント返信消失修正
✏️ Edit ツールのファイル変更対応修正
🔢 科学的記数法の環境変数パース修正

コードサンプル

Bash スクリーンリーダーモード有効化 
# コマンドラインフラグ
claude --ax-screen-reader

# 環境変数
export CLAUDE_AX_SCREEN_READER=1
claude
JSON settings.json 設定例 
{
  "axScreenReader": true,
  "vimMode": true,
  "vimInsertModeRemaps": {
    "jj": "Escape",
    "jk": "Escape"
  }
}
