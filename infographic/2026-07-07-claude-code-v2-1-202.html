新機能

NEW

Dynamic workflow size 設定

/config から「Dynamic workflow size」を設定可能に。small / medium / large の 3 段階でワークフロー実行時のエージェント数の目安を制御できる。タスクの規模やリソースに応じた柔軟な運用が可能

NEW

OpenTelemetry 属性の追加

ワークフローから生成されたエージェントのテレメトリに workflow.run_id および workflow.name 属性が追加。分散トレーシングツールでワークフロー単位のトレースを可視化・分析可能に

CHANGE

/review コマンドの動作変更

/review が高速な単一パスレビューに戻された。マルチエージェントレビューを利用する場合は /code-review を使用

バグ修正

Remote Control 関連

  • インタラクティブセッションへのコマンド送信が "Unknown command" エラーで失敗する問題
  • キャプションなしで送信された画像やファイルがサイレントにドロップされる問題
  • /remote-control セッションで誤ったパーミッションモードが表示される問題

バックグラウンドエージェント関連

  • /rename でのセッション名変更がジョブ再起動時にリセットされる問題
  • claude agents からチャットを開く際のクラッシュ/リスポーンループ
  • 多数の git worktree を持つリポジトリでの重大なパフォーマンス問題

その他の修正

  • インライン Ctrl+R 履歴検索でのクラッシュ
  • mTLS ハンドシェイクの一時的な失敗
  • SSH 上でサインイン URL がクリック不能になる問題
  • ワークフロースクリプト内の Unicode クォートエスケープ破損
  • 音声入力の無制限リトライ
  • プロキシ/ネットワーク断でのダウンロード即時失敗
  • スキル再呼び出し時の指示重複追加

改善

  • /workflows エージェントリスト表示: タイトル幅拡大、専用の時間カラム追加、モデル名の短縮
  • MCP エラーメッセージ: url があるが type がない場合のエラーが明確化され "type": "http" の追加を提案
  • /review コマンド: 高速な単一パスレビューに戻し、マルチエージェントは /code-review に分離

コード例

bash - ワークフロー管理と Remote Control
# Dynamic workflow size を設定
claude /config
# 設定画面で "Dynamic workflow size" を選択し、small / medium / large を指定

# ワークフローの実行状況を確認
claude /workflows

# Remote Control セッションを開始
claude /remote-control

# サインイン (SSH 環境でもクリック可能な URL が表示される)
claude auth login

# /review コマンドの使い分け
/review 123              # 高速な単一パスレビュー
/code-review high 123    # マルチエージェントレビュー
json - MCP サーバー設定 (url を使う場合は type: "http" が必要)
{
  "mcpServers": {
    "my-server": {
      "type": "http",
      "url": "https://my-mcp-server.example.com/mcp"
    }
  }
}
