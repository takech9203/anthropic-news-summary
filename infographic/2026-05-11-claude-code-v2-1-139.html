Agent View: 全セッションの一元管理

Research Preview として導入。claude agents で全 Claude Code セッション (実行中・入力待ち・完了) を一覧表示。複数の長時間タスクを並行管理するダッシュボード。

/goal コマンド: 完了条件による自律実行

完了条件を宣言すると、Claude が条件達成まで複数ターンにわたり自律的に作業を継続。インタラクティブ、-p フラグ、Remote Control の全モードで動作。リアルタイムでオーバーレイ表示。

🪝

Hook システムの強化

args: string[] によるシェルレス実行と continueOnBlock による柔軟なブロック制御。ガードレール付き自律実行の基盤が強化されました。

🔗

サブエージェントのトレーサビリティ

API リクエストに x-claude-code-agent-id / x-claude-code-parent-agent-id ヘッダーを追加。OTEL スパンにもエージェント ID 属性を含み、マルチエージェント構成のデバッグが容易に。

exec 形式 (args: string[]) でシェルを経由せずコマンド実行。パスのクォートが不要に。

🔄 continueOnBlock

PostToolUse フックに設定すると、拒否理由を Claude にフィードバックしてターンを継続。

ユースケース

🤖 マルチセッション並行管理

Agent View で複数のタスクを並行して走らせ、各セッションの進捗を一目で把握。入力待ちのセッションにはワンクリックで応答し、完了したセッションの結果をすぐに確認できます。

⚙️ CI/CD パイプラインでの自律タスク

/goal と -p フラグの組み合わせで、「全テストが通り、カバレッジ 80% 以上」のような完了条件を宣言的に指定。Claude が自律的にコードを修正し、条件達成まで作業を継続します。

🛡️ ガードレール付き自律実行

Hook の continueOnBlock を活用して、セキュリティポリシー違反時にブロックせずフィードバック。Claude が代替アプローチを自動で試行し、ポリシーに準拠したコードを生成します。

動作変更の注意事項

⚠️ アップデート時に確認すべき点

  • ANTHROPIC_API_KEY 等が設定されていると、Claude.ai ログインがあっても Remote Control / /schedule / 通知が無効化されます
  • フックはターミナルアクセスなしで実行されます。ターミナルに出力していたフックはファイル/ログへの書き込みに変更が必要です
  • 既存の command フィールドは動作しますが、パスにスペース/特殊文字がある場合は args フィールドへの移行を推奨します
