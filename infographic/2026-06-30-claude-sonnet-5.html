モデルリリース

Claude Sonnet 5 — 次世代 Sonnet モデルのリリース

2026-06-30

Opus 4.8 に近い性能を Sonnet 価格帯で実現する「最もエージェンティックな Sonnet」。1M コンテキスト、128k 出力、新トークナイザー搭載

概要

🤖

最もエージェンティックな Sonnet

計画立案、ツール使用、自律的な動作において最高レベルの Sonnet モデル。エージェントタスクに特化した強化が施されている。

⚡

Opus 4.8 に近い性能を低価格で

推論、ツール使用、コーディング、知識作業のすべてで Sonnet 4.6 を厳密に上回り、多くのユースケースで Opus 4.8 に匹敵する性能を提供。

📐

1M コンテキスト / 128k 出力

最大 1,000,000 トークンのコンテキストウィンドウと 128,000 トークンの出力に対応。大規模なコードベースやドキュメントの処理に最適。

🧠

アダプティブシンキング デフォルト有効

手動 Extended Thinking は廃止され、アダプティブシンキングがデフォルトで有効化。モデルが自動的に思考の深さを調整する。

🔤

新トークナイザー

同じテキストに対して約 30% 多くのトークンを生成。導入価格はこのコスト増加を相殺するように設計されている。

Sonnet 4.6 vs Sonnet 5

Before (Sonnet 4.6)

  • 価格: $3/$15 per MTok
  • Extended Thinking: budget_tokens で予算指定可能
  • サンプリング: temperature, top_p, top_k 設定可能
  • トークナイザー: 従来版
  • サイバーセーフガード: オプション

After (Sonnet 5)

  • 導入価格: $2/$10 per MTok
  • Adaptive Thinking: 自動有効、手動は 400 エラー
  • サンプリング: 非デフォルト値で 400 エラー
  • 新トークナイザー: 約 30% トークン増加
  • サイバーセーフガード: デフォルト有効

主要機能

🧠

アダプティブシンキング DEFAULT ON

デフォルトで有効化。モデルが自動的に思考の深さを調整し、タスクの複雑さに応じた最適な推論を行う。手動の Extended Thinking は廃止。

🛡️

サイバーセーフガード DEFAULT ON

デフォルトで有効化。悪意あるリクエストの拒否能力が向上し、全体的に望ましくない動作が Sonnet 4.6 と比較して減少。

🔤

新トークナイザー

同じ入力テキストに対してトークン数が約 1.0-1.35 倍に増加。導入価格期間中はおおむねコストニュートラルとなるよう設計。

🤖

エージェント能力

計画立案、ブラウザやターミナルのツール使用、自律的な動作において最高レベル。複雑なワークフローを自律的に遂行する能力が大幅に向上。

📐

1M コンテキストウィンドウ

最大 1,000,000 トークンのコンテキストと 128,000 トークンの出力をサポート。大規模なコードベースやドキュメント全体の処理に最適。

コードサンプル

基本的な API 呼び出し

Python Sonnet 5 基本呼び出し 
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-sonnet-5",
    max_tokens=8192,
    messages=[
        {"role": "user", "content": "Explain quantum computing in simple terms."}
    ]
)
print(message.content[0].text)

移行: Extended Thinking の対応

Python 変更前 (Sonnet 4.6) - 手動 Extended Thinking 
# 変更前 (Sonnet 4.6) - 手動 Extended Thinking
message = client.messages.create(
    model="claude-sonnet-4-6-20260220",
    max_tokens=8192,
    thinking={"type": "enabled", "budget_tokens": 4096},
    messages=[...]
)
Python 変更後 (Sonnet 5) - アダプティブシンキング自動有効 
# 変更後 (Sonnet 5) - アダプティブシンキングが自動で有効
message = client.messages.create(
    model="claude-sonnet-5",
    max_tokens=8192,
    messages=[...]
)

移行: サンプリングパラメータの削除

Python 変更前 (Sonnet 4.6) - カスタムサンプリング

ストリーミング使用例

Python ストリーミング 
import anthropic

client = anthropic.Anthropic()

with client.messages.stream(
    model="claude-sonnet-5",
    max_tokens=8192,
    messages=[
        {"role": "user", "content": "Write a detailed analysis of recent AI trends."}
    ]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

ツール使用の例

Python ツール使用 
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-sonnet-5",

    tools=[
        {
            "name": "get_weather",
            "description": "Get the current weather in a given location",
            "input_schema": {
                "type": "object",
                "properties": {
                    "location": {"type": "string", "description": "City name"}
                },
                "required": ["location"]
            }
        }
    ],
    messages=[
        {"role": "user", "content": "What's the weather like in Tokyo?"}
    ]
)
print(message.content)

ユースケース

エージェントワークフロー

計画立案、ブラウザやターミナルのツール使用、自律的な動作における最高レベルの能力を活かし、複雑なマルチステップタスクを自律的に遂行。

  • コードベース全体のリファクタリング
  • マルチツール連携によるデータ処理パイプライン
  • 自律的なバグ修正とテスト実行

複雑なコーディングタスク

Opus 4.8 に近いコーディング性能を Sonnet 価格帯で提供。大規模なコードベースの理解と生成に最適。

  • 1M コンテキストを活用した大規模コードレビュー
  • 複数ファイルにまたがる設計とリファクタリング
  • テスト生成とバグの発見

ナレッジワーク

推論能力の向上により、複雑な分析、要約、調査タスクにおいて高い精度を発揮。

  • 大量ドキュメントの分析と要約
  • 複雑なリサーチと情報統合
  • 技術ドキュメントの作成と校正

移行ガイド

1

モデル ID の更新 必須

claude-sonnet-4-6-20260220 から claude-sonnet-5 への移行を実施。

2

Extended Thinking の削除 必須

高 手動の thinking パラメータを使用している場合は削除が必要。設定すると 400 エラーが返される。アダプティブシンキングが自動で動作する。

3

サンプリングパラメータの見直し 必須

高 temperature、top_p、top_k をカスタマイズしている場合はデフォルト値に戻す。非デフォルト値の設定は 400 エラーを返す。

4

トークン使用量の監視 推奨

中 新トークナイザーにより同じ入力でもトークン数が増加するため、コスト監視を強化。導入価格期間中に影響を評価することを推奨。

5

コスト試算の更新 推奨

導入価格期間 (2026/8/31 まで) と標準価格適用後 (2026/9/1 以降) の両方でコスト試算を実施し、予算計画を更新。

まとめ

Claude Sonnet 5 は、Sonnet モデルファミリーの大幅な進化を示すリリースである。Opus 4.8 に近い性能を Sonnet 価格帯で提供し、特にエージェントタスクにおいて顕著な改善を実現している。

開発者にとって最も重要な変更点は、アダプティブシンキングのデフォルト有効化と手動 Extended Thinking の廃止、サンプリングパラメータのカスタマイズ不可、新トークナイザーによるトークン数の増加 (約 30%)、そして導入価格による移行期間のコスト緩和である。

導入価格は 2026 年 8 月 31 日まで有効であり、この期間中にトークナイザー変更の影響を評価し、標準価格適用後のコストを試算することが推奨される。エージェントシステムを構築している開発者にとっては、Sonnet 5 の強化されたツール使用能力と自律的な動作能力が大きなメリットとなる。
