Claude Code v2.1.133: Worktree 設定の柔軟化とプロキシ/セッション修正

2026-05-07

概要

🌳

worktree.baseRef 設定の追加

Worktree のブランチ元を fresh (origin/default) と head (ローカル HEAD) から選択可能に。デフォルトは fresh に戻される

🔌

プロキシ/mTLS の MCP OAuth 完全対応

HTTP(S)_PROXY / NO_PROXY / mTLS が OAuth フロー全体 (ディスカバリ、動的クライアント登録、トークン交換、リフレッシュ) に適用

🔑

並列セッション認証競合の解消

リフレッシュトークンの競合により全セッションが 401 エラーで停止する問題を修正。トークンリフレッシュの排他制御を改善

⚡

フックへのエフォートレベル伝達

$CLAUDE_EFFORT 環境変数と effort.level JSON フィールドでフックからエフォートレベルを参照可能に

🧠

メモリ使用量の改善

メモリ圧力下でウォームスペアのバックグラウンドワーカーを解放し、長時間セッションのメモリ消費を削減

Before / After: Worktree ブランチ元

Before (v2.1.128 - v2.1.132)

  • EnterWorktree はローカル HEAD からブランチ
  • 未プッシュのコミットが常に含まれる
  • クリーンな状態で始めたい場合は手動操作が必要
  • 設定で制御する方法がなかった

After (v2.1.133)

  • worktree.baseRef で制御可能に
  • fresh: origin/default から (デフォルト)
  • head: ローカル HEAD から
  • --worktree、EnterWorktree、エージェント分離に適用

⚠️ 移行の注意点

v2.1.128 以降の head 動作に依存している場合は、アップデート後に worktree.baseRef: "head" を .claude/settings.json に追加してください。

VS Code 修正

新機能

🌳 worktree.baseRef

fresh または head を指定し、Worktree のブランチ元を制御。--worktree、EnterWorktree、エージェント分離に適用

🔒 サンドボックスパス設定

Linux/WSL 向けに sandbox.bwrapPath と sandbox.socatPath でカスタムバイナリパスを指定可能

🏢 parentSettingsBehavior

管理者ティアキー。SDK managedSettings を first-wins または merge でポリシーマージに参加させる

⚡ $CLAUDE_EFFORT 環境変数

フックが effort.level と $CLAUDE_EFFORT でエフォートレベルを受信。Bash ツールからも参照可能

📖 --help の改善

claude --help に --remote-control オプションが一覧表示されるように

バグ修正

  • 🔑 並列セッションでリフレッシュトークン競合により全セッションが 401 エラーで停止する問題
  • 📁 Edit/Write の許可ルールがドライブルート (C:\ / POSIX /) にスコープされた場合に常にプロンプトが表示される問題
  • 🔐 ファイルロックがクロックスキューやディスク遅延で侵害された場合の ECOMPROMISED 未処理例外
  • 💬 コンパクション中の Esc キーで偽の「Error compacting conversation」通知が表示される問題
  • 🌐 HTTP(S)_PROXY / NO_PROXY / mTLS が MCP OAuth フロー全体で未適用だった問題
  • 💾 --add-dir で渡されたネットワークドライブで Read/Write/Edit が拒否される問題
  • 🎮 Remote Control の stop/interrupt が CLI セッションを完全にキャンセルしない問題
  • ⚙️ /effort の変更が他の同時実行セッションに干渉し、IDE 変更が無言で破棄される問題
  • 🤖 サブエージェントが Skill ツールでプロジェクト/ユーザー/プラグインスキルを発見できない問題

fi

対象ユーザーと影響

🏢 エンタープライズ環境

プロキシ経由の MCP OAuth 認証が正常動作、カスタムサンドボックスバイナリパス指定、SDK 管理設定のポリシーマージ制御が可能に。企業ネットワーク内での Claude Code 運用が大幅に改善されます。

👥 並列セッション利用者

リフレッシュトークンの競合による 401 エラーが解消。複数のターミナルで同時に Claude Code を使用する開発者にとって重要な修正です。エフォートレベルのセッション間干渉も修正されました。

🔧 フック開発者

$CLAUDE_EFFORT 環境変数と effort.level JSON フィールドにより、エフォートレベルに応じたフックの条件分岐が可能に。高エフォート時のみ追加テストを実行するといったワークフローを構築できます。

🌳 Worktree ベースの開発者

worktree.baseRef 設定により、クリーンな状態からの作業とローカルコミットの引き継ぎを自由に選択可能。チームの開発ワークフローに合わせた柔軟な運用ができます。
