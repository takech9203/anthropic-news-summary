Before / After

Before - /simplify

  • コードのクリーンアップと修正提案
  • エフォートレベル指定なし
  • PR コメント機能なし
  • 簡素化の提案に特化

After - /code-review

  • 正確性バグの検出に特化
  • low/medium/high/max レベル指定
  • --comment で PR にインライン投稿
  • CI/CD パイプライン統合可能

主な機能

🔍 /code-review

正確性バグ検出に特化した新コマンド。エフォートレベルで検出範囲を制御し、--comment で PR にインラインコメントを直接投稿

📌 ピン留めセッション

claude agents で Ctrl+T により重要なセッションを永続化。アイドル中も生存し、アップデート時にはインプレースで再起動

🔄 自動アップデーター

ネットワーク障害のリトライ、エラーカテゴリと OS エラーコードの報告。アップデート失敗時に現バージョンも表示

⚡ diff レンダリング

大規模ファイル編集時の差分表示パフォーマンスが改善。大きな変更の確認がスムーズに

📝 履歴重複排除

矢印キーで呼び出したプロンプトの再送信時に連続する重複エントリが記録されなくなった

🔒 ログイン制限強化

エンタープライズログインポリシーがサードパーティプロバイダーと API キーセッションにも正しく適用

/code-review エフォートレベル

レベル 検出対象 ユースケース
low / medium 少数の高確信度な正確性バグ クイックチェック、軽量 CI
high / max 幅広いカバレッジ (不確実な指摘含む) 重要な PR、リリース前レビュー
指定なし バランスの取れた検出 日常的なコードレビュー

コードサンプル

Bash /code-review コマンドの使用例 
# 正確性バグをデフォルトの深度で検出
/code-review

# 高エフォートレベルで詳細なバグ検出
/code-review high

# 低エフォートレベル (高確信度の問題のみ)
/code-review low

# GitHub PR にインラインコメントとして結果を投稿
/code-review --comment

# 高エフォート + PR コメント投稿の組み合わせ
/code-review high --comment
Bash ピン留めセッションと緊急アップデート 
# claude agents でセッション一覧を表示
claude agents

# セッション上で Ctrl+T を押下してピン留め
# → アイドル時も生存
# → アップデート時にインプレース再起動
# → メモリプレッシャー時は非ピン留めセッションが先に回収

claude update

ユースケース

🔄 PR レビューパイプラインへの統合

/code-review high --comment を CI/CD パイプラインに組み込み、PR 作成時に自動的にコードレビューコメントを投稿。レビュアーの負担を軽減しつつ、正確性バグの早期検出を実現

📌 長時間稼働モニタリングセッション

claude agents でモニタリングセッションをピン留め。アイドル状態でも自動終了されず、Claude Code アップデート時にもインプレースで再起動されるため、手動再構築が不要に

🪟 Windows 開発環境の安定利用

7 件の Windows/PowerShell 修正により、winget/Microsoft Store 経由の pwsh、NTFS ジャンクション、CJK 文字環境での安定性が大幅に向上。Windows での Claude Code 体験が改善
