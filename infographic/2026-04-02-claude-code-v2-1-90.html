概要

🎓

/powerup コマンド

Claude Code の機能をインタラクティブに学習できるレッスン機能が追加されました。アニメーションデモ付きで、各機能の使い方を実践的に習得できます

⚡

パフォーマンス大幅改善

SSE トランスポートとトランスクリプト書き込みの二次関数的ボトルネック (O(n^2)) を線形時間 (O(n)) に最適化。MCP ツールスキーマの不要な JSON.stringify も排除

🐛

重大なバグ修正

レートリミットダイアログの無限ループ、--resume のプロンプトキャッシュミス (v2.1.69 リグレッション)、Auto モードのユーザー境界遵守を修正

🔒

セキュリティ強化

PowerShell のツールパーミッションチェックを強化。バックグラウンドジョブバイパス、デバッガーハング、TOCTOU 脆弱性など複数の問題を修正

🛡️

プライバシー保護

DNS キャッシュコマンド (Get-DnsClientCache、ipconfig /displaydns) が自動許可対象から除外され、実行前にユーザー確認が必要に

Before / After 比較

SSE トランスポート

Before

ストリーミングフレームの結合処理が文字列の再割り当てを繰り返し、フレームサイズに対して O(n^2) の処理時間がかかっていた

After

バッファリング戦略の改善により O(n) に最適化。大きなレスポンスのストリーミング時に顕著な速度向上

トランスクリプト書き込み

Before

長い会話では各ターンでトランスクリプト全体を書き換えていたため、会話が長くなるほど書き込みが O(n^2) で遅くなっていた

After

差分書き込みまたはインクリメンタルな追記方式に変更され、長時間セッションでも安定したパフォーマンスを実現

新機能 (Added)

🎓

/powerup コマンド

Claude Code の機能をインタラクティブに学習できるレッスン機能。アニメーションデモ付きで、初心者から上級者まで実践的にスキルを習得可能

📦

マーケットプレイスキャッシュ保持

CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE 環境変数により、git pull 失敗時でも既存のマーケットプレイスキャッシュを保持。オフライン環境で特に有用

🐶

.husky 保護ディレクトリ

acceptEdits モードにおいて .husky ディレクトリが保護対象に追加。Husky の Git フック設定が意図せず変更されることを防止

パフォーマンス改善

  • 🔌MCP ツールスキーマの JSON.stringify 削除: キャッシュキー検索時にターンごとに実行されていた MCP ツールスキーマの JSON.stringify を排除。多数の MCP ツールを使用する環境で顕著な改善
  • 📡SSE トランスポートの線形時間化: 大きなストリームフレームを O(n^2) から O(n) の線形時間で処理するように最適化。大きなレスポンスのストリーミング時に顕著な速度向上
  • 📝トランスクリプト書き込みの最適化: 長い会話を持つ SDK セッションでトランスクリプト書き込みが二次関数的に遅くなる問題を解消
  • 🔄/resume のプロジェクトセッション並列読み込み: 全プロジェクト表示時にプロジェクトセッションを並列で読み込むようになり、多数のプロジェクトを持つユーザーの読み込み時間が改善

バグ修正 (Fixed)

重大な修正
  • レートリミットダイアログの無限ループ: 使用量上限に達した後、レートリミットオプションダイアログが繰り返し自動的に開き続け、最終的にセッションがクラッシュする問題を修正
  • --resume のプロンプトキャッシュミス: 遅延ツール、MCP サーバー、またはカスタムエージェントを使用している環境で、--resume 時に最初のリクエストでプロンプトキャッシュが完全にミスする問題を修正 (v2.1.69 からのリグレッション)
  • Edit/Write ツールの "File content has changed" エラー: PostToolUse のフォーマットオンセーブフックが連続する編集の間にファイルを書き換えた際に発生するエラーを修正
  • Auto モードのユーザー境界遵守: Auto モードが「push しないで」「X の前に Y を待って」などの明示的なユーザー境界を無視する問題を修正
Hooks 関連
  • PreToolUse フックの JSON 出力処理: stdout に JSON を出力して終了コード 2 で終了する PreToolUse フックが、ツール呼び出しを正しくブロックしない問題を修正
UI 関連
  • 折りたたみバッジの重複表示: フルスクリーンスクロールバックで CLAUDE.md ファイルがツール呼び出し中に自動読み込みされた際に、折りたたみ検索/読み取りサマリーバッジが複数回表示される問題を修正
  • ライトテーマでのホバーテキスト: ライトターミナルテーマでクリックして展開するホバーテキストがほぼ見えない問題を修正
  • パーミッションダイアログの UI クラッシュ: 不正なツール入力がパーミッションダイアログに到達した際の UI クラッシュを修正
  • 選択画面のヘッダー消失: /model、/config およびその他の選択画面でスクロール時にヘッダーが消える問題を修正
セキュリティ関連
  • PowerShell ツールパーミッションチェックの強化: 末尾の & によるバックグラウンドジョブバイパス、-ErrorAction Break によるデバッガーハング、アーカイブ展開の TOCTOU 脆弱性、パース失敗時の拒否ルール劣化を修正

変更・削除

--resume ピッカーの変更 (Changed)

claude -p や SDK 呼び出しで作成されたセッションが --resume ピッカーに表示されなくなりました。手動セッションのみが表示されるため、セッション選択が見やすくなります

DNS キャッシュコマンドの自動許可除外 (Removed)

Get-DnsClientCache と ipconfig /displaydns が自動許可対象から削除されました。DNS キャッシュにはプライバシーに関わる情報が含まれるため、実行前にユーザーの確認が必要になります

開発者への影響

--resume を頻繁に使用するユーザー

v2.1.69 以降 --resume のレスポンスが遅いと感じていた場合、本バージョンでプロンプトキャッシュミスが修正され、パフォーマンスが回復しました。特に MCP サーバーやカスタムエージェントを利用する環境で効果が顕著です

長時間セッションを行うユーザー

SSE トランスポートとトランスクリプト書き込みの O(n^2) ボトルネック解消により、長時間セッションでのパフォーマンス低下が大幅に改善されました。レートリミットダイアログの無限ループ修正もセッション安定性に貢献します

Windows / PowerShell ユーザー

PowerShell のセキュリティ強化により、一部のコマンドパターンでパーミッション確認の挙動が変わる可能性があります。Get-DnsClientCache と ipconfig /displaydns は自動許可されなくなったため、実行時にパーミッション確認が表示されます
