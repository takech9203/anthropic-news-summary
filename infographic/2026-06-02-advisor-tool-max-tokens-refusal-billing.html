API UPDATE

Advisor Tool の max_tokens パラメータと Refusal 課金の廃止

2026-06-02 | Claude API Release Notes

概要

🎯

Advisor Tool に max_tokens パラメータ追加

アドバイザーモデルの出力トークン数を制限し、レイテンシとコストの最適化が可能に

💰

Refusal レスポンスの課金廃止

出力を生成しなかったリクエスト (stop_reason: "refusal") は課金対象外に

⚡

エージェントワークロードのコスト効率向上

Haiku + Opus の組み合わせで max_tokens により Opus のコストを制御可能

主な変更点

🔧 Advisor Tool max_tokens

  • tools[].max_tokens でアドバイザーモデルの出力をキャップ可能
  • フルレングスのレスポンスが不要なワークロードでレイテンシと出力トークンコストを削減

🚫 Refusal 課金廃止

  • stop_reason: "refusal" を返し、出力を生成しなかったリクエストは課金対象外に
  • stop_details フィールドで category (cyber, bio, null) と explanation を返す

Before / After

Before

Advisor Tool: 出力トークン数の制限なし。アドバイザーモデルがフルレスポンスを生成しコスト増大


Refusal: 出力なしでもリクエストに課金発生

After

Advisor Tool: max_tokens で出力を制限。簡潔なガイダンスだけ取得しコスト削減


Refusal: 出力なしのリクエストは課金対象外

背景情報

  • Advisor Tool は 2026 年 4 月 9 日にパブリックベータとして発表
  • エグゼキューターモデルとアドバイザーモデルのペアリングアーキテクチャ
  • Beta header: advisor-tool-2026-03-01

Python Refusal Detection 
response = client.messages.create(...)
if response.stop_reason == "refusal":
    # No billing for this request
    print(f"Refusal category: {response.stop_details.category}")
    print(f"Explanation: {response.stop_details.explanation}")

ユースケース

🤖 エージェントワークロードの最適化

長時間実行されるエージェントタスクで、アドバイザーからの簡潔なガイダンスだけが必要な場合にコストを削減

💳 コスト管理の改善

Refusal 時の無駄な課金がなくなり、安全性フィルタが頻繁にトリガーされるワークロードのコスト予測が容易に

🔀 ハイブリッドモデル戦略

Haiku (安価・高速) + Opus (高精度) の組み合わせで、max_tokens により Opus のコストを制御

開発者への影響

🔧

Advisor Tool ユーザー

max_tokens パラメータ追加でコスト最適化が可能に。既存の Advisor Tool 実装にパラメータを追加するだけで効果を得られる

👥

全 API ユーザー

Refusal 時の無駄な課金がなくなる。特に安全性フィルタが頻繁にトリガーされるユースケースで恩恵が大きい
