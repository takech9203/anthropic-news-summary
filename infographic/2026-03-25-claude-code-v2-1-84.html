Claude Code v2.1.84

Windows PowerShell ツール、3P モデルカスタマイズ、アイドル復帰プロンプト、IME 入力修正を含む 40 件超の改善

2026-03-25

概要

⚡

PowerShell ツール (Windows オプトインプレビュー)

Windows 環境で PowerShell コマンドをネイティブに実行できる新しいツールが追加されました。pwsh.exe (PowerShell 7 以上) を優先検出し、フォールバックとして powershell.exe (5.1) を使用します。

🎛️

3P モデル能力カスタマイズ環境変数

Bedrock、Vertex、Foundry 経由のモデルで effort/thinking 能力の自動検出を上書きできる ANTHROPIC_DEFAULT_{OPUS,SONNET,HAIKU}_MODEL_SUPPORTS 環境変数が追加されました。

⏰

アイドル復帰プロンプト

75 分以上離席して戻ったユーザーに /clear を促すプロンプトが表示されるようになりました。古いセッションでの不要なトークン再キャッシュを削減します。

🇯🇵

IME 入力修正 (CJK インラインレンダリング)

ネイティブターミナルカーソルがテキスト入力キャレットを追跡するようになり、日本語、中国語、韓国語の入力がインラインで正しくレンダリングされます。

主要機能

⚡ PowerShell ツール

Windows ネイティブの PowerShell コマンド実行が可能に。pwsh.exe (7+) を優先検出し、powershell.exe (5.1) にフォールバック。Bash ツールと併存し、既存ワークフローに影響なし。

  • オートモード非対応
  • サンドボックス非対応
  • WSL 非対応
Preview

🎛️ 3P モデル能力カスタマイズ

_MODEL_SUPPORTS で effort/thinking 能力検出を上書き。_MODEL_NAME と _DESCRIPTION で /model ピッカーのラベルもカスタマイズ可能。エンタープライズ環境でのモデル管理が大幅に改善されます。

New

⏰ アイドル復帰プロンプト

75 分以上のアイドル後にセッションに戻ると、古いキャッシュの無効化による再キャッシュコストを防ぐため /clear を促すプロンプトを表示。不要なトークン消費を削減します。

New

🪝 TaskCreated hook

TaskCreate でタスクが作成された際に発火する新しい hook。ワークフロー自動化の柔軟性が向上します。WorktreeCreate hook も HTTP 対応し、レスポンス JSON でワークツリーパスを返せます。

New

🔒 allowedChannelPlugins

チームおよびエンタープライズ管理者がチャンネルプラグインの許可リストを定義可能に。組織のセキュリティポリシーに沿ったプラグイン管理を実現します。

New

コードサンプル

Bash PowerShell ツールの有効化 
# 環境変数で有効化
export CLAUDE_CODE_USE_POWERSHELL_TOOL=1
claude

# PowerShell ツールを有効にして起動 (Windows)
CLAUDE_CODE_USE_POWERSHELL_TOOL=1 claude
JSON settings.json での PowerShell 有効化 
{
  "env": {
    "CLAUDE_CODE_USE_POWERSHELL_TOOL": "1"
  }
}

# Bedrock の Opus モデルの能力を上書き
export ANTHROPIC_DEFAULT_OPUS_MODEL_SUPPORTS="effort,thinking"
export ANTHROPIC_DEFAULT_OPUS_MODEL_NAME="Bedrock Opus"
export ANTHROPIC_DEFAULT_OPUS_MODEL_DESCRIPTION="Bedrock us-east-1 Opus"

# Sonnet モデルの設定
export ANTHROPIC_DEFAULT_SONNET_MODEL_SUPPORTS="effort"

ユースケース

Windows 開発者

PowerShell ツールのオプトインプレビューにより、Windows ユーザーが PowerShell コマンドをネイティブに実行できるようになりました。Git Bash を介さない直接的な Windows 操作が可能になり、pwsh.exe (PowerShell 7 以上) を優先検出し powershell.exe (5.1) へのフォールバックも備えています。hooks やスキルのフロントマターで shell: powershell を指定することで、個別のコマンドを PowerShell で実行することも可能です。

Bedrock / Vertex / Foundry 利用組織

3P プロバイダ経由でモデルを使用する場合、Claude Code の effort/thinking 能力の自動検出がプロバイダ側の制約により正しく動作しないケースがありました。ANTHROPIC_DEFAULT_{OPUS,SONNET,HAIKU}_MODEL_SUPPORTS 環境変数で自動検出を上書きし、_MODEL_NAME と _DESCRIPTION で /model ピッカーのラベルをカスタマイズすることで、組織内でのモデル識別と管理が容易になります。

CJK 入力ユーザー

日本語、中国語、韓国語を入力するユーザーにとって、IME コンポジションのインラインレンダリング対応はターミナルでの入力体験を大幅に改善します。ネイティブターミナルカーソルがテキスト入力キャレットを追跡するようになり、IME コンポジションウィンドウが正しい位置に表示され、スクリーンリーダーも入力位置を追跡できるようになりました。
