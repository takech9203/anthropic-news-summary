Claude API キー有効期限設定機能を追加

2026-07-08

概要

⏰

有効期限の設定が可能に

API キー作成時に有効期限を設定可能。セキュリティポリシーに合わせたキー管理が実現します。

📋

3 種類の期間選択

プリセット期間、カスタム期間、無期限 (Never) の 3 種類から選択可能。用途に応じた柔軟な設定ができます。

📧

期限切れ前のメール通知

7 日以上の有効期限を持つキーは、期限切れ前にメールで通知。予期しないサービス停止を防止します。

✅

既存キーへの影響なし

破壊的変更なし。既存の API キーは従来通り動作し、影響を受けません。

Before / After

Before

  • 有効期限なし
  • 手動管理が必要
  • 漏洩リスクの長期化

After

  • 有効期限設定可能
  • 自動通知で管理負荷軽減
  • コンプライアンス対応

主な機能

⏳ 有効期限設定

API キー作成時に有効期間を指定。期限が過ぎたキーは自動的に無効化されます。

📅 期間選択

プリセット (30 日、60 日、90 日など)、カスタム日付指定、Never (無期限) の 3 種類から選択。

🔔 メール通知

7 日以上の有効期限を持つキーは、期限切れ前に管理者へメール通知を送信します。

🔧 Admin API

Admin API レスポンスに expires_at フィールドが追加。プログラマティックなキー管理が可能に。

  "expires_at": null,
  "status": "active"
}

推奨事項

🏭 本番環境

90 日程度の有効期限を設定し、定期的なローテーションを実施。セキュリティとオペレーション効率のバランスを確保します。

🛠 開発・テスト環境

30 日程度の短い有効期限を設定。不要なキーが残存するリスクを最小化します。

🔄 自動ローテーション

Admin API で expires_at を監視し、期限切れ前に新しいキーを自動生成するローテーション戦略を構築します。
