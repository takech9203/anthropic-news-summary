Before / After: MCP ツール結果

Before v2.1.91

  • MCP ツール結果にサイズ制限あり
  • DB スキーマなどの大規模結果は自動的に切り詰め
  • 情報の欠落が発生
  • MCP サーバー側でサイズ制御不可

After v2.1.91

  • _meta アノテーションでサイズ上限を指定可能
  • 最大 500K 文字まで結果を保持
  • 完全な情報を損失なく取得
  • MCP サーバー側で柔軟に制御

✨ 新機能

📋 MCP 結果オーバーライド

MCP サーバーがツール結果に _meta["anthropic/maxResultSizeChars"] を付与することで、最大 500K 文字まで結果を拡張可能。DB スキーマや大規模データに最適。

🔒 シェル実行無効化

disableSkillShellExecution 設定で、スキル・カスタムコマンド・プラグインからのインラインシェル実行を制限。セキュリティポリシーの厳しい環境向け。

📝 ディープリンク複数行対応

claude-cli://open?q= ディープリンクでエンコードされた改行 %0A が利用可能に。複数行プロンプトの共有が容易に。

📦 プラグイン bin/ 対応

プラグインが bin/ 配下に実行ファイルを同梱し、Bash ツールからベアコマンドとして直接呼び出し可能。独自 CLI ツールの提供が容易に。

🐛 バグ修正

  • --resume のトランスクリプトチェーン断裂: 非同期トランスクリプト書き込みがサイレントに失敗した際に会話履歴が失われる問題を修正
  • cmd+delete キーバインド: iTerm2、kitty、WezTerm、Ghostty、Windows Terminal で行頭まで削除しない問題を修正
  • リモートセッションのプランモード: コンテナ再起動後にプランファイルを見失い、パーミッションプロンプトやプラン承認モーダルが空になる問題を修正
  • permissions.defaultMode の JSON スキーマ: settings.json で "auto" を指定した際のバリデーションエラーを修正
  • Windows バージョンクリーンアップ: アクティブバージョンのロールバックコピーが保護されない問題を修正

⚡ 改善

  • /feedback コマンドの UX: 利用不可の場合、スラッシュメニューから消えるのではなく理由を説明するように変更
  • /claude-api スキルガイダンス強化: ツールサーフェスの決定、コンテキスト管理、キャッシュ戦略を含むエージェント設計パターンのガイダンスを改善
  • Bun パフォーマンス向上: stripAnsi を Bun.stripANSI にルーティングすることで高速化
  • Edit ツール最適化: old_string アンカーを短縮し、出力トークン数を削減

ユースケース

🗄️ データベーススキーマの全体取得

MCP サーバーが返す DB スキーマが従来のサイズ制限で切り詰められていた場合、_meta["anthropic/maxResultSizeChars"] を設定することで完全なスキーマ情報を Claude Code に渡せます。大規模なデータベース設計の理解やマイグレーション作業に有効です。

🔒 エンタープライズ環境でのセキュリティ強化

disableSkillShellExecution を有効にすることで、スキルやプラグインからの意図しないシェルコマンド実行を防止できます。セキュリティポリシーが厳格な組織での Claude Code 導入に適しています。

📦 プラグインによる独自 CLI ツール提供

プラグイン開発者は bin/ ディレクトリに実行ファイルを同梱することで、Claude Code のワークフロー内でシームレスに独自ツールを呼び出せます。コード解析ツールやフォーマッターなど、特定のワークフローに特化した機能を提供可能です。
