☁️ Claude Platform on AWS の提供開始

📅 2026-05-11

概要

🏢

Anthropic 運用のインフラを AWS 経由で利用

Claude Platform on AWS は、Anthropic が推論スタックを運用しつつ、AWS IAM 認証と AWS Marketplace 課金でアクセスできる第 3 の統合オプションです。

⚡

フルプラットフォーム体験を即日提供

Messages API、Files API、Message Batches API、Claude Managed Agents、Agent Skills、コード実行など、Anthropic プラットフォームの全機能にアクセス可能。ベータヘッダーも対応。

🔑

AWS 認証・課金でシームレスに統合

SigV4 または API キー認証、IAM ベースのアクセス制御、AWS Marketplace 経由の課金に対応。既存の AWS 管理フローをそのまま活用できます。

🔒

独立キャパシティと PrivateLink 対応

ファーストパーティ API とも Bedrock とも別のキャパシティプールで動作。AWS PrivateLink によるプライベートネットワーク接続にも対応。

Before / After

❌ Before: Bedrock のみ

  • AWS がインフラを運用・管理
  • 新機能は Bedrock リリース待ち
  • ベータヘッダー非対応
  • Files API / Managed Agents 利用不可
  • Agent Skills 利用不可

✅ After: Platform on AWS

  • Anthropic が推論スタックを運用
  • 最新機能に通常即日アクセス
  • ベータヘッダー完全対応
  • Files API / Managed Agents 対応
  • Agent Skills / コード実行対応

主な機能

📨 Messages API

Anthropic ファーストパーティと同一の /v1/messages エンドポイント。SDK の AnthropicAWS クライアントで接続。

📁 Files API

ファイルのアップロードと Messages API での参照。PDF、画像、テキストドキュメントの処理が可能。

📋 Message Batches API

大量メッセージの非同期バッチ処理。標準 API の 50% のコストで大規模処理を実行。

🤖 Claude Managed Agents

完全マネージドなエージェントハーネス。セキュアなサンドボックス、ビルトインツール、SSE ストリーミング。

🎯 Agent Skills

PowerPoint、Excel、Word、PDF 操作などのプリビルトスキル。カスタムスキルも API 経由でアップロード可能。

🧪 ベータ機能アクセス

anthropic-beta ヘッダーで最新ベータ機能を即日利用。Bedrock では非対応のベータ機能にもアクセス可能。

セットアップ手順

1

AWS Console でサインアップ

Claude Platform on AWS サービスページから利用規約に同意し、AWS Marketplace サブスクリプションを開始

2

Anthropic 組織セットアップ

AWS アカウントに紐づく新しい Anthropic 組織を構成。既存のプライベートオファーがある場合は自動検出

3

ワークスペース ID を記録

AWS Console または Claude Console でワークスペース ID を確認。リージョン、IAM スコープの設定

4

Claude Console にサインイン

AWS IAM 経由のフェデレーテッドアクセスで Claude Console にログイン。aws-external-anthropic:AssumeConsole 権限が必要

コードサンプル

Python 基本的な Messages API リクエスト 
from anthropic import AnthropicAWS

# Claude Platform on AWS クライアント (ベータ)
client = AnthropicAWS(
    aws_region="us-east-1",
    # AWS 認証情報は環境変数または IAM ロールから自動取得
)

# Messages API の呼び出し
message = client.messages.create(
    model="claude-opus-4-6-20250410",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Hello, Claude on AWS!"}
    ],
)
print(message.content[0].text)
Python ベータ機能 + Files API の利用 
from anthropic import AnthropicAWS

client = AnthropicAWS(aws_region="us-east-1")

# ファイルのアップロード
with open("document.pdf", "rb") as f:
    file = client.files.create(file=f, purpose="messages")

# ファイルを参照してメッセージ送信
message = client.messages.create(
    model="claude-opus-4-6-20250410",
    max_tokens=2048,
    messages=[{
        "role": "user",
        "content": [
            {"type": "file", "source": {"type": "file", "file_id": file.id}},
            {"type": "text", "text": "Summarize this document."},
        ],
    }],
)
print(message.content[0].text)
Bash curl での直接リクエスト 
curl https://api.anthropic.com/v1/messages \
  --aws-sigv4 "aws:amz:us-east-1:anthropic" \
  --user "$AWS_ACCESS_KEY_ID:$AWS_SECRET_ACCESS_KEY" \
  -H "x-amz-security-token: $AWS_SESSION_TOKEN" \
  -H "content-type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-opus-4-6-20250410",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "Hello, Claude Platform on AWS!"}
    ]
  }'

データ処理とセキュリティ

📊

データプロセッサ: Anthropic

推論の入出力データは Anthropic がデータプロセッサとして処理。AWS は課金・ID メタデータのみを処理。Anthropic のデータ使用規約が適用されます。

🌐

データレジデンシの注意点

データは AWS 内に留まるとは限りません。推論は Anthropic のプライマリクラウドにルーティングされる場合があり、サブサービスは予告なく変更される可能性があります。

🔐

AWS PrivateLink 対応

VPC からプライベートネットワーク経由で Claude Platform on AWS エンドポイントに接続可能。パブリックインターネットを経由しないセキュアな通信が実現します。

⚠️

Bedrock を選択すべきケース: FedRAMP High、IL4、IL5、HIPAA など規制産業で AWS を唯一のデータプロセッサとする必要がある場合は、引き続き Amazon Bedrock を使用してください。Bedrock は全て AWS 管理インフラで動作します。

ユースケース

🏢 エンタープライズ: AWS 課金でフル機能を活用

AWS Marketplace を通じた一元課金で、Files API、Message Batches API、Claude Managed Agents、Agent Skills をフル活用。既存の AWS アカウント管理フローをそのまま維持しながら、Anthropic プラットフォームの全機能にアクセスできます。

⚡ 先進的開発チーム: ベータ機能への即日アクセス

anthropic-beta ヘッダーで最新機能を即日利用可能。Bedrock では提供スケジュールが遅れる機能も、Claude Platform on AWS なら通常即日アクセスできるため、競争優位性を確保できます。

🤖 エージェント構築: Managed Agents + Agent Skills

これまで Bedrock では利用できなかった Claude Managed Agents と Agent Skills が利用可能に。サンドボックス化されたコード実行環境でエージェントを安全に運用し、PowerPoint、Excel、PDF 操作なども実行できます。

移行ガイド

📡 ファーストパーティ API からの移行

クライアントクラスを Anthropic → AnthropicAWS に変更。認証を AWS 認証に切り替え。リクエストボディ・レスポンス処理は変更不要。全機能がそのまま利用可能。

☁️ Amazon Bedrock からの移行

クライアントを AnthropicBedrock / AnthropicBedrockMantle → AnthropicAWS に変更。ベータヘッダー利用が可能に。Files API、Batches API、Managed Agents が新たに利用可能。
