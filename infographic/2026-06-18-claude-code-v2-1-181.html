Claude Code v2.1.181 リリース: 設定コマンド強化、プロンプトキャッシュ修正、ファイル書き込み信頼性向上

2026-06-18

概要

⚙

/config key=value 構文で設定を即座に変更可能

interactive、-p、Remote Control の全モードで利用可能。プロンプトから直接設定を変更できるようになりました。

💰

プロンプトキャッシュがカスタム ANTHROPIC_BASE_URL / Foundry で修正

アテステーショントークンをキャッシュキーから除外することで、コスト大幅削減を実現。

💾

ネットワークドライブ/クラウド同期フォルダでのファイル破損問題修正

アトミック書き込みにより、0 バイトファイル生成の問題を解消しました。

⚡

起動パフォーマンス改善

最大 15 秒ブロック解消、120ms 回帰修正。設定フェッチをバックグラウンドに移行し即座に TUI を表示。

🚀

Bun ランタイム 1.4 にアップグレード

最新の Bun ランタイムへのアップグレードにより、全体的なパフォーマンスが向上。

🔄

自動リトライ改善: thinking 中の接続断から自動回復

ストリーミング中にネットワーク接続が途切れても、自動的にリトライして復旧します。

Before / After

起動パフォーマンス

Before

起動に最大 15 秒ブロック + 120ms 回帰。設定フェッチが完了するまで TUI が表示されない。

After

即座に TUI 表示。設定はバックグラウンドフェッチに移行し、ユーザーはすぐに操作可能。

プロンプトキャッシュ (Foundry / カスタム URL)

Before

Foundry でプロンプトキャッシュ不動作。毎ターンキャッシュミスが発生し、コストが増大。

After

アテステーショントークンをキャッシュキーから除外。キャッシュが正常に動作しコスト削減。

ネットワークドライブでのファイル書き込み

Before

ネットワークドライブで 0 バイトファイルが生成される。データ消失のリスク。

After

アトミック書き込みで安全にファイルを保存。データ整合性を保証。

新機能

⚙ /config key=value

プロンプトから直接設定を変更可能。/config thinking=false や /config model=claude-sonnet-4-6 のように使用できます。interactive、-p、Remote Control の全モードで対応。

🍎 sandbox.allowAppleEvents

macOS で Apple Events の送信を許可する新しい設定。サンドボックス環境でも他のアプリケーションとの連携が可能になります。

📴 CLAUDE_CLIENT_PRESENCE_FILE

モバイル通知の抑制を制御する環境変数。デスクトップで作業中にモバイルへの通知を抑制し、集中を妨げません。

コードサンプル

Bash 設定変更とプレゼンスファイルの使用例 
# プロンプトから設定を変更する例
/config thinking=false
/config model=claude-sonnet-4-6

# Apple Events を許可する場合 (macOS のみ)
# settings.json に追加:
# "sandbox.allowAppleEvents": true

# プッシュ通知の抑制
export CLAUDE_CLIENT_PRESENCE_FILE="$HOME/.claude-present"
touch "$HOME/.claude-present"

開発者への影響

🏭 Foundry / カスタム URL ユーザー

プロンプトキャッシュが正常に動作するようになり、API コストが大幅に削減されます。特に長いコンテキストを使用するワークフローでは、キャッシュヒット率の改善により顕著なコスト削減効果が期待できます。ANTHROPIC_BASE_URL を設定しているユーザーは、アップデートするだけで自動的に恩恵を受けられます。

📁 ネットワークドライブ / クラウド同期ユーザー

OneDrive、Google Drive、NFS などのネットワークドライブやクラウド同期フォルダで作業している開発者にとって、ファイル破損のリスクが解消されます。アトミック書き込みにより、同期タイミングの衝突による 0 バイトファイル生成が防止されます。

🍎 macOS ユーザー

Apple Events エラー -600 の解消により、macOS でのサンドボックス実行が安定します。また sandbox.allowAppleEvents 設定により、Automator やショートカットとの連携が可能になり、ワークフロー自動化の幅が広がります。Spotlight によるフリーズ問題も修正されています。
