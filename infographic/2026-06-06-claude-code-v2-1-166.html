新機能

🔄 fallbackModel 設定

最大 3 つのフォールバックモデルを優先順位付きで設定。--fallback-model がインタラクティブセッションにも対応

🛡️ deny ルールのグロブ

ツール名に "*" グロブパターン対応。不明なツール名は起動時に警告表示

🔒 クロスセッション強化

SendMessage リレーメッセージの権限を剥奪。auto モードでの中継メッセージをブロック

🧠 thinking 無効化

MAX_THINKING_TOKENS=0 や --thinking disabled でデフォルト thinking を無効化可能

⚡ フォールバックリトライ

非リトライアブルエラー時にフォールバックモデルで 1 回自動リトライ

設定例

JSON settings.json - フォールバックモデル設定 
{
  "model": "claude-opus-4-8",
  "fallbackModel": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5"
  ]
}
JSON deny ルールのグロブパターン 
{
  "permissions": {
    "deny": [
      { "tool": "*", "description": "全ツールを拒否" }
    ],
    "allow": [
      { "tool": "Read", "description": "読み取り許可" },
      { "tool": "Grep", "description": "検索許可" }
    ]
  }
}
Bash thinking 無効化オプション 
# 環境変数で無効化
MAX_THINKING_TOKENS=0 claude

# CLI フラグで無効化
claude --thinking disabled

# フォールバックモデル指定
claude --fallback-model claude-sonnet-4-6

主なバグ修正

JetBrains 2026.1+ でターミナルがちらつく問題を同期出力の有効化で修正
macOS デーモン切断後に claude --bg-pty-host が CPU 100% で回転する問題を修正
Windows PowerShell コマンドバリデーションが子プロセスのパイプ保持でハングする問題を修正
Terminal Kitty プロトコル端末で Shift+非 ASCII 文字が無視される問題を修正
Agents git worktree に入ったバックグラウンドエージェントのクラッシュループを修正
Agents リモートセッションがバックエンド障害時に永久スタックする問題を修正
画像処理 処理不可能な画像の繰り返しエラーと余分なトークン消費を修正
Settings 無効なエントリが managed settings の残りのポリシー適用を無効化する問題を修正

影響を受けるユーザー

エンタープライズチーム

クロスセッションセキュリティ強化により、権限エスカレーション攻撃のリスクが大幅に低減

JetBrains IDE / Windows / macOS ユーザー

プラットフォーム固有のバグ修正により、各環境での安定性が大幅に向上
