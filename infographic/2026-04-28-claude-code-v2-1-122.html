バグ修正 12 件

💾 セッション / コマンド 3
  • /branch で巻き戻し済みタイムラインのエントリを含むセッションからフォークすると失敗する問題を修正
  • /model が Bedrock 推論プロファイル ARN に対して Effort オプションを表示しない問題を修正
  • !exit / !quit が bash モードで CLI 自体を終了してしまう問題を修正
☁️ Vertex AI / Bedrock 2
  • Vertex AI / Bedrock で構造化出力クエリ時に invalid_request_error が返される問題を修正
  • プロキシゲートウェイ経由で Vertex AI の count_tokens が 400 エラーを返す問題を修正
🔌 MCP / ツール 2
  • nonblocking モードでセッション開始後に接続した MCP ツールが ToolSearch で検出されない問題を修正
  • spinnerTipsOverride.excludeDefault が時間ベースのスピナーティップスを抑制しない問題を修正
🎨 UI / 表示 3
  • 画像が正しい最大値 2000px ではなく 2576px にリサイズされていた問題を修正 (トークン消費の削減)
  • リモートコントロールセッションのアイドルステータスが毎秒 2 回再描画される問題を修正 (tmux -CC パイプ溢れ対策)
  • 古いビュー設定によりアシスタントメッセージが空白表示される問題を修正
⚙️ 設定 / 音声 2
  • settings.json の不正な hooks エントリがファイル全体を無効化していた問題を修正
  • 音声モードで Caps Lock キーバインドに対するエラー表示を追加 (ターミナルは Caps Lock をキーイベントとして配信しないため)

コードサンプル

Bash Bedrock サービスティアの設定 
# コスト最適化ティア (バッチ処理向け)
export ANTHROPIC_BEDROCK_SERVICE_TIER=flex

# 優先ティア (低レイテンシ、リアルタイム向け)
export ANTHROPIC_BEDROCK_SERVICE_TIER=priority

# デフォルトティア (明示的に指定)
export ANTHROPIC_BEDROCK_SERVICE_TIER=default

# AWS プロファイルと組み合わせて使用
AWS_PROFILE=my-bedrock-profile ANTHROPIC_BEDROCK_SERVICE_TIER=priority claude
Bash PR URL からセッションを復元 
# Claude Code 起動後、/resume コマンドを使用
# 検索ボックスに PR URL を貼り付け

/resume
# → https://github.com/myorg/myrepo/pull/123 を貼り付け
# → その PR を作成したセッションが検索結果に表示される

# 対応プラットフォーム:
# GitHub:     https://github.com/{owner}/{repo}/pull/{number}
# GitLab:     https://gitlab.com/{group}/{project}/-/merge_requests/{number}
# Bitbucket:  https://bitbucket.org/{workspace}/{repo}/pull-requests/{number}

まとめ

Claude Code v2.1.122 は、新機能 2 件、改善 4 件、バグ修正 12 件 を含むリリースです。ANTHROPIC_BEDROCK_SERVICE_TIER 環境変数による Bedrock サービスティア選択と、/resume に PR URL を貼り付けてセッションを復元する機能が新たに追加されました。

バグ修正では、画像リサイズの最大値が 2576px から正しい 2000px に修正され、不要なトークン消費が削減されます。Vertex AI / Bedrock の構造化出力エラー、/branch のフォーク失敗、settings.json の hooks エントリ問題など、幅広い領域で安定性が改善されました。

全ての Claude Code ユーザーに対してアップデートを推奨します。特に Bedrock ユーザー は新しいサービスティア機能の活用を検討してください。
