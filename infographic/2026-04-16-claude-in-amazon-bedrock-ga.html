💻 コードサンプル

Python レガシー Bedrock 統合 
from anthropic import AnthropicBedrock

# 従来の Bedrock クライアント (InvokeModel API を使用)
client = AnthropicBedrock(aws_region="us-west-2")

message = client.messages.create(
    model="global.anthropic.claude-opus-4-6-v1",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello, Claude"}],
)
print(message.content[0].text)
Python 🆕 新しい Bedrock Mantle 統合 (GA) 
from anthropic import AnthropicBedrockMantle

# 新しい Bedrock Mantle クライアント (Messages API を直接使用)
# GA により 27 リージョンから選択可能
client = AnthropicBedrockMantle(aws_region="ap-northeast-1")

message = client.messages.create(
    model="anthropic.claude-opus-4-7",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello, Claude"}],
)
print(message.content[0].text)
TypeScript 🆕 Bedrock Mantle 統合 
import { AnthropicBedrockMantle } from "@anthropic-ai/bedrock-sdk";

const client = new AnthropicBedrockMantle({
  awsRegion: "ap-northeast-1",
});

const message = await client.messages.create({
  model: "anthropic.claude-opus-4-7",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello, Claude" }],
});

console.log(message.content[0].text);
Bash curl での直接リクエスト 
curl https://bedrock-mantle.ap-northeast-1.api.aws/anthropic/v1/messages \
  --aws-sigv4 "aws:amz:ap-northeast-1:bedrock-mantle" \
  --user "$AWS_ACCESS_KEY_ID:$AWS_SECRET_ACCESS_KEY" \
  -H "x-amz-security-token: $AWS_SESSION_TOKEN" \
  -H "content-type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "anthropic.claude-opus-4-7",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "Hello, Claude"}
    ]
  }'

🔄 移行ガイド

⚠️ 変更が必要

  • クライアントクラスを AnthropicBedrockMantle に変更
  • エンドポイントが bedrock-mantle.{region}.api.aws に
  • モデル ID を anthropic. プレフィックス付きに変更

✅ 変更不要

  • リクエストボディの構造はそのまま
  • レスポンスの処理ロジックはそのまま
  • ストリーミング処理は標準 SSE に自動対応

💡 ユースケース

🏢 エンタープライズ AI アプリケーション

AWS セキュリティ境界内でオペレーターアクセスゼロの環境で Claude を利用。データ所在地要件に合わせたリージョン選択や ZDR オプションにより、規制産業でのデプロイが容易に

🌏 グローバル展開

27 リージョンのグローバルエンドポイントにより、ユーザーに最も近いリージョンへ自動ルーティング。追加料金なしで低レイテンシを実現

🔀 マルチクラウド移植性

Anthropic API と同一のリクエスト形式により、ファーストパーティ API と Bedrock Mantle 間でコードの変更を最小限に抑えたプラットフォーム切り替えが可能

📏 クォータ

📥 デフォルト TPM

200 万入力トークン/分。追加承認なしで最大 400 万 TPM までリクエスト可能

🔒 データ保持

Amazon Bedrock のデータ保護ポリシーに準拠。ZDR は AWS サポートへの問い合わせで有効化可能

📊 オブザーバビリティ

CloudWatch と CloudTrail にログを出力。30 日間のローリングベースでの保持を推奨
