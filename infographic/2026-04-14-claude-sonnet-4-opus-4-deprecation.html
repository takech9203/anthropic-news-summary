Claude Sonnet 4 / Opus 4 の非推奨化を発表

2026-04-14

概要

⚠️

非推奨化発表: Sonnet 4 と Opus 4

Anthropic は Claude Sonnet 4 (claude-sonnet-4-20250514) および Claude Opus 4 (claude-opus-4-20250514) の非推奨化を発表しました。

📅

廃止日: 2026 年 6 月 15 日

約 2 か月の移行期間が設けられています。廃止後、これらのモデル ID へのリクエストは失敗します。Anthropic のモデル非推奨化ポリシー (60 日前通知) に準拠したスケジュールです。

🚀

移行先: 4.6 世代モデル

Sonnet 4 には claude-sonnet-4-6、Opus 4 には claude-opus-4-6 がそれぞれ推奨されています。

💰

価格据え置き

移行先の 4.6 世代モデルは旧モデルと同一価格です。移行によるコスト増加はありません。

Before / After: モデル ID の変更

Before -- 非推奨モデル

  • claude-sonnet-4-20250514
  • claude-opus-4-20250514
  • 2026 年 6 月 15 日以降は使用不可
  • アシスタントプレフィル対応

After -- 推奨移行先

  • claude-sonnet-4-6
  • claude-opus-4-6
  • 1M トークンコンテキスト (標準)
  • Adaptive Thinking 対応

必要なアクション

1. モデル ID の更新

コードベース内のすべてのモデル ID を新しい 4.6 モデルに変更してください。

2. アシスタントプレフィルの削除

4.6 モデルではプレフィルが 400 エラーを返します。構造化出力や output_config.format に移行してください。

3. Adaptive Thinking への移行

thinking: {type: "adaptive"} と effort パラメータへの移行を推奨します。

4. 新モデルでのテスト

本番環境に適用する前に、開発環境で十分なテストを実施してください。

5. SDK のアップデート

最新の Anthropic SDK にアップデートして、新しいモデル ID と機能に対応してください。

6. Opus 4.6 の動作差異を確認

Adaptive Thinking の推奨、プレフィル非対応、ツールパラメータの JSON エスケープ差異などを把握してください。

破壊的変更と推奨変更

破壊的変更

変更内容 影響度 対応
アシスタントプレフィル非対応 高 構造化出力または output_config.format に移行
ツールパラメータの JSON エスケープ差異 中 標準 JSON パーサーを使用していれば影響なし

推奨変更

変更内容 優先度 対応
Adaptive Thinking への移行 高 thinking: {type: "adaptive"} に変更
effort ベータヘッダーの削除 中 betas=["effort-2025-11-24"] を削除
interleaved thinking ベータヘッダーの削除 中 betas=["interleaved-thinking-2025-05-14"] を削除
output_format から output_config.format への移行 中 パラメータ名を変更

コードサンプル: Sonnet 4 から Sonnet 4.6 への移行

Python 変更前: Sonnet 4 
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=8192,
    messages=[
        {
            "role": "user",
            "content": "TypeScript で REST API サーバーを作成してください。"
        }
    ]
)

print(message.content[0].text)
Python 変更後: Sonnet 4.6 
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=8192,
    thinking={"type": "adaptive"},
    output_config={"effort": "medium"},
    messages=[
        {
            "role": "user",
            "content": "TypeScript で REST API サーバーを作成してください。"
        }
    ]
)

print(message.content[0].text)

コードサンプル: Opus 4 から Opus 4.6 への移行

Python 変更前: Opus 4 (プレフィル使用) 
import anthropic

client = anthropic.Anthropic()

# Opus 4 ではアシスタントプレフィルが使用可能だった
message = client.messages.create(
    model="claude-opus-4-20250514",
    max_tokens=16000,
    messages=[
        {
            "role": "user",
            "content": "この論文を分析してください。"
        },
        {
            "role": "assistant",
            "content": "## 分析結果\n\n"
        }
    ]
)
Python 変更後: Opus 4.6 (プレフィル削除、システムプロンプトで代替) 
import anthropic

client = anthropic.Anthropic()

# Opus 4.6 ではプレフィル不可。システムプロンプトで出力形式を指定
message = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=16000,
    thinking={"type": "adaptive"},
    output_config={"effort": "high"},
    system="回答は必ず '## 分析結果' という見出しから開始してください。",
    messages=[
        {
            "role": "user",
            "content": "この論文を分析してください。"
        }
    ]
)
