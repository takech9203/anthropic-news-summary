CLI Tool Update

Claude Code v2.1.146

2026-05-21

v2.1.146

概要

🔄

/simplify を /code-review にリネーム

コマンドの意図をより明確にし、エフォートレベル指定 (low / high) で レビュー深度を制御可能に

🤖

Auto モードの AskUserQuestion 修正

スキルやユーザーが明示的に依存している場合、Auto モードでも質問が抑制されなくなった

🪟

Windows 安定性の大幅改善

PowerShell リグレッション修正、ストロボ表示、NTFS ジャンクション、GNOME Terminal ペーストなど複数修正

🔌

MCP ページネーション修正

resources/list、templates/list、prompts/list で 2 ページ目以降がドロップされる問題を解消

Before / After

Before

  • /simplify - 単一深度のみ
  • Auto モードで全ての AskUserQuestion が抑制
  • MCP ページネーションで 2 ページ目以降消失
  • winget の pwsh で "command line is invalid"

After

  • /code-review [low|high] - 深度制御可能
  • スキル依存時は質問が正しく表示
  • 全ページのアイテムを正しく収集
  • 全インストール方法の pwsh が正常動作

主な変更点

📝 /code-review コマンド

/simplify から改名。オプションでエフォートレベルを指定し、レビューの深さを low / high で制御可能

🧠 Auto モード修正

AskUserQuestion の抑制ロジックを改善。スキルが明示的に依存する場合は抑制しない条件付きロジックを導入

🔌 MCP ページネーション

resources/list、resources/templates/list、prompts/list で 2 ページ目以降のアイテム取得を修正

🪟 PowerShell 修正

winget / Microsoft Store 経由の pwsh で発生していた v2.1.124 リグレッションを解消

🔄 自動アップデーター強化

ネットワーク障害時のリトライ対応。即座に失敗する代わりに一時的な障害をリトライ

⚡ diff パフォーマンス改善

大規模ファイル編集時の diff レンダリングを高速化。大きな差分表示がスムーズに

バグ修正一覧

  • 🔧 WindowsPowerShell ツールの "command line is invalid" エラー修正 (winget/Store 経由の pwsh)
  • 🔧 MCPresources/list、templates/list、prompts/list のページネーションで 2 ページ目以降がドロップ
  • 🔧 Windowsアタッチされたバックグラウンドセッションでフルスクリーンのストロボ表示
  • 🔧 General自動アップデーターのステータスラインにバージョンが表示されない問題
  • 🔧 WindowsNTFS ジャンクションを辿ってメインリポジトリのファイルを削除する問題
  • 🔧 General/background がスキルやカスタムスラッシュコマンドのみのセッションを拒否
  • 🔧 Generalバックグラウンドセッションで許可済みツール権限の再プロンプト
  • 🔧 General/theme のカラーエディターが Esc キーに反応しない
  • 🔧 GeneralAgent SDK ストリーミングセッション終了時のキャッチされない例外
  • 🔧 GeneralforceLoginOrgUUID / forceLoginMethod がサードパーティセッションに未適用
  • 🔧 LinuxGNOME Terminal の右クリック/中クリックペーストが動作しない
  • 🔧 GeneralCLAUDE_CODE_SUBAGENT_MODEL が子プロセスに転送されない

コードサンプル

Bash /code-review コマンドの使用例 
# デフォルトのレビュー深度でコードレビューを実行
/code-review

# 詳細レビュー (再利用性、品質、効率性を網羅的にチェック)
/code-review high

# 軽量レビュー (明らかな問題のみ)
/code-review low
TypeScript MCP ページネーション - 修正後の動作 
// MCP サーバーがページネーションを使用する場合
// v2.1.146 以降、全ページのアイテムが正しく取得される

server.setRequestHandler(ListResourcesRequestSchema, async (request) => {
  const page = request.params?.cursor
    ? parseInt(request.params.cursor) : 0;
  const pageSize = 10;
  const resources = getAllResources();
  const start = page * pageSize;
  const end = start + pageSize;

  return {
    resources: resources.slice(start, end),
    nextCursor: end < resources.length
      ? String(page + 1) : undefined,
  };
});

// Claude Code v2.1.146:
// 全 100 件が正しく取得される
// (v2.1.145 以前は最初の 10 件のみ)
Bash サブエージェントモデルの環境変数設定 
# マルチエージェントセッションでサブエージェントのモデルを指定
export CLAUDE_CODE_SUBAGENT_MODEL="claude-sonnet-4-6"

# v2.1.146 以降、この環境変数が子プロセスに正しく転送される
claude --agent-mode

開発者への影響

🎯 /simplify ユーザーへの影響

ドキュメントやチーム手順で /simplify を参照している場合は /code-review に更新が必要。エフォートレベル指定でより柔軟なレビューが可能に。

🔌 MCP サーバー開発者への影響

ページネーションを使用する MCP サーバーを運用している場合、アップデート後は全アイテムが正しく取得される。以前欠落していたリソースやプロンプトが表示されるようになる。

🪟 Windows 開発者への影響

winget / Microsoft Store 経由で PowerShell をインストールしている場合、v2.1.124 以降のエラーが解消。バックグラウンドセッションのストロボ表示も修正。

必要なアクション

  • 📦 全ユーザー: claude update で v2.1.146 にアップデート
  • 📝 /simplify ユーザー: /code-review に移行。エフォートレベルは任意
  • 🪟 Windows + winget/Store ユーザー: アップデートで PowerShell 問題が解消
  • 🔌 MCP サーバー運用者: ページネーション対応の確認推奨
  • 🤖 マルチエージェント利用者: CLAUDE_CODE_SUBAGENT_MODEL の転送が正常化
