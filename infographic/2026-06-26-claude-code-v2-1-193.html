Claude Code v2.1.193 アップデート

2026-06-26

主要ポイント

🔍

autoMode.classifyAllShell

全シェルコマンドをオートモード分類器に通す新設定。従来のパターンマッチングをバイパスし、すべての Bash/PowerShell コマンドをセキュリティ分類の対象にできる。

📊

OpenTelemetry レスポンスログ

アシスタントレスポンスの OTel ログイベント追加。OTEL_LOG_ASSISTANT_RESPONSES=1 で有効化、エンタープライズ環境の可観測性を向上。

🧹

メモリプレッシャーリーピング

アイドルバックグラウンドシェルの自動解放。メモリプレッシャー検出時にリソースを自動的に最適化し、長時間セッションの安定性を向上。

🔐

MCP 認証自動再接続

401/403 エラー時にヘルパーが自動再実行。MCP サーバーとの認証が切れた場合も自動的に再接続される。

Before / After

Before

  • 任意コード実行パターンのみ分類器を通過
  • OTel にレスポンスログなし
  • バックグラウンドシェルが無制限にメモリ消費
  • MCP 認証切れ時に手動再接続が必要

After

  • 全シェルコマンドを分類可能
  • レスポンスログ出力対応
  • メモリプレッシャー時に自動リーピング
  • 401/403 時に自動再接続

主な機能

⚙️ autoMode.classifyAllShell

すべての Bash/PowerShell コマンドをオートモード分類器にルーティング。セキュリティポリシーの適用範囲を拡大。

👁️ 拒否理由可視化

分類器がコマンドを拒否した理由がトランスクリプト、トースト通知、/permissions に表示。

📊 OTel レスポンスログ

claude_code.assistant_response イベント追加。デフォルトではリダクト、環境変数で有効化。

📁 ファイルパスオートコンプリート

bash モードでファイルパスのライブオートコンプリートが利用可能に。開発効率が向上。

🔔 MCP 認証通知

起動時に MCP サーバーが認証を必要とする場合、/mcp コマンドへの誘導通知を表示。

🧹 メモリプレッシャーリーピング

アイドル状態のバックグラウンドシェルに対してメモリプレッシャー時に自動リソース解放。

コードサンプル

JSON .claude/settings.json 
{
  "autoMode": {
    "classifyAllShell": true
  }
}
Bash 環境変数の設定 
# レスポンスログを明示的に有効化
export OTEL_LOG_ASSISTANT_RESPONSES=1

# レスポンスログを無効化 (プロンプトログのみ維持)
export OTEL_LOG_ASSISTANT_RESPONSES=0

# メモリプレッシャーリーピングを無効化
export CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1

ユースケース

🏢 エンタープライズセキュリティ

autoMode.classifyAllShell を有効にすることで、すべてのシェルコマンドがセキュリティ分類器を通過。正規表現パターンでは捕捉できない潜在的に危険なコマンドも事前に評価され、組織のセキュリティガバナンスが強化される。

📈 可観測性向上

OpenTelemetry レスポンスログにより、AI アシスタントの出力を監査・モニタリング可能に。プロンプトとレスポンスの両方をトレースすることで、AI システムの動作を包括的に把握し、品質改善やコンプライアンス対応に活用できる。

💻 リソース最適化

メモリプレッシャーリーピングにより、長時間稼働するセッションでのメモリ消費が自動最適化。アイドル状態のバックグラウンドシェルが不要なメモリを占有し続ける問題が解消され、開発環境の安定性が向上する。
