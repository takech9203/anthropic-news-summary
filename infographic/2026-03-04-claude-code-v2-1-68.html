Opus 4.6 デフォルトエフォートレベルの変更

2026-03-04

v2.1.68 v2.1.66

概要

🎯

デフォルトが Medium に変更

Max および Team サブスクライバー向けに、Opus 4.6 のデフォルトエフォートレベルが「medium」に変更された。速度と精度のバランスが最適化され、日常的なタスクで効率的に動作する。

⚡

ultrathink キーワードの再導入

複雑なタスクや深い分析が必要な場面で「ultrathink」キーワードを入力することで、次のターンのみ高エフォートモードを即座に有効化できる。

🔄

旧モデルの統一

Opus 4 および Opus 4.1 がファーストパーティ API から削除され、ピン留めユーザーは自動的に Opus 4.6 へ移行される。

🧹

エラーログの削減

v2.1.66 でスプリアスなエラーログ出力が削減され、ログの可読性が向上した。

Before / After

Before

  • デフォルト: High エフォート
  • 日常タスクでも高い計算リソースを消費
  • Opus 4 / 4.1 / 4.6 が混在
  • スプリアスなエラーログが出力

After

  • デフォルト: Medium エフォート
  • 速度と精度の最適バランス
  • Opus 4.6 に統一
  • 不要なエラーログを削減

主な変更点

⚙️ エフォートレベル変更

Opus 4.6 のデフォルトが「medium」に。/model コマンドでいつでも変更可能。多くの日常タスクで速度が向上する。

🧠 ultrathink 再導入

「ultrathink」キーワードで次のターンのみ高エフォートを有効化。複雑な問題解決や大規模リファクタリングに最適。

🗑️ 旧モデル削除

Opus 4 / Opus 4.1 がファーストパーティ API から削除。ピン留めユーザーは自動的に Opus 4.6 へ移行される。

📋 ログ改善

v2.1.66 でスプリアスなエラーログを削減。ログの可読性が向上し、デバッグ時のノイズが減少した。

エフォートレベルシステム

3 段階のエフォートレベルで計算リソースと応答速度を制御する。

レベル 説明 用途
Low 最小限の計算リソースで高速応答 単純な質問、軽微な修正
Medium 新デフォルト 速度と精度のバランスが最適 一般的な開発タスク全般
High 最大限の計算リソースで深い分析 複雑な問題解決、大規模リファクタリング

変更方法: /model コマンドでデフォルトを変更、または ultrathink キーワードで一時的に High を有効化

開発者への影響とユースケース

💻 日常的な開発タスク

Medium エフォートがデフォルトとなったことで、コードレビュー、軽微なリファクタリング、ドキュメント作成などの日常タスクが高速化される。特に Max および Team プランのサブスクライバーが恩恵を受ける。

🔬 複雑な問題解決

大規模リファクタリングや難解なバグ調査には「ultrathink」キーワードを入力するだけで、そのターンのみ High エフォートを有効化できる。柔軟な使い分けが可能。

🔄 モデル移行の確認

Opus 4 / Opus 4.1 をピン留めしていたユーザーは自動的に Opus 4.6 へ移行される。手動操作は不要だが、タスク実行結果とワークフローへの影響を確認することを推奨。

必要なアクション

  • 1 エフォートレベルの確認: デフォルトが「medium」に変更されたため、常に高エフォートが必要なワークフローでは /model コマンドで設定を調整する
  • 2 ultrathink の活用: 複雑なタスクでは「ultrathink」キーワードを入力して、そのターンのみ高エフォートモードを有効化する
  • 3 モデル設定の確認: Opus 4 / Opus 4.1 をピン留めしていた場合、自動的に Opus 4.6 に移行されているため、動作に問題がないか確認する
