MCP サーバー管理の改善

/mcp コマンドがツールカウントを表示し、workspace が予約名に。再接続時のノイズも削減されました。

プラグイン ZIP アーカイブ対応

--channels コンソール認証対応

API キーによるコンソール認証環境でも --channels が利用可能に。マネージド設定で channelsEnabled: true が必要。

🔒

OTEL 環境変数のサブプロセス分離

Bash、フック、MCP、LSP のサブプロセスが OTEL_* 環境変数を継承しなくなり、テレメトリの混在を防止。

⚡

並列シェル呼び出しの安定化

読み取り専用コマンドの失敗が兄弟コマンドをキャンセルしなくなり、並列実行が安定動作。

機能詳細

🔌 MCP 関連 - 4 件

📊 ツールカウント表示

/mcp コマンドが接続済みサーバーのツール数を表示。ツール数 0 のサーバーにはフラグが付き、接続問題の早期発見が可能に。

🚫 workspace 予約名

MCP サーバー名 "workspace" が予約名に。既存の同名サーバーはスキップされ警告が表示されます。

🔇 再接続時通知抑制

MCP サーバー再接続時に完全なツール名リストが表示されなくなり、サーバープレフィックスごとの要約表示に変更。

🖼️ 画像欠落修正

MCP ツール結果で構造化コンテンツとコンテンツブロックの両方を返した場合に画像がドロップされる問題を修正。

📦 プラグインシステム - 3 件

🗜️ ZIP アーカイブ対応

--plugin-dir がディレクトリに加えて .zip プラグインアーカイブを受け付け。配布とバージョン管理が容易に。

🔧 Components パネル修正

--plugin-dir で読み込んだプラグインに "Marketplace 'inline' not found" と表示される問題を修正。

🔄 npm 更新検出修正

/plugin update で npm ソースのプラグインの新バージョンが検出されない問題を修正。

🖥️ UI/UX 改善 - 8 件

🎨 /color ランダム選択

引数なしで /color を実行するとランダムなセッションカラーが選択されるように。

📋 /model ピッカー改善

重複する Opus 4.7 エントリが統合され、現在の Opus が "Opus" と表示されるように修正。

👁️ フォーカスモード修正

新しいプロンプト送信時に前のレスポンスが一瞬暗くなる問題を修正。

🐱 Kitty 通知修正

/exit 時に "4;0;" というデスクトップ通知が表示される問題を修正。

🔗 長い URL クリック修正

フルスクリーンモードで折り返された長い URL が各行で個別にクリック可能に。

🏷️ リンクラベル表示改善

OSC 8 非対応ターミナルでリンクが label (url) 形式で表示されるように改善。

⏳ プログレスインジケーター

ツール呼び出し間で OSC 9;4 プログレスインジケーターがちらつく問題を修正。

⌨️ vim モード修正

NORMAL モードで Space キーがカーソルを右に移動するよう修正。標準 vi/vim の動作と一致。

🤖 Auto モード / SDK - 5 件

🔗 --channels コンソール認証

API キーによるコンソール認証で --channels が動作。マネージド設定で channelsEnabled: true の追加が必要。

サブプロセスが OTEL_* 環境変数を継承しなくなり、テレメトリデータの意図しない送信先を防止。

SDK ホストで "Always allow" 選択時に .claude/settings.local.json に書き込まれるように。

分類器がアクション評価不可時にリトライ、/compact、--debug のヒントを表示。

🔁 サマリー発火制限

静的なサブエージェントトランスクリプトでサマリーが繰り返し発火する問題を修正。アイドル時のトークンコストを制限。

移行ガイド / 破壊的変更

MCP サーバー名 "workspace" の予約

Before

MCP サーバー名として "workspace" が使用可能

After

"workspace" は予約名としてスキップされる。別の名前への変更が必要。

EnterWorktree のブランチ作成元

Before

EnterWorktree が origin/<default-branch> から新しいブランチを作成 (未プッシュコミットが消失)

After

ローカル HEAD から新しいブランチを作成。ドキュメント通りの動作に修正。

--channels のコンソール組織設定

Before

--channels はコンソール認証 (API キー) 環境で利用不可

After

API キー環境で利用可能。マネージド設定に channelsEnabled: true を追加。
