Claude Code v2.1.153
バックグラウンドセッション安定化と MCP 修正を含む大型アップデート

2026-05-28

主要ポイント

🔄

バックグラウンドセッション安定化

11 件のバグ修正により、/bg 応答継続、/btw ショートカット、クリップボード、IME、tmux 対応など包括的に改善。バックグラウンドセッションの信頼性が大幅に向上

🔌

MCP サーバー 5 件の修正

ステートフル MCP サーバーの再接続ループ修正、認証トークンのルーティング修正 (セキュリティ)、サブエージェント MCP 設定の問題を解消

🪟

Windows 対応改善

PowerShell インストーラーの誤報告修正、アップデートロールバック機能、VS Code プロセスの正常終了を実現

🔑

セキュリティ修正: OAuth トークン誤送信

カスタム API ゲートウェイにユーザーの Anthropic OAuth 認証情報が誤送信されるリグレッションを修正。該当ユーザーは速やかなアップデートが推奨される

⚙️

/model コマンドの動作変更

選択したモデルが新規セッションのデフォルトとして保存される動作に変更。キーバインド modelPicker:setAsDefault を modelPicker:thisSessionOnly へ移行が必要

新機能

📦 skipLfs オプション

github/git プラグインのマーケットプレイスソースに skipLfs オプションを追加。クローンおよびアップデート時の Git LFS ダウンロードをスキップ可能に

📢 npm 更新通知

自動更新ができない npm グローバルインストール環境で、一度だけ通知を表示。/doctor で修正方法を確認可能

📟 ステータスライン環境変数

ステータスラインコマンドに COLUMNS と LINES 環境変数を渡し、ターミナル幅に応じた出力サイズ調整が可能に

🔍 agents オートコンプリート

ディスパッチ入力でネイティブスラッシュコマンドとバンドルスキルを候補に表示。プロジェクトスキルだけでなく全スキルが候補に

📊 PR カラム表示改善

claude agents の PR カラムが単一 PR の場合は「PR #N」、複数の場合は「N PRs」と表示されるよう改善

⚙️ /model 動作変更

選択したモデルが新規セッションのデフォルトとして保存される動作に変更。現在のセッションのみ変更する場合はピッカーで「s」を押す

🩺 /doctor 更新情報

前回のアップデート試行結果を表示。更新に失敗した場合の原因特定が容易に

🍎 macOS バックグラウンド改善

Privacy & Security に "Claude Code" として表示され、アップグレード後もパーミッション付与が維持される

🔄 バックグラウンドセッション (11 件)

  • 💤 /bg コマンド実行時に応答をドロップせずバックグラウンドで継続するよう修正
  • ⌨️ /btw ショートカットがタスク実行中に応答しなくなる問題を修正
  • 📋 $CLAUDE_JOB_DIR への一時ファイル書き込み時の "sensitive file" パーミッションプロンプトを修正

  • 🎨 256 色ターミナルでの背景色にじみを修正

  • 👻 Remote Control 使用時のゾンビセッションエントリの残留を修正

  • 🔁 ステートフル MCP サーバーの再接続ループ修正 (v2.1.147 リグレッション)
  • 🔑 カスタム API ゲートウェイへの OAuth 認証トークン誤送信を修正 (セキュリティ)
  • 🤖 サブエージェントの MCP 設定が --strict-mcp-config やポリシーを無視する問題を修正

/model コマンドの動作変更

選択したモデルが新規セッションのデフォルトとして保存されるように変更。IDE と同じ動作に統一された。

変更前 (v2.1.152 以前)

  • ✗ /model で選択 = 現在のセッションのみ
  • ✗ デフォルト保存には「d」キーが必要
  • ✗ キーバインド名: modelPicker:setAsDefault

変更後 (v2.1.153)

  • ✓ /model で選択 = デフォルトとして保存
  • ✓ 現在のセッションのみは「s」キー
  • ✓ キーバインド名: modelPicker:thisSessionOnly

移行ガイド: キーバインド変更

modelPicker:setAsDefault をカスタマイズしていた場合、以下の変更が必要です。

JSON keybindings.json - 変更前 
{
  "modelPicker:setAsDefault": "d"
}
JSON keybindings.json - 変更後 
{
  "modelPicker:thisSessionOnly": "s"
}

BASH v2.1.153 へのアップデート 
claude update

開発者への影響とユースケース

バックグラウンドセッション利用者

claude --bg や /bg を活用している開発者にとって、応答継続・クリップボード・IME・tmux 対応など 11 件の修正により、安定したバックグラウンド作業が可能に。特に macOS では Privacy & Security でのパーミッション管理が改善された。

エンタープライズ・カスタム API ゲートウェイ利用者

OAuth 認証トークンがサードパーティのゲートウェイに誤送信されるセキュリティ問題が修正された。カスタム API ゲートウェイを使用している組織は、速やかに v2.1.153 へのアップデートが推奨される。

MCP サーバー開発者

ステートフル MCP サーバーの再接続ループ (v2.1.147 リグレッション) が修正され、サブエージェントの MCP 設定が --strict-mcp-config やエンタープライズポリシーを正しく尊重するようになった。

Windows 環境の開発者

PowerShell インストーラーの誤報告修正、アップデート失敗時のロールバック機能追加、VS Code プロセスの正常終了により、Windows 環境での開発体験が大幅に向上した。

大規模セッションを運用する開発者

セッション再開時の数 GB レベルのメモリ消費が解消され、多数のセッションを持つマシンでもスムーズな操作が可能に。古いデーモンの問題やハング修正も含まれる。

/model コマンドをカスタマイズしていた開発者

キーバインド名が変更されたため、keybindings.json の更新が必要。modelPicker:setAsDefault を modelPicker:thisSessionOnly にリネームすること。
