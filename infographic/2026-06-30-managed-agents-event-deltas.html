✨ 新機能詳細

⚡ イベントデルタ (Event Deltas)

GET /v1/sessions/{id}/events/stream エンドポイントで event_deltas[] クエリパラメータを指定し、event_start と event_delta イベントによりリアルタイムでテキストをプレビューできる。

🔄 逆方向ページネーション

GET /v1/sessions が next_page に加えて prev_page カーソルを返すようになった。page パラメータとして渡すことで前のページに戻れる。

🎛️ セッション設定オーバーライド

type: "agent_with_overrides" を指定することで、モデル、システムプロンプト、ツール、MCP サーバー、スキルを単一セッションに対してのみ変更できる。A/B テストやデバッグに有用。

🔐 Vault injection_location

環境変数クレデンシャルの値をヘッダー (headers)、ボディ (body)、またはその両方 (both) にインジェクションするかを制御できる。

🔔 拡張 Webhook イベント

Agent イベント (バージョン公開)、Deployment イベント (一時停止、再開)、Deployment run イベント (スケジュール実行の成功、失敗) に対応。ポーリング不要のイベント駆動型アーキテクチャを実現。

🏗️ 技術的な詳細

イベントデルタのストリーミングフロー

Server-Sent Events (SSE) プロトコルを使用して配信される。クライアントは event_deltas[] パラメータで対象イベントタイプを指定し、ストリームに接続する。

1. event_start

イベントの開始を通知。クライアントはレンダリング準備を開始する。

2. event_delta (複数回)

テキストの差分をリアルタイムで配信。各デルタは前回からの追加テキストを含む。

3. agent.message (最終)

完全なメッセージイベント。最終的な応答テキスト全体を含む。

セッション設定オーバーライドの構造

オーバーライド可能なフィールド: model、system_prompt、tools、mcp_servers、skills

Vault injection_location のオプション

headers: リクエストヘッダーにのみインジェクション / body: リクエストボディにのみインジェクション / both: ヘッダーとボディの両方にインジェクション

📡 デプロイメント監視

拡張 Webhook イベントにより、ポーリングなしでエージェントやデプロイメントのライフサイクルイベントを監視できる。

📝 まとめ

今回の Managed Agents アップデートは、プラットフォームの成熟度を大きく向上させるものである。イベントデルタによるリアルタイムストリーミングは、エージェントベースのアプリケーションにおけるユーザー体験を劇的に改善する。セッション設定オーバーライドにより、本番エージェント定義を変更することなく柔軟な実験が可能になる。拡張 Webhook イベントはポーリングの必要性を排除し、よりリアクティブなシステム設計を促進する。これらの機能を組み合わせることで、開発者はより洗練されたエージェントアプリケーションを構築できるようになる。
