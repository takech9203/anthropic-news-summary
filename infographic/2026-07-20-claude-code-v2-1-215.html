Claude Code v2.1.215 — /verify と /code-review の自動実行廃止

2026-07-20

概要

🎯

ユーザー主導の明示的呼び出しに変更

Claude が /verify および /code-review スキルを自発的に実行しなくなった。ユーザーが明示的にスラッシュコマンドで呼び出した場合のみ実行される。

💰

不要なトークン消費の削減

コストの高い検証・レビュー操作が自動実行されなくなり、ユーザーが必要と判断した場合のみコストが発生する。

🔄

予測可能な動作の実現

Claude の動作がより予測可能になり、自動的に追加されていた検証やレビューのステップが排除される。

Before / After

Before

  • Claude が自発的に /verify を実行
  • Claude が自発的に /code-review を実行
  • 意図しないタイミングでトークン消費
  • ワークフローの中断・遅延

After

  • ユーザーが /verify で明示的に呼び出し
  • ユーザーが /code-review で明示的に呼び出し
  • 必要な時だけコストが発生
  • 中断のないスムーズなワークフロー

変更の意義

🎮 ユーザーコントロールの強化

コストの高い操作の実行タイミングをユーザーが完全にコントロールできるようになった。不要なタイミングでの自動実行による作業中断がなくなる。

📉 コスト効率の改善

/verify や /code-review は比較的多くのトークンを消費するスキル。自動実行されなくなることで全体的なトークン消費が最適化される。

🔮 予測可能な動作

ユーザーが期待する通りのワークフローを維持できる。セッションの流れがより明確になり、意図しない追加ステップがなくなる。

使用方法

Shell スラッシュコマンドの使用例 
# コードレビューを実行したい場合
> /code-review

# 検証を実行したい場合
> /verify

# 自動化スクリプトでの対応例
claude "変更を実装してください。完了したら /verify を実行してください"
Shell アップデート方法 
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# Homebrew でのアップデート
brew upgrade claude-code

# バージョン確認
claude --version

影響を受けるユースケース

🧑‍💻 日常的な開発ワークフロー

コード変更後に Claude が自動的にレビューを実行していた場合、今後は明示的に /code-review を呼び出す必要がある。必要な時だけレビューを実行することで、作業の流れを中断せずに済む。

🤖 自動化パイプライン

Claude Code をスクリプトやパイプラインで使用している場合、検証やレビューが必要なタイミングでプロンプトに明示的にコマンドを含める必要がある。

💡 コスト最適化

不要なトークン消費が自動的に削減される。特に長時間のセッションや大規模なコードベースで作業する場合、コスト効率が大幅に改善される。
