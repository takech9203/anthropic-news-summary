Claude Code v2.1.107-v2.1.108 リリース

2026-04-14

v2.1.105 → v2.1.107 → v2.1.108

概要

🗄️

プロンプトキャッシュ TTL 制御

ENABLE_PROMPT_CACHING_1H で全バックエンド統一の 1 時間 TTL、FORCE_PROMPT_CACHING_5M で 5 分 TTL を強制設定。キャッシュ運用がより柔軟に

📋

セッションリキャップ機能

セッションに戻った際にコンテキスト要約を提供する /recap コマンド。/config で設定可能、テレメトリ無効環境では CLAUDE_CODE_ENABLE_AWAY_SUMMARY で有効化

🔍

スラッシュコマンド Skill ツール経由自動発見

モデルが Skill ツールを通じて /init、/review、/security-review などの組み込みコマンドを自動発見・実行。自然言語リクエストからの最適コマンド選択が可能に

💾

メモリフットプリント削減

言語グラマーをオンデマンドで読み込む方式に変更。セッション開始時の一括読み込みを廃止し、必要な言語のみ読み込むことでメモリ使用量を削減

🔄

セッション復元 14 件のバグ修正

--resume と --teleport に関する 6 件を含む 14 件のバグ修正。セッション名・カラー保持、前提条件エラー表示、トランケーション防止など、復元の信頼性が包括的に改善

DISABLE_PROMPT_CACHING* 環境変数でプロンプトキャッシュが無効化されている場合、起動時に警告を表示

改善 5 件

💡

Thinking ヒントの早期表示 (v2.1.107)

長時間の操作中に Thinking ヒントがより早く表示されるようになりました

⚠️

/model のモデル切り替え警告

会話途中でモデルを切り替える前に警告を表示。次のレスポンスで会話履歴全体をキャッシュなしで再読み込みするため、コスト増加の可能性を事前に通知

📂

/resume ピッカーのディレクトリフィルタリング

デフォルトで現在のディレクトリのセッションのみ表示。Ctrl+A で全プロジェクトのセッションに切り替え可能

📨

エラーメッセージの分類改善

サーバーレートリミットとプラン使用量リミットを区別。5xx/529 エラーでは status.claude.com へのリンクを表示。未知のコマンドでは最も近いコマンドを提案

💾

メモリフットプリントの削減

ファイルの読み取り、編集、シンタックスハイライトで使用される言語グラマーがオンデマンドで読み込まれるようになり、メモリ使用量が削減

バグ修正 14 件

🔄 セッション・テレポート修正 6

  • /login コードプロンプトでペーストが機能しない問題を修正 (v2.1.105 リグレッション)
  • claude --resume <session-id> 使用時にセッション名とカラーが失われる問題を修正
  • --teleport 後にターミナルエスケープコードがプロンプト入力に文字化けとして表示される問題を修正
  • --teleport と --resume <id> の前提条件エラーがサイレントに終了する問題を修正
  • --resume のセッショントランケーション問題を修正 (自己参照メッセージが含まれる場合)
  • Remote Control セッションタイトルが 3 番目のメッセージ以降に自動生成タイトルで上書きされる問題を修正

🗄️ キャッシュ・テレメトリ修正 2

  • DISABLE_TELEMETRY 設定時にサブスクライバーが 1 時間の TTL ではなく 5 分の TTL にフォールバックする問題を修正
  • トランスクリプト書き込みエラーがサイレントにドロップされる代わりにログに記録されるように修正

🔧 ツール・環境修正 4

  • Agent ツールが安全性分類器のトランスクリプト超過時に自動モードで権限を要求する問題を修正
  • CLAUDE_ENV_FILE が # コメント行で終わる場合に Bash ツールが出力を生成しない問題を修正
  • /feedback の送信失敗後に説明を編集せずに再送信できるように修正
  • language 設定時にレスポンスからダイアクリティカルマークが削除される問題を修正

📌 その他修正 2

  • 最初のメッセージが短い挨拶の場合にセッションタイトルにプレースホルダーテキストが表示される問題を修正
  • 別プロジェクトから実行時にポリシー管理プラグインが自動更新されない問題を修正

コードサンプル

Bash プロンプトキャッシュ TTL の制御 
# 1 時間のプロンプトキャッシュ TTL を有効化
# API キー、Bedrock、Vertex、Foundry のすべてで有効
export ENABLE_PROMPT_CACHING_1H=1
claude

# 5 分間の TTL を強制 (デバッグ / テスト用)
export FORCE_PROMPT_CACHING_5M=1
claude

# キャッシュが無効化されている場合は起動時に警告が表示される
# 意図的に無効化する場合
export DISABLE_PROMPT_CACHING=1
claude
# Warning: Prompt caching is disabled via DISABLE_PROMPT_CACHING
Bash リキャップ機能の使用 
# セッション内で手動リキャップを実行
# Claude Code のプロンプトで以下を入力
> /recap
# セッションのコンテキスト要約が表示される

# /config からリキャップ機能を設定
> /config
# リキャップ機能のオン/オフを切り替え

# テレメトリ無効化環境で強制有効化
CLAUDE_CODE_ENABLE_AWAY_SUMMARY=1 claude
Bash アップデートコマンド 
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# Homebrew でのアップデート
brew upgrade claude-code

# 現在のバージョン確認
claude --version

開発者への影響

👥 全ての Claude Code ユーザー

14 件のバグ修正と 5 件の改善により、全体的な安定性と使用体験が向上。言語グラマーのオンデマンド読み込みによるメモリフットプリント削減の恩恵を受けます

☁️ API キー / Bedrock / Vertex / Foundry ユーザー

ENABLE_PROMPT_CACHING_1H により全バックエンドで統一的にプロンプトキャッシュの 1 時間 TTL を有効化可能。従来の ENABLE_PROMPT_CACHING_1H_BEDROCK は非推奨になりました

🔒 テレメトリ無効化ユーザー

DISABLE_TELEMETRY 設定時のキャッシュ TTL フォールバック問題が修正。リキャップ機能を利用するには CLAUDE_CODE_ENABLE_AWAY_SUMMARY を設定してください

🔄 セッション復元を頻繁に使用するユーザー

--resume と --teleport に関する 6 件のバグ修正により、セッション復元の信頼性が大幅に改善

🌍 多言語環境のユーザー

language 設定使用時にダイアクリティカルマークが削除される問題が修正。アクセント、ウムラウト、セディーユなどを含む言語を使用するユーザーに影響

🖥️ Remote Control ユーザー

Web UI で設定したセッションタイトルが自動生成タイトルで上書きされなくなりました
