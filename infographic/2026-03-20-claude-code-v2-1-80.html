概要

📡

MCP エコシステムの拡張

--channels (research preview) により、MCP サーバーからセッションへのプッシュメッセージが可能になりました。リアルタイムイベント受信や非同期ワークフローの構築が期待できます。

📊

レートリミットの可視化

rate_limits フィールドにより、Claude.ai の使用状況をステータスラインでリアルタイムに監視できます。5 時間/7 日間ウィンドウの使用率とリセット時刻を確認可能です。

💾

大規模リポジトリのパフォーマンス最適化

25 万ファイル規模のリポジトリで起動時メモリ使用量が約 80MB 削減されました。v2.1.79 での 18MB 削減に続く大幅な最適化です。

🔧

広範なバグ修正

--resume のパラレルツール結果欠落、ボイスモードの WebSocket 障害、API プロキシ経由のストリーミングエラーなど 7 件のバグが修正されています。

新機能

📡 --channels

MCP サーバーがセッションにメッセージをプッシュできる新機能。サーバー側からのプッシュ通知が可能になり、リアルタイムイベントの受信や非同期ワークフローの構築が期待できます。

Research Preview

📊 rate_limits フィールド

ステータスラインスクリプト向けに Claude.ai のレートリミット使用状況を表示。5 時間ウィンドウと 7 日間ウィンドウの used_percentage と resets_at を確認できます。

New

🔌 source: settings

settings.json 内にプラグインエントリをインラインで宣言できるプラグインマーケットプレースソース。外部ファイルを参照せずにプラグイン設定を一元管理できます。

New

💡 CLI ツール使用検出

ファイルパターンマッチングに加え、CLI ツールの使用状況を検出してプラグインの推奨が表示されるようになりました。

New

⚙️ effort フロントマター

スキルやスラッシュコマンドに effort フロントマターを指定することで、呼び出し時にモデルのエフォートレベルを上書きできるようになりました。

New

バグ修正

セッション・データ関連
🔄

--resume のパラレルツール結果欠落を修正

パラレルツールコールを含むセッションを再開した際、全ての tool_use / tool_result ペアが正しく復元されるようになりました。従来は [Tool result missing] プレースホルダが表示される問題がありました。

⚙️

管理設定の起動時適用を修正

enabledPlugins、permissions.defaultMode、ポリシー設定の環境変数が、remote-settings.json がキャッシュされている場合に起動時に適用されない問題を修正しました。

通信・API 関連
🎤

ボイスモード WebSocket 障害を修正

Cloudflare のボット検出が非ブラウザ TLS フィンガープリントをブロックすることで発生していた WebSocket 接続失敗を修正しました。

📡

ストリーミングの 400 エラーを修正

API プロキシ、Bedrock、Vertex 経由でファイングレインドツールストリーミングを使用した際の 400 エラーを修正しました。

UI 関連
🖥️

/remote-control の不要な表示を修正

ゲートウェイやサードパーティプロバイダーのデプロイメントで機能しない /remote-control が表示される問題を修正しました。

📑

/sandbox のタブ切替を修正

Tab キーや矢印キーでのタブ切替が応答しない問題を修正しました。

📏

/effort の表示を修正

auto が実際に解決される値を表示するようになり、ステータスバーインジケーターと一致するようになりました。

改善

💾 メモリ使用量削減

大規模リポジトリでの起動時メモリ使用量が大幅に削減。25 万ファイル規模のリポジトリで約 80MB の節約を確認。v2.1.79 での 18MB 削減に続く最適化です。

🔍 @ ファイル補完の応答性向上

大規模 Git リポジトリでの @ ファイルオートコンプリートの応答性が改善されました。

🔐 /permissions のナビゲーション改善

Tab キーと矢印キーでリスト内からタブを切替できるようになりました。

📋 バックグラウンドタスクパネル改善

左矢印キーでリストビューからパネルを閉じられるようになりました。

💡 プラグインインストール簡素化

2 ステップのフローから単一の /plugin install コマンドに簡素化されました。

npm update -g @anthropic-ai/claude-code

# 現在のバージョン確認
claude --version

claude --channels

# セッション再開 (パラレルツール結果が正しく復元される)
claude --resume

# プラグインのインストール (簡素化されたフロー)
/plugin install my-plugin

開発者への影響

--resume でセッション再開時に [Tool result missing] が表示される場合

パラレルツール結果の復元が修正されています。早急なアップデートを推奨します。

API プロキシ、Bedrock、Vertex 経由でストリーミングエラーが発生する場合

ファイングレインドツールストリーミングの 400 エラーが修正されています。

大規模リポジトリで起動が遅い場合

メモリ使用量の大幅削減 (約 80MB) により起動パフォーマンスが向上しています。

ボイスモードの接続が不安定な場合

Cloudflare ボット検出に起因する WebSocket 障害が修正されています。

管理設定が起動時に反映されない場合

キャッシュされた remote-settings.json の問題が修正されています。

MCP サーバーを開発・運用している場合

--channels (research preview) により、サーバーからセッションへのプッシュメッセージが可能になります。今後の API 変更の可能性に留意してください。

スキルやスラッシュコマンドを作成している場合

effort フロントマターによりスキル単位のモデルエフォートレベル制御が可能になりました。

レートリミットを意識して作業している場合

rate_limits フィールドにより使用状況をステータスラインでリアルタイムに監視できます。
