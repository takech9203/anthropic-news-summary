API / プラットフォーム

📨 Messages API が Amazon Bedrock でリサーチプレビューとして利用可能に

📅 2026-04-07

概要

📨

新エンドポイント /anthropic/v1/messages

Anthropic のファーストパーティ Claude API と同一のリクエスト形式を AWS マネージドインフラ上で利用可能。アダプターコード不要でクロスプラットフォーム開発を実現。

☁️

AWS マネージドインフラ、オペレーターアクセスゼロ

Anthropic スタッフによるインフラへのアクセスが一切なく、全てが AWS セキュリティ境界内で動作。セキュリティ要件の厳しいエンタープライズ環境に最適。

🔧

AnthropicBedrockMantle - 新 SDK クライアント

従来の AnthropicBedrock クライアントに代わる新しい SDK クライアント。クライアント初期化の変更だけで移行可能、リクエストボディやレスポンス処理はそのまま流用可能。

🌐

us-east-1 リージョンで利用可能

リサーチプレビューとして US East (N. Virginia) リージョンのみで提供。専用 AWS アカウントが必要で、アクセスは申請制。

🆕 新統合の主要機能

📨 同一リクエスト形式

Anthropic API と同じリクエストシェイプを使用。messages、max_tokens 等のパラメータをそのまま流用可能。

🔒 オペレーターアクセスゼロ

Anthropic スタッフがインフラにアクセスできないゼロアクセス設計。全てが AWS セキュリティ境界内で完結。

📡 SSE ストリーミング

AWS 独自イベントストリームではなく、標準的な Server-Sent Events を使用。既存のストリーミング処理コードをそのまま利用可能。

💾 プロンプトキャッシング

繰り返しのプロンプトをキャッシュしてコストとレイテンシーを削減。長いシステムプロンプトの効率化に有効。

🧠 拡張思考

Extended Thinking 機能をサポート。複雑な推論タスクでモデルの思考プロセスを可視化し、出力品質を向上。

📋 構造化出力

Structured Outputs に対応。JSON スキーマに準拠した出力を確実に取得でき、アプリケーション統合が容易に。

💻 コード例

🔶 従来の Bedrock 統合 (InvokeModel API)

from anthropic import AnthropicBedrock

# 従来の Bedrock クライアント (InvokeModel API を使用)
client = AnthropicBedrock(aws_region="us-west-2")

message = client.messages.create(
    model="global.anthropic.claude-opus-4-6-v1",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello, Claude"}],
)
print(message.content[0].text)

🟢 新しい Claude in Amazon Bedrock (Messages API)

from anthropic import AnthropicBedrockMantle

# 新しい Bedrock Mantle クライアント (Messages API を直接使用)
client = AnthropicBedrockMantle(aws_region="us-east-1")

message = client.messages.create(
    model="CLAUDE_MODEL_ID",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello, Claude"}],
)
print(message.content[0].text)

🌐 curl での直接リクエスト

curl https://bedrock-mantle.us-east-1.api.aws/anthropic/v1/messages \
  --aws-sigv4 "aws:amz:us-east-1:bedrock-mantle" \
  --user "$AWS_ACCESS_KEY_ID:$AWS_SECRET_ACCESS_KEY" \
  -H "x-amz-security-token: $AWS_SESSION_TOKEN" \
  -H "content-type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "CLAUDE_MODEL_ID",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "Hello, Claude"}
    ]
  }'

📋 サポート状況

✅ サポートされる機能

  • ✅ Messages API (/v1/messages)
  • ✅ プロンプトキャッシング
  • ✅ 拡張思考 (Extended Thinking)
  • ✅ ツール使用 (クライアント定義ツール)
  • ✅ 引用 (Citations)
  • ✅ 構造化出力 (Structured Outputs)
  • ✅ リージョン内推論

❌ サポートされない機能

  • ❌ Anthropic 定義ツール (Web Search、Web Fetch、Remote MCP、Memory、Files API、Computer Use、Skills、Code Execution)
  • ❌ Agent API
  • ❌ Message Batches API
  • ❌ /v1/users エンドポイント

🔑 認証方式

推奨

Bedrock サービスロール

AWS マネージドキーを使用した最も安全な長期アクセス方式。

任意

IAM 引き受けロール

ID フェデレーション経由のアクセス。最大 12 時間セッション。

非推奨

Bearer トークン

IAM ロールなしの短期アクセス。最大 12 時間、非推奨。

🎯 ユースケース

🔄 クロスプラットフォームコードの移植性

Anthropic API と同一のリクエスト形式を使用するため、Anthropic 直接 API 向けに書かれたコードを最小限の変更で Bedrock 上で実行可能。マルチクラウド戦略を採用する企業にとって、ベンダーロックインを回避しつつ同一コードベースを維持できる。

🔒 エンタープライズセキュリティ

オペレーターアクセスゼロの AWS セキュリティ境界内で Claude を利用可能。Anthropic スタッフがインフラにアクセスできないため、金融、医療、政府機関などセキュリティ要件の厳しい業界に最適。全推論データが AWS ストレージ内に保持される。

🚀 レガシー Bedrock からの移行

従来の InvokeModel API / AnthropicBedrock クライアントから AnthropicBedrockMantle への移行は、クライアント初期化の変更が中心。リクエストボディの構造、レスポンスの処理ロジック、ストリーミング処理は変更不要で、段階的な移行が可能。

🔄 移行ガイド

1

アクセスの申請

Anthropic アカウントエグゼクティブに連絡してリサーチプレビューへのアクセスをリクエスト。

2

専用 AWS アカウントの準備

us-east-1 リージョンに専用の AWS アカウントを作成。アイソレーションのため専用アカウントが必要。

3

許可リスト登録の待機

アカウント ID を提出後、Bedrock Marketplace チームによる許可リスト登録を待機 (通常 24 時間以内)。

4

SDK の更新

Anthropic SDK の Bedrock 対応パッケージをインストールし、AnthropicBedrock を AnthropicBedrockMantle に変更。

📝 まとめ

🎯

Anthropic API 互換の Bedrock 統合

Messages API の Amazon Bedrock リサーチプレビューは、Anthropic のファーストパーティ API と同一のリクエスト形式を AWS マネージドインフラ上で利用可能にする重要なアップデートです。新エンドポイント /anthropic/v1/messages と AnthropicBedrockMantle クライアントにより、開発者はアダプターコードなしで Claude API のコードを Bedrock 上で直接実行できるようになります。オペレーターアクセスゼロの AWS セキュリティ境界内で動作するため、セキュリティ要件の厳しいエンタープライズ環境にも適しています。
