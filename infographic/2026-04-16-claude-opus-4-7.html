Claude Opus 4.7 リリース

2026-04-16

高解像度画像対応・タスクバジェット・新エフォートレベルを搭載、Opus 4.6 からの破壊的変更あり

概要

🖼️

高解像度画像サポート

最大解像度が 1568px / 1.15MP から 2576px / 3.75MP へ約 3.3 倍に向上。実ピクセルとの 1:1 座標マッピングで、スケールファクターの計算が不要に。

⚡

新しい xhigh エフォートレベル

high と max の中間に位置する新レベル。コーディングやエージェントユースケースに推奨。知能重視のタスクには最低 high 以上の使用が推奨。

📊

タスクバジェット機能 (ベータ)

エージェントループ全体 (思考 + ツール呼び出し + 出力) にトークン目標を設定可能。最小 20k トークンから設定でき、ハードキャップではなくアドバイザリーな目標値。

💰

価格据え置き

入力 $5 / 出力 $25 (100 万トークンあたり) で Opus 4.6 と同一価格を維持。性能向上とコスト据え置きの両立。

破壊的変更: Opus 4.6 vs 4.7

Before (Opus 4.6)

  • Extended Thinking: budget_tokens で予算指定可能
  • サンプリング: temperature, top_p, top_k 設定可能
  • Thinking 表示: デフォルトで表示
  • トークナイザー: 従来版

After (Opus 4.7)

  • Adaptive Thinking のみ: budget_tokens は 400 エラー
  • パラメータ廃止: 非デフォルト値で 400 エラー
  • Thinking 非表示: display: "summarized" が必要
  • 新トークナイザー: 最大 35% 増加

新機能

🖼️

高解像度画像サポート

Claude モデルとして初めてハイレゾ画像に対応。最大 2576px / 3.75MP をサポートし、1:1 ピクセルマッピングによる正確な座標検出が可能。低レベルの知覚タスクとバウンディングボックス検出が大幅に改善。

⚡

xhigh エフォートレベル NEW

high と max の中間に位置する新レベル。コーディングやエージェントユースケースに最適化されており、知能と速度のバランスを追求。output_config で指定可能。

📊

タスクバジェット BETA

エージェントループ全体のトークン目標を設定。モデルが残りバジェットを参照しながら動作。task-budgets-2026-03-13 ベータヘッダーが必要。最小 20k トークン。max_tokens とは異なるアドバイザリー値。

📄

ナレッジワーク

.docx のレッドライン、.pptx の編集、チャート/図表の分析が向上

💾

メモリ

ファイルシステムベースのメモリ、スクラッチパッド、構造化メモリストアの活用が向上

👁️

動作の変化

📏

より文字通りの指示追従: プロンプトの指示をより正確に、文字通り解釈して実行するようになりました。

📐

レスポンス長の自動調整: タスクの複雑さに応じてレスポンス長が最適化されます。冗長な出力が減少し、必要に応じて詳細度をプロンプトで指定可能。

🔧

ツール呼び出し回数の減少: モデルがツールを呼ぶ前に推論で解決しようとする傾向が強化。ツールの積極的な利用が必要な場合はプロンプトで明示的に指示が必要。

💬

より直接的なトーン: より意見を持った直接的なトーンに変化。絵文字の使用が減少し、プロフェッショナルな表現に。

📋

進捗報告の改善: 長いエージェントトレースで、より定期的に進捗を報告するようになりました。

🔀

サブエージェント生成の抑制: デフォルトでのサブエージェント生成数が減少。並列処理が必要な場合は明示的な指示が必要。

コードサンプル

Python: Opus 4.6 から Opus 4.7 への基本移行

Python 変更前 (Opus 4.6) 
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=16000,
    temperature=0.7,
    thinking={"type": "enabled", "budget_tokens": 32000},
    messages=[
        {
            "role": "user",
            "content": "このコードベースをレビューし、リファクタリング計画を提案してください。"
        }
    ]
)

# Thinking コンテンツへの直接アクセス
for block in message.content:
    if block.type == "thinking":
        print(f"思考: {block.thinking}")
    elif block.type == "text":
        print(f"回答: {block.text}")
Python 変更後 (Opus 4.7) 
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=21600,  # 35% の余裕を確保: 16000 * 1.35
    # temperature は省略 (非デフォルト値は 400 エラー)
    thinking={"type": "adaptive", "display": "summarized"},
    output_config={"effort": "high"},
    messages=[
        {
            "role": "user",
            "content": "このコードベースをレビューし、リファクタリング計画を提案してください。"
        }
    ]
)

for block in message.content:
    if block.type == "thinking":
        print(f"思考: {block.thinking}")  # display: "summarized" で取得可能
    elif block.type == "text":
        print(f"回答: {block.text}")

移行ガイド

1

破壊的変更の対応 必須

高 Extended Thinking バジェット廃止: thinking: {type: "adaptive"} に変更
高 サンプリングパラメータ廃止: temperature、top_p、top_k を削除
中 Thinking デフォルト非表示: display: "summarized" を追加 (必要な場合)
中 トークナイザー更新: max_tokens を余裕を持って設定

2

新機能の活用 任意

高解像度画像: 自動適用 (追加設定不要)
xhigh エフォートレベル: output_config: {effort: "xhigh"}
タスクバジェット: ベータヘッダー + output_config.task_budget

3

動作変化への適応 任意

ツール呼び出し頻度の減少への対応: 必要に応じてプロンプトで明示的にツール利用を指示
レスポンス長の最適化: 詳細度をプロンプトで調整
サブエージェント生成の抑制: 並列処理が必要な場合は明示的に指示

開発者への影響

アップグレード対象

Opus 4.6 を使用中のすべての開発者

claude-opus-4-6 を使用しており、Opus 4.7 へのアップグレードを検討しているすべての開発者が対象です。以下のユースケースに該当する場合は特に注意が必要です。

  • Extended Thinking のバジェット指定 (budget_tokens) を使用しているアプリケーション
  • temperature、top_p、top_k パラメータをカスタマイズしているアプリケーション
  • Thinking ブロックの内容を参照しているアプリケーション

画像処理の高精度化を必要とするユースケース

高解像度画像サポートにより、ポインティング、測定、カウント、バウンディングボックス検出が大幅に改善。1:1 ピクセルマッピングでスケールファクターの計算が不要になります。

エージェントループのトークン管理

タスクバジェット機能 (ベータ) により、エージェントループ全体のトークン使用量を制御できるようになりました。長時間実行されるエージェントタスクのコスト管理に有効です。

必要なアクション

1️⃣

Extended Thinking バジェットの削除: thinking: {"type": "enabled", "budget_tokens": N} を thinking: {"type": "adaptive"} に変更。Adaptive Thinking はデフォルトでオフのため、明示的に有効化が必要。

2️⃣

サンプリングパラメータの削除: temperature、top_p、top_k を非デフォルト値で指定しているコードから、これらのパラメータを完全に削除。

3️⃣

Thinking 表示の対応: Thinking コンテンツを利用している場合は display: "summarized" を追加。

4️⃣

max_tokens の調整: 新しいトークナイザーにより最大 35% のトークン増加があるため、max_tokens に余裕を持たせる。

5️⃣

テストの実施: 本番環境に適用する前に、開発環境で十分なテストを実施。特に画像処理、エージェント動作、トークン使用量に注意。

まとめ

Claude Opus 4.7 は、Anthropic の最も高性能な GA モデルとして、複雑な推論やエージェントコーディングにおける能力を大幅に向上させたリリースです。高解像度画像サポート (2576px / 3.75MP)、新しい xhigh エフォートレベル、タスクバジェット機能 (ベータ) といった注目の新機能が追加されています。

一方で、Messages API に複数の破壊的変更が含まれている点は特に注意が必要です。Extended Thinking のバジェット指定の廃止、サンプリングパラメータの使用不可、Thinking コンテンツのデフォルト非表示化、そして新トークナイザーによるトークン消費量の増加 (最大 35%) は、既存のアプリケーションに直接影響を与えます。

価格は Opus 4.6 と同一の $5/$25 per MTok に据え置かれており、コスト面での追加負担はありません。ただし、トークナイザーの変更により実質的なトークン消費が増加する可能性があるため、使用量のモニタリングを推奨します。

アップグレードを検討する開発者は、まず移行ガイドを確認し、破壊的変更への対応を完了した上で、開発環境でのテストを十分に行ってから本番環境に適用することを強く推奨します。
