Claude Code Update

Claude Code v2.1.117

2026-04-21

⚡ ネイティブビルドでの検索高速化 (bfs/ugrep)、Opus 4.6/Sonnet 4.6 デフォルトエフォート引き上げ、Opus 4.7 コンテキストウィンドウ修正を含む大型アップデート

Key Points

🔍

ネイティブビルドで bfs/ugrep 組み込み - 検索パフォーマンス革新

macOS/Linux ネイティブビルドで Glob/Grep ツールが組み込みの bfs と ugrep に置き換えられました。ツール往復通信が不要になり、Bash ツールから直接高速検索が可能になります。

📈

Pro/Max デフォルトエフォートレベルが high に引き上げ

Opus 4.6 および Sonnet 4.6 のデフォルトエフォートレベルが medium から high に変更されました。より深い思考によるデフォルトでの高品質な応答が得られます。

🤖

Opus 4.7 コンテキストウィンドウ修正: 200K → 1M

Opus 4.7 のコンテキストウィンドウが 200K として誤計算されていた重大なバグが修正され、本来の 1M トークンが正しく認識されるようになりました。/context の表示精度改善と不必要な早期自動コンパクションが解消されます。

🔀

フォーク型サブエージェントの外部ビルド対応

環境変数 CLAUDE_CODE_FORK_SUBAGENT=1 を設定することで、外部ビルドでもフォーク型サブエージェントを有効化できるようになりました。プロセス起動のオーバーヘッド削減とリソース効率の向上が実現します。

🔌

MCP 同時接続によるスタートアップ高速化

ローカルと claude.ai の両方の MCP サーバーが設定されている場合、同時接続がデフォルトになりました。直列接続から並列接続への移行でスタートアップ時間が短縮されます。

🐛

15 件のバグ修正で安定性が大幅向上

OAuth トークンリフレッシュ、WebFetch の大規模 HTML ハング、Bedrock Opus 4.7 推論プロファイル、サブエージェントのマルウェア誤検出、Linux メモリ増加など幅広い修正が含まれています。

技術的な詳細

⚡ ネイティブビルドでの bfs/ugrep 組み込み

v2.1.117 の最も重要な技術的変更です。従来のアーキテクチャでは Glob と Grep は個別ツールとして実装されており、モデルがこれらを呼び出す際にツール往復通信が必要でした。v2.1.117 ではネイティブビルドに bfs (高速ファイル検索) と ugrep (高速テキスト検索) がバイナリとして直接組み込まれ、Bash ツールを通じて直接実行されるためオーバーヘッドが排除されます。

  • macOS ネイティブビルド: 適用
  • Linux ネイティブビルド: 適用
  • Windows: 変更なし (従来のツールを維持)
  • npm インストール版: 変更なし (従来のツールを維持)

📈 デフォルトエフォートレベルの変更

Pro/Max サブスクライバー向けに、Opus 4.6 および Sonnet 4.6 のデフォルトエフォートレベルが medium から high に引き上げられました。エフォートレベルはモデルが応答生成に費やす「思考の深さ」を制御するパラメータです。high エフォートではモデルがより深く思考するため応答品質が向上しますが、トークン使用量と応答時間は増加する可能性があります。

🤖 Opus 4.7 コンテキストウィンドウの修正

v2.1.117 以前では Opus 4.7 のコンテキストウィンドウサイズが 200K トークンとして計算されていましたが、ネイティブコンテキストウィンドウは 1M トークンです。この誤認識により以下の問題が発生していました。

  1. /context の不正確な表示: 実際のコンテキスト使用率より大幅に高い値が表示 (例: 5% なのに 25% と表示)
  2. 早期自動コンパクション: 上限に達していないにもかかわらず自動コンパクションが実行

🔀 フォーク型サブエージェントの外部ビルド対応

フォーク型サブエージェントはメインプロセスからフォークされたサブプロセスとして実行されるエージェントです。従来は内部ビルドのみで利用可能でしたが、CLAUDE_CODE_FORK_SUBAGENT=1 で外部ビルドでも有効化可能に。プロセス起動のオーバーヘッド削減、メモリ空間共有によるリソース効率の向上、コンテキストの高速引き継ぎがメリットです。

🔑 OAuth トークンリフレッシュの改善

Plain-CLI の OAuth セッションでアクセストークンが期限切れになるとセッションが終了する問題がありました。v2.1.117 では API からの 401 レスポンスを検出した際にトークンを反応的にリフレッシュするメカニズムが導入され、長時間セッションでもトークン期限切れによる中断が発生しなくなります。

# ネイティブビルドでは bfs/ugrep が組み込まれている
# 従来の Glob/Grep ツール呼び出しなしに直接検索可能

# bfs: 高速なファイル検索
bfs . -name "*.py" -type f -newer ./setup.py

# ugrep: 高速なテキスト検索
ugrep -rn "def main" --include="*.py"

# パイプラインで組み合わせ
bfs . -name "*.ts" -type f | xargs ugrep -l "import.*React"

  • 🔀 フォーク型サブエージェントの検討: パフォーマンス向上が必要な場合、CLAUDE_CODE_FORK_SUBAGENT=1 の設定を検討してください
