2026-03-17

概要

🔌

プラグインエコシステム強化

CLAUDE_PLUGIN_DATA による永続データ管理、StopFailure フックイベント、エージェントフロントマターサポートなど、プラグイン開発の基盤が大幅に拡張されました。

🛡️

⚡

ストリーミング改善

レスポンステキストが行単位でストリーミング表示されるようになり、大規模セッション再開時のメモリ使用量と起動時間も改善されました。

🖥️

互換性向上

tmux パススルー対応、WSL2 ボイスモード修正、VS Code の Homebrew PATH 解決など、多様な環境での安定性が向上しました。

🎨

カスタムモデル対応

ANTHROPIC_CUSTOM_MODEL_OPTION 環境変数により、/model ピッカーにカスタムモデルエントリを追加可能になりました。

セキュリティ修正

Before / After

Before

  • sandbox.enabled: true でも依存関係不足時にサイレントに無効化
  • ユーザーはサンドボックスが有効と信じたまま動作
  • セキュリティリスクに気づけない

After

  • 依存関係が不足している場合、起動時に警告を表示
  • サンドボックスの実際の状態をユーザーに通知
  • セキュリティ設定の信頼性が向上

新機能

⚡ StopFailure フック

API エラー (レート制限、認証失敗など) によりターンが終了した際に発火する新しいフックイベント。カスタムリカバリ処理やログ記録が可能になります。

💾 CLAUDE_PLUGIN_DATA

プラグインの永続状態を保存するデータディレクトリパス。プラグイン更新後もデータが保持され、アンインストール時には確認プロンプトが表示されます。

📝 エージェントフロントマター

プラグイン提供エージェントに effort、maxTurns、disallowedTools のフロントマター設定が可能に。エージェントの動作を細かく制御できます。

🎨 ANTHROPIC_CUSTOM_MODEL_OPTION

/model ピッカーにカスタムエントリを追加する環境変数。_NAME および _DESCRIPTION サフィックスで表示名と説明をカスタマイズ可能です。

セキュリティ修正

⚠️ サンドボックスのサイレント無効化修正

sandbox.enabled: true が設定されているにもかかわらず依存関係が不足している場合に、サンドボックスが警告なしに無効化されていた問題を修正。起動時に警告が表示されるようになりました。

🔒 保護ディレクトリの書き込み制御

bypassPermissions モードで .git、.claude などの保護ディレクトリがプロンプトなしで書き込み可能になっていた問題を修正。意図しない書き込みが防止されます。

🚫 MCP deny ルール修正

deny: ["mcp__servername"] の権限ルールがツールリストから除外されず、モデルがブロック済みツールを認識・使用を試みることが可能だった問題を修正。ツールリスト自体から除外されるようになりました。

コードサンプル

JSON StopFailure フックの設定例 
// .claude/hooks/hooks.json
{
  "StopFailure": [
    {
      "command": "notify-send 'Claude Code' 'API エラーでターンが終了しました'"
    }
  ]
}
Bash CLAUDE_PLUGIN_DATA の使用例 
# プラグインスクリプト内での使用例
DATA_DIR="${CLAUDE_PLUGIN_DATA}"
mkdir -p "${DATA_DIR}"

# 設定やキャッシュの保存
echo '{"setting": "value"}' > "${DATA_DIR}/config.json"
Bash tmux パススルーの有効化 
# tmux.conf に追加
set -g allow-passthrough on

# 既存セッションに即時反映
tmux set -g allow-passthrough on

開発者への影響

🔌 プラグイン開発者

CLAUDE_PLUGIN_DATA による永続データ管理、StopFailure フックイベント、エージェントフロントマターのサポートにより、プラグインの状態管理・エラーハンドリング・エージェント制御がより柔軟に行えるようになりました。

🏢 エンタープライズ管理者

サンドボックスのサイレント無効化、bypassPermissions モードでの保護ディレクトリ書き込み、MCP deny ルールの 3 つのセキュリティ修正が含まれます。特にサンドボックスを前提としたセキュリティ運用環境では早急なアップデートを推奨します。

🖥️ 多様な環境のユーザー

tmux パススルー、WSL2 ボイスモード、VS Code の Homebrew PATH 解決、truecolor 非対応ターミナルでのカラー表示など、多様な環境での利用体験が改善されました。

📊 大規模セッションユーザー

行単位ストリーミングにより長い出力のリアルタイム確認が可能になり、大規模セッションの再開パフォーマンスも向上しています。--resume を頻繁に使用するユーザーにとって大きな改善です。
