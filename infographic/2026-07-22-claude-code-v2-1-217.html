v2.1.217 - 20 項目の改善

概要

🤖

サブエージェント同時実行制御

デフォルト 20 の同時実行上限、予算連動停止、ネスト禁止の 3 つのメカニズムによるリソース管理の強化

🏢

エンタープライズ環境修正

mTLS、TLS-verify、OAuth スコープ、プロキシ設定、OTEL テレメトリエンドポイントの統合管理が正しく動作

🧠

メモリリーク修正

MCP ツール出力の切り詰め後に完全なデータが保持され続けるメモリリークを解消。長時間セッションの安定性が向上

🛡️

セキュリティ強化

バックグラウンドセッションのシンボリックリンク正規化により、ワークスペース外へのエスケープを防止

✨

UX 改善

絵文字ショートコード補完、トランスクリプト書き込み警告、スクリーンリーダー改善、Windows 自動復旧

Before / After

Before

  • サブエージェントの同時実行数に上限なし
  • --max-budget-usd がバックグラウンドエージェントを停止できない
  • サブエージェントが無制限にネスト可能
  • MCP ツール出力がメモリに残り続ける
  • シンボリックリンクでワークスペース外にエスケープ可能

After

  • デフォルト 20 の同時実行キャップ
  • 予算上限到達でバックグラウンドエージェントも停止
  • ネスト禁止がデフォルト、環境変数で制御
  • 切り詰め後に元データの参照を適切に解放
  • シンボリックリンクを正規化してエスケープ防止

新機能

😊 絵文字ショートコード補完

プロンプト入力で :heart: と入力すると ❤️ が挿入される。途中入力でサジェスト表示。emojiCompletionEnabled で無効化可能。

⚠️ トランスクリプト警告

ディスク容量不足や環境変数の継承でセッション保存が無効になっている場合に警告を表示。トランスクリプトが無言で失われなくなった。

🔢 同時実行キャップ

サブエージェントの同時実行数をデフォルト 20 に制限。CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS でカスタマイズ可能。

🚫 ネスト禁止

サブエージェントからのサブエージェント生成をデフォルトで禁止。CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH で深度を設定可能。

重要な修正

🧠 MCP メモリリーク

切り詰められた MCP ツール出力が完全なデータをセッション全体でメモリに保持し続ける問題を修正。

🪟 Windows 自動復旧

オートアップデート失敗時に claude.exe が消失する問題を修正。失敗時は保存された実行ファイルを自動復元。

🔗 シンボリックリンク正規化

バックグラウンドセッション分離でシンボリックリンクされた作業ディレクトリが正規化されていない問題を修正。

🏢 企業ネットワーク設定

Claude Desktop セッションで mTLS、TLS-verify、OAuth スコープ、プロキシ設定が無視される問題を修正。

📊 OTEL テレメトリ

マネージド設定の OTEL_EXPORTER_OTLP_ENDPOINT が下位スコープのオーバーライドにリダイレクトされる問題を修正。

💰 予算連動停止

--max-budget-usd の上限到達時にバックグラウンドサブエージェントも停止するよう修正。

その他の修正と改善

♿

スクリーンリーダー改善

起動時のアナウンスが最初のプロンプトレンダリングで途切れる問題と、思考ステータス行が数秒ごとに再レンダリングされる問題を修正

🖥️

Remote Control セッション

権限プロンプトが表示された後に接続したビューアーにプロンプトやダイアログが表示されない問題を修正

📄

Bedrock オートコンパクト

Claude Opus 4.8 on Bedrock でオートコンパクトがトリガーされず、/compact が上限超過後に失敗する問題を修正

🔄

--resume の TypeError 修正

トランスクリプトに不正な添付エントリがある場合に --resume/--continue や /resume が TypeError で失敗する問題を修正

📝

ブレース展開の予算制限

CLAUDE.md や SKILL.md の paths フロントマターに多数のブレースグループが含まれる場合の OOM キル/ストールを防止

🔗

PR バッジリンクの改善

フッターの PR バッジリンクがターミナルサポートを検出できない環境でもクリック可能に。FORCE_HYPERLINK=0 でオプトアウト

⏰

ログイン期限警告の変更

期限切れの 5 日前ではなく 3 日前に警告を表示するよう変更

export FORCE_HYPERLINK=0

開発者への影響

🤖 サブエージェント並列処理ユーザー

デフォルトの同時実行上限 (20) やネスト禁止が既存ワークフローに影響する可能性がある。必要に応じて CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS や CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH で調整。

🏢 企業ネットワーク環境ユーザー

Claude Desktop セッションで mTLS やプロキシ設定が正しく動作するようになった。以前ワークアラウンドを適用していた場合は再評価を推奨。

📊 テレメトリ管理者

マネージド設定でテレメトリエンドポイントを管理している場合、シグナル固有のオーバーライドが正しく統制されているか確認。

必要なアクション

  • 1 Claude Code を v2.1.217 以上にアップデート。特にメモリリーク修正とセキュリティ修正の恩恵を受けるため。
  • 2 サブエージェント設定の確認: デフォルトの同時実行上限やネスト禁止が既存ワークフローに影響しないか確認。
  • 3 企業ネットワーク設定のワークアラウンドを再評価。mTLS やプロキシが正しく動作するため。
  • 4 OTEL テレメトリ設定: マネージドエンドポイントとシグナル固有オーバーライドの関係を確認。
