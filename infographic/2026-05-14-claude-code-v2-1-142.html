CLAUDE CODE UPDATE

Claude Code v2.1.142 - エージェントディスパッチ設定の大幅拡張

2026-05-14

概要

🏷️

8 つの新フラグ追加

claude agents コマンドに --add-dir、--settings、--mcp-config、--plugin-dir、--permission-mode、--model、--effort、--dangerously-skip-permissions の 8 フラグを追加し、バックグラウンドセッションの構成を完全に制御可能に

⚡

Fast Mode Opus 4.7 移行

Fast Mode のデフォルトモデルが Opus 4.7 に変更。推論品質と速度の両方が向上し、環境変数でのオプトアウトも提供

📦

プラグインスキル検出簡素化

ルートレベルに SKILL.md を持つプラグインが skills/ サブディレクトリなしでも自動的にスキルとして検出されるように変更

🛡️

デーモン安定性向上

macOS スリープ復帰後のセッション消失、バイナリアップグレード後のクラッシュループ、Windows ネットワークドライブでのデッドロックなど包括的に修正

📊

合計 26 件の変更

新機能 5 件、バグ修正 18 件、改善 3 件を含む本リリースは、エージェントワークフローの本番運用における信頼性を大きく向上

主な機能

🏷️ エージェントディスパッチフラグ

claude agents dispatch に 8 つの新フラグを追加。モデル選択、MCP 設定、パーミッションモード、作業ディレクトリなどをディスパッチ時に一括指定可能に

⚡ Fast Mode Opus 4.7

デフォルトモデルが Opus 4.7 に変更。推論速度向上により品質を維持しつつ応答速度も改善。CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1 で Opus 4.6 に固定可能

📦 プラグインスキル検出

ルートレベルに SKILL.md を持ち skills/ サブディレクトリを持たないプラグインが自動的にスキルとして検出。シンプルなプラグイン構造が可能に

🔌 LSP サーバー表示

/plugin 詳細ペインと claude plugin details コマンドでプラグインが提供する LSP サーバーを表示

⚠️ Web セットアップ警告

/web-setup が既存の GitHub App 接続を置換する前に警告を表示。意図しない接続の上書きを防止

コードサンプル

Bash エージェントディスパッチの新フラグ 
# カスタム設定でバックグラウンドエージェントをディスパッチ
claude agents dispatch "テストスイートを実行して結果を報告" \
  --add-dir /path/to/shared-libs \
  --settings /team/claude-settings.json \
  --mcp-config /team/mcp-servers.json \
  --permission-mode auto \
  --model opus-4-7 \
  --effort high

# プラグインディレクトリを指定してディスパッチ
claude agents dispatch "コードレビューを実施" \
  --plugin-dir /team/plugins \
  --model opus-4-7

# CI/CD 環境向け: パーミッションスキップ付きディスパッチ
claude agents dispatch "ビルドとデプロイを実行" \
  --dangerously-skip-permissions \
  --mcp-config ./ci/mcp-config.json \
  --effort medium
Bash Fast Mode のモデルオーバーライド 
# Opus 4.6 に固定する場合
export CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1

# 確認: Fast Mode で使用されるモデル
claude config get fast_mode_model

# 環境変数を解除して Opus 4.7 に戻す
unset CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE
JSON プラグインのシンプル構造 
// plugin.json - skills/ ディレクトリなしでスキルが認識される
{
  "name": "my-simple-plugin",
  "version": "1.0.0",
  "description": "A simple plugin with a single skill",
  "main": "src/index.ts"
}
Bash MCP タイムアウトの設定 
# MCP ツールタイムアウトを 5 分に設定 (修正前は 60 秒で打ち切られていた)
export MCP_TOOL_TIMEOUT=300000

# リモート MCP サーバーへの長時間ツール呼び出しが正常に動作
claude "大規模データセットの分析を実行して"

バグ修正

🖥️ デーモン/バックグラウンド (8 件)

  • macOS スリープ復帰後の接続問題修正 - デーモンがクロックジャンプを検出するロジックに変更し、スリープ中の経過時間がアイドルタイムとして加算される問題を解消
  • バイナリアップグレード後のクラッシュループ修正 - brew upgrade 後に旧デーモンがクリーンに終了せず、エージェントが削除済みパスで起動を繰り返す問題を修正
  • Git ワークツリー認識問題修正 - バックグラウンドセッションが既存の Git ワークツリーを認識せず、Edit がブロックされる問題を修正
  • Chrome 拡張接続時のクラッシュ修正 - Claude-in-Chrome 拡張が共有タブなしで接続された状態でのクラッシュループを修正
  • --dangerously-skip-permissions の永続化修正 - リタイア/ウェイクサイクルでフラグが失われる問題を修正
  • Windows ネットワークドライブでのデッドロック修正 - ネットワークドライブ上で claude agents がデッドロックする問題を修正
  • ヘッドレスブラウザ Shim の修正 - アタッチ済みセッションでバックグラウンドワーカーの Shim が誤って適用される問題を修正
  • エディタ設定の修正 - "v to open in editor" がシェルの $EDITOR/$VISUAL ではなくデーモンのデフォルトエディタを使用する問題を修正

🔌 MCP/プラグイン (4 件)

  • MCP_TOOL_TIMEOUT がリモート HTTP/SSE MCP サーバーのフェッチタイムアウトを引き上げず、60 秒で打ち切られる問題を修正
  • skills: ["./"] を使用するプラグインで偽の "path escapes plugin directory" エラーが表示される問題を修正
  • インストールメタデータが存在しない場合にアクティブなプラグインバージョンディレクトリが削除される問題を修正
  • デフォルトフォルダをシャドウする plugin.json キーをすべて名前付きで表示するように修正

🎨 UI/UX (6 件)

  • 256 色ターミナルでの背景色ブリード修正 - Apple Terminal などでの claude agents セッションアタッチ時の表示崩れを修正
  • セッションタイトル修正 - 最初のメッセージがリンクの場合にタイトルが URL から派生される問題を修正
  • 重複モデルリクエスト修正 - リモートクライアントからの冗長な set_model リクエストが重複ブレッドクラムを挿入する問題を修正
  • プラグインインストール数表示修正 - 新規公開プラグインが "0 installs" と表示される問題を修正
  • リアクティブコンパクション改善 - 最初の要約試行がオーバーフローサイズをシードとして使用し、無駄なリトライを回避
  • コンテキストがほぼ満杯の状態での長時間セッションのコンパクション効率が向上
