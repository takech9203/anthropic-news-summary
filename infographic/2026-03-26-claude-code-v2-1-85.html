主要機能

🎯 hooks 条件付き if フィールド

パーミッションルール構文 (Bash(git *) など) で hook の実行条件を指定。条件不一致時はプロセス生成自体をスキップし、多数の hooks 定義時のオーバーヘッドを最小化します。

🔑 MCP headersHelper 環境変数

CLAUDE_CODE_MCP_SERVER_NAME と CLAUDE_CODE_MCP_SERVER_URL が headersHelper スクリプトに提供され、1 つのスクリプトで複数の MCP サーバーに対応できます。

🔐 MCP OAuth RFC 9728

RFC 9728 Protected Resource Metadata ディスカバリーに対応。.well-known/oauth-protected-resource エンドポイントから認可サーバーを自動検出します。

🤖 PreToolUse AskUserQuestion

PreToolUse hooks が AskUserQuestion をインターセプトし、updatedInput と permissionDecision: "allow" で回答を提供。ヘッドレス統合が可能になります。

🔗 ディープリンク 5,000 文字対応

claude-cli://open?q=... ディープリンクが最大 5,000 文字をサポート。長いプロンプトには「スクロールして確認」の警告が表示されます。

🛡️ 組織プラグインブロック

managed-settings.json で組織ポリシーによりブロックされたプラグインはインストール・有効化が不可に。マーケットプレイスビューからも非表示になります。

OpenTelemetry の tool_parameters が OTEL_LOG_TOOL_DETAILS=1 環境変数によるゲーティングに変更。セキュリティとプライバシーが向上しました。

JSON hooks 条件付き if フィールドの設定 
{
  "hooks": {
    "PreToolUse": [
      {
        "if": "Bash(git *)",
        "command": "/path/to/git-audit-hook.sh",
        "timeout": 5000
      },
      {
        "if": "AskUserQuestion",
        "command": "/path/to/auto-answer.sh",
        "timeout": 10000
      }
    ]
  }
}
Bash MCP headersHelper - 複数サーバー対応スクリプト 
#!/bin/bash
# headers-helper.sh - 1 つのスクリプトで複数の MCP サーバーに対応

case "$CLAUDE_CODE_MCP_SERVER_NAME" in
  "production-server")
    echo '{"Authorization": "Bearer prod-token-xxx"}'
    ;;
  "staging-server")
    echo '{"Authorization": "Bearer staging-token-xxx"}'
    ;;
  *)
    echo '{"Authorization": "Bearer default-token-xxx"}'
    ;;
esac

# CLAUDE_CODE_MCP_SERVER_URL も利用可能
# echo "Server URL: $CLAUDE_CODE_MCP_SERVER_URL" >&2

バグ修正 (17 件)

📋 セッション・コマンド関連

  • /compact のコンテキスト超過修正: 会話が大きくなりすぎてコンパクトリクエスト自体がコンテキストに収まらない場合に失敗する問題を修正
  • /plugin enable/disable の修正: プラグインのインストール場所と設定での宣言場所が異なる場合に失敗する問題を修正
  • --worktree の非 git リポジトリ対応: WorktreeCreate hook 実行前に --worktree が非 git リポジトリでエラー終了する問題を修正
  • プロンプトキューのスタック修正: 特定のスラッシュコマンド実行後にプロンプトがキューに残り、上矢印キーで取得できない問題を修正
  • Remote Control セッションステータス修正: パーミッション解決後も「Requires Action」のままになる問題を修正

🔌 MCP・認証関連

  • deniedMcpServers 設定の修正: deniedMcpServers 設定が claude.ai の MCP サーバーをブロックしない問題を修正
  • MCP ステップアップ認証の修正: リフレッシュトークンが存在する場合にステップアップ認証が失敗する問題を修正。サーバーが 403 insufficient_scope でスコープ昇格を要求した際、再認可フローが正しくトリガーされるように
  • Python Agent SDK MCP 修正: type:'sdk' の MCP サーバーが --mcp-config 経由で渡された場合、起動時にドロップされる問題を修正

🌐 ネットワーク・セッション関連

  • リモートセッションのメモリリーク修正: ストリーミングレスポンスが中断された際のメモリリークを修正
  • ECONNRESET エラーの修正: エッジ接続チャーン時に発生する持続的な ECONNRESET エラーを、リトライ時に新しい TCP 接続を使用することで修正

🖥️ ターミナル・表示関連

  • SSH/VSCode ターミナルでの生キーシーケンス修正: SSH 接続時や VSCode 統合ターミナルでプロンプトに生のキーシーケンスが表示される問題を修正
  • 拡張キーボードモードの修正: Ghostty、Kitty、WezTerm で終了後にターミナルが拡張キーボードモードのまま残る問題を修正。Ctrl+C および Ctrl+D が正しく動作するように
  • ストリーミング中のスクロールアップ修正: スクロールアップ時に古いコンテンツが透けて見える問題を修正
  • shift+enter/meta+enter の修正: タイプアヘッドサジェスチョンが shift+enter および meta+enter をインターセプトし、改行挿入の代わりに動作する問題を修正
  • diff シンタックスハイライトの修正: 非ネイティブビルドで diff のシンタックスハイライトが動作しない問題を修正
  • マルチモニターの switch_display 修正: computer-use ツールの switch_display がマルチモニター環境で「not available in this session」を返す問題を修正
  • OTEL エクスポーター none 設定のクラッシュ修正: OTEL_LOGS_EXPORTER、OTEL_METRICS_EXPORTER、OTEL_TRACES_EXPORTER が none に設定されている場合のクラッシュを修正
