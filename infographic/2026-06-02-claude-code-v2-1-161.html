Before / After: 並列ツール呼び出し

❌

Before

1 つの Bash コマンドが失敗すると、同じバッチの他ツールもキャンセルされる

✅

After

各ツールが独立して実行・完了。1 つの失敗が他に波及しない

機能改善

📊 OTEL メトリクス

カスタムディメンション対応。team, repo 等のラベルで使用量をスライス可能に

📈 エージェント進捗表示

claude agents の進捗が done/total 形式で表示されるようになった

📂 MCP コネクタ折りたたみ

🔄 並列ツール独立性

並列ツール呼び出しで各ツールが独立して動作。1 つの失敗が他に波及しない
