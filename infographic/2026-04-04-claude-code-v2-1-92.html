コードサンプル

JSON forceRemoteSettingsRefresh ポリシー設定 
{
  "forceRemoteSettingsRefresh": true
}

この設定を有効にすると、CLI は起動時にリモート設定サーバーから最新の設定を取得するまでブロックされます。取得に失敗した場合は CLI が終了し、古い設定での動作を防止します。

Bash Remote Control セッション名のカスタマイズ 
# デフォルト: ホスト名がプレフィックスとして使用される
# 例: myhost-graceful-unicorn
claude --remote-control

# カスタムプレフィックスを指定
claude --remote-control --remote-control-session-name-prefix "dev-team"
# 例: dev-team-graceful-unicorn

ユースケース・開発者への影響

🏢 エンタープライズセキュリティ管理者

forceRemoteSettingsRefresh ポリシーを有効にすることで、リモートマネージド設定の適用をフェイルクローズ方式で強制できます。セキュリティポリシーの確実な適用が必要な環境で、古い設定や未取得状態での CLI 動作リスクを排除します。

☁️ Amazon Bedrock 新規ユーザー

インタラクティブセットアップウィザードにより、AWS 認証からモデルピニングまで対話的に設定可能。手動設定のエラーを減らし、Bedrock 経由の Claude Code 利用をスムーズに開始できます。

💳 サブスクリプションユーザー・Pro ユーザー

/cost コマンドのモデル別・キャッシュヒット別内訳表示により、コスト最適化の意思決定が容易に。また、プロンプトキャッシュ期限切れ時のトークン数ヒントにより、不要なコスト発生を事前に把握できます。
