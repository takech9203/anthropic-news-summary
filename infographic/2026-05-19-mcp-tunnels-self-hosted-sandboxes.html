機能詳細

🔐 MCP トンネル

プライベートネットワーク内の MCP サーバーに接続するための安全なトンネル機能。

  • cloudflared + プロキシスタック
  • 3 層セキュリティ構成
  • インバウンドポート不要
  • Research Preview 段階

🖥️ セルフホストサンドボックス

自社インフラでツール実行環境を運用するパターン。

  • Always-on: ant CLI ポーリング
  • Always-on: SDK EnvironmentWorker
  • Webhook トリガー
  • 4 プラットフォーム対応

🔄 動的 MCP 設定更新

セッション中の MCP サーバー/ツール設定をライブ更新。再起動なしでサーバー追加、ツール設定変更が可能。

📦 大規模出力自動スピル

100K トークンを超える出力を自動的にファイルへ書き出し。モデルはトランケートされたプレビューとファイルパスを取得して後続処理に活用。

Python MCP トンネル経由のツール呼び出し 
import anthropic

client = anthropic.Anthropic()

response = client.beta.messages.create(
    model="claude-opus-4-7",
    max_tokens=1000,
    messages=[{"role": "user", "content": "Use the hello tool to greet tunnel."}],
    mcp_servers=[
        {
            "type": "url",
            "url": "https://echo.YOUR_TUNNEL_DOMAIN_HERE/mcp",
            "name": "echo",
        }
    ],
    tools=[{"type": "mcp_toolset", "mcp_server_name": "echo"}],
    betas=["mcp-client-2025-11-20"],
)

ユースケース

🏢 エンタープライズ内部システム連携

社内のプライベートネットワークにある既存 MCP サーバー (データベース、CRM、社内ツール) に MCP トンネル経由で安全にアクセス。ファイアウォール設定の変更やポート開放なしに Claude を社内業務に統合可能。

🔐 規制対応のセルフホスト環境

金融、医療、政府機関などデータの外部送信が厳しく制限される環境で、セルフホストサンドボックスを使用。ツール実行とデータ処理を完全に自社インフラ内で完結し、コンプライアンス要件を満たす。

⚡ 大規模データ処理パイプライン

大量のデータ分析やコード生成タスクで 100K トークンを超える出力が発生する場合、自動スピル機能によりメモリ制限を気にせず処理を継続。ファイルベースで後続ステップに結果を引き渡し可能。
