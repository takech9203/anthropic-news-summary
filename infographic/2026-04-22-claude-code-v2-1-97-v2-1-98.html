🔒 セキュリティ修正 -- 重大

🚨
Bash ツールのパーミッションシステムに対する包括的なセキュリティ監査の結果、複数の攻撃ベクターが同時に修正されました。任意コード実行に至る脆弱性を含むため、全ての Claude Code ユーザーに対して速やかなアップデートを強く推奨します。

🚨 任意コード実行につながる脆弱性

  • バックスラッシュエスケープ攻撃: エスケープされたフラグが読み取り専用コマンドとして自動許可され、任意コード実行が可能になる脆弱性
  • 複合コマンドによるプロンプト回避: 複合 Bash コマンドが auto モードおよび bypass-permissions モードで安全チェックと明示的 ask ルールの強制パーミッションプロンプトをバイパス

🔍 パーミッション判定の不備

  • 環境変数プレフィックスの悪用: 任意の環境変数プレフィックスが安全とみなされ、読み取り専用判定を迂回可能
  • /dev/tcp, /dev/udp リダイレクト: ネットワークリダイレクトがファイルシステム操作として自動許可
  • grep -f / rg -f パターンファイル: 作業ディレクトリ外のパターンファイルをプロンプトなしで読み取り可能

🔧 パーミッションモードの整合性

  • --dangerously-skip-permissions のダウングレード: 保護パスへの書き込み承認後、accept-edits モードにサイレントにダウングレード
  • エージェントチームのモード継承: チームメンバーがリーダーのパーミッションモードを継承しない問題

📋 ルール処理の不備

  • 管理設定の allow ルール残留: 管理者が削除した allow ルールがプロセス再起動まで有効のまま残留
  • ワイルドカードルールのスペース/タブ不一致: Bash(cmd:*) が余分なスペースやタブを含むコマンドにマッチしない
  • deny ルールのプロンプトダウングレード: cd と他のセグメントを混在させたパイプコマンドに対して deny がプロンプトに降格
  • JavaScript プロトタイププロパティ衝突: toString 等のプロパティ名が settings.json をサイレントに無視させる

CLAUDE_CODE_PERFORCE_MODE を設定すると、Edit/Write/NotebookEdit が読み取り専用ファイルに対して p4 edit ヒントと共にエラーを返します。
