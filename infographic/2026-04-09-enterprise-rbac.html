エンタープライズ機能

Enterprise プラン向け RBAC

2026-04-09

グループ、カスタムロール、SCIM 連携による細粒度アクセス管理で、大規模組織における Claude 導入のセキュリティとガバナンスを強化

概要

👥

グループ管理

ユーザーをグループに整理し、部門・チーム・プロジェクト単位で管理。手動作成と SCIM 自動同期の両方に対応

🎭

カスタムロール

Claude の各機能へのアクセス権限を定義するカスタムロールを作成し、各グループに割り当て可能

🔒

部門ごとの機能制限

部門の業務要件に応じて利用可能な Claude 機能を制限。Cowork の有効化やチャット機能のみの許可など柔軟に構成

🔄

SCIM 連携

Okta、Microsoft Entra ID、OneLogin などの IdP と SCIM プロトコルで連携し、グループ構成を自動反映

🤝

Cowork チームレベル制御

Claude Cowork を特定チームに対してのみ有効化または無効化。段階的な導入をサポート

💰

グループ単位の支出制限

グループごとに月間利用上限を設定し、部門予算に応じたコスト管理を実現

Before / After

Before -- RBAC 導入前

  • 全ユーザーに同一権限が付与
  • 部門ごとの利用方針に合わせた制限が困難
  • ユーザー数増加で手動管理が限界に
  • IdP のグループ情報を Claude に反映する自動化手段が不足
  • 部門・チーム単位での支出制御が不可

After -- RBAC 導入後

  • グループとカスタムロールによる細粒度制御
  • 部門ごとに利用可能な機能を柔軟に設定
  • SCIM 連携でグループ管理を自動化
  • IdP 側の操作だけでユーザー・グループ管理が完結
  • グループ単位の支出上限でコストガバナンスを実現

主要機能

👥 グループ管理

管理者がユーザーをグループに整理。手動でのグループ作成・ユーザー割り当てに加え、SCIM を通じた IdP からの自動同期にも対応

🎭 カスタムロールの作成

Cowork、API アクセス、ファイルアップロードなどの Claude 機能に対するアクセス権限を定義するカスタムロールを作成し、各グループに割り当て

🔒 部門ごとの機能制限

エンジニアリングチームには Cowork を有効化しつつ、マーケティングチームにはチャット機能のみを許可するといった構成が可能

🔄 SCIM 連携による自動同期

Okta、Microsoft Entra ID、OneLogin などの主要な IdP と SCIM プロトコルで連携し、グループ構成を Claude に自動反映

🤝 Cowork のチームレベル制御

Claude Cowork (協調作業機能) を特定チームに対してのみ有効化または無効化できる柔軟な制御

💰 グループ単位の支出制限

グループごとに月間利用上限を設定し、利用状況のモニタリングと上限到達時の自動制限で部門予算を管理

技術詳細

RBAC 3 層アーキテクチャ

👤

ユーザー層

組織に所属する個々のユーザー。1 つ以上のグループに所属可能

👥

グループ層

ユーザーをまとめる論理的なグループ (部門、チーム、プロジェクトなど)

🎭

ロール層

Claude の機能アクセスを定義するカスタムロール

権限のマージルール

ユーザーが複数のグループに所属する場合、最も権限の広いロールが適用されます (許可的マージ)。

SCIM 連携の詳細

SCIM (System for Cross-domain Identity Management) は、ユーザーとグループの情報を IdP と Claude 間で自動同期するための業界標準プロトコルです。以下の機能が提供されます。

  • グループの自動作成・更新・削除: IdP 側でグループを変更すると、Claude 側に自動反映される
  • ユーザーの自動プロビジョニング・デプロビジョニング: 新規ユーザーの追加や退職者の無効化が IdP の操作だけで完結する
  • リアルタイム同期: IdP での変更が即座に Claude 側に反映される

カスタムロールで制御可能な機能

💬 Claude チャット
🤝 Claude Cowork
📁 ファイルアップロードと分析
🔍 Web 検索
⚡ コードの実行とプレビュー
🔗 インテグレーション

支出制限の管理

グループ単位の支出制限では、以下の設定が可能です。

  • 月間利用上限の設定
  • 利用状況のモニタリング
  • 上限到達時の通知と自動制限

ユースケース

部門別アクセス管理

エンジニアリングチームには Claude Cowork、コード実行、Web 検索を含むフルアクセスを付与し、マーケティングチームにはチャットとファイルアップロードのみを許可。法務チームにはチャット機能のみの制限付きロールを割り当てるなど、部門の業務要件に応じた機能配分を実現する。

段階的なロールアウト

まず小規模なパイロットチームに Claude Cowork を有効化し、利用状況とセキュリティ面での問題がないことを確認してから、徐々に対象チームを拡大する。SCIM 連携により、IdP 側でグループメンバーを変更するだけで Claude 側のアクセス権が自動更新されるため、運用負荷を最小限に抑えられる。

コストガバナンス

各部門のグループに月間支出上限を設定し、部門予算に応じた利用管理を実施する。利用状況のモニタリングと上限到達時の自動制限により、予期しないコスト超過を防止。全社的な AI 投資の可視化と最適化を推進する。

開発者への影響

対象

  • 🏢
    Enterprise プランの管理者: 組織全体のアクセスポリシーを設計・運用する IT 管理者やセキュリティチーム
  • 📊
    部門マネージャー: 自チームの Claude 利用範囲と予算を管理する責任者
  • 🔐
    IdP 管理者: Okta、Microsoft Entra ID、OneLogin などの ID 基盤を管理する担当者
  • 📋
    コンプライアンス担当者: 組織のセキュリティポリシーに基づいた AI ツールのガバナンスを担当する人材

必要なアクション

1

アクセスポリシーの設計

組織の部門構成と各部門の Claude 利用要件を整理し、必要なグループとカスタムロールの構成を設計する

2

SCIM 連携の設定 (推奨)

既存の IdP と Claude の SCIM 連携を設定し、グループの自動同期を有効化する。手動管理と比較して運用負荷が大幅に削減される

3

カスタムロールの作成

設計したポリシーに基づいてカスタムロールを作成し、各グループに割り当てる

4

支出制限の設定

部門予算に応じて各グループの支出上限を設定する

5

段階的なロールアウト

小規模なパイロットチームから開始し、利用状況を確認しながら徐々に対象チームを拡大することを推奨する

まとめ

Enterprise プラン向け RBAC の導入は、大規模組織における Claude の本格展開を支える重要な基盤機能です。この機能は Enterprise プランの管理者コンソールから即座に利用可能であり、既存の Enterprise 契約に追加料金なく含まれています。

注目すべきポイント

  • 🔄
    IdP 連携による運用自動化: SCIM プロトコルを通じた既存 ID 基盤との連携により、ユーザーやグループの管理が IdP 側の操作だけで完結。手動管理の負荷を大幅に削減し、ヒューマンエラーのリスクを低減
  • 🎭
    細粒度のアクセス制御: カスタムロールにより、Claude Cowork、Web 検索、ファイルアップロードなどの機能を個別に制御。部門の業務要件やセキュリティポリシーに合わせた柔軟な構成が可能
  • 📈
    段階的な導入の支援: チームレベルでの Cowork 有効化や部門ごとの機能制限により、パイロットチームから段階的に利用範囲を拡大する運用が容易に
  • 💰
    コストガバナンス: グループ単位の支出制限により、部門予算に応じた利用管理が可能。予期しないコスト超過を防止
