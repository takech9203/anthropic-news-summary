Claude Code v2.1.169: セーフモード導入と大規模バグ修正

disableBundledSkills 設定

バンドルされたスキル、ワークフロー、組み込みスラッシュコマンドをモデルから隠す設定を追加

Before / After

Before

  • 問題の切り分けにカスタマイズを手動で 1 つずつ無効化
  • ディレクトリ変更でキャッシュが破棄される
  • MCP ポリシーが再接続時に適用漏れ
  • Windows で claude -p がハング

After

  • --safe-mode で全カスタマイズを一括無効化
  • /cd でキャッシュ維持のままディレクトリ移動
  • 全経路で MCP ポリシーが一貫して適用
  • Windows のパフォーマンスリグレッション解消

新機能

🛡️ --safe-mode

全カスタマイズを無効化してトラブルシューティング。環境変数 CLAUDE_CODE_SAFE_MODE でも有効化可能

📂 /cd コマンド

プロンプトキャッシュを維持したままセッション中にワーキングディレクトリを変更

🎯 disableBundledSkills

設定または環境変数 CLAUDE_CODE_DISABLE_BUNDLED_SKILLS でバンドルスキルを非表示に

主要バグ修正

修正内容 環境
エンタープライズ MCP ポリシーが再接続時・IDE 設定・初回セッションで適用されない 全環境
claude -p がスラッシュコマンドスキャン待ちで低速化/ハング (v2.1.161 リグレッション) Windows
macOS で各ターン開始時に ~30-50ms の UI 停止 macOS
リモートコントロールが OAuth トークンリフレッシュ時に "reconnecting" で固着 全環境
Git Credential Manager のポップアップが不要に表示 Windows
バックグラウンドエージェントがプロジェクト env 値を無視 全環境
agents ビューで WSL/Windows Terminal でフレームが garbled WSL
MCPB プラグインキャッシュの不要な無効化 Windows
OTEL クライアント証明書パスが信頼確認なしに設定可能 全環境

改善

  • レスポンスストリーミング中およびスピナーアニメーション中の CPU 使用率を低減
  • Vertex/Foundry で 5 分のアイドルタイムアウトを復元 (無限ハング防止)
  • TaskCreate の信頼性向上: 不正な入力を自動修復
  • リモート管理設定の部分適用: 無効エントリがあっても有効ポリシーを適用
  • バックグラウンドセッションが --ide, --chrome 等のフラグを retire/wake 間で保持
  • "CLAUDE.md is too long" 警告がモデルのコンテキストウィンドウに応じてスケール
  • /workflows がターン進行中でも即座に表示
  • API キー認証無効時のエラーメッセージにガイダンスを追加

コードサンプル

Bash セーフモードと新機能の使用例 
# セーフモードで起動 (全カスタマイズ無効)
claude --safe-mode

# 環境変数でセーフモードを有効化
CLAUDE_CODE_SAFE_MODE=1 claude

# セッション中にワーキングディレクトリを変更
# (プロンプトキャッシュを維持)
/cd /path/to/new/project

# バンドルスキルを無効化して起動
CLAUDE_CODE_DISABLE_BUNDLED_SKILLS=1 claude

# Vertex/Foundry でアイドルタイムアウトを無効化
API_FORCE_IDLE_TIMEOUT=0 claude

# バックグラウンドエージェントの全セッションを表示
claude agents --json --all
JSON settings.json 設定例 
{
  "disableBundledSkills": true
}

ユースケース

🏢 エンタープライズ: MCP ポリシーの確実な適用

セキュリティチームが設定した allowedMcpServers / deniedMcpServers が再接続時・IDE 設定・初回セッションの全経路で一貫して適用されるようになった。追加の設定変更は不要

🔍 開発者: カスタマイズの問題切り分け

claude --safe-mode で起動し、問題が再現するか確認。再現しなければカスタマイズが原因。CLAUDE.md、プラグイン、フック、MCP サーバーを順次有効化して原因を特定

💻 Windows ユーザー: パフォーマンス問題の解消

v2.1.161 以降で発生していた claude -p のハング/低速化問題、Git Credential Manager のポップアップ、MCPB プラグインキャッシュの不要な再展開が全て解消

Claude Code のアップデート

Vertex/Foundry ユーザー

信頼されていないプロジェクト設定から OTEL クライアント証明書パスを設定する場合、信頼確認が必要に
