Claude API

Rate Limits API リリース -- レートリミットをプログラマティックに取得可能に

2026-04-24

概要

Anthropic は組織およびワークスペースに設定された API レートリミットをプログラマティックに取得できる Rate Limits API をリリースしました。Claude Console の Limits ページで確認できる情報と同等のデータを API 経由で取得できます。

📊

組織・ワークスペースのリミット取得

組織レベルとワークスペースレベルの両方のレートリミットを API 経由で一覧取得可能。ワークスペースのレスポンスには org_limit フィールドが含まれ、オーバーライド値と組織リミットを比較できます。

🔑

Admin API キー必須

Rate Limits API の利用には Admin API キー (sk-ant-admin...) が必要です。管理者ロールを持つユーザーのみがこのキーをプロビジョニングできます。通常の API キーでは利用不可です。

📖

読み取り専用 API

現時点では読み取り専用の API です。レートリミットの変更は API からは行えず、引き続き Claude Console から行う必要があります。

主要機能

📡 エンドポイント

組織レベル: GET /v1/organizations/rate_limits とワークスペースレベル: GET /v1/organizations/workspaces/{id}/rate_limits の 2 つの主要エンドポイントを提供。

🔍 フィルタオプション

?model= で特定モデルを検索、?group_type= で model_group、batch、token_count、files、skills、web_search をフィルタ可能。

🔐 認証

Admin API キー (sk-ant-admin...) による認証が必須。管理者ロールのみがプロビジョニング可能。通常の sk-ant-api... キーでは利用不可。

📄 ページネーション

page パラメータと next_page フィールドによるページネーションをサポート。将来の拡張に備えてループ処理の実装を推奨。

エンドポイント 説明
GET /v1/organizations/rate_limits 組織レベルのレートリミットを一覧取得
GET ...rate_limits?model={model} 特定モデルのリミットを検索
GET .../workspaces/{id}/rate_limits ワークスペースのオーバーライド一覧
?group_type={type} グループタイプでフィルタ

レスポンス構造

  • group_type: リミットのグループタイプ (model_group、batch など)
  • models[]: 対象モデルの一覧
  • limits[]: {type, value} ペアの配列 (requests_per_minute、input_tokens_per_minute、output_tokens_per_minute)
  • org_limit: ワークスペースレスポンスのみ -- 組織レベルのリミット値を参照可能
⚠
制限事項: レートリミットの更新は API からは行えません。変更は Claude Console から行う必要があります。ワークスペースレスポンスにグループが含まれていない場合は、組織レベルのリミットがそのまま継承されていることを意味します。

Before / After

Before

  • Claude Console の Limits ページで手動確認が必要
  • ゲートウェイやプロキシにレートリミット値をハードコード
  • Anthropic 側でリミット変更時にドリフトが発生
  • 内部アラート生成に手動でリミット値を管理
  • 複数ワークスペースのオーバーライド検証が困難

After

  • API 経由でプログラマティックにリミットを取得
  • ゲートウェイ・プロキシ設定を動的に同期
  • リミット変更を自動検知してドリフトを防止
  • Usage and Cost API と組み合わせてアラート自動化
  • ワークスペース設定の一元的な監査が可能

Python レートリミット取得とアラート生成の例 
import requests

ADMIN_KEY = "sk-ant-admin-..."
BASE_URL = "https://api.anthropic.com/v1"

headers = {
    "anthropic-version": "2023-06-01",
    "x-api-key": ADMIN_KEY,
}


def get_org_rate_limits(model=None):
    """組織レベルのレートリミットを取得する"""
    url = f"{BASE_URL}/organizations/rate_limits"
    if model:
        url += f"?model={model}"

    all_limits = []
    while url:
        response = requests.get(url, headers=headers)
        response.raise_for_status()
        data = response.json()
        all_limits.extend(data["data"])
        # ページネーション対応
        next_page = data.get("next_page")
        if next_page:
            url = f"{BASE_URL}/organizations/rate_limits?page={next_page}"
        else:
            url = None

    return all_limits


def get_workspace_rate_limits(workspace_id):
    """ワークスペースのオーバーライドを取得する"""
    url = f"{BASE_URL}/organizations/workspaces/{workspace_id}/rate_limits"

    all_limits = []
    while url:
        response = requests.get(url, headers=headers)
        response.raise_for_status()
        data = response.json()
        all_limits.extend(data["data"])
        next_page = data.get("next_page")
        if next_page:
            url = (
                f"{BASE_URL}/organizations/workspaces/"
                f"{workspace_id}/rate_limits?page={next_page}"
            )
        else:
            url = None

    return all_limits


def check_usage_against_limits(usage_rpm, model="claude-opus-4-7", threshold=0.8):
    """使用量がリミットの閾値を超えていないか確認する"""
    limits = get_org_rate_limits(model=model)
    for group in limits:
        for limit in group["limits"]:
            if limit["type"] == "requests_per_minute":
                ratio = usage_rpm / limit["value"]
                if ratio >= threshold:
                    print(
                        f"[ALERT] {model}: RPM usage {usage_rpm}/{limit['value']} "
                        f"({ratio:.0%}) exceeds threshold {threshold:.0%}"
                    )
                else:
                    print(
                        f"[OK] {model}: RPM usage {usage_rpm}/{limit['value']} "
                        f"({ratio:.0%})"
                    )


# 使用例
org_limits = get_org_rate_limits()
for group in org_limits:
    print(f"Group: {group['group_type']}")
    print(f"  Models: {', '.join(group['models'])}")
    for limit in group["limits"]:
        print(f"  {limit['type']}: {limit['value']}")

# アラートチェック
check_usage_against_limits(usage_rpm=3500, model="claude-opus-4-7")

ユースケース

🌐 ゲートウェイ・プロキシ設定の自動同期

ゲートウェイやプロキシにハードコードされたレートリミット値を Rate Limits API からの動的取得に切り替えることで、Anthropic 側のリミット変更時に発生するドリフトを防止できます。起動時またはスケジュール実行で最新のリミット値を取得し、ローカル設定を自動更新します。

🔔 内部アラートの生成

Usage and Cost API と Rate Limits API を組み合わせることで、リミット到達前のアラートを自動生成できます。使用量がリミットの閾値 (例: 80%) を超えた場合に通知を送信し、API 利用の安定性を確保します。

📋 ワークスペース設定の監査

複数ワークスペースのオーバーライド設定が期待値と一致しているかを自動検証できます。ワークスペースレスポンスの org_limit フィールドにより、オーバーライド値と組織レベルのリミットを容易に比較でき、IaC ツールとの統合による設定の一貫性検証も可能です。

開発者への影響と導入ステップ

1

Admin API キーの取得

Claude Console で管理者ロールを持つユーザーが Admin API キーをプロビジョニングします。

2

既存のハードコード値の確認

ゲートウェイやプロキシにハードコードされたレートリミット値がある場合、Rate Limits API からの動的取得に切り替えを検討します。

3

監視・アラートの統合

Usage and Cost API と Rate Limits API を組み合わせた監視ダッシュボードやアラートの構築を検討します。

4

ページネーション対応

将来の拡張に備えて、next_page フィールドを使ったループ処理を実装します。

ℹ️
破壊的変更なし: Rate Limits API は新規追加の読み取り専用 API であるため、既存のアプリケーションへの破壊的変更はありません。段階的に導入できます。
