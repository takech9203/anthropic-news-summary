Before / After

Before

  • 末尾システムコンテキストが毎回新規トークンとして課金
  • 並列セッションがスリープ復帰後に全て同時ログアウト

After

  • 正しくキャッシュされ低コストで処理
  • トークンリフレッシュの競合を解消

主な機能・修正

📡 サブエージェントテキスト転送

--forward-subagent-text フラグおよび CLAUDE_CODE_FORWARD_SUBAGENT_TEXT 環境変数でサブエージェントの出力と思考を stream-json に含める

💰 プロンプトキャッシュ課金修正

Bedrock/Vertex/Mantle/Foundry で末尾システムコンテキストブロックの毎回課金を修正し、コスト最適化を実現

🔄 並列セッション安定化

スリープ復帰後の同時ログアウト防止。トークンリフレッシュの競合状態を解消

🛡️ 権限プレビューサニタイズ

Unicode 制御文字による承認メッセージの視覚的改ざんを防止。安全な権限確認を保証

🪝 PreToolUse フック ask 判定

auto モードでもフックの ask 判定を確実に尊重。意図しないツール実行を防止

コードサンプル

Bash サブエージェントテキスト転送 
# フラグでサブエージェントのテキスト・思考を転送
claude --output-format stream-json --forward-subagent-text "タスクを実行してください"

# 環境変数で設定
export CLAUDE_CODE_FORWARD_SUBAGENT_TEXT=1
claude --output-format stream-json "タスクを実行してください"
Bash 環境変数の新しい数値フォーマット 
# 環境変数の新しい数値フォーマット
export CLAUDE_CODE_MAX_TOKENS=1e6
export CLAUDE_CODE_TIMEOUT=64_000

サブエージェントの中間出力をリアルタイム監視してデバッグ効率を向上。stream-json と組み合わせて複雑なワークフローの可視化を実現

🏢 Bedrock/Vertex エンタープライズユーザー

プロンプトキャッシュ修正によるコスト最適化。末尾システムコンテキストの不要課金が解消され、大規模運用のコストを削減

🖥️ マルチセッション利用者

安定したバックグラウンドエージェント運用。スリープ復帰後のセッション断を防ぎ、長時間タスクを安心して実行可能に
