新機能ハイライト

🌐 ゲートウェイモデル選択

ANTHROPIC_BASE_URL のゲートウェイ /v1/models エンドポイントからモデル一覧を自動取得。エンタープライズ環境で利用可能なモデルを直接選択可能に。

🗑️ project purge

プロジェクトの全状態を一括削除。--dry-run、--yes、--interactive、--all オプション対応。CI/CD パイプラインのクリーンアップに最適。

🔓 skip-permissions 拡張

--dangerously-skip-permissions が .claude/、.git/、.vscode/ への書き込みもバイパス。壊滅的削除は安全対策として維持。

🔑 OAuth 手動コード入力

claude auth login でブラウザコールバック不可環境 (WSL2、SSH、コンテナ) でのコード貼り付け認証に対応。

📊 OTEL スキルイベント

claude_code.skill_activated が全スラッシュコマンドで発火。invocation_trigger 属性で発火元を識別可能。

🔴 Auto モード改善

権限チェック停滞時にスピナーが赤色に変化。ツール実行中との混同を防止。

セキュリティ修正

🔒 マネージド設定のサンドボックス制限修正

優先度の高いマネージド設定ソースに sandbox ブロックが含まれていない場合、allowManagedDomainsOnly / allowManagedReadPathsOnly が無視される問題を修正。マルチテナント環境でのセキュリティポリシーの一貫性が確保されます。マネージド設定を使用する環境では速やかなアップデートを推奨。

Windows 対応改善

🪟 PowerShell 7 検出

Microsoft Store、PATH なし MSI、.NET グローバルツールでインストールされた PowerShell 7 を自動検出。

⚡ PowerShell 優先シェル

PowerShell ツール有効時、Bash の代わりに PowerShell がプライマリシェルとして使用されます。

📋 クリップボードセキュリティ

コピー内容がプロセスコマンドライン引数に露出する問題を修正。EDR/SIEM テレメトリ対策。22KB 超のクリップボードも対応。

🇯🇵 CJK 文字化け修正

Windows の no-flicker モードで日本語、韓国語、中国語がガーブル表示される問題を修正。

コード例

Bash claude project purge の使用例 
# プロジェクト状態のパージ（ドライラン - 実際には削除しない）
claude project purge /path/to/project --dry-run

# プロジェクト状態のパージ（確認なし）
claude project purge /path/to/project --yes

# インタラクティブモードでパージ（削除対象を選択）
claude project purge /path/to/project --interactive

# 全プロジェクトの状態をパージ
claude project purge --all
Bash ゲートウェイ連携の設定例 
# カスタムゲートウェイを設定
export ANTHROPIC_BASE_URL="https://gateway.example.com/api"

# Claude Code を起動
claude

# /model コマンドでゲートウェイのモデル一覧が表示される

# ターミナルに OAuth コードを貼り付けて認証を完了
