2026-07-07

📋 概要

Claude Apps に 2 つの重要なアップデートが発表された。Claude Cowork がマルチデバイス対応し、Web およびモバイルで利用可能に。さらに Microsoft 365 コネクタが書き込み対応に大幅拡張され、メール送信、カレンダー管理、ファイル操作が可能になった。

📱

Cowork マルチデバイス対応

デスクトップ専用だった Cowork が Web ブラウザおよびモバイルアプリで利用可能に。あらゆるデバイスからアクセスできる

☁️

リモートセッション実行

セッションがリモートサーバー上で実行されるため、ラップトップを閉じても作業が継続。デバイスオフラインでもタスクを自動実行する

✏️

Microsoft 365 書き込みツール

メール送信、カレンダー管理、OneDrive/SharePoint へのファイル作成・更新が可能に。読み取り専用から大幅拡張

🏠

統合ホーム画面

Chat と Cowork が 1 つのホーム画面を共有し、プロジェクトとアーティファクトを一元管理できる

🔄 Before / After

❌ Before

  • Cowork はデスクトップアプリ専用
  • ローカルマシンのリソースに依存
  • ラップトップを閉じるとセッション中断
  • MS365 は検索と読み取りのみ
  • メール送信やファイル作成は不可
  • Chat と Cowork が別画面

✅ After

  • Web/モバイル/デスクトップ全対応
  • リモートサーバーで実行、デバイス非依存
  • バックグラウンドで作業継続
  • MS365 メール送信・カレンダー管理可能
  • OneDrive/SharePoint ファイル操作可能
  • 統合ホーム画面で一元管理

# Microsoft Entra での権限同意 (PowerShell)
# Claude の Microsoft 365 コネクタ用サービスプリンシパルに対して
# 書き込み権限を付与する管理者操作の例

# 1. Azure AD モジュールのインポート
Import-Module Microsoft.Graph

# 2. 管理者としてサインイン
Connect-MgGraph -Scopes "Application.ReadWrite.All"

# 3. Claude コネクタのサービスプリンシパルを確認
$sp = Get-MgServicePrincipal -Filter "displayName eq 'Claude for Microsoft 365'"

# 4. 付与される権限の確認
# - Mail.ReadWrite (メール読み書き)
# - Mail.Send (メール送信)
# - Calendars.ReadWrite (カレンダー読み書き)
# - Files.ReadWrite.All (OneDrive/SharePoint ファイル読み書き)
# - MailboxSettings.ReadWrite (メールボックス設定)

# 5. 管理者同意の付与
# Azure Portal > Enterprise Applications > Claude for Microsoft 365
# > Permissions > Grant admin consent

💡 ユースケース

📱 クロスデバイスワークフロー

オフィスのデスクトップで Cowork セッションを開始し、移動中にモバイルから進捗確認、帰宅後に Web ブラウザから結果を確認。デバイスに縛られない連続的な作業が可能になる

✉️ エンタープライズメール自動化

Claude に定例報告メールの作成・送信を依頼。カレンダーの空き時間を確認し、会議を自動スケジュール。議事録を OneDrive に自動保存するエンドツーエンドのワークフロー

⏰ スケジュール実行による自動タスク

毎朝のレポート集計、定期的なデータ分析、週次のファイル整理など、cron のようにデバイスがオフラインでもタスクを自動実行。真の「バックグラウンド AI ワーカー」として機能する

🏢 IT 管理者によるセキュアな展開

Microsoft Entra 管理者の同意と組織管理者の有効化という 2 段階承認で、セキュリティとコンプライアンスの要件を満たしながら段階的に導入可能

👨‍💻 対象ユーザーと必要なアクション

⭐ Max プランユーザー

Cowork の Web/モバイル対応が最初にロールアウト。即座に新機能を利用可能

🏢 エンタープライズ管理者

MS365 書き込みツールの有効化に Entra 管理者の同意と組織管理者の設定が必要

📱 モバイルワーカー

デスクトップを利用できない環境でも Cowork セッションにアクセスし継続作業可能

🔒 IT セキュリティチーム

新たな書き込み権限の評価とリスクアセスメントが必要

▶ 必要なアクション

  • Max プランユーザー: Web ブラウザまたはモバイルアプリから Cowork 機能を試用可能
  • 既存 Cowork ユーザー: デスクトップセッションがクラウドに同期され、他のデバイスからアクセス可能に
  • Microsoft Entra 管理者: 更新されたパーミッションセットに同意する必要あり
  • 組織の Claude 管理者: 書き込みツールを明示的に有効化する必要あり
  • Teams 利用者: Teams は引き続き読み取り専用のため、書き込み操作は対象外
