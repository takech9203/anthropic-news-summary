主要バグ修正 (18 件)

🗂️ コンテキスト・セッション管理

1M コンテキスト固着: 使用量クレジットなしの 1M セッションが自動コンパクト化されるように

画像処理エラー: 複数画像含む会話での繰り返しエラーを解消

🤖 バックグラウンドエージェント

スピナー残留: ワーカー応答後 30 秒間の Working 表示を修正

設定誤読: プレウォームワーカーでの別ディレクトリ設定読み込みを修正

EAUTH エラー: デーモン自動更新後の接続失敗を修正

停止後アクティブ表示: ネストエージェント停止後のパネル表示を修正

🎛️ モデル選択 (6 件)

availableModels: サブエージェント・アドバイザーへの制限適用、1M 行表示、Bedrock 未提供モデル、ID 二重化、opusplan 設定を一括修正

🔐 パーミッション・ワークフロー

ワイルドカードドメイン: WebFetch ドメインルール・パス中間ワイルドカードのマッチング修正

ワークフロー検証: コメント内の Date.now() 等でのバリデーション失敗を修正

🖥️ UI・インタラクション

プロンプト履歴・メモリ・マウス・プラグイン UI: サブエージェントタブ、リモートメモリ、Windows コンソール、プラグインブラウザの各種修正

改善 (8 件)

⚡ パフォーマンス: 長い会話での冗長メッセージ正規化削除、ストリーミング状態差分検出でレスポンスタイム改善

💻 CPU 使用量: /goal チップの 5 Hz 再レンダリング廃止、サブエージェント並列実行中の不要 UI 更新削減

🌐 Chrome ツール: ブラウザツールが単一バッチ呼び出しで読み込みに。個別呼び出しから最適化

💬 拒否メッセージ: Usage Policy 拒否時に新セッション開始/モデル変更を提案

🔍 /code-review: ultra オプションが未サインイン時も表示、アカウント要件を説明

📡 Remote Control: フッターインジケーターを「/rc active」に短縮、狭いターミナルで非表示

🔁 /loop 表示: リモートセッションでの不適切な推奨を停止

🖥️ VS Code: PowerShell ツール呼び出しを適切なコマンド表示に、ANSI コード除去

ユースケース

🏗️ 大規模リファクタリング

メインエージェントが全体のアーキテクチャ設計を担当し、サブエージェントが各コンポーネントの実装を並列で分担。さらに各サブエージェントがテスト生成を委託する 3 階層構造

🔬 並列コードレビュー

コードレビューエージェントがセキュリティ監査、パフォーマンス分析、スタイルチェックを別々のサブエージェントで並列実行。結果を統合して最終レビューを生成

☁️ Bedrock + AWS Config 連携

AWS_REGION を設定せずに ~/.aws/config のリージョン設定に依存する環境で、追加設定なしで Claude Code が正しいリージョンを検出

推奨アクション

claude update で v2.1.172 に更新

Bedrock ユーザー: ~/.aws/config のリージョン設定が正しく検出されることを /status で確認

1M コンテキスト利用者: スタックしていたセッションが自動解消されることを確認

管理者: availableModels がサブエージェント・アドバイザーにも適用されるため設定を確認

パーミッション設定: ワイルドカードドメインルールが正しく機能するようになったことを確認
