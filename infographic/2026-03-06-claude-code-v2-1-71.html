新機能 v2.1.71

🔄 /loop コマンド

プロンプトやスラッシュコマンドを一定間隔で繰り返し実行する機能。

  • 例: /loop 5m check the deploy
  • セッション内で定期タスクを自動化

⏰ cron スケジューリング

セッション内で定期実行プロンプトをスケジュール可能に。より柔軟なタスク自動化を実現します。

🎤 音声キーバインド

音声モードのアクティベーションキーを keybindings.json でカスタマイズ可能に。デフォルトはスペースキー。

🛡️ Bash 自動承認リスト拡張

fmt、comm、cmp、numfmt、expr、test、printf、getconf、seq、tsort、pr を追加。

VS Code 強化 v2.1.70

📋 セッションリスト

アクティビティバーにスパークアイコンを追加し、全 Claude Code セッションを一覧表示。セッション管理が直感的に。

📄 プランビュー

プランのフルマークダウン表示とコメント追加機能。作業計画の確認と共有が容易に。

🔌 MCP 管理ダイアログ

/mcp コマンドでサーバーの有効化・無効化、再接続、OAuth 認証管理がネイティブダイアログで可能に。

Before / After

Before

  • 長時間セッションで stdin がフリーズ
  • 音声モード有効時に 5-8 秒の起動遅延
  • OAuth トークンリフレッシュで UI フリーズ
  • スリープ復帰後の再接続に最大 10 分
  • CJK 文字がクリップボードで文字化け

After

  • stdin フリーズを完全に解消
  • CoreAudio 初期化を非同期化し遅延なし
  • OAuth トークンリフレッシュを直列化
  • ブリッジ再接続が数秒に短縮
  • PowerShell Set-Clipboard で正常動作

主要な修正

v2.1.71 で 17 件、v2.1.70 で 19 件の修正が含まれています。

⌨️ stdin フリーズ: 長時間セッションでキーストロークが処理されなくなる問題を解消
🔊 起動時フリーズ: 音声モード有効時の CoreAudio 初期化がメインスレッドをブロックする問題を解消
🔑 OAuth 再接続: claude.ai コネクタが OAuth トークンリフレッシュ後に再接続できない問題を修正
🍴 フォーク会話: /fork で分岐した会話が同じプランファイルを共有し、編集が上書きされる問題を修正
🖼️ Read ツール画像処理: 画像処理失敗時にオーバーサイズの画像がコンテキストに挿入される問題を修正
🔌 プラグインインストール消失: 複数の Claude Code インスタンスを同時実行した際にプラグインが失われる問題を修正
🌐 API 400 エラー: ANTHROPIC_BASE_URL でサードパーティゲートウェイ使用時のツール検索エラーを解消
📋 Windows/WSL クリップボード: CJK 文字や絵文字が文字化けする問題を PowerShell Set-Clipboard で修正
💾 プロンプトキャッシュ無効化: MCP サーバーの instructions 接続時にキャッシュが破棄される問題を修正

パフォーマンス改善

🖥️ 再レンダリング削減

ターン中のプロンプト入力の再レンダリングを約 74% 削減。入力中のちらつきが大幅に軽減されます。

💾 起動メモリ削減

カスタム CA 証明書未使用ユーザーのメモリを約 426KB 削減。ネイティブ画像プロセッサの読み込みを初回使用時まで遅延。

📡 ポーリング最適化

Remote Control の /poll レートを接続中は 10 分に 1 回に削減し、サーバー負荷を約 300 倍低減。

🗜️ コンパクション改善

画像を保持してプロンプトキャッシュの再利用を可能に。長いセッションでのコスト効率が向上。

技術的な詳細

🔄

CoreAudio 初期化の非同期化

macOS でのシステムウェイク後の音声モード初期化をメインスレッドから分離し、起動時のブロッキングを解消。

🔒

OAuth トークンリフレッシュの直列化

多数のコネクタが同時にリフレッシュする際の競合を防止。UI フリーズの根本原因を解消。

🛡️

画像処理のフォールバック改善

画像処理失敗時にオーバーサイズのデータがコンテキストに流入しないようガードを追加。

📡

ポーリング戦略の最適化

接続中のポーリング頻度を大幅に削減しつつ、接続断時には即座に高速ポーリングに切り替え。

CLI ヘビーユーザー

Claude Code CLI を日常的に利用している開発者は、stdin フリーズの修正とパフォーマンス改善により、長時間セッションでの作業体験が大幅に向上します。/loop コマンドで定期タスクの自動化も可能になりました。

VS Code ユーザー

カスタム Bedrock エンドポイントやサードパーティゲートウェイを利用しているチームは、API 400 エラーや effort パラメータエラーの修正により、より安定した運用が可能になります。プラグインの消失問題も解消されました。

まとめ

Claude Code v2.1.70-v2.1.71 は、合計 57 件の変更を含む大規模なリリースです。v2.1.71 で追加された /loop コマンドにより、デプロイ監視やテスト実行などの定期タスクをセッション内で自動化できるようになりました。安定性の面では、長時間セッションでの stdin フリーズ、音声モードの起動遅延、OAuth 再接続の失敗など、日常的な使用体験に直結する問題が多数修正されています。VS Code 拡張機能もセッション管理、プランビュー、MCP サーバー管理の 3 つの新機能で大幅に強化されており、全ての Claude Code ユーザーに早期のアップデートを推奨します。

全てのユーザーに早期アップデートを推奨します: npm update -g @anthropic-ai/claude-code
