概要

🧠

Memory Store の導入

ワークスペーススコープのテキストドキュメントコレクションを作成可能。Claude に最適化されたストレージとして、エージェントが自然にファイルとして読み書きできる形式を採用しています。

🔄

イミュータブルバージョン管理

すべての変更が memver_... プレフィックスのバージョンとして記録され、30 日間の履歴を保持。完全な監査証跡が確保されます。

🌐

多言語 SDK 対応

Python、TypeScript、Go、Java、C#、PHP、Ruby の各 SDK および ant CLI で利用可能。managed-agents-2026-04-01 ヘッダーで有効化します。

🔒

セキュリティ制御

read_write と read_only のアクセスレベルにより、エージェントの権限を適切に管理。プロンプトインジェクションからの保護が可能です。

🔏

楽観的並行性制御

content_sha256 プリコンディションによるセーフな更新をサポート。複数セッションからの同時更新を安全に処理できます。

Before (Memory なし)

  • ❌ セッションごとに独立した記憶
  • ❌ 前回の学びを引き継げない
  • ❌ 外部 DB の構築が必要
  • ❌ ユーザー情報の再入力
→ Memory 導入

After (Memory 有効)

  • ✅ セッション横断の記憶保持
  • ✅ ユーザーの好み・慣例を永続化
  • ✅ ファーストパーティのストレージ
  • ✅ /mnt/memory/ でファイル操作

Memory Store アーキテクチャ

Memory Store は 3 階層構造で構成されます。Store がコレクションを管理し、Memory が個々のドキュメントを保持し、Version が変更履歴を記録します。

Store

🗂️ Memory Store

ワークスペーススコープのコレクション。組織あたり最大 1,000 個作成可能。名前と説明を持ち、説明はシステムプロンプトに自動追加されます。

Memory

📝 Memory

各ストア内に最大 2,000 個保存可能なテキストドキュメント。パスで識別され、1 つあたり最大 100 kB。ファイルツールで読み書きを行います。

Version

🔖 Memory Version

メモリの各変更を記録するイミュータブルなスナップショット。memver_... プレフィックスの ID で管理され、30 日間保持されます。

セキュリティとアクセス制御

🔓 read_write

用途: ユーザー固有の学習、動的な知識蓄積

リスク: プロンプトインジェクションによる汚染リスクあり

推奨: 信頼された入力を処理するエージェント向け

🔒 read_only

用途: 参照資料、静的なガイドライン、テンプレート

リスク: 書き込み不可のため汚染リスクなし

推奨: 信頼されていないユーザー入力を処理するエージェント向け

⚠️ プロンプトインジェクションに関する重要な注意事項

Memory Store はデフォルトで read_write アクセスでアタッチされます。エージェントが信頼されていない入力を処理する場合、プロンプトインジェクションによって悪意のあるコンテンツが Memory Store に書き込まれるリスクがあります。参照用の Memory Store には必ず read_only を指定することが強く推奨されます。

コード例

Python Memory Store の作成からセッションアタッチまで 
import anthropic

client = anthropic.Anthropic()

# 1. Memory Store を作成
store = client.beta.memory_stores.create(
    name="User Preferences",
    description="Per-user preferences and project context.",
)

print(f"Memory Store created: {store.id}")

# 2. 初期コンテンツをシード
client.beta.memory_stores.memories.create(
    store.id,
    path="/formatting_standards.md",
    content="All reports use GAAP formatting. Dates are ISO-8601...",
)

client.beta.memory_stores.memories.create(
    store.id,
    path="/user/preferences.md",
    content="Preferred language: Japanese\nTimezone: Asia/Tokyo\n",
)

# 3. セッションに Memory Store をアタッチ
session = client.beta.sessions.create(
    agent=agent.id,
    environment_id=environment.id,
    resources=[
        {
            "type": "memory_store",
            "memory_store_id": store.id,
            "access": "read_write",
            "instructions": (
                "User preferences and project context. "
                "Check before starting any task."
            ),
        }
    ],
)

print(f"Session created with memory: {session.id}")

Python 楽観的並行性制御によるセーフな更新 
import hashlib

# 現在のメモリを取得
memory = client.beta.memory_stores.memories.retrieve(
    store.id,
    path="/user/preferences.md",
)

# コンテンツの SHA256 ハッシュを計算
content_hash = hashlib.sha256(memory.content.encode()).hexdigest()

# ハッシュをプリコンディションとして指定して更新
# 他のプロセスが同時に更新した場合はエラーになる
client.beta.memory_stores.memories.update(
    store.id,
    path="/user/preferences.md",
    content="Updated preferences...",
    content_sha256=content_hash,
)

🎯 ドメイン特化型エージェント

ビジネスルール、業界固有の知識、フォーマット基準などを Memory Store に格納。ドメインエキスパートとしての出力品質を維持し、過去のミスから学習して精度を向上させます。

Claude Managed Agents の Memory 機能がパブリックベータとして公開されました。managed-agents-2026-04-01 ヘッダーで利用可能なこの機能により、エージェントはセッションを跨いでユーザーの好み、プロジェクト慣例、過去の学びなどを永続化できるようになります。
