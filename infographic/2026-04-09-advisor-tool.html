API 新機能

Advisor Tool -- エグゼキューターモデルにアドバイザーモデルの知性を融合

2026-04-09

高速なエグゼキューターモデルに、より高性能なアドバイザーモデルの戦略的ガイダンスを組み合わせる新しいツール

概要

🧠

マルチモデル連携

高速なエグゼキューター (Sonnet 4.6 / Haiku 4.5) が、生成途中でアドバイザー (Opus 4.6) に戦略的な指針を求める新パターン

🔄

単一リクエストで完結

全ての処理が 1 回の /v1/messages リクエスト内で完了。開発者側での追加ラウンドトリップは不要

⚡

コストと品質の最適化

Sonnet エグゼキューター + Opus アドバイザーで、Opus 単体に近い品質をより低コストで実現

🤖

自律的な判断

エグゼキューターがタスクの複雑さに応じてアドバイザーの呼び出しタイミングを自動決定

Before / After

Before

  • 全タスクを Opus で処理 = 高コスト
  • Sonnet / Haiku で妥協 = 品質低下
  • モデル選択は二者択一
  • 複雑なオーケストレーションが必要

After

  • 必要な時だけ Opus に相談
  • 大部分は Sonnet レートで処理
  • Opus 単体に近い品質を実現
  • 単一 API リクエストで完結

動作の仕組み

1

エグゼキューターの呼び出し

エグゼキューターが server_tool_use ブロックを生成。name: "advisor"、input: {} で空の入力。タイミングはエグゼキューターが自律的に判断する

2

アドバイザーの推論

Anthropic がサーバーサイドでアドバイザーモデルに対して別途推論を実行。システムプロンプト、全ツール定義、全ターン、全ツール結果を含む完全なトランスクリプトを参照する

3

アドバイスの返却

アドバイザーの応答が advisor_tool_result ブロックとして返される。シンキングブロックはドロップされ、アドバイステキストのみがエグゼキューターに到達する

4

エグゼキューターの継続

エグゼキューターがアドバイスを踏まえて生成を継続。アドバイザーの計画に基づいてツール呼び出しや実装を進める

モデル互換性

アドバイザーはエグゼキューターと同等以上の性能を持つモデルである必要があります。

エグゼキューターモデル アドバイザーモデル
Claude Haiku 4.5 claude-haiku-4-5-20251001 Claude Opus 4.6 claude-opus-4-6
Claude Sonnet 4.6 claude-sonnet-4-6 Claude Opus 4.6 claude-opus-4-6
Claude Opus 4.6 claude-opus-4-6 Claude Opus 4.6 claude-opus-4-6

ツールパラメータ

パラメータ 型 状態 説明
type string 必須 "advisor_20260301" を指定
name string 必須 "advisor" を指定
model string 必須 アドバイザーモデル ID (例: "claude-opus-4-6")
max_uses integer 任意 1 リクエストあたりの呼び出し上限
caching object | null 任意 プロンプトキャッシングの有効化

主な機能

🔗 他ツールとの共存

Web Search、カスタムツールと同じ tools 配列で共存可能。アドバイザーの計画がツール選択にも影響

💾 プロンプトキャッシング

エグゼキューター側とアドバイザー側の 2 つの独立したキャッシングレイヤーで最適化

🛡️ ZDR 対応

Zero Data Retention 契約がある場合、データは API レスポンス返却後に保存されない

📊 Batch 処理対応

バッチ処理に対応。usage.iterations がアイテムごとに報告される

🎯 コスト制御

max_uses でリクエストレベル、クライアント側カウントで会話レベルの制御が可能

⏸️ ストリーミング対応

アドバイザーのサブ推論中はストリームが一時停止し、完了後に結果が一括到着

エラーハンドリング

アドバイザー呼び出しが失敗しても、リクエスト自体は失敗しません。エグゼキューターがアドバイスなしで処理を継続します。

エラーコード 意味
max_uses_exceeded max_uses の上限に到達
too_many_requests アドバイザーのサブ推論がレートリミットに該当
overloaded キャパシティ上限に到達
prompt_too_long コンテキストウィンドウを超過
execution_time_exceeded サブ推論がタイムアウト
unavailable その他のアドバイザー障害

課金の仕組み

  • 💰 アドバイザー呼び出しはアドバイザーモデルのレートで別途課金
  • 📊 使用量は usage.iterations[] 配列で報告。type: "advisor_message" と type: "message" で分離
  • 📈 トップレベルの usage フィールドはエグゼキュータートークンのみを反映
  • 🎛️ max_tokens はエグゼキューター出力にのみ適用。アドバイザーのトークンには適用されない

コードサンプル

Python 基本的な使用方法 
import anthropic

client = anthropic.Anthropic()

response = client.beta.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=4096,
    betas=["advisor-tool-2026-03-01"],
    tools=[
        {
            "type": "advisor_20260301",
            "name": "advisor",
            "model": "claude-opus-4-6",
        }
    ],
    messages=[
        {
            "role": "user",
            "content": "Build a concurrent worker pool in Go with graceful shutdown.",
        }
    ],
)

print(response)
Python マルチターン会話 
tools = [
    {
        "type": "advisor_20260301",
        "name": "advisor",
        "model": "claude-opus-4-6",
    }
]

messages = [
    {"role": "user", "content": "Build a concurrent worker pool in Go."}
]

response = client.beta.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=4096,
    betas=["advisor-tool-2026-03-01"],
    tools=tools,
    messages=messages,
)

# advisor_tool_result ブロックを含む完全なレスポンスを追加
messages.append({"role": "assistant", "content": response.content})
messages.append({"role": "user", "content": "Now add a max-in-flight limit of 10."})

response = client.beta.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=4096,
    betas=["advisor-tool-2026-03-01"],
    tools=tools,
    messages=messages,
)

ユースケース

🖥️ コーディングエージェント

Sonnet エグゼキューターがコードを生成し、複雑なアーキテクチャ判断が必要な場面で Opus アドバイザーに相談。テスト結果やエラーの分析もアドバイザーが支援し、全体の品質を向上させる

🔍 マルチステップ調査パイプライン

複数の情報源を横断する調査タスクで、アドバイザーが調査戦略の立案と軌道修正を担当。エグゼキューターは個別の情報取得を高速に処理する

🖱️ コンピューター操作エージェント

画面操作の大部分は機械的だが、複雑なフロー判断が必要な場面でアドバイザーが戦略的ガイダンスを提供。コストを抑えつつ操作精度を向上させる

💡 不向きなケース

  • 単一ターンの Q&A (計画する対象がない)
  • ユーザーがモデルを選択するパススルー型
  • 全ターンでアドバイザーの全能力が必要なケース

開発者への必要なアクション

1️⃣

ベータヘッダーの追加

リクエストに anthropic-beta: advisor-tool-2026-03-01 ヘッダーを含める

2️⃣

ツール定義の追加

tools 配列に advisor_20260301 タイプのツールを追加する

3️⃣

SDK の更新

最新の Anthropic SDK にアップデート (Python、TypeScript、Go、C#、PHP、Ruby に対応)

4️⃣

マルチターン会話の対応

advisor_tool_result ブロックを含む完全なコンテンツを後続ターンで返す

5️⃣

課金ロジックの更新

usage.iterations[] を使用したコスト追跡ロジックの実装を検討する
