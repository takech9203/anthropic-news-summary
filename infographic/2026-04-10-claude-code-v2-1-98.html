セキュリティ修正 -- 重要

⚠ 14 件のセキュリティ修正には、任意コード実行につながる重大な脆弱性の修正が含まれています。特に --dangerously-skip-permissions や自動モードを使用している環境では、速やかなアップデートが必須です。
1

Bash ツール権限バイパスの修正

バックスラッシュエスケープされたフラグ (例: rm \-rf) が読み取り専用として自動許可され、任意コード実行につながる脆弱性を修正

2

複合 Bash コマンドのセキュリティチェック回避修正

パイプ、セミコロン、&& で連結された複合コマンドが自動モードで強制権限プロンプトをバイパスする問題を修正

3

/dev/tcp および /dev/udp リダイレクトの権限チェック

/dev/tcp/HOST/PORT や /dev/udp/HOST/PORT へのリダイレクトが自動許可されていた問題を修正。権限プロンプトが表示されるように変更

4

--dangerously-skip-permissions のサイレントダウングレード修正

Bash 経由で保護パスへの書き込み承認後、権限モードがサイレントに accept-edits モードにダウングレードされる問題を修正

5

ワイルドカード権限ルールのマッチング修正

Bash(cmd:*) や Bash(git commit *) のルールがコマンド内の余分なスペースやタブでマッチしない問題を修正

⚠ 残り 9 件のセキュリティ修正

環境変数プレフィックス付きコマンドの権限チェック、managed-settings の allow ルール即時反映、additionalDirectories のセッション中反映、Bash deny ルールのダウングレード修正、Bash 権限の誤検出修正、JavaScript プロトタイププロパティ名の権限ルール修正、エージェントチームメンバーの権限継承修正、grep -f / rg -f の権限チェックなど。詳細はレポートを参照してください。

新機能

☁️ Vertex AI セットアップウィザード

ログイン画面の「3rd-party platform」からインタラクティブにセットアップ。GCP 認証、プロジェクト設定、リージョン選択、認証情報検証、モデルピンニングを対話的にガイド

📂 Perforce モード

CLAUDE_CODE_PERFORCE_MODE 環境変数で有効化。読み取り専用ファイルへの Edit/Write が p4 edit のヒントを表示してエラーを返す

📡 Monitor ツール

バックグラウンドスクリプトからイベントをストリーミングする新しい Monitor ツール。リアルタイムでバックグラウンド処理を監視可能

🔐 サブプロセスサンドボックス

CLAUDE_CODE_SUBPROCESS_ENV_SCRUB で PID namespace 分離を有効化。CLAUDE_CODE_SCRIPT_CAPS でスクリプト実行回数を制限可能

セキュリティ修正: Before / After

Before -- v2.1.97

  • バックスラッシュエスケープで権限バイパス
  • 複合コマンドでチェック回避
  • /dev/tcp リダイレクトが自動許可
  • ワイルドカードルールがスペースでマッチ失敗
  • 権限モードがサイレントにダウングレード

After -- v2.1.98

  • エスケープされたフラグも正しく権限チェック
  • 複合コマンドを個別にセキュリティチェック
  • /dev/tcp リダイレクトで権限プロンプト表示
  • 余分なスペース/タブでも正しくマッチ
  • 権限モードの不正なダウングレードを防止

開発者への影響とアクション

🔒 全ての Claude Code ユーザー

14 件のセキュリティ修正は全ユーザーに影響。特に --dangerously-skip-permissions や自動モードを使用している環境では、任意コード実行につながる重大な脆弱性の修正が含まれており、最優先でアップデートが必要

速やかなアップデート必須

☁️ Google Cloud / Vertex AI ユーザー

新しいインタラクティブセットアップウィザードにより、GCP 認証からモデルピンニングまでの設定が大幅に簡素化。手動での環境変数設定が不要に

ウィザードを試す

📂 Perforce ユーザー

CLAUDE_CODE_PERFORCE_MODE 環境変数の設定により、Perforce 管理下のファイルへの意図しない上書きを防止。安全な開発ワークフローを実現

環境変数を設定

🏢 エンタープライズ環境

サブプロセスサンドボックス (CLAUDE_CODE_SUBPROCESS_ENV_SCRUB) とスクリプト実行回数制限 (CLAUDE_CODE_SCRIPT_CAPS) により、セキュリティポリシーへの準拠が容易に

サンドボックス設定を検討

🔌 MCP サーバー利用者

OAuth トークンリフレッシュの修正により、ADFS 等の IdP での再起動後のトークン問題が解消。Remote Control のメモリリーク修正も適用済み

再起動後の動作を確認

💻 Windows / macOS / kitty ターミナルユーザー

kitty キーボードプロトコルでの大文字入力修正、macOS テキスト置換修正、VS Code Windows の git-bash エラー修正など、プラットフォーム固有の問題が解消

確認推奨

📋 権限ルール使用者への注意

toString 等の JavaScript プロトタイププロパティ名を権限ルールに使用していた場合、settings.json がサイレントに無視されていた可能性があります。アップデート後に設定が正しく適用されているか確認してください。
