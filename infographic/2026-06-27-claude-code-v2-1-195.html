Claude Code v2.1.195

2026-06-27 | フックマッチャー修正、音声入力改善、バックグラウンドエージェント安定性向上

概要

Claude Code v2.1.195 は、フックマッチャーのハイフン付き識別子における部分一致バグの修正、音声入力の複数プラットフォームにおける改善、バックグラウンドエージェントの安定性向上を中心としたリリースである。特に日本語を含むスペースなし言語での音声入力自動送信が修正され、日本語話者にとって重要なアップデートとなっている。

主なハイライト

🎯

フックマッチャーの完全一致化

ハイフン付き識別子の部分一致バグを修正し、完全一致に変更。意図しないツールへのマッチを防止し、より厳密な制御が可能に。

🎙️

音声入力の改善

日本語、中国語、タイ語などスペースなし言語での自動送信修正。macOS での長時間セッション中の無音キャプチャ問題も解決。

🔄

バックグラウンドエージェントの安定性向上

ジョブ消失、クラッシュ後の空白画面、デーモン到達不能など複数の問題を修正。長時間稼働エージェントの信頼性が大幅向上。

🖱️

マウスクリック無効化オプション追加

CLAUDE_CODE_DISABLE_MOUSE_CLICKS 環境変数でフルスクリーンモードのマウス操作を無効化可能に。tmux との干渉を回避。

フックマッチャーの動作変更

v2.1.195 以前では、フックマッチャーがハイフン付き識別子に対して部分一致を行っていた。本バージョンからは完全一致に変更され、より厳密な制御が可能となった。

変更前 (部分一致)

  • brave-search で指定すると:
  • mcp__brave-search__web_search にマッチ
  • mcp__brave-search-extended__deep_search にもマッチ (意図しない)

変更後 (完全一致)

  • brave-search で指定すると:
  • brave-search のみにマッチ
  • 全ツール対象には mcp__brave-search__.* を使用

変更カテゴリ別詳細

✨ 新機能

フルスクリーンモード制御の新環境変数

  • マウスクリック、ドラッグ、ホバーを無効化
  • ホイールスクロールは引き続き有効
  • tmux などターミナルマルチプレクサとの干渉回避に有用

🐛 バグ修正

信頼性とセキュリティの向上

  • フックマッチャーの完全一致化
  • macOS 音声入力の無音キャプチャ修正
  • スペースなし言語の自動送信修正
  • 外部プラグインの同意プロンプト修正
  • /plugin の有効化/無効化修正
  • バックグラウンドジョブの消失修正
  • クラッシュタスクの再表示修正
  • デーモン到達不能状態の修正

📈 改善

診断と表示の品質向上

  • Linux 音声モードの診断改善
  • claude agents 完了リストの表示改善
  • Remote セッション起動時のチェックリスト表示

🔒 セキュリティ

プラグインの安全性強化

  • 外部プラグインの明示的インストール同意を全ローダーパスで要求
  • プラグイン名の不一致によるバイパスを防止

コード例

bash マウスクリック無効化とアップデート 
# フルスクリーンモードでマウスクリックを無効化 (ホイールスクロールは有効)
export CLAUDE_CODE_DISABLE_MOUSE_CLICKS=1
claude

# アップデートの実行
claude /update
json .claude/settings.json - フックマッチャーの正しい設定例 
{
  "hooks": {
    "tool_use": {
      "matchers": [
        "code-reviewer",
        "mcp__brave-search__.*"
      ],
      "action": "allow"
    }
  }
}

code-reviewer は完全一致で特定ツールにマッチし、mcp__brave-search__.* はワイルドカードで brave-search MCP サーバーの全ツールにマッチする。

技術的な詳細

フックマッチャーの動作変更

v2.1.195 以前では部分一致により brave-search が mcp__brave-search-extended__deep_search のような意図しないツールにもマッチしていた。本バージョンからは完全一致に変更され、ワイルドカードで明示的にパターンを指定する必要がある。

音声入力の自動送信ロジック

従来の自動送信ロジックはスペース文字の入力をトリガーとして使用していたが、日本語、中国語、タイ語ではスペースなしで文章が構成されるため機能しなかった。新しい実装では言語固有の区切り検出メカニズムが導入されている。

開発者への影響

⚠️

フック設定の確認が必要

ハイフン付き識別子を使用しているフックマッチャーが完全一致に変更されたため、既存の設定を確認すること。特に MCP サーバーの全ツールにマッチさせたい場合は正規表現パターンへの変更が必要。

🌐

日本語話者: 音声入力が正常動作

スペースなし言語での自動送信が正しく機能するようになり、音声入力体験が大幅に改善。

🔧

バックグラウンドエージェント利用者

ジョブ消失やデーモン到達不能の修正により安定性が大幅向上。以前消失したジョブがある場合は更新後に状態を確認することを推奨。

🖥️

環境変数の活用

フルスクリーンモードでマウス操作が干渉する場合は CLAUDE_CODE_DISABLE_MOUSE_CLICKS=1 を設定して解決可能。

想定されるユースケース

MCP ツールを選択的に制御したい開発者

フックマッチャーの完全一致化により、特定のツールのみを許可/制限する設定が明確に記述できるようになった。mcp__brave-search__web_search のみを許可し、他のツールはブロックするといった精密な制御が可能。

日本語環境で音声入力を活用する開発者

日本語での音声入力後、自動送信が正しくトリガーされるようになった。従来はスペースベースの検出のため手動送信が必要だったが、言語固有の区切り検出により自然なワークフローが実現。

長時間稼働のバックグラウンドタスクを運用する開発者

バージョン更新時のジョブ消失やデーモンの到達不能状態が修正され、本番環境での継続的なエージェント運用がより信頼できるものとなった。

tmux 上で Claude Code を使用する開発者

CLAUDE_CODE_DISABLE_MOUSE_CLICKS=1 を設定することで、フルスクリーンモードのマウスイベントが tmux のペイン操作と干渉する問題を回避できる。

まとめ

Claude Code v2.1.195 は、信頼性と安定性を重視したリリースである。フックマッチャーの完全一致への変更は破壊的変更となる可能性があるため、ハイフン付き識別子を使用しているフック設定の見直しを推奨する。日本語を含むスペースなし言語での音声入力修正は、日本語環境での Claude Code 利用体験を大きく改善するものである。バックグラウンドエージェント関連の複数の修正により、長時間稼働するエージェントの安定性が向上し、本番環境での利用がより信頼できるものとなった。
