Before / After

MCP 接続ハンドリング

Before

SSE/HTTP トランスポートでサーバー接続がレスポンス途中で切断された場合、MCP ツール呼び出しが無限にハングアップ。非ストリーミングフォールバックも API 到達不能時に数分間ハング。stdio MCP サーバーが非 JSON 行を出力すると即座に切断

After

タイムアウト検出により接続断を適切に検出し、リトライまたはエラーハンドリングに移行。非ストリーミングフォールバックも迅速にエラーフィードバックを返却。stdio MCP サーバーの非 JSON 出力を許容して継続

Ctrl+O の動作変更

Before

Ctrl+O がトランスクリプト表示とフォーカスビューの両方を制御。機能が混在し、個別の制御が不可能

After

Ctrl+O は通常トランスクリプトと verbose トランスクリプトの切り替えのみ。フォーカスビューは新しい /focus コマンドで独立して制御

バグ修正 24 件

🔗 MCP / 接続修正 3

  • SSE/HTTP トランスポートでレスポンス中にサーバー接続が切断された場合に、MCP ツール呼び出しが無限にハングアップする問題を修正
  • API に到達できない場合に非ストリーミングフォールバックのリトライが数分間ハングする問題を修正
  • stdout に JSON 以外の行を出力する stdio MCP サーバーが最初の不正行で切断される問題を修正 (v2.1.105 リグレッション)

🖥 フルスクリーン / レンダリング修正 5

  • セッションリキャップ、ローカルスラッシュコマンド出力、その他のシステムステータス行がフォーカスモードで表示されない問題を修正
  • ツール実行中にテキストが選択されている場合にフルスクリーンモードで CPU 使用率が高くなる問題を修正
  • macOS Terminal.app および同期出力をサポートしないターミナルでの起動時にレンダリングが文字化けする問題を修正
  • フルスクリーンモードでリストがモーダルからはみ出る場合に /skills メニューがスクロールしない問題を修正
  • マルチツールコール中にキューに入れられたメッセージが一時的に 2 回表示される問題を修正

🔒 セキュリティ / パーミッション修正 3

  • 「エディタで開く」アクションが信頼されていないファイル名からのコマンドインジェクションに対して堅牢化
  • PermissionRequest フックが返す updatedInput が permissions.deny ルールに対して再チェックされるように。setMode:'bypassPermissions' の更新も disableBypassPermissionsMode を尊重
  • ツール呼び出しが失敗した場合に PreToolUse フックの additionalContext がドロップされる問題を修正

📂 セッション / Resume 修正 4

  • 実行中または異常終了したセッションで --resume を使用する際に、/rename で設定した名前ではなく最初のプロンプトが表示される問題を修正
  • サブエージェントのトランスクリプトを含むセッションディレクトリ全体が適切にクリーンアップされるように修正
  • セッションが古すぎる場合に汎用エラーではなく再ログインのプロンプトが表示されるように修正
  • claude.ai からのセッション名変更がローカル CLI セッションに永続化されるように修正

🔌 プラグイン / Skill 修正 2

  • マーケットプレイスエントリが省略している場合でも、plugin.json で宣言された依存関係がインストール時に尊重されるように修正
  • メッセージ途中で /<skill> 経由で呼び出された場合に disable-model-invocation: true が設定された Skill が失敗する問題を修正

⚙ SDK / headless 修正 2

  • CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC または CLAUDE_CODE_DISABLE_TERMINAL_TITLE 設定時に、自動タイトル生成で余分な Haiku リクエストが発行される問題を修正
  • パイプ出力の Ink 出力に単一の非常に幅の広い行が含まれる場合に過剰なメモリ割り当てが発生する問題を修正

⌨ 入力 / キーストローク修正 2

  • /tui やプロバイダーセットアップウィザードなどで CLI が再起動した後にキーストロークがドロップされる問題を修正
  • v2.1.108 のセッションリキャップ機能が Bedrock、Vertex、Foundry、DISABLE_TELEMETRY 環境でもデフォルトで有効に

📌 その他修正 3

  • 二重メッセージ表示: マルチツールコール中にキューに入れられたメッセージが一時的に 2 回表示される問題を修正
  • Ctrl+G 外部エディタでのレスポンス表示が有効化可能に
  • --resume/--continue でのスケジュールタスク復元をサポート

ユースケース

🖥 フルスクリーン開発者体験の最適化

/tui fullscreen でちらつきのないレンダリングに切り替え、autoScrollEnabled で長い出力のスクロール位置を固定、/focus でフォーカスビューを独立制御。macOS Terminal.app の文字化け修正や CPU 使用率修正と合わせ、快適なフルスクリーン開発環境を構築

📱 モバイルからの非同期タスク管理

Remote Control を有効にしてプッシュ通知を設定すれば、長時間ビルドやテスト完了時にモバイルで通知を受け取れます。/context、/exit、/reload-plugins もリモートから実行可能で、離席中のタスク管理が容易に

🔍 CI/CD パイプラインでの分散トレース統合

TRACEPARENT/TRACESTATE 環境変数を設定して claude --headless を実行すれば、Claude Code のセッションを上位の OpenTelemetry トレースに関連付け可能。CI/CD 全体のオブザーバビリティが向上

🛡 MCP エコシステムの安定運用

SSE/HTTP 接続断でのハングアップ修正と stdio MCP サーバーの非 JSON 出力への耐性回復により、MCP サーバーとの接続安定性が向上。/doctor で重複定義を検出し、設定ミスを事前に発見

移行ガイド

⚠ Ctrl+O とフォーカスビューの分離

従来 Ctrl+O はトランスクリプト表示とフォーカスビューの両方を制御していましたが、v2.1.110 では分離されました。Ctrl+O は通常トランスクリプトと verbose トランスクリプトの切り替えのみになり、フォーカスビューは新しい /focus コマンドで独立して切り替えます。

⚠ テレメトリ無効化環境でのリキャップオプトアウト

Bedrock、Vertex、Foundry、DISABLE_TELEMETRY 環境のユーザーはセッションリキャップがデフォルト有効になりました。不要な場合は CLAUDE_CODE_ENABLE_AWAY_SUMMARY=0 を設定するか、/config からオプトアウトしてください。
