安全性の強化

🚫 破壊的 Git コマンドのブロック

git reset --hard、git checkout -- .、git clean -fd、git stash drop がユーザーの明示的指示なしではブロックされる

🔒 git commit --amend の制限

現セッションでエージェント自身が作成したコミット以外への --amend を防止。過去のコミットの意図しない上書きを排除

🏗️ インフラ破棄コマンドの制御

terraform destroy、pulumi destroy、cdk destroy は対象スタックの明示指定が必須に

設定・UX の改善

⚠️ 非推奨モデル警告

リクエストモデルが deprecated または自動更新された場合に stderr で警告。print モードとエージェントフロントマターにも対応

🔗 attribution.sessionUrl 設定

Web / Remote Control セッションのコミットや PR から claude.ai セッションリンクを省略可能に

📋 /config --help

/config key=value で使用可能なすべてのショートハンドキーを一覧表示

🎛️ /config トグル動作変更

Enter/Space で設定変更、Esc で保存して閉じる動作に統一

🧹 起動画面の簡素化

"setup issues" 行を削除。設定確認は /doctor または --debug で実施

バグ修正

🔧

thinking.disabled.display 400 エラー: サブエージェント生成時・セッションタイトル生成時のエラーを修正

👥

⏰

スケジュールタスク分類: webhook トリガーがキーボード入力として扱われる問題を修正。タスク通知として正しく分類

影響を受けるユースケース

🤖 自動モードでの長時間作業

自動モードで複雑なタスクを実行する際、エージェントが誤って git reset --hard を実行してローカル変更を失うリスクが排除される。明示的に「変更を破棄してください」と指示した場合のみ実行が許可される。

👥 チームでの tmux チームメイト利用

シェルの rc ファイル初期化が遅い環境でもチームメイトペインの起動が安定し、バックグラウンドタスクがチームメイトのターン終了時に kill されなくなった。

🏢 CI/CD でのヘッドレス運用

MCP サーバーの認証スタブが SDK モードで露出する問題が修正され、ヘッドレス環境でのセキュリティが向上。非推奨モデル警告も stderr に出力されるため、CI ログで確認可能に。

必要なアクション

1
アップデートの実施: claude update コマンドで最新版に更新
2
/config の動作変更への適応: Esc キーが「保存して閉じる」に変更されたため、キャンセルには値の手動リセットが必要
3
attribution.sessionUrl の確認: コミットメッセージから claude.ai セッション URL を省略したい場合は新設定を有効化
4
/doctor の定期実行: 起動時の自動表示が削除されたため、設定問題は /doctor で能動的に確認

コードサンプル

Bash アップデートと設定確認 
# Claude Code を最新版にアップデート
claude update

# 設定の問題がないか確認
claude /doctor

# 新しい設定オプションの確認
claude /config --help

# セッション URL を省略する設定
claude /config attribution.sessionUrl=false
