Before / After: 権限チェックの変化

❌ Before (v2.1.212 以前)

  • 10,000 文字超のコマンド → 不正確な解析 → 自動実行
  • zsh 変数サブスクリプト → 無害テキストと誤判定
  • FD リダイレクト → パーサー差異で自動実行
  • dir/** → ツリー全体の dir/ にマッチ
  • docker --url → 権限なしで実行可能

✅ After (v2.1.214)

  • 10,000 文字超 → 常にプロンプト表示
  • zsh 構文 → 承認を要求
  • FD リダイレクト → fail closed
  • dir/** → cwd/dir のみにマッチ
  • docker --url → 権限プロンプト必須

🛡️ セキュリティ修正

  • dir/** 許可ルールのスコープ修正 — Edit(src/**) が cwd/src 配下のみに限定。ツリー内の任意の src/ への書き込みを防止
  • Windows PowerShell 5.1 権限バイパス — PowerShell 5.1 セッションでの権限チェックバイパスを修正
  • FD リダイレクト形式 — Bash のパーサーと権限アナライザの差異を突いたバイパスを修正
  • 長大コマンド判定 — 10,000 文字超のコマンドは常にプロンプト表示に変更
  • zsh 変数サブスクリプト — [[ ]] 内の zsh 構文が無害テキスト扱いされていた問題を修正
  • help/man コマンド — 特定のオプションやコマンド置換を含む help/man が自動承認されていた問題を修正
  • リモートセッション権限プロンプト — ローカル確認前に進行可能だった問題を修正
  • docker デーモンリダイレクト — --url, --connection, --identity 付き docker コマンドに権限プロンプトを追加

💻 コードサンプル

JSON settings.json — 権限ルール移行例 
{
  "permissions": {
    "allow": [
      "Edit(**/src/**)",
      "Edit(**/tests/**)",
      "Bash(npm test)",
      "Bash(npm run build)"
    ],
    "deny": [
      "Bash(rm -rf /)",
      "Edit(.env)"
    ]
  }
}

⚠️ v2.1.214 以降、Edit(src/**) は <cwd>/src 配下のみにマッチ。
ネストされた src/ ディレクトリにもマッチさせる場合は Edit(**/src/**) に変更が必要。

Bash OpenTelemetry コンテンツ長の設定 
# デフォルト (60 KB) を 120 KB に拡張
export CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH=122880

# アップデート実行
npm update -g @anthropic-ai/claude-code

# バージョン確認
claude --version

📋 移行ガイド: dir/** パターン

パターン 修正前 修正後 対応
Edit(src/**) allow 全ての src/ にマッチ cwd/src のみ 意図通りなら変更不要
hook if: src/** 全ての src/ にマッチ cwd/src のみ **/src/** に変更
deny src/** 任意の深さ 任意の深さ 変更不要

⚠️ 動作変更

📁 dir/** フック条件

単一セグメントの dir/** が cwd/dir のみにマッチ。任意深さには **/dir/** と明示的に記述が必要

📄 file コマンド権限

-m/--magic-file や -f/--files-from を使用する file コマンドは権限確認が必要に変更

🔌 接続プーリング

古い接続エラー後にプーリングを無効化。リトライは新しいソケットで実行

🔀 SessionStart フック

フォーク開始セッションの source が "resume" から "fork" に変更
