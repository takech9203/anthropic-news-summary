Claude Code v2.1.79 リリース

メモリ使用量 18MB 削減、VSCode リモートコントロール、認証・安定性の改善

2026-03-18

概要

💾

メモリ使用量 18MB 削減

全てのシナリオで起動時のメモリ使用量が約 18MB 削減されました。コンテナ環境やメモリ制約のあるサーバーで特に効果を発揮します。

🔑

--console フラグによる認証

claude auth login --console で Anthropic Console (API 課金) による認証が簡単に行えるようになりました。

📱

VSCode リモートコントロール

/remote-control コマンドにより、VSCode のセッションを claude.ai/code にブリッジし、ブラウザやスマートフォンから継続操作が可能になりました。

🔧

10 件のバグ修正

claude -p のサブプロセスハング、ボイスモードの起動不具合、エンタープライズのレート制限リトライなど、幅広い環境での安定性が向上しました。

Before / After

Before

  • claude -p がサブプロセスとして起動時にハング
  • 非ストリーミング API フォールバックにタイムアウトなし
  • エンタープライズユーザーが 429 エラー時にリトライ不可
  • ボイスモードが起動時にアクティブ化されない

After

  • stdin なしのサブプロセス起動でもハングせず動作
  • 1 回の試行あたり 2 分のタイムアウトを導入
  • レート制限エラー時のリトライが正常に動作
  • voiceEnabled: true で正しくアクティブ化

新機能

🔑 --console フラグ

claude auth login --console で Anthropic Console (API 課金) による認証が可能に。従来の OAuth 認証に加え、API キーベースの認証フローを簡単に選択できます。

⏱️ ターン所要時間表示

/config メニューに "Show turn duration" トグルが追加。各ターンの処理時間を確認でき、パフォーマンスモニタリングに活用できます。

📱 /remote-control

VSCode のセッションを claude.ai/code にブリッジし、ブラウザやスマートフォンから継続操作が可能に。デスクから離れてもセッションを監視・操作できます。

📝 AI セッションタイトル

VSCode のセッションタブに、最初のメッセージに基づいた AI 生成タイトルが表示されるようになりました。

パフォーマンス改善

💾

起動時メモリ使用量の削減

全てのシナリオで起動時のメモリ使用量が約 18MB 削減されました。起動時の初期化プロセスの最適化によるもので、特にコンテナ環境やメモリ制約のあるサーバーで効果を発揮します。多数のセッションを並行実行する環境では、累積的なメモリ節約が期待できます。

⏳

非ストリーミング API フォールバック

ストリーミングが利用できない場合のフォールバック処理に、1 回の試行あたり 2 分のタイムアウトが導入されました。従来はタイムアウトが設定されておらず、ネットワーク障害時にセッションが無期限にハングする可能性がありました。

プラグイン改善

🔌

CLAUDE_CODE_PLUGIN_SEED_DIR の複数ディレクトリ対応

プラットフォームのパス区切り文字 (Unix では :、Windows では ;) で区切ることで、複数のシードディレクトリを指定できるようになりました。組織全体の共有プラグインディレクトリとチーム固有のプラグインディレクトリを同時に指定するなど、柔軟なプラグイン管理が実現します。

バグ修正

🖥️ CLI 関連

claude -p のサブプロセスハング修正

明示的な stdin なしでサブプロセスとして起動された場合 (例: Python の subprocess.run) に claude -p がハングする問題を修正。CI/CD パイプラインや自動化スクリプトでの利用が安定します。

claude -p の Ctrl+C 修正

print モードで Ctrl+C が動作しない問題を修正しました。

/btw コマンドの修正

ストリーミング中に /btw を実行した際、サイド質問への回答ではなくメインエージェントの出力が返される問題を修正しました。

🎨 UI・操作関連

ボイスモード起動の修正

voiceEnabled: true が設定されている場合に、起動時にボイスモードが正しくアクティブ化されない問題を修正しました。

/permissions のタブナビゲーション修正

左右矢印キーによるタブナビゲーションが正しく動作するようになりました。

ターミナルタイトル制御の修正

CLAUDE_CODE_DISABLE_TERMINAL_TITLE が起動時のターミナルタイトル設定を防止しない問題を修正しました。

カスタムステータスライン修正

ワークスペーストラストがブロックしている場合にカスタムステータスラインが何も表示しない問題を修正しました。

🏢 エンタープライズ・セッション関連

エンタープライズユーザーのレート制限リトライ修正

エンタープライズユーザーがレート制限 (429) エラー時にリトライできない問題を修正しました。

SessionEnd フック修正

インタラクティブな /resume でセッションを切り替えた際に SessionEnd フックが発火しない問題を修正しました。

💻 VSCode 関連

thinking pill の表示修正

レスポンス完了後に "Thinking" と表示される問題を修正し、"Thought for Ns" と正しい所要時間が表示されるようになりました。

セッション diff ボタン修正

左サイドバーからセッションを開いた際にセッション diff ボタンが表示されない問題を修正しました。

コードサンプル

Bash --console フラグによる認証 
# Anthropic Console (API 課金) で認証
claude auth login --console

# 従来の OAuth 認証 (変更なし)
claude auth login
Bash claude -p をサブプロセスとして使用 
# Anthropic Console 認証で claude -p をサブプロセスとして使用
claude auth login --console

# Python からの呼び出し (v2.1.79 でハング問題が修正)
python3 -c "
import subprocess
result = subprocess.run(
    ['claude', '-p', 'Hello, Claude!'],
    capture_output=True,
    text=True
)
print(result.stdout)
"
Bash 複数プラグインシードディレクトリの設定 
# Unix 環境: コロンで区切る
export CLAUDE_CODE_PLUGIN_SEED_DIR="/org/shared/plugins:/team/plugins:/user/plugins"

# Windows 環境: セミコロンで区切る
set CLAUDE_CODE_PLUGIN_SEED_DIR="C:\org\plugins;C:\team\plugins"
Bash /remote-control の使用 
# VSCode 内の Claude Code で実行
/remote-control

# 表示されるリンクをブラウザまたはモバイルデバイスで開く
# claude.ai/code からセッションを継続操作
Bash アップデート方法 
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# 現在のバージョン確認
claude --version

開発者への影響

🤖 自動化環境のユーザー

claude -p のサブプロセスハング問題と Ctrl+C の修正により、CI/CD パイプラインや Python の subprocess.run からの呼び出しが安定しました。自動化ワークフローで Claude Code を活用しているユーザーは早急なアップデートを推奨します。

💻 VSCode ユーザー

/remote-control によるブラウザ・モバイルからのセッション継続、AI 生成セッションタイトル、thinking pill の正しい表示、diff ボタンの修正など、VSCode 拡張の利便性が大幅に向上しました。

🏢 エンタープライズ管理者

レート制限 (429) エラー時のリトライが正常に動作するようになりました。大規模チームでの運用安定性が向上しています。

🔌 プラグイン開発者

CLAUDE_CODE_PLUGIN_SEED_DIR で複数ディレクトリの指定が可能になり、組織共有とチーム固有のプラグインディレクトリを同時に管理できます。

🎤 ボイスモードユーザー

voiceEnabled: true を設定しているユーザーは、起動時のアクティブ化が正しく動作するようになりました。

🔄 フック活用ユーザー

SessionEnd フックがインタラクティブな /resume によるセッション切り替え時にも正しく発火するようになりました。

まとめ

⚡

パフォーマンス最適化

起動時メモリ使用量が約 18MB 削減され、非ストリーミング API フォールバックに 2 分のタイムアウトが導入されました。リソース制約のある環境でのメモリ効率とネットワーク障害時のセッション安定性が向上しています。

🆕

新しい認証・操作機能

--console フラグによる Anthropic Console 認証、/config のターン所要時間表示トグル、VSCode の /remote-control コマンドと AI 生成セッションタイトルが追加されました。特に /remote-control は、デスクから離れた際にブラウザやスマートフォンからセッションを継続できる実用的な機能です。

🔧

広範な安定性修正

claude -p のサブプロセスハング問題とエンタープライズユーザーのレート制限リトライの修正が特に重要です。CI/CD パイプラインや自動化環境で claude -p を活用しているユーザーは、早急なアップデートを推奨します。全ての Claude Code ユーザーにアップデートを推奨します。
