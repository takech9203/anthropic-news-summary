概要

🔑

セッション ID ヘッダー

API リクエストに X-Claude-Code-Session-Id ヘッダーが追加。プロキシサーバーがボディ解析不要でセッション単位の集約が可能に

📁

🎯

トークン使用量の最適化 4 件

@ メンション JSON エスケープ廃止、プロンプトキャッシュヒット率向上、スキル説明文 250 文字制限、Read ツールコンパクトフォーマット

🔧

重要なバグ修正 16 件

--resume 互換性、Windows 設定ファイル破損、メモリリーク、VSCode Max プランモデルデフォルト問題など

新機能

🔑 Session-Id ヘッダー

API リクエストに X-Claude-Code-Session-Id ヘッダーが自動付与されます。プロキシサーバーがリクエストボディを解析せずにセッション単位でリクエストを集約でき、レートリミット、コスト配分、使用量モニタリングが容易になります。

📂 VCS ディレクトリ除外

.jj (Jujutsu) と .sl (Sapling) が VCS ディレクトリ除外リストに追加されました。Grep やファイルオートコンプリートがこれらのメタデータディレクトリに降下しなくなり、検索パフォーマンスが向上します。

Before / After: トークン最適化

Before

  • @ メンション: JSON エスケープ済み
  • ツール説明文: 動的コンテンツ含む
  • Read ツール: 冗長な行番号
  • スキル説明文: 制限なし

After

  • @ メンション: 生の文字列
  • ツール説明文: 静的コンテンツのみ
  • Read ツール: コンパクトフォーマット
  • スキル説明文: 250 文字制限

改善・変更 8 件

🎯 トークン使用量の最適化

  • 📝 @ メンション時のトークンオーバーヘッド削減 — ファイル内容が JSON エスケープされなくなり、生の文字列として渡されるように
  • 💾 Bedrock / Vertex / Foundry ユーザー向けプロンプトキャッシュヒット率改善 — ツール説明文から動的コンテンツを除去
  • 📏 スキル説明文の 250 文字制限 — /skills リスト内のコンテキスト使用量を削減
  • 📖 Read ツールのコンパクト行番号フォーマット — 未変更ファイルの重複再読み込みも排除

⚡ パフォーマンス / UX 改善

  • 🚀 起動時のイベントループストール削減 — macOS キーチェーンキャッシュ有効期間を 5 秒から 30 秒に延長
  • 🧠 メモリファイル名のインタラクティブ表示 — 「Saved N memories」通知内でホバーハイライト、クリックで開く
  • 📋 /skills メニューのアルファベット順ソート
  • 🤖 Auto モードのプラン制限メッセージ改善 — 「temporarily unavailable」→「unavailable for your plan」

バグ修正 16 件

🔄 セッション・コマンド関連

  • v2.1.85 以前のセッションで --resume が「tool_use ids were found without tool_result blocks」で失敗する問題
  • 条件付きスキル/ルール設定時に ~/.claude/CLAUDE.md など外部ファイルの Write / Edit / Read が失敗する問題
  • --bare モードで MCP ツールがドロップされ、ターン途中のメッセージが黙って破棄される問題

💽 設定・パフォーマンス関連

  • スキル呼び出しのたびに不要な設定ファイルのディスク書き込みが発生 — Windows での設定破損を引き起こす可能性
  • /feedback で非常に長いセッションの大きなトランスクリプトファイルによるメモリ不足クラッシュ
  • Markdown / ハイライトレンダーキャッシュがコンテンツ文字列を保持し続けるメモリリーク

🔐 認証・入力関連

  • c ショートカットで OAuth ログイン URL が約 20 文字しかコピーされない問題
  • マスク入力で狭いターミナルの複数行折り返し時にトークン先頭部分が漏洩する問題

🧩 プラグイン・UI・VSCode

  • macOS / Linux で公式マーケットプレイスプラグインスクリプトが「Permission denied」— v2.1.83 からのリグレッション
  • 複数インスタンス実行時に別セッションのモデルがステータスラインに表示される問題
  • 長い会話でスクロール追従が停止する問題
  • /plugin アンインストールダイアログの動作修正
  • クリック後 Enter 押下でトランスクリプトが空白になるリグレッション
  • ultrathink ヒントがキーワード削除後も残り続ける問題
  • [VSCode] 長時間操作中に誤って「Not responding」が表示される問題
  • [VSCode] Max プランユーザーが OAuth トークンリフレッシュ後に Sonnet にデフォルト変更される問題

@app.middleware("http")
async def track_session_cost(request: Request, call_next):
    session_id = request.headers.get(
        "X-Claude-Code-Session-Id", "unknown"
    )
    response = await call_next(request)
    # セッション単位でコストを記録
    await record_cost(session_id, response)
    return response

ユースケース

🏢 企業のインフラチーム: プロキシサーバーでのセッション追跡

X-Claude-Code-Session-Id ヘッダーにより、API プロキシサーバーでリクエストボディの解析なしにセッション単位のレートリミット、コスト配分、使用量モニタリングを実装できます。TLS 終端前でもヘッダーは参照可能です。

⚡ Bedrock / Vertex / Foundry ユーザー: コスト削減

ツール説明文から動的コンテンツが除去されたことで、プロンプトキャッシュのヒット率が向上。API コスト削減とレスポンス速度向上が期待できます。@ メンションの JSON エスケープ廃止と Read ツールのコンパクトフォーマットも合わせて、トークン使用量が総合的に削減されます。

🖥️ Windows / 長時間セッションユーザー: 安定性向上

設定ファイルの不要なディスク書き込み排除で Windows での設定破損が解消。メモリリーク修正と /feedback クラッシュ修正で、長時間セッションの安定性が大幅に向上します。

アップデート推奨ケース

🚨

早急なアップデートを推奨

• v2.1.85 以前のセッションを --resume で再開できない場合
• Windows で設定ファイルが破損する場合
• macOS / Linux でマーケットプレイスプラグインが Permission denied で失敗する場合
• VSCode 拡張で Max プランなのに Sonnet に戻される場合
• 長時間セッションでメモリ使用量が増加し続ける場合
• OAuth ログイン URL が正しくコピーされない場合
