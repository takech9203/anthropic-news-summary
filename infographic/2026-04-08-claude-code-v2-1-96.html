概要

🚨

v2.1.94 でリグレッション発生

2026 年 4 月 7 日にリリースされた v2.1.94 は、Mantle 対応を含む大型アップデートでした。しかし、この変更により Bedrock 認証の処理にリグレッションが発生しました。

🔧

v2.1.96 で緊急修正

AWS_BEARER_TOKEN_BEDROCK または CLAUDE_CODE_SKIP_BEDROCK_AUTH を使用している環境で Bedrock リクエストが 403 "Authorization header is missing" エラーで失敗する問題を修正しました。

⚠️

v2.1.95 はスキップ

v2.1.95 は Changelog にエントリがなく、スキップされています。内部的なビルド番号の問題、または公開前に取り下げられたバージョンと考えられます。

主な変更点

🐛 Bedrock 認証の修正 (バグ修正)

AWS_BEARER_TOKEN_BEDROCK または CLAUDE_CODE_SKIP_BEDROCK_AUTH を使用している環境で、Bedrock リクエストが 403 "Authorization header is missing" エラーで失敗する問題を修正しました。v2.1.94 で追加された Mantle 対応 (CLAUDE_CODE_USE_MANTLE=1) の変更が、既存の Bedrock 認証フローに影響を与えていました。

技術詳細

v2.1.94 で追加された Amazon Bedrock powered by Mantle 対応の変更が、既存の Bedrock 認証フローに影響を与えました。以下の環境変数を使用した認証パスにおいて、Authorization ヘッダーが正しくリクエストに付与されなくなっていました。

環境変数 用途 影響
AWS_BEARER_TOKEN_BEDROCK Bedrock へのベアラートークン認証 403 エラーで全リクエスト失敗
CLAUDE_CODE_SKIP_BEDROCK_AUTH Bedrock 認証のスキップ設定 403 エラーで全リクエスト失敗
原因: v2.1.94 で追加された CLAUDE_CODE_USE_MANTLE=1 (Amazon Bedrock powered by Mantle 対応) の処理が、ベアラートークン認証および認証スキップの処理パスに影響を与え、Authorization ヘッダーが欠落するようになっていました。

コードサンプル

Bash ベアラートークン認証の設定 
# AWS_BEARER_TOKEN_BEDROCK を使用した Bedrock 認証
export AWS_BEARER_TOKEN_BEDROCK="your-bearer-token"
export ANTHROPIC_MODEL="us.anthropic.claude-sonnet-4-6-20260220-v1:0"

# Claude Code を起動 (v2.1.96 で正常に動作)
claude
Bash 認証スキップの設定 
# カスタム認証プロキシを使用する場合など
export CLAUDE_CODE_SKIP_BEDROCK_AUTH=1
export ANTHROPIC_BASE_URL="https://your-proxy.example.com"

# Claude Code を起動
claude
Bash バージョン確認とアップデート 
# 現在のバージョンが 2.1.94 かどうか確認
claude --version

# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# Homebrew でのアップデート
brew upgrade claude-code

# アップデート後のバージョン確認
claude --version
# 期待される出力: 2.1.96

影響を受けるユーザーと推奨アクション

🏢 ベアラートークン認証ユーザー

AWS_BEARER_TOKEN_BEDROCK を使用して Bedrock に接続しているユーザーは、v2.1.94 で全てのリクエストが 403 エラーで失敗する状態でした。直ちに v2.1.96 にアップデートしてください。

🔐 認証スキップ設定ユーザー

CLAUDE_CODE_SKIP_BEDROCK_AUTH を設定してカスタム認証プロキシ等を使用しているユーザーも同様に影響を受けます。v2.1.96 へのアップデートが必要です。

✅ 標準の IAM 認証ユーザー

上記の環境変数を使用していない標準の IAM 認証ユーザーは、本リグレッションの影響を受けていない可能性があります。ただし、最新版へのアップデートは推奨されます。

⏩ v2.1.92 以前のユーザー

v2.1.94 をスキップして直接 v2.1.96 にアップデートすることで、大型アップデートの恩恵を受けつつリグレッションを回避できます。
