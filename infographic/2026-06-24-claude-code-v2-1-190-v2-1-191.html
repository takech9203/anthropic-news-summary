Before / After

Before

  • - /clear 後の会話復元不可
  • - 各テキストチャンク毎に UI 更新
  • - MCP 接続エラーで即失敗
  • - サンドボックス許可が毎回プロンプト
  • - 長時間セッションでメモリ増加

After

  • + /rewind で会話状態を復元
  • + 100ms バッチで CPU 37% 削減
  • + ショートバックオフ付きリトライ
  • + 許可がセッション中記憶される
  • + ターミナルキャッシュのメモリリーク修正

主な機能と改善

⏪ /rewind コマンド

/clear で消去した会話を復元可能。誤操作からのリカバリーが容易に

🔄 MCP リトライ機構

tools/list、prompts/list、resources/list がネットワークエラー時にショートバックオフ付きリトライ

🔑 MCP OAuth 改善

ヘッドレス環境でブラウザポップアップをスキップし URL 貼り付けに直接移行

🏠 サンドボックス記憶

許可したホストをセッション終了まで記憶し、毎回の再プロンプトを排除

📋 管理設定キャッシュ

forceRemoteSettingsRefresh が MDM/ファイルポリシー経由で有効化。Cache-Control: no-cache で最新取得

✏️ vim モード改善

NORMAL / でのプロンプト履歴検索時にスラッシュコマンドへの到達ヒントを表示

技術詳細: パフォーマンス最適化

flowchart LR
    subgraph Before["Before: 毎チャンク更新"]
        A1["API Stream"] --> B1["Chunk 1"]
        B1 --> C1["UI Update"]
        A1 --> B2["Chunk 2"]
        B2 --> C2["UI Update"]
        A1 --> B3["Chunk 3"]
        B3 --> C3["UI Update"]
    end
    subgraph After["After: 100ms バッチ"]
        A2["API Stream"] --> D1["Buffer"]
        D1 --> E1["100ms Timer"]
        E1 --> F1["Batch UI Update"]
    end

従来は各テキストチャンクの受信ごとに UI を更新していたが、v2.1.191 では 100ms 間隔でバッチ処理を行うことで、レンダリング回数を大幅に削減。これにより約 37% の CPU 使用量削減を達成。

コマンド例

Bash アップデートと新機能の使用 
# Claude Code を最新バージョンに更新
claude update

# 新機能: /rewind で会話を復元
# セッション内で /clear 実行後に
/rewind

# バージョン確認
claude --version
# v2.1.191

開発者への影響

長時間コーディングセッション

CPU 37% 削減とメモリリーク修正により、長時間の開発セッションがより快適に。システムリソースの効率的な利用でバッテリー持続時間も改善

MCP サーバー利用者

一時的なネットワーク障害に対する耐性が強化。CI/CD パイプラインなどのヘッドレス環境での OAuth フローが容易に

エンタープライズ管理者

MDM やファイルポリシー経由の管理設定が正しく反映されるようになり、組織全体での一貫した設定管理が可能に
