主要ポイント

🚀

Auto モード全プラットフォーム開放

Bedrock、Vertex AI、Foundry で CLAUDE_CODE_ENABLE_AUTO_MODE 環境変数なしに Auto モードが利用可能に。段階展開の完了により全プラットフォームでデフォルト有効化された。

🤖

デフォルトモデル Claude Opus 4.8

Bedrock、Vertex、Claude Platform on AWS のデフォルトモデルが Claude Opus 4.8 に統一された。Claude Platform (API 直接利用) と同等の体験を全プラットフォームで提供。

🔒

プラグインセキュリティ強化

シェルインジェクション防止と設定読み込みスコープの制限により、サプライチェーン攻撃のリスクが排除された。プラグイン開発者は exec フォームへの移行が必要。

Before / After

Before - プラグイン設定

  • command フィールドでシェルフォーム使用
  • ${user_config.*} がシェル経由で展開
  • シェルインジェクション攻撃が可能

After - プラグイン設定

  • exec フォーム (args 配列) を使用
  • $CLAUDE_PLUGIN_OPTION_* 環境変数
  • インジェクション攻撃が構造的に不可能

Before - Auto モード設定

  • .claude/settings.local.json から読み込み
  • リポジトリ内から制御可能
  • 悪意あるリポジトリによる操作リスク

After - Auto モード設定

  • ~/.claude/settings.json のみ有効
  • ユーザーレベルで明示的に管理
  • リポジトリからの操作不可能

新機能・変更

🌐 Auto モード全開放

Bedrock、Vertex AI、Foundry で環境変数設定なしに Auto モードが利用可能。無効化する場合は disableAutoMode を設定。

⭐ Opus 4.8 デフォルト化

Bedrock、Vertex、Claude Platform on AWS のデフォルトモデルが Claude Opus 4.8 に変更され、全プラットフォームで統一。

📁 設定読み込み元変更

Auto モードは .claude/settings.local.json から autoMode を読み込まなくなった。~/.claude/settings.json を使用する必要がある。

🛡️ シェルインジェクション修正

シェルフォームでの ${user_config.*} が拒否され、exec フォームまたは環境変数の使用が必須に。コマンドインジェクション防止。

🔐 プラグインスコープ制限

pluginConfigs がプロジェクトレベルの .claude/settings.json から読み込まれなくなった。ユーザー設定、--settings、管理設定のみ有効。

移行ガイド

JSON プラグイン移行: シェルフォーム → exec フォーム 
// 変更前 (v2.1.206 以前) - 脆弱なシェルフォーム
{
  "hooks": {
    "on_save": {
      "command": "curl -H 'Authorization: ${user_config.api_key}' https://api.example.com"
    }
  }
}

// 変更後 (v2.1.207) - 安全な exec フォーム
{
  "hooks": {
    "on_save": {
      "command": "curl",
      "args": ["-H", "Authorization: $CLAUDE_PLUGIN_OPTION_API_KEY", "https://api.example.com"]
    }
  }
}

ユースケース

Bedrock / Vertex AI / Foundry ユーザー

Auto モードがオプトインなしで利用可能に。デフォルトモデルも Claude Opus 4.8 に更新。既存の CLAUDE_CODE_ENABLE_AUTO_MODE 環境変数設定は削除可能。

プラグイン開発者

シェルフォームでの ${user_config.*} が使用不可に。exec フォーム (args 配列) または $CLAUDE_PLUGIN_OPTION_* 環境変数への移行が必要。pluginConfigs のプロジェクトレベル読み込みも廃止。

エージェントチーム利用者

クラッシュループの修正により、不正なメールボックスメッセージによるシステム停止が防止される。バックグラウンドセッションの名前表示とブランク復帰も修正。

Remote Control 利用者

ネットワーク復帰時のステータス消失とデスクトップアプリからの進捗表示問題が修正。安定したリモート操作が可能に。

Windows ユーザー

AWS クレデンシャル解決の停滞による無期限ハングが修正。credential_process が応答しない場合もタイムアウトが適切に処理される。

Deep Research 利用者

Fetch フェーズエージェントが "unknown" ではなくソースのホスト名で表示されるようになり、調査の透明性が向上。

全ユーザー

ストリーミング中のターミナルフリーズが解消。長大な出力 (数百行のリスト、大きなテーブル、コードブロック) でもスムーズな操作が可能。差分レンダリングにより入力遅延を防止。

推奨アクション

  • 1 プラグインの移行: シェルフォームで ${user_config.*} を使用しているプラグインは、exec フォームまたは環境変数に移行する
  • 2 Auto モード設定の確認: .claude/settings.local.json で autoMode を設定している場合、~/.claude/settings.json に移動する
  • 3 不要な環境変数の削除: CLAUDE_CODE_ENABLE_AUTO_MODE を設定している場合は削除可能
  • 4 Auto モードの無効化: 組織ポリシーとして無効化する場合は disableAutoMode を設定する
