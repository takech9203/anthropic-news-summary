Before / After: /fork コマンド

Before

  • セッション内サブエージェント起動
  • 親セッションのトークンバジェットを消費
  • claude agents に表示されない
  • 親セッションが終了するとサブエージェントも終了

After

  • 独立バックグラウンドセッションにコピー
  • 独自のトークンバジェット
  • claude agents に独自の行として表示
  • 親とは独立したライフサイクル
  • 従来動作は /subtask で利用可能

主な機能

🔀 /fork バックグラウンドセッション化

会話をバックグラウンドにコピーして独立実行。元セッションは中断なく継続

🔍 WebSearch キャップ

デフォルト 200 回/セッション。CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION で調整

🤖 サブエージェントキャップ

デフォルト 200 回/セッション。CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION でオーバーライド

⏱️ MCP 自動バックグラウンド

2 分超の呼び出しを自動移行。CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS で閾値変更

🔄 /resume セッションピッカー

削除済みセッション含む過去セッションからピッカーで選択再開

🔧 claude auto-mode reset

auto-mode 設定をデフォルトに復元

コードサンプル

Bash 暴走ループキャップの設定 
# WebSearch の上限を 500 に引き上げ (大規模リサーチ向け)
export CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION=500

# サブエージェントの上限を 50 に引き下げ (コスト制御重視)
export CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION=50

# MCP 自動バックグラウンドの閾値を 5 分に変更
export CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS=300000

# MCP 自動バックグラウンドを無効化
export CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS=0
