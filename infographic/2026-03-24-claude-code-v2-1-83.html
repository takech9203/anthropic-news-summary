Claude Code v2.1.83

マネージド設定のドロップインディレクトリ、30 件超のバグ修正、セキュリティとパフォーマンスの大幅改善

2026-03-24

概要

🏢

エンタープライズポリシー管理

managed-settings.d/ ドロップインディレクトリにより、セキュリティチーム、インフラチーム、プロジェクトチームがそれぞれ独立したポリシーファイルを管理できるようになりました。アルファベット順にマージされるため、数字プレフィックスで優先順位を制御できます。

🔐

セキュリティ強化

CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1 でサブプロセスへの認証情報漏洩を防止。sandbox.failIfUnavailable でサンドボックスなし実行を禁止。--mcp-config のポリシーバイパスも修正されました。

🔧

30 件超のバグ修正

macOS 終了時ハング、起動時 UI フリーズ、大規模ファイル diff ハング、サンドボックスでのパイプコマンド問題、バックグラウンドエージェント不可視化、MCP SSE 接続断時のハングなど、広範な領域をカバーしています。

⚡

パフォーマンス改善

--bare -p の 14% 高速化、非ストリーミングフォールバックのトークン上限 3 倍拡張 (21k -> 64k)、Bedrock SDK コールドスタート改善、プラグインのディスクキャッシュ化など、起動速度と処理能力が全般的に向上しています。

主要機能

📁 managed-settings.d/

ドロップインディレクトリから複数のポリシーフラグメントをアルファベット順にマージ。00-security.json、10-infra.json、20-project.json のようにチーム別の独立管理が可能になりました。

New

🔐 ENV_SCRUB

CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1 で Bash ツール、hooks、MCP stdio サーバーなどのサブプロセス環境から Anthropic およびクラウドプロバイダーの認証情報を自動除去します。

Security

🔔 CwdChanged / FileChanged

ワーキングディレクトリの変更やファイルの変更を検知する hook イベント。direnv のようなリアクティブな環境管理ツールとの連携が可能になりました。

New

🔍 トランスクリプト検索

トランスクリプトモード (Ctrl+O) で / を押して検索し、n / N で結果を移動できるようになりました。

New

🛡️ sandbox.failIfUnavailable

サンドボックスが起動できない場合、サンドボックスなしで実行する代わりにエラーで終了。セキュリティポリシーの厳格な適用が可能です。

Security

🔌 プラグインオプション

manifest.userConfig が外部から利用可能に。sensitive: true の値は macOS ではキーチェーン、その他ではセキュアファイルに保存されます。

New

コードサンプル

Bash managed-settings.d/ の設定例 
# ドロップインディレクトリの作成
mkdir -p ~/.claude/managed-settings.d/

# セキュリティポリシー
cat > ~/.claude/managed-settings.d/00-security.json << 'EOF'
{
  "sandbox.failIfUnavailable": true,
  "allowedMcpServers": ["approved-server-1", "approved-server-2"]
}
EOF

# インフラチーム設定
cat > ~/.claude/managed-settings.d/10-infra.json << 'EOF'
{
  "disableDeepLinkRegistration": true
}
EOF
Bash ENV_SCRUB の使用例 
# 環境変数を設定してサブプロセスの認証情報を除去
export CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1
claude

# CI/CD での使用例
CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1 ANTHROPIC_API_KEY=sk-ant-xxx \
  claude -p --bare "コードをレビューしてください"
JSON CwdChanged / FileChanged hook の設定 
// .claude/settings.json
{
  "hooks": {
    "CwdChanged": [
      {
        "command": "direnv allow && direnv export json > /tmp/direnv-env.json"
      }
    ],
    "FileChanged": [
      {
        "command": "echo 'File changed: $CLAUDE_FILE_PATH'"
      }
    ]
  }
}

主要バグ修正

🍎 起動・終了関連
🔄

macOS での終了時ハング修正

Claude Code が macOS で終了時にハングする問題と、caffeinate プロセスが適切に終了しない問題を修正しました。

⏱️

起動時 UI フリーズ修正

ボイス入力有効時のオーディオモジュール先行読み込みによる 1 - 8 秒の UI フリーズ、claude.ai MCP 設定取得の約 3 秒遅延を修正しました。

🖥️ UI・操作関連
📄

大規模ファイル diff のハング修正

共通行の少ない大規模ファイルの diff でハングする問題を修正。5 秒のタイムアウトとフォールバック機構を導入しました。

📜

スクロールバック・UI 修正

画面フラッシュ、スクロールバックジャンプ、キューコマンドのフリッカー、ゴーストキャラクターなど多数の UI 問題を修正しました。

👻 バックグラウンドエージェント・サンドボックス
🔍

エージェント不可視化修正

コンテキスト圧縮後にバックグラウンドサブエージェントが不可視になり、重複エージェントがスポーンされる問題を修正しました。

🐧

サンドボックスパイプコマンド修正

Linux のサンドボックスモードで rg ... | wc -l などのパイプコマンドがハングして 0 を返す問題を修正しました。

🔌 MCP・Remote Control 関連
📡

MCP SSE 接続断のハング修正

SSE 接続がコール途中でドロップした際に MCP ツールコールが無期限にハングする問題を修正しました。

💾

Remote Control メモリリーク修正

リモートセッションでツール使用 ID が無制限に蓄積するメモリリーク、コンテナ再起動後の履歴消失を修正しました。

🎤 ボイスモード関連
🔊

ALSA エラー・SoX 検出・CJK IME 修正

オーディオハードウェアのない Linux での ALSA エラー、Termux/Android での SoX 検出、CJK IME 全角スペースでのホールドトゥトーク問題を修正しました。

改善・変更

🚀 --bare -p 14% 高速化

SDK パターンでの API リクエストまでの時間が 14% 短縮。claude -p 起動も未認証 MCP サーバーで約 600ms 短縮されました。

📦 非ストリーミング拡張

トークン上限が 21k から 64k に (約 3 倍)、タイムアウトが 120 秒から 300 秒に延長。ネットワーク不安定な環境での信頼性が向上しました。

💾 プラグイン起動改善

コマンド、スキル、エージェントがディスクキャッシュから再フェッチなしでロード。Bedrock SDK のコールドスタートレイテンシも改善されました。

🌐 WebFetch User-Agent 更新

Claude-User として識別され、サイト運営者が robots.txt で Claude Code トラフィックを認識・許可リストに追加できます。

⌨️ キーバインド変更

バックグラウンドエージェント停止が Ctrl+F から Ctrl+X Ctrl+K に変更。Ctrl+L は画面クリアと完全な再描画を強制するようになりました。

📝 MEMORY.md 制限強化

200 行に加えて 25KB でもトランケート。TaskOutput ツールは非推奨化され、Read での直接読み取りが推奨されます。

🖥️ VSCode 拡張改善

60 秒応答なしで赤色「Not responding」表示、Esc 2 回押しで巻き戻しピッカー、セッション履歴の修正が行われました。

👁️ bash ゴーストテキスト

直前に送信したコマンドが即座にゴーストテキスト提案に含まれるように。/status は Claude の応答中でも即座に実行されます。

ユースケース

エンタープライズポリシー管理

managed-settings.d/ ドロップインディレクトリにより、セキュリティチーム (00-security.json)、インフラチーム (10-infra.json)、プロジェクトチーム (20-project.json) がそれぞれ独立したポリシーファイルを管理できます。単一ファイルでのコンフリクトを回避し、組織内の設定管理を大幅に簡素化します。sandbox.failIfUnavailable と組み合わせることで、セキュリティポリシーの厳格な適用が可能です。

セキュリティ強化

CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1 により、Bash ツール、hooks、MCP stdio サーバーなどのサブプロセスから ANTHROPIC_API_KEY や AWS 認証情報を自動除去できます。悪意のある MCP サーバーやフック経由での認証情報窃取を防ぐ防御層として機能し、サプライチェーン攻撃のリスクを低減します。

CI/CD パイプライン高速化

--bare -p の 14% 高速化と非ストリーミングフォールバックの大幅拡張により、CI/CD パイプラインでの Claude Code の利用がより効率的になりました。コードレビュー、テスト生成、ドキュメント自動生成などの自動化タスクで、起動オーバーヘッドの削減と大規模レスポンスの確実な受信が実現します。
