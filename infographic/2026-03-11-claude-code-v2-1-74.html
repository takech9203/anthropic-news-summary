Claude Code Update

Claude Code v2.1.74 リリース:
メモリリーク修正、セキュリティ強化、RTL 言語対応

2026-03-11

セキュリティ、安定性、プラットフォーム対応の 3 つの柱で重要な改善を実現したリリース。メモリリーク修正、マネージドポリシーのバイパス修正、RTL テキストレンダリング対応など多数の変更を含みます。

概要 - キーポイント

🛡

セキュリティ修正

マネージドポリシーの ask ルールがユーザーの allow ルールやスキルの allowed-tools によってバイパスされる問題を修正。エンタープライズ環境でのポリシー管理の信頼性が向上。

🧠

メモリリーク修正

ストリーミング API レスポンスバッファが解放されず RSS メモリが無制限に増加する問題を修正。Node.js/npm コードパスの安定性が大幅に改善。

🔑

MCP OAuth 修正

コールバックポート競合によるハングと、HTTP 200 でエラーを返す OAuth サーバー (Slack など) でのリフレッシュトークン期限切れ処理の 2 つの問題を修正。

🌐

プラットフォーム対応

RTL テキストレンダリング (ヘブライ語・アラビア語)、Windows LSP サーバー、macOS 音声モードの権限プロンプトなど多様な環境での利用体験を改善。

✨

新機能

/context コマンドの最適化提案機能と autoMemoryDirectory 設定を追加。コンテキスト負荷の特定や自動メモリの保存先カスタマイズが可能に。

Before / After

Before (v2.1.73 以前)

  • ストリーミングバッファ未解放で RSS メモリが無制限に増加
  • マネージドポリシーの ask ルールがバイパス可能
  • MCP OAuth がポート競合時にハング
  • RTL テキストが正しく表示されない
  • SessionEnd フックが 1.5 秒で強制終了

After (v2.1.74)

  • ジェネレータ終了時にバッファを適切に解放
  • 管理者ポリシーが確実に適用される
  • ポート競合時のフォールバック処理を実装
  • Windows Terminal / VS Code で RTL 正常表示
  • 環境変数でタイムアウトをカスタマイズ可能

主な変更点

New Feature

📊 /context 最適化提案

コンテキスト負荷の高いツール、メモリの肥大化、キャパシティ警告を特定し、具体的な最適化のヒントを提示する機能を追加。

New Feature

🗂 autoMemoryDirectory

自動メモリのストレージディレクトリをカスタム指定可能に。チームやプロジェクトごとのメモリ管理が柔軟に。

Security

🛡 マネージドポリシー修正

マネージドポリシーの ask ルールがユーザーの allow やスキルの allowed-tools でバイパスされる脆弱性を修正。

Performance

🧠 メモリリーク修正

ストリーミング API レスポンスバッファのメモリリークを修正。長時間セッションでの RSS メモリ無制限増加を解消。

MCP

🔑 OAuth 認証ハング修正

コールバックポートが既に使用中の場合に MCP OAuth 認証がハングする問題を修正。

MCP

🔄 OAuth リフレッシュ修正

HTTP 200 でエラーを返す OAuth サーバー (Slack など) でリフレッシュトークン期限切れ後に再認証プロンプトが表示されない問題を修正。

Platform

📝 RTL テキスト対応

ヘブライ語、アラビア語などの RTL テキストが Windows Terminal、conhost、VS Code 統合ターミナルで正しくレンダリングされるように修正。

Platform

🪟 Windows LSP 修正

不正なファイル URI により Windows 上で LSP サーバーが動作しない問題を修正。

Platform

🎤 macOS 音声モード修正

ネイティブバイナリでマイク権限が未付与のターミナルで音声モードがサイレントに失敗する問題を修正。audio-input エンタイトルメント追加で権限プロンプトを表示。

Agent

🤖 フルモデル ID 対応

エージェントフロントマターの model: フィールドおよび --agents JSON 設定でフルモデル ID がサイレントに無視される問題を修正。

Hooks

🪝 SessionEnd タイムアウト

終了時に hook.timeout 設定に関わらず 1.5 秒で強制終了される問題を修正。環境変数で設定可能に。

Hooks

🔌 プラグイン修正

REPL 内でのマーケットプレイスプラグインのインストール失敗、サブモジュール同期、不明コマンドの入力保持など複数の修正。

Other

⚙ その他の改善

--plugin-dir の優先順位変更、VS Code セッション削除ボタン修正、統合ターミナルのスクロール操作改善。

技術的な詳細

🧠 ストリーミングバッファのメモリリーク

Node.js/npm コードパスにおいて、ストリーミング API レスポンスのジェネレータが早期終了 (例: ユーザーによるキャンセル) した際、レスポンスバッファが解放されずにメモリが蓄積し続ける問題がありました。長時間のセッションや頻繁な API 呼び出しのキャンセルが発生する環境で RSS メモリが無制限に増加する原因となっていました。

🛡 マネージドポリシーのセキュリティ強化

管理者が ask ルールとして設定したポリシーが、ユーザーレベルの allow ルールやスキルの allowed-tools 設定によって迂回される脆弱性が修正されました。エンタープライズ環境でのポリシー管理において重要な修正です。

🔑 MCP OAuth のエッジケース対応

OAuth フローにおける 2 つの異なる問題が修正されました。1 つ目はコールバックポート競合によるハング、2 つ目は HTTP 200 ステータスでエラーを返す OAuth サーバー (Slack が該当) でのリフレッシュトークン期限切れ処理です。

開発者への影響

💻 Claude Code CLI ユーザー

日常的に Claude Code を利用している開発者全般。メモリリーク修正により長時間セッションの安定性が向上し、/context コマンドの最適化提案で効率的な開発が可能に。

🏢 エンタープライズチーム

マネージドポリシーを運用しているチーム。ask ルールのバイパス修正により、管理者が設定したセキュリティポリシーが確実に適用されるようになりました。

🔗 MCP 連携ユーザー

MCP 連携で OAuth 認証を利用しているユーザー。特に Slack 連携でリフレッシュトークンの問題が発生していた場合に直接的な恩恵があります。

🪟 Windows / RTL 言語ユーザー

Windows 環境で Claude Code を使用している開発者や、ヘブライ語・アラビア語などの RTL 言語圏のユーザー。LSP サーバーの動作改善と RTL テキスト表示の修正を含みます。

🎤 macOS 音声モードユーザー

macOS ネイティブバイナリで音声モードを使用しているユーザー。audio-input エンタイトルメント追加により、macOS が正しく権限プロンプトを表示するようになりました。

早急なアップデートを推奨するケース

  • 長時間セッションでメモリ使用量の増加が観察されている場合
  • マネージドポリシーでセキュリティルールを設定している環境
  • MCP OAuth 連携 (特に Slack) で認証の問題が発生している場合

まとめ

Claude Code v2.1.74 は、セキュリティ、安定性、プラットフォーム対応の 3 つの柱で重要な改善を実現したリリースです。最も注目すべきは、マネージドポリシーの ask ルールがバイパスされるセキュリティ問題の修正で、エンタープライズ環境でのポリシー管理の信頼性が向上しています。

ストリーミング API レスポンスバッファのメモリリーク修正により、長時間セッションでの RSS メモリ無制限増加が解消され、Node.js/npm コードパスの安定性が大幅に改善されました。MCP OAuth 認証に関する 2 つの修正も、Slack などの外部サービスとの連携信頼性を向上させています。

プラットフォーム対応としては、RTL テキストレンダリング、Windows LSP サーバー、macOS 音声モードの権限プロンプトなど、多様な環境での利用体験が改善されています。全ての Claude Code ユーザーに早期のアップデートを推奨します。
