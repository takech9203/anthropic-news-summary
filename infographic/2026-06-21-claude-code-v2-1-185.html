Claude Code v2.1.185: ストリームストールヒントの UX 改善

2026-06-21

概要

💬

ヒントメッセージの文言改善

「No response from API」という接続エラーを示唆する表現を「Waiting for API response」という正確な待機状態の表現に変更。ユーザーの不安を軽減し、システムが正常に動作中であることを明確に伝達する。

⏱

トリガー閾値を 10 秒から 20 秒に延長

複雑なリクエストや低速なネットワーク環境での偽陽性ヒント表示を削減。正常な処理中に不要な警告が表示される頻度を大幅に低減する。

Before / After 比較

Before (v2.1.184 以前)

No response from API · Retrying in 5s
閾値: 10 秒間の無応答で表示

接続が失敗したかのような誤解を与える表現

After (v2.1.185)

Waiting for API response · will retry in 5s
閾値: 20 秒間の無応答で表示

正常に待機中であることを正確に伝達

主な変更点

💬 メッセージ文言の変更

「No response」(否定的・断定的) から「Waiting for response」(待機中・継続的) へ変更。「Retrying in」を「will retry in」に変更し、まだリトライしていないことを明確化。

⏱ トリガー閾値の延長

ストリームストールヒントの表示タイミングを 10 秒から 20 秒に延長。大規模コードベース解析や多数のツール呼び出しを伴うタスクでの偽陽性を大幅に削減。

技術詳細

ストリームストールヒントは、API からのストリーミングレスポンスにおいてチャンクの到着が途絶えた場合に発動する UX メカニズムである。以下のシーケンス図は、改善後のフローを示す。

開発者への影響

低速ネットワーク環境のユーザー

VPN 経由や地理的に遠いリージョンへの接続で作業する開発者にとって、不要な警告ヒントが表示される頻度が大幅に減少する。

大規模コードベースの解析タスク

複雑なプロンプトや多数のツール呼び出しを伴うタスクでは、モデルの初回レスポンスまでに 10 秒以上かかることがある。20 秒の閾値により偽陽性が解消される。

日常的な Claude Code ユーザー全般

メッセージが「接続失敗」ではなく「待機中」と正確に表示されることで、不要な不安やワークフローの中断が軽減される。特別な設定変更は不要。
