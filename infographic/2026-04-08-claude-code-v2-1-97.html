主な機能

新機能 (Added)

👁️ フォーカスビュー

Ctrl+O で NO_FLICKER モードのフォーカスビューをトグル。プロンプト + 1 行ツールサマリー (diffstats 付き) + 最終応答のみ表示。長いセッションでの情報ノイズを削減します。

NEW

🔄 refreshInterval ステータスライン

ステータスラインコマンドを N 秒ごとに再実行する設定を追加。ビルド状態やテスト結果などの動的情報をリアルタイム表示できます。

NEW

🌳 git_worktree ステータス入力

リンクされた git worktree 内にいる場合、ステータスライン JSON 入力に workspace.git_worktree が自動設定されます。

NEW

● /agents ライブインジケーター

/agents コマンドでエージェントタイプの横に「N running」インジケーターが表示。実行中のサブエージェント数を一目で確認できます。

NEW

📜 Cedar シンタックスハイライト

.cedar および .cedarpolicy ファイルに対するシンタックスハイライトを追加。Cedar ポリシーファイルの可読性が向上します。

NEW

権限・セキュリティ修正 (6 件)

🛡️ skip-permissions ダウングレード修正

保護パスへの書き込み承認後に --dangerously-skip-permissions がサイレントに accept-edits モードにダウングレードされる脆弱性を修正。

SECURITY

⚡ Bash ツール権限強化

環境変数プレフィックスやネットワークリダイレクトのチェックを厳格化。一般的なコマンドでの誤った権限プロンプト表示も削減。

SECURITY

🔧 プロトタイプ汚染対策

toString 等の JavaScript プロトタイププロパティ名が権限ルールと一致し、settings.json がサイレントに無視される問題を修正。

SECURITY

📋 管理設定の即時反映

管理者が削除した managed-settings の allow ルールがプロセス再起動まで有効だった問題を修正。additionalDirectories のセッション中反映も対応。

SECURITY

📂 --add-dir と additionalDirectories の競合修正

settings から additionalDirectories を削除した際、--add-dir で渡された同じディレクトリへのアクセスまで取り消される問題を修正。

SECURITY

MCP・ネットワーク修正 (5 件)

💾 MCP メモリリーク修正

HTTP/SSE 接続のサーバー再接続時に約 50 MB/時のバッファが蓄積する問題を修正。複数 MCP サーバー環境で特に重大な影響がありました。

FIX

🔑 OAuth トークンリフレッシュ修正

oauth.authServerMetadataUrl が再起動後のトークンリフレッシュ時に参照されない問題を修正。ADFS 等の IdP で影響がありました。

FIX

⏱️ 429 リトライ: 指数バックオフ

サーバーが小さな Retry-After 値を返した場合、全リトライが約 13 秒で消費される問題を修正。指数バックオフが最小値として適用されます。

FIX

📊 レートリミットオプション消失修正

コンテキスト圧縮後にレートリミットのアップグレードオプションが消える問題を修正。

FIX

☁️ Bedrock SigV4 空文字列対応

AWS_BEARER_TOKEN_BEDROCK 等が空文字列の場合 (GitHub Actions で未設定の入力) に SigV4 認証が失敗する問題を修正。

FIX

セッション・Resume 修正 (4 件)

📋 /resume ピッカー複数修正

--resume で読み取り専用になる問題、Ctrl+A リロードで検索クリア、空リストがナビゲーションを飲み込む問題、タスクステータス表示の問題を一括修正。

FIX

📄 大容量ファイル diff 修正

--resume 時に 10 KB を超えるファイルの edit diff が表示されない問題を修正。

FIX

💿 キャッシュミスと入力消失修正

添付メッセージがトランスクリプトに保存されないことによるキャッシュミスと、mid-turn 入力の消失を修正。

FIX

💬 作業中メッセージの永続化

Claude が作業中に入力されたメッセージがトランスクリプトに永続化されない問題を修正。

FIX

フック・サブエージェント修正 (3 件)

🪝 Stop/SubagentStop フック修正

長時間セッションで Stop/SubagentStop フックが失敗する問題を修正。エラー時に実際のメッセージが表示されるように改善。

FIX

📂 サブエージェント作業ディレクトリリーク修正

worktree 隔離または cwd: オーバーライドを持つサブエージェントが、親セッションの Bash ツールに作業ディレクトリをリークする問題を修正。

FIX

📝 圧縮時の重複トランスクリプト修正

prompt-too-long リトライ時にマルチ MB のサブエージェントトランスクリプトファイルが重複して書き込まれる問題を修正。

FIX

プラグイン修正 (2 件)

🔌 plugin update 最新版検出修正

git ベースのマーケットプレイスプラグインで、リモートに新しいコミットがあっても「already at the latest version」と報告される問題を修正。

FIX

📝 YAML ブーリアンキーワード修正

プラグインの name が YAML ブーリアン (true, false, yes, no) の場合にスラッシュコマンドピッカーが壊れる問題を修正。

FIX

改善 (12 件)

✅ Accept Edits 自動承認強化

LANG=C rm foo や timeout 5 mkdir out のような安全な環境変数/プロセスラッパー付きコマンドを自動承認。

ユースケース

🔒 セキュリティ管理者: 権限システムの堅牢化

企業環境で Claude Code の権限設定を管理する管理者にとって、6 件のセキュリティ修正は極めて重要です。--dangerously-skip-permissions のサイレントダウングレード修正、managed-settings の即時反映、プロトタイプ汚染対策により、設定が意図通りに適用されることが保証されます。settings.json で toString 等のプロトタイププロパティ名を使用していた場合は、設定が無視されていた可能性があるため確認が必要です。

🔌 MCP プラグイン開発者: メモリリーク解消と安定性向上

複数の MCP サーバーを接続して長時間セッションを実行するプラグイン開発者にとって、約 50 MB/時のメモリリーク修正は重大な改善です。OAuth トークンリフレッシュの修正により ADFS 等の IdP を使用する環境での安定性も向上。また、plugin update コマンドが git リモートの新コミットを正しく検出するようになり、プラグインの更新管理も改善されました。

🇯🇵 日本語/CJK ユーザー: 入力と表示の大幅改善

NO_FLICKER モードでの CJK テキスト文字化け修正により、日本語・韓国語テキストのコピーが正確に動作します。さらに日本語・中国語の句読点の後でもスラッシュコマンドや @ メンション補完がトリガーされるようになり、日本語入力中にわざわざスペースを入力する必要がなくなりました。日常的な使用体験が大幅に向上しています。
