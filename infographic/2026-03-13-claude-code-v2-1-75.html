Claude Code v2.1.75 リリース:
Opus 4.6 の 1M コンテキストウィンドウ対応とトークン推定修正

2026-03-13

v2.1.75

概要

Claude Code v2.1.75 が 2026 年 3 月 13 日にリリースされました。Opus 4.6 の 1M コンテキストウィンドウのデフォルト開放、トークン推定精度の改善、多数のバグ修正が含まれています。

🧠

Opus 4.6 の 1M コンテキストウィンドウ

Max、Team、Enterprise プランで Opus 4.6 の 1M コンテキストウィンドウがデフォルトで利用可能に。追加使用量として別途課金が不要になり、大規模コードベースの解析やリファクタリングがより効率的に行えます。

📊

トークン推定の過剰カウント修正

thinking ブロックおよび tool_use ブロックのトークン推定が過剰にカウントされ、早期にコンテキストコンパクションが発生する問題を修正。コンテキストウィンドウをより効率的に活用できるようになりました。

⚠️

Windows マネージド設定パスの破壊的変更

非推奨だった C:\ProgramData\ClaudeCode\ のフォールバックが削除されました。C:\Program Files\ClaudeCode\ への移行が必須です。

⚠️ 破壊的変更

Windows マネージド設定ファイルパスの変更

非推奨だった旧パスへのフォールバックが v2.1.75 で完全に削除されました。Windows 環境でマネージド設定を運用しているエンタープライズユーザーは、アップデート前に設定ファイルの移行を完了させてください。

C:\ProgramData\ClaudeCode\managed-settings.json → C:\Program Files\ClaudeCode\managed-settings.json

破壊的変更

新機能

🧠 Opus 4.6 1M コンテキスト

Max、Team、Enterprise プランで Opus 4.6 の 1M コンテキストウィンドウがデフォルトで有効化。大規模モノリポやフレームワーク全体の解析に最適です。

🎨 /color コマンド

全ユーザーがセッションのプロンプトバーの色をカスタマイズ可能に。複数セッションを同時に使用する際の視覚的な区別に活用できます。

📛 セッション名のプロンプトバー表示

/rename でセッション名を設定した際、プロンプトバーにセッション名が表示されるようになりました。

🕐 メモリファイルのタイムスタンプ

メモリファイルに最終更新日時が追加され、Claude がどのメモリが新しいか古いかを判断できるようになりました。

🔍 フックソースの表示

権限確認プロンプトにおいて、フックの出所 (settings/plugin/skill) が表示されるようになりました。

バグ修正ハイライト

📊

トークン推定の過剰カウント修正

thinking ブロックおよび tool_use ブロックのトークン推定が過剰にカウントされ、早期にコンテキストコンパクションが発生する問題を修正しました。

💻

Bash ツールの ! 文字化け修正

パイプコマンド内の ! が正しく処理されない問題を修正。jq 'select(.x != .y)' などが正常に動作するようになりました。

🎤

音声モードの初回起動修正

新規インストール環境で /voice を 2 回トグルしないと音声モードが起動しない問題を修正しました。

🏷️

モデル名表示の更新修正

/model や Option+P でモデル切り替え後、ヘッダーに表示されるモデル名が更新されない問題を修正しました。

💥

セッションクラッシュ修正

添付メッセージの計算が undefined を返した際にセッションがクラッシュする問題を修正しました。

🔌

マネージド無効プラグインの非表示化

組織によって強制無効化されたプラグインが /plugin のインストール済みタブに表示される問題を修正しました。

📂

マーケットプレイス設定パスの修正

破損したマーケットプレイス設定パスのハンドリングを修正しました。

🔄

/resume セッション名の喪失修正

フォークまたは継続されたセッションを /resume した際にセッション名が失われる問題を修正しました。

⌨️

/status ダイアログの Esc キー修正

Config タブを表示した後に Esc キーでダイアログが閉じない問題を修正しました。

✅

プラン承認・拒否時の入力処理修正

プランの承認または拒否時の入力ハンドリングを修正しました。

📝

エージェントチームのフッターヒント修正

"↓ to expand" と表示されていた箇所を正しい "shift + ↓ to expand" に修正しました。

パフォーマンス改善

🍎

macOS 起動パフォーマンス改善

非 MDM マシンで不要なサブプロセスの起動をスキップし、起動時間を短縮しました。

🔇

非同期フック完了メッセージの抑制

非同期フックの完了メッセージがデフォルトで非表示に変更。--verbose またはトランスクリプトモードで確認可能です。

コードサンプル

Bash アップデートと新機能の使用例 
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# 現在のバージョン確認
claude --version

# /color コマンドでセッションのプロンプトバー色を設定
/color blue

# /rename でセッション名を設定 (プロンプトバーに表示される)
/rename "feature-auth-refactor"

# Opus 4.6 の 1M コンテキストウィンドウを活用
# Max/Team/Enterprise プランでは追加設定不要

# 修正された Bash ツールでの jq コマンド例
jq 'select(.status != "completed")' results.json

# 非同期フックの完了メッセージを確認する場合
claude --verbose
PowerShell Windows マネージド設定の移行 
# 1. 旧パスにファイルが存在するか確認
Test-Path "C:\ProgramData\ClaudeCode\managed-settings.json"

# 2. 新しいパスにディレクトリを作成
New-Item -ItemType Directory -Force -Path "C:\Program Files\ClaudeCode"

# 3. 設定ファイルを移動
Move-Item "C:\ProgramData\ClaudeCode\managed-settings.json" "C:\Program Files\ClaudeCode\managed-settings.json"

# 4. 旧ディレクトリが空であれば削除
Remove-Item "C:\ProgramData\ClaudeCode" -ErrorAction SilentlyContinue

移行ガイド: Windows マネージド設定パス

対象: Windows 環境でマネージド設定ファイルを C:\ProgramData\ClaudeCode\ に配置しているエンタープライズユーザー

旧パスの存在確認

Test-Path "C:\ProgramData\ClaudeCode\managed-settings.json" を実行して旧パスにファイルが存在するか確認します。

新しいディレクトリを作成

New-Item -ItemType Directory -Force -Path "C:\Program Files\ClaudeCode" で新しいパスにディレクトリを作成します。

設定ファイルを移動

Move-Item コマンドで設定ファイルを旧パスから新パスへ移動します。

旧ディレクトリのクリーンアップ

旧ディレクトリが空であれば Remove-Item で削除します。

開発者への影響

大規模コードベースを扱うチーム

Opus 4.6 の 1M コンテキストウィンドウのデフォルト開放により、大規模なモノリポやフレームワーク全体の解析、リファクタリングがより効率的に行えるようになります。追加の設定や課金は不要です。

コンテキストコンパクションに悩んでいた開発者

トークン推定の過剰カウント修正により、コンテキストウィンドウの実効的な利用可能量が増加し、不要なコンパクションが削減されます。会話の途中で情報が失われる問題が改善されます。

複数セッションを並行して使用する開発者

/color コマンドによるプロンプトバーの色分けと、/rename によるセッション名表示で、複数セッションの識別が容易になります。

まとめ

Claude Code v2.1.75 は、Opus 4.6 の 1M コンテキストウィンドウのデフォルト開放を筆頭に、トークン推定精度の改善やユーザー体験の向上を実現した重要なリリースです。Max、Team、Enterprise プランのユーザーは、追加の設定なしで大規模コードベースの解析やリファクタリングをより効率的に行えるようになります。

トークン推定における thinking ブロックと tool_use ブロックの過剰カウント修正により、コンテキストウィンドウの実効的な利用可能量が増加し、不要なコンパクションが削減されます。Bash ツールの ! 文字化け修正や音声モードの初回起動問題の解消など、日常的な開発体験を改善する多数のバグフィックスも含まれています。

Windows 環境でマネージド設定を C:\ProgramData\ClaudeCode\ に配置しているエンタープライズユーザーは、C:\Program Files\ClaudeCode\ への移行が必須です。破壊的変更のため、アップデート前に設定ファイルの移行を完了させてください。
