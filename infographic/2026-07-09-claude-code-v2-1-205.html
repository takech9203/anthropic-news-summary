Claude Code v2.1.205 アップデート

2026-07-09

📋 概要

🛡️

セキュリティ多層防御の導入

トランスクリプト改ざん防止、偽造承認のブロック、未解決変数に対する rm -rf ガードの 3 層でオートモードの安全性を大幅に強化

💾

自動アップデーターのメモリ最適化

バイナリダウンロードをストリーミング方式に変更し、ピークメモリ使用量を約 400 MB 削減。CI/CD 環境での安定性が向上

🖥️

エージェントビューの UI 刷新

PR リンクの自動表示、カラー状態ワード、分類器によるヘッドライン生成により、バックグラウンドジョブの管理が効率化

🩺

/doctor コマンドの診断ツール化

フルセットアップ診断ツールに昇格し、問題の検出と修正を一括で実行可能に。/checkup エイリアスも利用可能

🔄 Before / After: 自動アップデーター

❌ v2.1.204 以前

  • バイナリ全体をメモリにバッファリング
  • ピークメモリ +400 MB
  • メモリ制約環境で不安定
  • CI/CD ランナーで OOM リスク

✅ v2.1.205 以降

  • チャンクごとにディスクへ直接ストリーミング
  • ピークメモリ大幅削減
  • コンテナ環境でも安定動作
  • 低スペックマシンでも安全にアップデート

✨ 主な機能

🛡️

セキュリティ 3 層防御

トランスクリプト書き込みブロック、偽造承認の明示的排除、未解決変数の rm -rf 実行前確認によりプロンプトインジェクション耐性を強化

🩺

/doctor 診断ツール

フルセットアップチェックアップツールに昇格。問題の検出だけでなく修正まで一括実行可能。/checkup エイリアスも対応

🔗

エージェントビュー: PR リンク

PR の編集、マージ、コメント、プッシュを行ったセッションに対して、該当 PR へのリンクが自動表示される

🎨

エージェントビュー: カラー状態表示

カラー付き状態ワードと分類器が生成したヘッドラインにより、生のツールコール文字列の代わりに人間可読な情報を提供

📛

MCP サーバー名の予約

"Claude Browser" と "Claude Preview" が予約名に。Claude Desktop ペインのリネームに先行した措置

⚡

ストリーミングダウンロード

自動アップデーターがチャンクごとにディスクに直接書き込む方式に変更。ピークメモリ約 400 MB 削減

💻 コードサンプル

Bash v2.1.205 アップデートと診断 
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# Homebrew でのアップデート
brew upgrade claude-code

# 現在のバージョン確認
claude --version

# /doctor コマンドで環境診断と修正
claude
/doctor

# エイリアスも利用可能
/checkup

# エージェントビューで PR リンク付きの状態確認
claude agents

# json-schema オプションの使用 (format キーワード対応)
claude --json-schema '{"type": "object", "properties": {"email": {"type": "string", "format": "email"}}}'

# MCP サーバー名の確認
claude mcp list
Bash MCP サーバー名の予約対応 
# 以下の名前は予約済みとなり、使用不可
# - "Claude Browser"
# - "Claude Preview"

# カスタム MCP サーバーでこれらの名前を使用している場合は変更が必要
claude mcp remove "Claude Browser"
claude mcp add "My Custom Browser" --command "..."

👨‍💻 ユースケース

🤖 オートモード利用者: セキュリティ強化による安心感

トランスクリプト改ざん防止と偽造承認のブロックにより、プロンプトインジェクション攻撃に対する耐性が向上。自律的な実行環境での安全性が確保される

☁️ CI/CD 環境・コンテナ利用者: メモリ効率の改善

自動アップデーターのストリーミング化により、メモリ制約のある GitHub Actions ランナーやコンテナ環境でも安定したアップデートが可能に

📋 バックグラウンドエージェント管理者: UI 刷新による効率化

PR リンク自動表示、カラー状態ワード、分類器ヘッドラインにより、多数のジョブの状況把握が容易になり認知負荷が軽減される

🪟 Windows ユーザー: 安定性向上

NTFS ジャンクションでのワークツリー外ファイル削除とディレクトリ削除時のクラッシュが修正され、Windows 環境での信頼性が向上
