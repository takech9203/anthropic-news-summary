Claude Code v2.1.163: エンタープライズバージョン管理、プラグイン強化、CI/CD 修正

2026-06-05

概要

🏢

エンタープライズバージョン管理

requiredMinimumVersion / requiredMaximumVersion マネージド設定により、組織は承認済みバージョン範囲を強制可能。範囲外では起動を拒否

🧩

プラグイン管理の強化

/plugin list コマンドで --enabled / --disabled フィルター対応。インストール済みプラグインの状態を一目で確認可能

🪝

フックシステム拡張

Stop/SubagentStop フックが additionalContext を返却可能に。エラーではなくフィードバックとして Claude にコンテキストを伝えてターンを継続

🔧

CI/CD パイプライン修正

claude -p のハング、Bedrock/Vertex/Foundry 認証エラー、$TMPDIR リグレッションなど致命的な問題を修正

🖥️

クロスプラットフォーム修正

Windows OneDrive 環境での EEXIST エラー、$HOME 参照による deny ルールのバイパスを修正

新機能

🔒 バージョン範囲強制

requiredMinimumVersion / requiredMaximumVersion マネージド設定で、セキュリティパッチ未適用や未検証バージョンの使用を防止

📋 /plugin list

インストール済みプラグインの一覧表示。--enabled / --disabled フィルターでステータス別に絞り込み

📋 "c to copy" ショートカット

/btw の回答を Raw Markdown でクリップボードにコピー。ペースト先でフォーマットが保持される

🪝 additionalContext

Stop/SubagentStop フックがフィードバックを返却してターンを継続可能に。品質ゲートの実装に最適

💲 \$ エスケープ構文

Skills のコマンドボディで数字の前にリテラルな $ を記述するための \$ エスケープをサポート

🔗 MCP セッション ID

--resume 使用時に stdio MCP サーバーがフック/Bash と同じ CLAUDE_CODE_SESSION_ID を受信

Before / After

Before v2.1.163

  • claude -p がバックグラウンドプロセスで永久にハング
  • Bedrock/Vertex で CI=true 時に認証エラー
  • $TMPDIR が全コマンドでオーバーライド
  • フックの Stop は「成功 or エラー」の二択
  • バージョン管理はユーザー任せ

After v2.1.163

  • stdin クローズ後 ~5 秒でバックグラウンドシェル停止
  • Bedrock/Vertex/Foundry で API キー不要
  • $TMPDIR はサンドボックスのみ
  • フックから additionalContext でフィードバック可能
  • マネージド設定でバージョン範囲を強制

主なバグ修正

🚨
claude -p ハング: バックグラウンドコマンドが終了しない場合に永久にハングする問題。stdin クローズ後 約 5 秒で停止するように修正
🚨
Bedrock/Vertex/Foundry 認証: CI=true かつ Anthropic API キー未設定時に "ANTHROPIC_API_KEY required" で失敗する問題を修正
🚨
$TMPDIR リグレッション: v2.1.154 で全 Bash コマンドに $TMPDIR オーバーライドが適用されていた問題。Bazel/EDR 保護環境の Go ワークフローが影響を受けていた
🔐
deny ルールの $HOME バイパス: Read(~/Desktop/**) の deny ルールが $HOME 経由の Bash コマンドをブロックしない問題を修正
🔐
org マネージド権限ルール: 新規設定ディレクトリで起動中にフェッチが完了した場合、セッション全体に権限ルールが適用されない問題を修正
🖥️
Windows EEXIST エラー: セッション環境ディレクトリに読み取り専用属性がある場合、または OneDrive 内にある場合の Bash コマンド失敗を修正
⌨️
キーボード無応答: ペースト操作の終了マーカーがドロップされた場合にキーボード入力が永久に無応答になる問題を修正
🪝
フック条件の誤発火: if: "Bash(...)" が $() や $VAR を含む全コマンドで発火する問題を修正

コードサンプル

JSON マネージド設定 - バージョン範囲の強制 
{
  "managedSettings": {
    "requiredMinimumVersion": "2.1.160",
    "requiredMaximumVersion": "2.1.170"
  }
}
JSON Stop フックで additionalContext を返却 
{
  "hooks": {
    "Stop": [
      {
        "command": "python3 check_quality.py"
      }
    ]
  }
}

// check_quality.py の出力:
{
  "hookSpecificOutput": {
    "additionalContext": "コードにハードコードされた認証情報があります。環境変数を使用してください。"
  }
}
Bash CI/CD での Bedrock 使用例 
# API キー不要で Bedrock 経由で実行
CI=true AWS_REGION=us-east-1 claude -p "テストを実行してください"

# プラグイン状態の確認
claude
# /plugin list --enabled
# /plugin list --disabled

# バージョン確認
claude --version

開発者への影響

🏢 エンタープライズ IT 管理者

requiredMinimumVersion / requiredMaximumVersion を活用してバージョンポリシーを設定。セキュリティパッチ適用済みの最小バージョンと検証完了済みの最大バージョンを定義し、組織全体のコンプライアンスを確保する

⚙️ CI/CD エンジニア

Bedrock/Vertex/Foundry 使用時に API キー未設定でも動作可能に。claude -p のハング問題が修正され、バックグラウンドプロセスを含むパイプラインの信頼性が向上。Bazel/EDR 環境での $TMPDIR 問題も解消

🪝 フック開発者

additionalContext により、品質ゲートやセキュリティチェックをフックで実装し、問題がある場合に Claude に修正を促すワークフローが実現可能。エラーによるターン終了ではなくフィードバックとして処理される

必要なアクション

1
アップデート実行: npm update -g @anthropic-ai/claude-code
2
エンタープライズ管理者: requiredMinimumVersion / requiredMaximumVersion でバージョンポリシーを設定
3
CI/CD ユーザー: claude -p でバックグラウンドプロセスが約 5 秒後に停止される新挙動に留意
4
Bazel/EDR 環境: v2.1.154 以降の $TMPDIR 問題が本リリースで解消
5
フック利用者: Stop/SubagentStop フックで additionalContext を活用した品質ゲートの実装を検討
