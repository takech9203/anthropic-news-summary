Model Deprecation

Claude Opus 4.1 モデルの非推奨化と廃止予定

2026-06-05

概要

⚠️

非推奨化の発表

Claude Opus 4.1 (claude-opus-4-1-20250805) が 2026 年 6 月 5 日に非推奨化。廃止日は 2026 年 8 月 5 日

🔄

推奨移行先

Claude Opus 4.8 (claude-opus-4-8) への移行が推奨。1M トークンコンテキスト、128k 最大出力、Adaptive Thinking 対応

💥

破壊的変更への対応が必要

temperature/top_p/top_k の廃止、Extended Thinking の廃止、プリフィルの廃止、新トークナイザーへの対応

📅

約 2 か月の移行期間

2026 年 8 月 5 日の廃止日までに移行を完了する必要あり。廃止後はリクエストが失敗する

Before / After

Before: Opus 4.1

  • temperature/top_p/top_k 使用可
  • Extended Thinking 対応
  • アシスタントプリフィル対応
  • 旧トークナイザー
  • 200k コンテキスト

After: Opus 4.8

  • サンプリングパラメータ廃止
  • Adaptive Thinking に移行
  • Structured Outputs 使用
  • 新トークナイザー (~35% 増)
  • 1M コンテキスト

破壊的変更

🚫 サンプリングパラメータ

temperature、top_p、top_k を非デフォルト値に設定すると 400 エラー。プロンプトで動作を制御する方式に変更

🧠 Extended Thinking 廃止

thinking: {type: "enabled"} は 400 エラー。Adaptive Thinking (thinking: {type: "adaptive"}) と effort パラメータを使用

📝 プリフィル廃止

アシスタントメッセージのプリフィルは 400 エラー。Structured Outputs や output_config.format を使用

📊 新トークナイザー

Claude Opus 4.7 以降は新トークナイザーを使用。テキスト処理で最大約 35% 多くのトークンを消費する可能性

コードサンプル

Python Before: Claude Opus 4.1 
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-1-20250805",
    max_tokens=16000,
    temperature=0.7,  # Opus 4.7+ では 400 エラー
    thinking={"type": "enabled", "budget_tokens": 10000},
    messages=[{"role": "user", "content": "Hello, Claude!"}],
)
Python After: Claude Opus 4.8 
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-8",
    max_tokens=16000,
    # temperature, top_p, top_k は削除
    thinking={"type": "adaptive"},  # Adaptive Thinking
    output_config={"effort": "high"},  # effort で制御
    messages=[{"role": "user", "content": "Hello, Claude!"}],
)

必要なアクション

⚠️ 移行チェックリスト

  • Claude Console の Usage ページから CSV エクスポートで利用箇所を特定
  • モデル ID を claude-opus-4-1-20250805 から claude-opus-4-8 に変更
  • temperature/top_p/top_k パラメータを削除
  • Extended Thinking を Adaptive Thinking に移行
  • プリフィルを Structured Outputs に切り替え
  • 新トークナイザーによるコスト影響を評価
  • 廃止日 (8/5) より十分前にテストを完了

影響を受けるユースケース

Claude API を直接利用するアプリケーション

claude-opus-4-1-20250805 をハードコードしているプロダクション環境。廃止日以降はリクエストが失敗するため、事前にモデル ID の更新が必要

サンプリングパラメータに依存するワークフロー

temperature や top_p でモデルの創造性を制御している場合。Opus 4.8 ではプロンプトエンジニアリングで代替する必要がある

Claude Platform on AWS / Microsoft Foundry ユーザー

Anthropic が運営するプラットフォームは同一スケジュールで廃止。Bedrock/Vertex AI はパートナー独自のスケジュール
