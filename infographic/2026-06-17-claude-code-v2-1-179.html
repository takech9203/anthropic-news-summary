Claude Code v2.1.179 リリース: 安定性とバグ修正のアップデート

2026-06-17

概要

🔌

接続・ストリーミングの安定化

ストリーミング中に接続が切断された場合でも、受信済みの部分レスポンスが保持されるようになり、作業内容が失われなくなりました。

📁

サンドボックスの大規模ディレクトリ対応

Linux 環境で大規模ディレクトリツリーに対する glob パターンのマッチングがセッションを使用不可能にする問題が解消されました。

🖥️

UI の 5 件の修正

WSL2 スクロール回帰、フィードバックサーベイ誤キャプチャ、プロモバナー重複、サブエージェントトランスクリプト、入力フォーカスの問題を修正。

🌐

リモートセッションの改善

バックグラウンドタスクの「still running」表示問題の修正と、プラグイン読み込みパフォーマンスの向上。

Before / After

Before

接続切断時

ストリーム中断で生のエラーメッセージが表示され、受信済みレスポンスが全て破棄。スピナーが「running tool」で停止し続ける。

After

接続切断時

部分レスポンスが保持され、それまでの出力が表示される。長時間の生成中に接続が不安定になっても作業内容が失われない。

Before

サンドボックス glob

大規模ディレクトリで denyRead/allowRead の glob マッチ結果が Bash ツール説明文に含まれ、トークン制限超過でセッション使用不可。

After

サンドボックス glob

glob マッチ結果を効率的な内部表現に変換。数千ファイルのディレクトリでもセッションが正常に動作。

Before

WSL2 スクロール

v2.1.172 以降、Windows Terminal / VS Code 上の WSL2 でマウスホイールスクロールが完全に動作しない回帰バグ。

After

WSL2 スクロール

マウスイベントパーサーが WSL2 環境を適切に処理。Windows Terminal と VS Code の異なるプロトコルに対応。

主な修正カテゴリ

🔌 接続・ストリーミング

ストリーム中断時のレスポンス保全メカニズム追加。フォールバック処理でバッファ蓄積済み部分レスポンスを保持。

📁 サンドボックス

Linux の大規模ディレクトリツリーでの glob 処理最適化。内部表現変換によりトークン制限超過を防止。

🖥️ UI 修正

  • WSL2 マウスホイールスクロール
  • フィードバックサーベイ誤キャプチャ
  • プロモバナー重複表示
  • Ctrl+O サブエージェント表示
  • プロンプト入力フォーカス復帰

🌐 リモートセッション

  • バックグラウンドタスクのステータス表示修正
  • プラグイン読み込みパフォーマンス向上

対象ユーザー

🪟 WSL2 環境の開発者

Windows Terminal または VS Code 上の WSL2 でマウスホイールスクロールが動作しなかった問題が修正されました。v2.1.172 以降この問題を経験していた場合は即座のアップデートを推奨します。

🐧 Linux でサンドボックス設定を使用する開発者

denyRead/allowRead の glob パターン設定後にセッションが使用不可能になっていた場合、本リリースで問題が解消されます。

🌐 不安定なネットワーク環境のユーザー

ストリーミング中の接続切断でレスポンスが失われていた問題が修正され、部分レスポンスが保持されるようになりました。

👥 リモートセッションを利用するチーム

バックグラウンドタスクの表示問題修正とプラグイン読み込みパフォーマンス向上により、リモートでの開発体験が改善されます。
