Claude Code v2.1.123
OAuth 認証の 401 リトライループ修正

2026-04-29

概要

🔐

OAuth 認証のリトライループ修正

CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 を設定している環境で OAuth 認証が 401 エラーを繰り返す問題を修正

⚡

ホットフィックスリリース

v2.1.122 (4 月 28 日) リリース翌日の緊急修正。単一バグ修正のみを含むシンプルなリリース

🎯

影響範囲

実験的ベータ機能を無効化しているユーザーのみ影響。該当する場合は速やかなアップデートを推奨

Before / After

Before (v2.1.122)

CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 を設定すると OAuth 認証が 401 エラーを繰り返しリトライし、認証が完了しない無限ループ状態に

After (v2.1.123)

実験的ベータを無効化した状態でも OAuth 認証に必要なパラメータが正しく送信され、認証フローが正常に完了

技術的な詳細

🔄 問題の原因

ベータ無効化時に認証リクエストの必要パラメータが欠落し、サーバーが 401 を返却。リトライロジックが 401 を再試行可能と判断

🛠️ 修正内容

実験的ベータの無効化設定に関わらず、OAuth 認証に必要なパラメータが正しく送信されるよう修正

コードサンプル

Bash アップデートと動作確認 
# Claude Code を最新バージョンに更新
npm update -g @anthropic-ai/claude-code

# バージョン確認
claude --version
# Claude Code v2.1.123

# 実験的ベータを無効化した状態で正常動作を確認
export CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1
claude
# 認証が正常に完了し、401 リトライループが発生しない

影響を受けるユーザー

🏢 実験的ベータを無効化しているエンタープライズ環境

安定性を重視して CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 を設定している企業環境では、この修正により OAuth 認証が正常に機能するようになります。速やかなアップデートを推奨します。

👤 OAuth 認証を使用している全てのユーザー

認証のリトライロジックが改善されるため、将来的な認証エラー時の安定性が向上します。通常のアップデートサイクルでの更新で問題ありません。
