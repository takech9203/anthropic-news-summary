API アップデート

Batch API の max_tokens 上限 300k 引き上げと 1M コンテキストベータの廃止予告

2026-03-30

概要

📈

Batch API の max_tokens を 300k に拡大

Message Batches API における max_tokens の上限が 300,000 トークンに引き上げられました。Claude Opus 4.6 および Sonnet 4.6 で利用可能で、output-300k-2026-03-24 ベータヘッダーの指定が必要です。

⚠️

1M コンテキストベータの廃止予告

Claude Sonnet 4.5 および Sonnet 4 向けの 1M トークンコンテキストウィンドウベータが 2026 年 4 月 30 日に廃止されます。以降は context-1m-2025-08-07 ベータヘッダーを含むリクエストが 400 エラーを返します。

🔄

移行先モデルではベータヘッダー不要

Claude Sonnet 4.6 および Opus 4.6 では 1M トークンコンテキストウィンドウが GA 済みのため、ベータヘッダーなしで標準 API から利用できます。API コールがシンプルになります。

Before / After: 1M コンテキストの移行

Before -- Sonnet 4.5 / Sonnet 4

  • ベータヘッダー context-1m-2025-08-07 が必須
  • client.beta.messages.create() を使用
  • 2026 年 4 月 30 日以降は 400 エラー

After -- Sonnet 4.6 / Opus 4.6

  • ベータヘッダー不要
  • client.messages.create() で OK
  • 1M トークンが標準機能として利用可能

主な変更点

📦 Extended Output ベータ

Message Batches API でシングルターン出力の上限が 300k トークンに拡大。書籍レベルの長文生成や大規模コード生成に対応。バッチ処理の 50% 割引価格が適用されます。

🚫 1M ベータ廃止予告

Claude Sonnet 4.5 / Sonnet 4 の 1M トークンコンテキストベータが 2026 年 4 月 30 日に廃止。Sonnet 4.6 または Opus 4.6 への移行が必要です。

🤖 対応モデル

Extended Output ベータは Claude Opus 4.6 と Sonnet 4.6 で利用可能。Claude API のみ対応で、Amazon Bedrock / Vertex AI / Microsoft Foundry は非対応です。

⏱️ 処理時間に注意

300k トークンの生成には 1 時間以上かかる場合があります。24 時間のバッチ処理ウィンドウを考慮したスケジューリングが推奨されます。

コードサンプル

Python Extended Output ベータを利用した Batch API リクエスト 
import anthropic
from anthropic.types.beta.messages.batch_create_params import Request
from anthropic.types.beta.message_create_params import MessageCreateParamsNonStreaming

client = anthropic.Anthropic()

# output-300k-2026-03-24 ベータヘッダーを指定して
# 最大 300k トークンの出力を生成
message_batch = client.beta.messages.batches.create(
    betas=["output-300k-2026-03-24"],
    requests=[
        Request(
            custom_id="long-form-request",
            params=MessageCreateParamsNonStreaming(
                model="claude-opus-4-6",
                max_tokens=300000,
                messages=[
                    {
                        "role": "user",
                        "content": "Python Web フレームワークの包括的なチュートリアルを"
                                   "作成してください。基礎から応用まで、コード例を含めて"
                                   "詳細に解説してください。",
                    }
                ],
            ),
        ),
    ]
)

print(f"Batch ID: {message_batch.id}")
print(f"Status: {message_batch.processing_status}")
Bash Extended Output ベータを利用した Batch API リクエスト (curl) 
curl https://api.anthropic.com/v1/messages/batches \
     --header "x-api-key: $ANTHROPIC_API_KEY" \
     --header "anthropic-version: 2023-06-01" \
     --header "anthropic-beta: output-300k-2026-03-24" \
     --header "content-type: application/json" \
     --data \
'{
    "requests": [
        {
            "custom_id": "long-form-request",
            "params": {
                "model": "claude-opus-4-6",
                "max_tokens": 300000,
                "messages": [
                    {
                        "role": "user",
                        "content": "大規模なコードベースの包括的なドキュメントを生成してください。"
                    }
                ]
            }
        }
    ]
}'

開発者への影響とユースケース

📖 書籍レベルの長文ドラフト生成

Extended Output ベータにより、技術ドキュメントや長文コンテンツのドラフトを一括生成できます。300k トークンの出力でバッチ処理の 50% 割引が適用されるため、大量のドキュメント生成をコスト効率よく実行できます。

💻 大規模コードスキャフォールド

プロジェクト全体のコードベースを一度に生成するユースケースに対応。最大 300k トークンの出力で、複数ファイルにまたがる大規模なコード生成が可能になりました。

📊 大規模構造化データの抽出

大量のドキュメントからの構造化データ抽出を Batch API で効率的に処理。長い推論チェーンの実行も 300k トークンの出力上限により柔軟に対応できます。

🔄 1M コンテキストベータからの移行

Claude Sonnet 4.5 / Sonnet 4 で 1M トークンコンテキストを利用中の開発者は、2026 年 4 月 30 日までに Sonnet 4.6 または Opus 4.6 へ移行が必要です。移行先ではベータヘッダー不要で、標準価格で 1M トークンコンテキストが利用可能です。
