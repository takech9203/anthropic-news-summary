CLAUDE CODE

Claude Code v2.1.216

2026-07-21

v2.1.216 | サンドボックス設定の柔軟化とパフォーマンス改善

概要

⚙️

sandbox.filesystem.disabled 設定の追加

ファイルシステム分離をスキップしつつ、ネットワークエグレス制御を維持する新しい設定オプション。CI/CD パイプラインなど柔軟なカスタマイズが可能に。

🚀

長時間セッションの O(n²) パフォーマンス問題を修正

メッセージ正規化処理がターン数に応じて二次関数的にコスト増大し、数秒のストールやレジューム遅延を引き起こしていた問題を解消。

🛡️

シンボリックリンク悪用の防止など複数のセキュリティ修正

.claude のシンボリックリンク経由の書き込みリダイレクト防止、/rewind のリンク保護、Windows ネットワークパスのアクセス制御など。

🌳

ワークツリー分離とバックグラウンドエージェントの安定性向上

git リダイレクト修正、エージェントプロンプト復元、バックグラウンドセッション削除問題の解消など多数の改善。

🪟

Windows/PowerShell の Unicode 対応修正

不可視 Unicode 文字を含むコマンドの権限検証を修正し、PowerShell 環境でのセキュリティを強化。

Before / After 比較

Before

長時間セッションでターン数に応じて O(n²) コスト増大。数百ターン以上で数秒のストールが発生し、セッション再開にも時間がかかる状態。

After

効率化された正規化処理により、長時間セッションでも安定したレスポンスを維持。ストールやレジューム遅延を解消。

主な変更カテゴリ

⚙️ サンドボックス

sandbox.filesystem.disabled 設定により、ファイルシステム分離を無効化しながらネットワークエグレス制御を維持。CI/CD 環境で有用。

🚀 パフォーマンス

メッセージ正規化の二次的コスト増大を修正。長時間セッションの体感速度が大幅に改善。

🛡️ セキュリティ

シンボリックリンク経由の書き込みリダイレクト防止、/rewind のリンク保護、Windows ネットワークパスのアクセス制御、PowerShell Unicode 対応。

🌳 ワークツリー

git -C/--git-dir/GIT_DIR 経由の共有チェックアウトへのリダイレクト修正。バックグラウンドセッション削除問題も解消。

🔑 OAuth / 認証

auto モードの OAuth トークン期限切れ対応、Claude-in-Chrome の 403 ループ修正、MCP 再認証の改善。

🎨 UX 改善

AskUserQuestion の中立的表現、/fork 確認メッセージ改善、/context のコンテキストウィンドウ超過警告。

コード例

新しい sandbox.filesystem.disabled 設定の例。ファイルシステム分離を無効化しながらネットワーク制御を維持する。

JSON settings.json - サンドボックス設定 
{
  "sandbox": {
    "filesystem": {
      "disabled": true
    }
  }
}

ユースケース / 対象ユーザー

💻 日常的な Claude Code ユーザー

パフォーマンス改善により長時間セッションでのストールが解消。UX 改善 (AskUserQuestion の中立表現、/context 警告) で日常的な操作がスムーズに。

⏳ 長時間セッション利用者

O(n²) コスト増大の修正により、数百ターン以上のセッションでも安定したレスポンスを維持。セッション再開の遅延も解消され、継続的な開発作業が快適に。

🏢 エンタープライズ / Windows ユーザー

シンボリックリンク保護やネットワークパスのアクセス制御により、セキュリティが強化。PowerShell の Unicode 対応修正で Windows 環境の信頼性が向上。

🔧 インフラ / DevOps エンジニア

sandbox.filesystem.disabled 設定により、CI/CD パイプラインなどファイルシステムへの完全アクセスが必要な環境でもネットワーク制御を維持可能。

その他の修正

🚨 デーモン安全性

claude daemon stop --any が古いレガシーデーモンロックファイル経由で無関係なプロセスを終了する可能性がある問題を修正。

📊 Prometheus メトリクス

無効な # UNIT 行の出力を修正。メトリクスエンドポイントの正確性が向上。

🔄 スキル動的反映

セッション中に変更されたスキルやコマンドが再起動なしにスラッシュメニューに表示されるよう修正。

🌐 RTL テキスト表示

VSCode のアラビア語、ヘブライ語、ペルシャ語が英語やコードと混在した場合の表示順序を修正。

☁️ クラウドセッション

コンテナがターン中に再起動した場合、中断されたターンがレジューム時に再実行されるようになった。

📝 Bash 権限チェック

&& リストや否定内のリダイレクトを含む複合文の権限チェックを修正。非 ASCII 文字のパースも改善。

まとめ

Claude Code v2.1.216 は、パフォーマンス、セキュリティ、安定性の 3 つの柱で大幅な改善を実現したリリースである。特に、長時間セッションにおける二次的コスト増大の修正は、日常的に Claude Code を使用する開発者にとって体感的な改善をもたらす。サンドボックス設定の柔軟化により、より多様な開発環境での活用が可能になり、シンボリックリンクや権限チェックに関するセキュリティ修正により、エンタープライズ環境での安全性も向上した。ワークツリー分離とバックグラウンドエージェントの安定性向上は、複雑なマルチリポジトリワークフローを持つチームにとって重要な改善である。
