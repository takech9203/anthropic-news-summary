機能改善 - 2 件

🔍 subagent_type マッチング強化

Agent ツールの subagent_type パラメータが、大文字小文字およびセパレータ (スペース、ハイフン、アンダースコア) の違いを吸収するようになりました。"Code Reviewer" と code-reviewer が同一として認識されます。

🎨 エージェントカラーパレット更新

複数エージェントを同時に使用する際の視覚的識別性が向上。新しいカラーパレットにより、各エージェントを直感的に区別しやすくなりました。

バグ修正 - 10 件

🔄 バックグラウンドサービス - 3 件

  • 修正claude --bg がバックグラウンドサービスのアイドル終了直前に "connection dropped mid-request" で失敗する問題
  • 修正エンタープライズエンドポイントセキュリティが導入されたマシンでバックグラウンドサービスの起動が失敗する問題
  • 修正/loop がバックグラウンドタスクに対して冗長なウェイクアップをスケジュールする問題

⚙️ 設定・権限 - 3 件

  • 修正設定ホットリロードのリグレッション: シンボリックリンクされた設定ファイルが誤った変更イベントを発生させる問題
  • 修正リモートマネージド設定が 401 エラー時にリトライしない問題
  • 修正マネージド extraKnownMarketplaces の自動更新ポリシーが永続化されない問題

🖥️ UI/UX - 3 件

  • 修正/goal が disableAllHooks または allowManagedHooksOnly 設定時にサイレントにハングする問題
  • 修正Read ツールの検証で、空白パディングまたは + プレフィックス付き offset が失敗する問題
  • 修正ネイティブターミナルでフォーカスを失った際にカーソルが入力キャレットの位置に留まらない問題

🪟 Windows - 1 件

  • 修正実行ファイルが見つからない場合に同期的な where.exe の再スポーンが発生し、イベントループが停止する問題

プラグインシステム

⚠️ デフォルトフォルダ無視時の警告表示

プラグインのデフォルトコンポーネントフォルダ (例: commands/) が、plugin.json で同じキーが明示的に設定されているためにサイレントに無視される場合、警告メッセージが表示されるようになりました。設定の意図しない競合を早期に発見できます。

ユースケース / 影響

🔄 バックグラウンドモード利用者

--bg フラグの接続安定性が向上。サービスのアイドル終了タイミングでの接続切断が解消され、長時間のバックグラウンドタスクが確実に完了するようになりました。

🪟 Windows ユーザー

イベントループ停止問題の解消により、実行ファイル検索時のパフォーマンスが改善。where.exe の同期的再スポーンによるフリーズが発生しなくなりました。

🏢 エンタープライズ環境

エンドポイントセキュリティソフトウェアとの互換性が向上し、マネージド設定のリトライ処理や自動更新ポリシーの永続化により、企業環境での信頼性が大幅に改善されました。
