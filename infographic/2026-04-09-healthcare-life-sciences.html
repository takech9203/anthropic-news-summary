製品発表・ヘルスケア

🏥 Anthropic がヘルスケアとライフサイエンス向け Claude を発表

📅 2026-04-09 ｜ Anthropic News

HIPAA 対応インフラと臨床試験・規制申請ツールを拡充。CMS、Medidata、ClinicalTrials.gov への新コネクタを導入

概要 -- 5 つのキーポイント

🏥

Claude for Healthcare を新設

HIPAA 対応インフラストラクチャを備えた、医療提供者・保険者・ヘルステック企業向けの新しい製品ライン

🔌

医療専用コネクタを追加

CMS Coverage Database、ICD-10、Medidata、ClinicalTrials.gov、ToolUniverse への新しいコネクタを導入

🤖

エージェントスキルの拡充

FHIR 開発、事前承認レビュー、臨床試験プロトコル生成、バイオインフォマティクスなど多数の新スキルを追加

💚

個人健康データ統合

プライバシーバイデザインの原則に基づき、ユーザーが自身の健康データを Claude と安全に共有可能

🧬

ライフサイエンス機能の拡充

臨床試験のプロトコル作成から試験運営、規制申請までの一連のワークフローを Claude で支援

Before / After

❌ Before

  • ライフサイエンス向けコネクタのみ
  • 医療固有のコネクタなし
  • HIPAA 対応機能なし
  • 個人健康データの接続不可
  • 臨床試験サポート限定的

✅ After

  • Healthcare + Life Sciences の 2 本柱
  • CMS、ICD-10、Medidata 等に対応
  • HIPAA 対応インフラストラクチャ
  • 個人健康データ安全統合
  • 臨床試験の全工程をカバー

🏥 Claude for Healthcare -- 新機能

🔌 ヘルスケアコネクタ

🏛️ CMS Coverage Database

Centers for Medicare & Medicaid Services のデータベースに接続。LCD / NCD を参照し、保険適用要件の確認、事前承認チェック、保険請求の異議申し立てを支援

📋 ICD-10

International Classification of Diseases 第 10 版に対応。Claude が診断コードおよび関連情報を検索・参照可能

📚 PubMed

3,500 万件以上の生物医学文献にアクセス。Enterprise 経由でヘルスケア向けにも提供開始

🤖 エージェントスキル

🔗 FHIR 開発スキル

Fast Healthcare Interoperability Resources 規格に基づく開発を支援。医療システム間の相互運用性を向上させ、より迅速なシステム連携を実現

📝 事前承認レビュースキル

保険適用要件、臨床ガイドライン、患者記録、異議申し立て文書を横断的に参照するサンプルスキル

🧬 Claude for Life Sciences -- 拡充

🔌 新しいコネクタ

📊 Medidata

臨床試験ソリューションプラットフォームに接続。過去の試験登録データやサイトパフォーマンスデータを活用可能

🏛️ ClinicalTrials.gov

米国の臨床試験レジストリに接続。医薬品・医療機器パイプライン調査、被験者募集計画、サイト選定に活用

🔬 ToolUniverse

実験室の機器接続を支援し、データの自動収集と分析を実現

🤖 新しいエージェントスキル

📋 臨床試験プロトコル生成

FDA / NIH 要件に準拠した臨床試験プロトコルのドラフトを作成。エンドポイント推奨、規制経路、競合分析を含む

🔄 Allotrope 変換

機器データを Allotrope 形式に変換するスキル

🧬 バイオインフォマティクス

scVI-tools および Nextflow デプロイメントバンドル。科学的課題選定スキルも追加

📦 既存コネクタ一覧

コネクタ用途
🧪 Benchling分子生物学のデータ管理
🧬 10x Genomicsシングルセルゲノミクス
📚 PubMed生物医学文献検索
🎨 BioRender科学イラスト作成
📊 Synapse.org研究データ共有プラットフォーム
📖 Wiley Scholar Gateway学術論文アクセス

👤 個人健康データ統合

ユーザーが自身の健康データを Claude に接続し、医療情報をより深く理解できる新機能

📊 医療履歴の要約

過去の診察記録や検査結果を整理・要約し、全体像を把握しやすく

🔍 検査結果の説明

医療検査の結果をわかりやすく平易な言葉で解説

📈 パターン検出

フィットネスデータや健康メトリクスを横断的に分析し、パターンを検出

🔒 プライバシー設計の原則

✅ 明示的なオプトイン
🔑 ユーザーが権限管理
🚫 トレーニング非使用
⚕️ 医療専門家への誘導

📊 ベンチマークパフォーマンス

Claude Opus 4.5 は医療ベンチマークにおいて大幅な改善を達成 (Extended Thinking 64k トークン + ネイティブツール使用で評価)

🧮 MedCalc

医療計算タスクの精度 (Python コード実行による医療計算)

🤖 MedAgentBench

医療エージェントタスクの完了率 (マルチステップ医療タスク)

🔬 SpatialBench

LatchBio による空間生物学分析ベンチマーク

💡 ヘルスケアのユースケース

📄 保険請求の異議申し立て支援

患者記録、保険適用ポリシー、臨床ガイドライン、過去の文書を統合し、より強力な異議申し立てを構築。保険者側の処理効率化にも貢献

🏨 ケアコーディネーションとトリアージ

患者ポータルのメッセージ、紹介状、引き継ぎを整理し、優先度の高いものを特定。見落としを防止

🚀 ヘルステックスタートアップ支援

Claude Developer Platform で、アンビエントスクライビング、カルテレビュー、臨床意思決定支援ツールなどの新しいプロダクトを構築

💡 ライフサイエンスのユースケース

📋 臨床試験プロトコルの作成

FDA および NIH の要件を考慮し、組織独自のテンプレートやポリシーを活用して臨床試験プロトコルのドラフトを作成

📈 臨床試験運営の効率化

Medidata の試験データを活用し、登録状況やサイトパフォーマンスなどの重要指標を追跡。タイムラインに影響する問題を早期に発見

📑 規制申請書類の準備

既存の規制文書のギャップを特定し、機関からのクエリへの回答をドラフト。FDA ガイドラインのナビゲーションを支援

コードサンプル

Python CMS Coverage Database コネクタの利用例 
import anthropic

client = anthropic.Anthropic()

# 事前承認チェックの例
message = client.messages.create(
    model="claude-opus-4-5-20250414",
    max_tokens=4096,
    tools=[
        {
            "type": "connector",
            "connector": {"id": "cms_coverage_database"}
        },
        {
            "type": "connector",
            "connector": {"id": "icd_10"}
        }
    ],
    messages=[
        {
            "role": "user",
            "content": (
                "患者の MRI 検査について事前承認の要件を確認してください。"
                "診断コード: M54.5 (腰痛)。"
                "Medicare の National Coverage Determination を参照して、"
                "必要な臨床基準を教えてください。"
            )
        }
    ]
)
print(message.content)
Python 臨床試験プロトコルドラフト生成 
message = client.messages.create(
    model="claude-opus-4-5-20250414",

    tools=[

        {"type": "connector", "connector": {"id": "medidata"}},
        {"type": "connector", "connector": {"id": "pubmed"}}
    ],
    messages=[
        {
            "role": "user",
            "content": (
                "2 型糖尿病に対する新規 GLP-1 受容体作動薬の"
                "第 III 相臨床試験プロトコルのドラフトを作成してください。"
                "FDA および NIH の要件に準拠し、"
                "ClinicalTrials.gov で類似試験のデザインを参照してください。"
            )
        }
    ]
)
