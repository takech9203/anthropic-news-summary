## 概要

Claude Code v2.1.128 が 2026 年 5 月 4 日にリリースされました。本リリースでは、MCP サーバー管理の改善、プラグインシステムの `.zip` アーカイブ対応、`--channels` のコンソール認証対応、OTEL 環境変数の分離、そして 20 件以上のバグ修正が含まれています。特に MCP 関連の安定性向上と、開発者体験を改善する多数の修正が注目されます。

## 詳細

### 背景

Claude Code はターミナルベースの AI 開発支援ツールとして、MCP (Model Context Protocol) サーバーとの連携やプラグインエコシステムの拡張が進んでいます。v2.1.128 では、MCP サーバーの接続状態の可視化改善、プラグイン配布の柔軟性向上、コンソール認証環境での channels 機能の解放に重点が置かれています。

### 主な変更点

#### MCP 関連の改善 - 4 件

1. **`/mcp` コマンドのツールカウント表示**: `/mcp` コマンドが接続済みサーバーのツール数を表示し、ツール数が 0 のサーバーにフラグを付けるようになりました。接続問題の早期発見に役立ちます

2. **`workspace` 予約サーバー名**: MCP サーバー名として `workspace` が予約名になりました。既存の同名サーバーはスキップされ、警告が表示されます

3. **再接続時のツールリスト通知抑制**: MCP サーバーの再接続時に、完全なツール名リストが会話に表示されなくなりました。再アナウンスされたツールはサーバープレフィックスごとに要約表示されます

4. **構造化コンテンツとコンテンツブロック併用時の画像欠落修正**: MCP ツール結果でサーバーが構造化コンテンツとコンテンツブロックの両方を返した場合に画像がドロップされる問題が修正されました

#### プラグインシステム - 3 件

1. **`--plugin-dir` の ZIP アーカイブ対応**: `--plugin-dir` オプションがディレクトリに加えて `.zip` プラグインアーカイブを受け付けるようになりました。プラグイン配布が容易になります

2. **`/plugin` Components パネルの修正**: `--plugin-dir` で読み込んだプラグインに対して "Marketplace 'inline' not found" と表示される問題が修正されました

3. **`/plugin update` の npm プラグイン更新検出修正**: npm ソースのプラグインで新バージョンが検出されない問題が修正されました

#### UI/UX 改善 - 8 件

1. **`/color` コマンドのランダム選択**: 引数なしで `/color` を実行するとランダムなセッションカラーが選択されるようになりました

2. **`/model` ピッカーの表示改善**: 重複する Opus 4.7 エントリが統合され、現在の Opus が "Opus 4.7" ではなく "Opus" と表示されるようになりました

3. **フォーカスモードの表示修正**: 新しいプロンプト送信時に前のレスポンスが一瞬暗くなる問題が修正されました

4. **Kitty ターミナルの通知修正**: `/exit` 時に "4;0;" というデスクトップ通知が表示される問題が修正されました (OSC 9 解釈に起因)

5. **長い URL のクリック修正**: フルスクリーンモードで折り返された長い URL が各行で個別にクリック可能になりました

6. **マークダウンリンクラベルの表示改善**: OSC 8 ハイパーリンク非対応ターミナルでリンクが `label (url)` 形式で表示されるようになりました (URL のみの表示から改善)

7. **ターミナルプログレスインジケーター修正**: ツール呼び出し間で OSC 9;4 プログレスインジケーターがちらつく問題が修正され、ターン全体で表示が維持されます

8. **vim モード修正**: NORMAL モードで `Space` キーがカーソルを右に移動するようになり、標準的な vi/vim の動作と一致します

#### Auto モード / SDK - 5 件

1. **`--channels` のコンソール認証対応**: `--channels` が API キーによるコンソール認証で動作するようになりました。マネージド設定を使用するコンソール組織は `channelsEnabled: true` の設定が必要です

2. **OTEL 環境変数の分離**: サブプロセス (Bash、フック、MCP、LSP) が `OTEL_*` 環境変数を継承しなくなりました。Bash ツール経由で実行される OTEL 計測済みアプリが CLI 自体の OTLP エンドポイントを使用する問題が解消されます

3. **SDK ホストの永続的 `localSettings` 提案**: Bash 権限プロンプトで "Always allow" を選択すると `.claude/settings.local.json` に書き込まれるようになりました

4. **`EnterWorktree` の動作修正**: 新しいブランチがドキュメント通りローカル HEAD から作成されるようになりました。従来は `origin/<default-branch>` から作成されていたため、未プッシュのコミットが消失する問題がありました

5. **Auto モードのエラーヒント**: 分類器がアクションを評価できない場合、エラーメッセージにヒント (リトライ、`/compact`、`--debug` での実行) が含まれるようになりました

#### バグ修正 - 14 件

1. **Remote Control のレート制限表示修正**: レート制限時に空の "Opening your options..." メッセージではなく、実行可能なアップセルオプションが表示されるようになりました

2. **ドラッグアンドドロップ画像アップロード修正**: 画像読み取り失敗時に "Pasting text..." で停止する問題が修正されました

3. **大容量パイプ入力のクラッシュ修正**: `claude -p` に 10MB を超える入力を stdin からパイプした際のクラッシュループが修正されました

4. **リスト内コードブロックのコピー修正**: リストアイテム内のフェンスドコードブロックをコピーペーストする際に先頭の空白が含まれる問題が修正されました

5. **`/config` タブナビゲーション修正**: タブヘッダーにフォーカスが維持されるようになり、矢印キーと Esc キーが正常に動作します

6. **1M コンテキストモデルの "Prompt is too long" 誤検知修正**: 小さい autocompact ウィンドウを持つ 1M コンテキストモデルのセッションで、実際の API 制限に到達する前にブロックされる問題が修正されました

7. **並列シェルツール呼び出しの修正**: 読み取り専用コマンド (grep、git diff、ls) の失敗が兄弟の呼び出しをキャンセルしなくなりました

8. **エフォート表示の修正**: エフォートをサポートしないモデルで "with X effort" バナーが表示される問題が修正されました

9. **`/fast` のファジーマッチ修正**: サードパーティプロバイダーで `/fast` が無関係なスキルにマッチする代わりに "not available" と表示されるようになりました

10. **Bedrock デフォルトモデル解決修正**: `global.*` ではなくリージョン適切なプレフィックスに解決されるようになりました

11. **`/rename` 修正**: 最後のエントリがコンパクト境界のリジュームセッションで引数なし `/rename` が失敗する問題が修正されました

12. **古い remote-control ステータス行の修正**: `--resume`/`--continue` 後に以前のセッションの "remote-control is active" が表示される問題が修正されました

13. **`installed_plugins.json` のクリーンアップ**: 削除済みキャッシュディレクトリを指す古いエントリが PATH を汚染しなくなりました

14. **MCP stdio サーバーの引数破損修正**: `CLAUDE_CODE_SHELL_PREFIX` 設定時にスペースやシェルメタ文字を含む引数が破損する問題が修正されました

#### パフォーマンス改善 - 2 件

1. **サブエージェントプロンプトキャッシュ改善**: サブエージェントの進捗サマリーでプロンプトキャッシュが欠落する問題が修正され、`cache_creation` が約 3 倍削減されました

2. **サブエージェントサマリーの繰り返し発火制限**: サブエージェントのトランスクリプトが静的な間にサマリーが繰り返し発火する問題が修正され、アイドル状態のサブエージェントの最悪ケースのトークンコストが制限されました

#### Headless モード修正 - 1 件

- **`--output-format stream-json`**: `init.plugin_errors` に依存関係の降格に加えて `--plugin-dir` のロード失敗も含まれるようになりました

### 技術的な詳細

**OTEL 環境変数の分離**: Claude Code のサブプロセスは従来、親プロセスの環境変数をすべて継承していました。OpenTelemetry で計測されたアプリケーションを Bash ツール経由で実行する場合、Claude Code CLI 自体の OTLP エンドポイント (`OTEL_EXPORTER_OTLP_ENDPOINT` など) が子プロセスに渡され、テレメトリデータが意図しないエンドポイントに送信される問題がありました。v2.1.128 では、Bash、フック、MCP、LSP のサブプロセス起動時に `OTEL_*` 環境変数が明示的に除外されます。

**`--plugin-dir` ZIP 対応**: プラグインディレクトリの指定に `.zip` アーカイブファイルのパスを渡せるようになりました。ZIP ファイルは実行時に展開され、通常のプラグインディレクトリと同様に扱われます。これにより、プラグインの配布やバージョン管理が容易になります。

**MCP `workspace` 予約名**: MCP サーバー設定で `workspace` という名前は内部用に予約されました。既存の設定でこの名前を使用している場合、サーバーはスキップされ警告が出力されます。別のサーバー名への変更が必要です。

**並列シェルツール呼び出しの改善**: 以前は並列で実行されるシェルコマンドのうち 1 つが失敗すると、同時に実行中の他のコマンドもキャンセルされていました。v2.1.128 では、読み取り専用コマンド (grep、git diff、ls など) の失敗は兄弟コマンドに影響しなくなりました。これにより、ファイル検索と diff の同時実行などが安定して動作します。

## 開発者への影響

### 対象

- **MCP サーバー利用者**: ツールカウント表示、再接続時のノイズ削減、`workspace` 予約名により管理性が向上します
- **プラグイン開発者/配布者**: ZIP アーカイブ対応により配布方法の選択肢が増えます
- **エンタープライズ環境のユーザー**: `--channels` のコンソール認証対応、Bedrock デフォルトモデル修正が影響します
- **OTEL 計測を行う開発者**: サブプロセスへの環境変数リーク修正により、テレメトリデータの混在が解消されます
- **全ユーザー**: 並列シェル呼び出し修正、vim モード修正、プログレスインジケーター改善など、全般的な使用体験が向上します

### 必要なアクション

以下のコマンドで最新バージョンに更新できます。

```bash
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# Homebrew でのアップデート
brew upgrade claude-code

# 現在のバージョン確認
claude --version
```

**MCP サーバー名に `workspace` を使用している場合は、別の名前に変更してください。**

### 移行ガイド (該当する場合)

- **MCP サーバー名 `workspace` の変更**: `workspace` は予約名になりました。この名前を使用しているサーバーは自動的にスキップされるため、MCP 設定ファイルでサーバー名を変更する必要があります
- **`--channels` のコンソール組織設定**: コンソール認証 (API キー) で `--channels` を使用する場合、マネージド設定に `channelsEnabled: true` を追加する必要があります
- **`EnterWorktree` の動作変更**: 新しいブランチがローカル HEAD から作成されるようになったため、`origin/<default-branch>` からの作成を前提としていたワークフローの見直しが必要な場合があります

## コード例

```bash
# --plugin-dir で ZIP アーカイブを指定
claude --plugin-dir /path/to/my-plugin.zip

# 複数のプラグインソースを組み合わせ
claude --plugin-dir /path/to/plugin-a/ --plugin-dir /path/to/plugin-b.zip

# --channels をコンソール認証で使用 (API キー環境)
export ANTHROPIC_API_KEY="sk-ant-..."
claude --channels

# マネージド設定で channels を有効にする例 (.claude/settings.json)
# {
#   "channelsEnabled": true
# }

# MCP サーバー名の変更例 (.claude/settings.json)
# 変更前: "workspace" は予約名のためスキップされる
# {
#   "mcpServers": {
#     "workspace": { ... }  <-- スキップされる
#   }
# }
#
# 変更後: 別の名前を使用
# {
#   "mcpServers": {
#     "my-workspace-tools": { ... }  <-- 正常に動作
#   }
# }

# /mcp コマンドでサーバー状態を確認
# > /mcp
# my-server: connected (12 tools)
# broken-server: connected (0 tools) ⚠️

# /color でランダムなセッションカラーを選択
# > /color
```

## まとめ

Claude Code v2.1.128 は、新機能・改善 11 件、バグ修正 20 件以上を含むリリースです。

主なハイライトは以下の通りです。

- **MCP 管理の改善**: `/mcp` コマンドのツールカウント表示、再接続時のノイズ削減、`workspace` 予約名の導入により、MCP サーバー管理の可視性と安定性が向上
- **プラグイン配布の柔軟化**: `--plugin-dir` が `.zip` アーカイブに対応し、プラグインの配布・管理が容易に
- **コンソール認証での channels 対応**: API キー認証環境でも `--channels` が利用可能になり、マネージド設定との組み合わせで柔軟な運用が実現
- **OTEL 環境変数の分離**: サブプロセスへの不要な環境変数の伝播を防止し、テレメトリの正確性を確保
- **並列シェル呼び出しの安定化**: 読み取り専用コマンドの失敗が他の並列コマンドに影響しなくなり、開発ワークフローが安定
- **サブエージェントのコスト最適化**: プロンプトキャッシュの改善とサマリー発火の制限により、トークン消費が大幅に削減

全般的に、日常的な開発体験の安定性と効率性が向上するアップデートとなっています。
