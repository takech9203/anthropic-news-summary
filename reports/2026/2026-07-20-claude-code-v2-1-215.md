## 概要

Claude Code v2.1.215 (2026 年 7 月 20 日) がリリースされた。本リリースの主な変更点は、Claude が `/verify` および `/code-review` スキルを自発的に実行しなくなった点である。これらのスキルはユーザーが明示的にスラッシュコマンドで呼び出した場合のみ実行される動作に変更された。

## 詳細

### 背景

従来の Claude Code では、Claude がワークフローの中で自発的に `/verify` (検証) や `/code-review` (コードレビュー) スキルを呼び出すことがあった。例えば、コード変更を行った後に自動的にコードレビューを実行したり、タスク完了時に自動的に検証を行ったりする動作が見られていた。

しかし、これらのスキルは処理に時間とトークンを消費する比較的コストの高い操作であり、ユーザーが意図しないタイミングで実行されることで以下の問題が生じていた。

- 不必要なトークン消費によるコスト増加
- ワークフローの中断や遅延
- ユーザーが求めていないタイミングでの実行による混乱

### 主な変更点

#### 動作変更: /verify と /code-review の自動実行廃止

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| `/verify` | Claude が自発的に実行可能 | ユーザーが `/verify` で明示的に呼び出した場合のみ実行 |
| `/code-review` | Claude が自発的に実行可能 | ユーザーが `/code-review` で明示的に呼び出した場合のみ実行 |

この変更により、ユーザーがこれらのコストの高い操作を実行するタイミングを完全にコントロールできるようになった。

### 技術的な詳細

この変更はスキルの実行制御に関する動作変更である。スキル自体の機能や出力に変更はなく、呼び出しのトリガー条件のみが変更された。

**変更の性質:**

- `/verify` スキル: コードの検証を行うスキル。変更後はユーザーが明示的に `/verify` コマンドを入力した場合にのみ実行される
- `/code-review` スキル: コードレビューを行うスキル。変更後はユーザーが明示的に `/code-review` コマンドを入力した場合にのみ実行される

**設計思想:**

本変更は「ユーザーの意図を尊重する」という原則に基づいている。コストの高い操作はユーザーが必要と判断したタイミングで実行すべきであり、Claude が自発的に判断すべきではないという方針の反映である。

## 開発者への影響

### 対象

- **全ての Claude Code ユーザー**: 日常的なワークフローにおいて、Claude が自動的に検証やコードレビューを実行しなくなるため、動作の変化を認識する必要がある
- **コスト意識の高いユーザー**: 不要なトークン消費が減少するため、コスト効率が改善される
- **自動化パイプライン利用者**: Claude Code をスクリプトやパイプラインで使用している場合、検証やレビューが必要なタイミングで明示的にコマンドを含める必要がある

### 必要なアクション

以下のコマンドで最新バージョンに更新できる。

```bash
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# Homebrew でのアップデート
brew upgrade claude-code

# 現在のバージョン確認
claude --version
```

**推奨される確認事項:**

- 従来 Claude が自動的にコードレビューや検証を実行していたワークフローがある場合、必要に応じて明示的に `/verify` や `/code-review` を呼び出すよう運用を調整する
- 自動化スクリプトで Claude Code を使用している場合、検証ステップが必要なタイミングでプロンプトに `/verify` や `/code-review` の実行指示を含める

### 移行ガイド (該当する場合)

本変更は即座に適用される動作変更であり、特別な移行作業は不要である。ただし、以下の点に留意されたい。

| 従来のワークフロー | 新しいワークフロー |
|-------------------|-------------------|
| コード変更後に Claude が自動でレビュー | コード変更後にユーザーが `/code-review` を実行 |
| タスク完了時に Claude が自動で検証 | タスク完了時にユーザーが `/verify` を実行 |

## コード例

### スラッシュコマンドの使用方法

```
# コードレビューを実行したい場合
> /code-review

# 検証を実行したい場合
> /verify
```

### 自動化スクリプトでの対応例

Claude Code をプログラマティックに使用している場合、検証ステップを明示的に含めることができる。

```bash
# プロンプトに検証指示を含める例
claude "変更を実装してください。完了したら /verify を実行してください"
```

## まとめ

Claude Code v2.1.215 は、`/verify` と `/code-review` スキルの実行制御をユーザーに委ねる動作変更を導入したリリースである。

この変更の意義は以下の 3 点に集約される。

第一に、**ユーザーコントロールの強化**。コストの高い操作の実行タイミングをユーザーが完全にコントロールできるようになった。Claude が不要なタイミングで検証やレビューを実行し、作業の流れを中断することがなくなる。

第二に、**コスト効率の改善**。`/verify` や `/code-review` は比較的多くのトークンを消費するスキルである。これらが自動実行されなくなることで、ユーザーが必要と判断した場合のみコストが発生するようになり、全体的なトークン消費の最適化が図られる。

第三に、**予測可能な動作の実現**。Claude の動作がより予測可能になり、ユーザーが期待する通りのワークフローを維持できるようになった。自動的に追加されていた検証やレビューのステップが排除されることで、セッションの流れがより明確になる。

本変更は破壊的変更ではないが、従来 Claude の自動レビューや自動検証に依存していたワークフローがある場合は、明示的なスラッシュコマンドの実行を運用に組み込むことを推奨する。
