## 概要

Claude Code v2.1.195 は、フックマッチャーのハイフン付き識別子における部分一致バグの修正、音声入力の複数プラットフォームにおける改善、バックグラウンドエージェントの安定性向上を中心としたリリースである。特に日本語を含むスペースなし言語での音声入力自動送信が修正され、日本語話者にとって重要なアップデートとなっている。

## 詳細

### 背景

Claude Code は継続的にアップデートされており、v2.1.193 でバックグラウンドエージェントや MCP 認証の改善が行われた。本バージョン v2.1.195 では、前バージョンで報告されたフックマッチャーの不具合やプラットフォーム固有の音声入力問題に対処し、全体的な信頼性と安定性を向上させている。

### 主な変更点

#### 新機能

- **`CLAUDE_CODE_DISABLE_MOUSE_CLICKS` 環境変数の追加**: フルスクリーンモードでマウスクリック、ドラッグ、ホバーを無効化し、ホイールスクロールのみを有効にする設定が追加された。ターミナルマルチプレクサ (tmux など) との干渉を避けたい場合に有用である。

#### バグ修正

- **フックマッチャーのハイフン付き識別子の修正**: `code-reviewer` や `mcp__brave-search` などハイフンを含む識別子が意図せず部分一致していた問題を修正。完全一致に変更され、MCP サーバーの全ツールにマッチさせたい場合は `mcp__brave-search__.*` のようにワイルドカードを使用する。
- **macOS 音声入力の無音キャプチャ修正**: 長時間セッション中にデフォルト入力デバイスが変更された後、音声入力が無音をキャプチャし続ける問題を修正。
- **スペースなし言語の音声入力自動送信修正**: 日本語、中国語、タイ語などスペースなしで記述される言語において、音声入力の自動送信が発火しなかった問題を修正。
- **外部プラグインの同意プロンプト修正**: プロジェクトの `.claude/settings.json` でのみ有効化された外部プラグインが、全てのローダーパスで明示的なインストール同意を要求しなかった問題を修正。
- **`/plugin` の有効化/無効化修正**: プラグインの `plugin.json` の `name` がマーケットプレイスエントリ名と異なる場合に Enable/Disable が機能しなかった問題を修正。
- **バックグラウンドジョブの消失修正**: 新しい Claude Code バージョンで書き込まれた際に `claude agents` からジョブが消失したりデータが失われたりする問題を修正。
- **クラッシュしたバックグラウンドタスクの再表示修正**: クラッシュしたタスクを再度開いた際、再起動表示までに最大 5 秒間空白画面が表示される問題を修正。
- **バックグラウンドエージェントデーモンの到達不能修正**: 制御ソケットの起動に失敗した際にデーモンが到達不能状態で実行され続け、再起動をブロックしていた問題を修正。

#### 改善

- **Linux 音声モードの診断改善**: SoX がインストール済みだがオーディオキャプチャデバイスが存在しない場合に「マイクなし」と「SoX 未インストール」を適切に区別するようになった。
- **`claude agents` 完了リストの表示改善**: 利用可能な縦方向のスペースを活用するようになり、短いターミナルではヘッダーがコンパクト化されてライブセッションが見える状態を維持する。
- **Remote セッション起動の改善**: コンテナ起動中にプロビジョニングチェックリストを表示するようになった。

### 技術的な詳細

#### フックマッチャーの動作変更

v2.1.195 以前では、フックマッチャーがハイフン付き識別子に対して部分一致 (substring-match) を行っていた。これにより、`brave-search` というマッチャーが `mcp__brave-search__web_search` だけでなく、`mcp__brave-search-extended__deep_search` のような意図しないツールにもマッチする可能性があった。

本バージョンからは完全一致 (exact-match) に変更され、より厳密な制御が可能となった。

#### 音声入力の自動送信ロジック

従来の自動送信ロジックはスペース文字の入力をトリガーとして使用していたが、日本語、中国語、タイ語ではスペースなしで文章が構成されるため、自動送信が機能しなかった。新しい実装では、言語固有の区切り検出メカニズムが導入されている。

## 開発者への影響

### 対象

- **フックを使用している全開発者**: マッチャーの動作変更によりフック設定の見直しが必要な場合がある
- **音声入力を使用する日本語話者**: 自動送信が正しく機能するようになる
- **バックグラウンドエージェントを活用する開発者**: 安定性が大幅に向上
- **プラグイン開発者**: プラグインの有効化/無効化やセキュリティモデルが改善
- **Linux で音声モードを使用する開発者**: エラー診断が改善

### 必要なアクション

1. **フック設定の確認**: ハイフン付き識別子を使用しているフックマッチャーが完全一致に変更されたため、既存の設定を確認すること。特に MCP サーバーの全ツールにマッチさせたい場合は正規表現パターンへの変更が必要である。
2. **環境変数の活用**: フルスクリーンモードでマウス操作が干渉する場合は `CLAUDE_CODE_DISABLE_MOUSE_CLICKS=1` を設定する。
3. **バックグラウンドエージェントの再確認**: 以前消失したジョブがある場合、本バージョンへの更新後に状態を確認する。

### 移行ガイド (該当する場合)

#### フックマッチャーの移行

**変更前** (部分一致):

```json
{
  "hooks": {
    "tool_use": {
      "matchers": ["mcp__brave-search"]
    }
  }
}
```

上記の設定は `mcp__brave-search` という名前のツールのみにマッチするようになった。

**変更後** (全ツールにマッチさせたい場合):

```json
{
  "hooks": {
    "tool_use": {
      "matchers": ["mcp__brave-search__.*"]
    }
  }
}
```

ワイルドカード `.*` を使用して、該当 MCP サーバーの全ツールにマッチさせる。

## コード例

```bash
# フルスクリーンモードでマウスクリックを無効化 (ホイールスクロールは有効)
export CLAUDE_CODE_DISABLE_MOUSE_CLICKS=1
claude

# アップデートの実行
claude /update
```

```json
// .claude/settings.json - フックマッチャーの正しい設定例
{
  "hooks": {
    "tool_use": {
      "matchers": [
        "code-reviewer",
        "mcp__brave-search__.*"
      ],
      "action": "allow"
    }
  }
}
```

## まとめ

Claude Code v2.1.195 は、信頼性と安定性を重視したリリースである。フックマッチャーの完全一致への変更は破壊的変更となる可能性があるため、ハイフン付き識別子を使用しているフック設定の見直しを推奨する。日本語を含むスペースなし言語での音声入力修正は、日本語環境での Claude Code 利用体験を大きく改善するものである。バックグラウンドエージェント関連の複数の修正により、長時間稼働するエージェントの安定性が向上し、本番環境での利用がより信頼できるものとなった。
