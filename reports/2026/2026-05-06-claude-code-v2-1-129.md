## 概要

Claude Code v2.1.129 が 2026 年 5 月 6 日にリリースされました。本リリースでは、URL からプラグインを直接読み込む `--plugin-url` フラグの追加、Homebrew/WinGet インストール環境向けの自動アップデート機能、プラグインマニフェストの `experimental` 配下への移行、Ctrl+R 履歴検索の動作改善、そして 17 件のバグ修正が含まれています。特にエンタープライズ環境向けの安定性向上と、日常的な使用体験を改善する修正が多く含まれています。

## 詳細

### 背景

Claude Code はターミナルベースの AI 開発支援ツールとして、プラグインエコシステムの拡張とエンタープライズ環境での安定性向上が継続的に進められています。v2.1.129 では、プラグインの配布・読み込み方法の拡充、パッケージマネージャーとの連携による自動アップデート、サードパーティデプロイメント環境での UX 改善に重点が置かれています。

### 主な変更点

#### プラグインシステム - 2 件

1. **`--plugin-url <url>` フラグの追加**: URL から `.zip` プラグインアーカイブを取得し、現在のセッションで使用できるようになりました。リモートホストされたプラグインを直接読み込めるため、チーム内でのプラグイン共有が容易になります

2. **プラグインマニフェストの `experimental` 配下への移行**: `themes` と `monitors` の宣言が `"experimental": { ... }` 配下に移動しました。トップレベルでの宣言は引き続き動作しますが、`claude plugin validate` が警告を出すようになりました

#### 環境変数・設定 - 3 件

1. **`CLAUDE_CODE_FORCE_SYNC_OUTPUT=1`**: 同期出力を強制的に有効化する環境変数が追加されました。自動検出が機能しないターミナル (Emacs `eat` など) で正しい出力表示を実現します

2. **`CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE`**: Homebrew または WinGet でインストールされた環境で設定すると、バックグラウンドでアップグレードコマンドを実行し、完了後に再起動を促すようになりました

3. **`CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1`**: Gateway `/v1/models` ディスカバリーが `/model` ピッカー向けにオプトイン方式に変更されました (v2.1.126-2.1.128 では自動的に有効でした)

#### UI/UX 改善 - 3 件

1. **Ctrl+R 履歴検索のデフォルト動作変更**: すべてのプロジェクトのプロンプト履歴を横断検索するデフォルト動作に戻りました (v2.1.124 以前の動作に一致)。Ctrl+S で現在のプロジェクトまたはセッションに絞り込めます

2. **サードパーティデプロイメントのスピナーティップス改善**: Bedrock、Vertex、Foundry、または `ANTHROPIC_BASE_URL` ゲートウェイ経由のデプロイメントで、ファーストパーティ Anthropic のサーフェスを参照するスピナーティップスが表示されなくなりました

3. **`skillOverrides` 設定の動作修正**: `off` でモデルと `/` から非表示、`user-invocable-only` でモデルからのみ非表示、`name-only` で説明を折りたたみ、という設定が正しく機能するようになりました

#### テレメトリ・デバッグ - 2 件

1. **`claude_code.pull_request.count` OTel メトリクスの改善**: PR/MR の作成カウントが MCP ツール経由の作成も含むようになりました (以前はシェルコマンド経由のみ)

2. **ポリシー拒否エラーメッセージの改善**: API Request ID が含まれるようになり、サポートへの問い合わせ時のデバッグが容易になりました

#### バグ修正 - 17 件

1. **API エラー表示の修正**: 認識されない 400 ステータスコードのエラーで、基底のエラーメッセージの代わりに生の JSON が表示される問題が修正されました

2. **`/clear` のターミナルタブタイトルリセット修正**: `/clear` 実行後にターミナルタブタイトルがリセットされない問題が修正されました

3. **`/rename` セッションタイトル表示修正**: 権限ダイアログやその他のダイアログがアクティブな間にセッションタイトルチップが消える問題が修正されました

4. **エージェントパネル表示修正**: サブエージェント実行中にプロンプト下のエージェントパネルが非表示になる問題が修正されました (v2.1.122 でのリグレッション)

5. **外部エディタハンドオフ修正**: Ctrl+G による外部エディタ呼び出し時にプロンプト上部の会話履歴が空白になる問題が修正されました

6. **`/context` のトークン消費修正**: `/context` がレンダリングされた ASCII 可視化グリッドを会話に挿入し、呼び出しごとに約 1.6k トークンを無駄にする問題が修正されました

7. **`/agents` ライブラリリストのナビゲーション修正**: リストがビューポートを超える場合に、矢印キーで選択したエージェントが画面外に出る問題が修正されました

8. **`/branch` 成功メッセージ修正**: 新しいブランチのセッション ID が `/resume` 用に含まれるようになりました

9. **絵文字を含むボールドヘッダーの表示修正**: フルスクリーンモードでキーキャップ/ZWJ/スキントーン絵文字を含むボールドヘッダーの末尾文字が欠落する問題が修正されました

10. **サーバー管理設定ポリシーの適用修正**: 保存された OAuth 認証情報に `user:inference` スコープがないエンタープライズ/チームユーザーに対してポリシーが適用されない問題が修正されました

11. **OAuth リフレッシュ競合修正**: スリープ復帰後の OAuth リフレッシュ競合により、すべての実行中セッションがログアウトされる問題が修正されました

12. **プロンプトキャッシュ TTL 修正**: 1 時間のプロンプトキャッシュ TTL が暗黙的に 5 分にダウングレードされる問題が修正されました

13. **キャッシュミス警告の誤表示修正**: `/clear` またはコンパクション後に `/effort` や `/model` を変更した際にキャッシュミス警告が不正に表示される問題が修正されました

14. **`Bash(mkdir *)` 等の許可ルール修正**: プロジェクト内パスに対する `Bash(mkdir *)`、`Bash(touch *)` などの許可ルールが正しく適用されるようになりました

15. **`deniedMcpServers` パターンマッチング修正**: `*://` スキームワイルドカードを含むパターンが大文字小文字混在のホスト名にマッチしない問題が修正されました

16. **ボイスモード時の WebSocket 警告修正**: `--debug` モードでボイスモード中に無害な WebSocket 警告がエラーとしてログされる問題が修正されました

17. **[VSCode] `/clear` の完全クリア修正**: `/clear` が会話コンテキストと表示されたトランスクリプトの両方を正しくクリアするようになりました

### 技術的な詳細

**`--plugin-url` のアーキテクチャ**: `--plugin-url` フラグは指定された URL から `.zip` アーカイブをダウンロードし、一時ディレクトリに展開してセッションスコープで読み込みます。v2.1.128 で追加された `--plugin-dir` の ZIP 対応を拡張し、リモート URL からの直接取得を可能にしたものです。セッション終了時に一時ファイルはクリーンアップされます。

**プロンプトキャッシュ TTL 修正**: 以前のバージョンでは、特定の条件下で 1 時間のプロンプトキャッシュ TTL が 5 分に暗黙的にダウングレードされていました。この問題により、キャッシュヒット率が大幅に低下し、不必要な `cache_creation` コストが発生していました。v2.1.129 では TTL が正しく維持され、キャッシュ効率が本来の設計通りに動作します。

**OAuth リフレッシュ競合の解決**: スリープ復帰時に複数のセッションが同時に OAuth トークンのリフレッシュを試行し、競合状態が発生していました。これにより有効なリフレッシュトークンが無効化され、すべてのセッションがログアウトされる問題がありました。v2.1.129 ではリフレッシュ処理にロック機構が導入され、一度に 1 つのセッションのみがリフレッシュを実行するようになりました。

**Gateway モデルディスカバリーのオプトイン化**: v2.1.126 で導入された Gateway `/v1/models` エンドポイントの自動ディスカバリーが、一部の環境で予期しないモデルリストの表示やパフォーマンスの問題を引き起こしていました。v2.1.129 では明示的に `CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1` を設定した場合のみ有効になり、デフォルトでは無効化されています。

## 開発者への影響

### 対象

- **プラグイン開発者/配布者**: `--plugin-url` による URL 配布対応と、マニフェストの `experimental` 配下への移行が必要です
- **Emacs ユーザー**: `CLAUDE_CODE_FORCE_SYNC_OUTPUT=1` により `eat` ターミナルでの表示問題が解消されます
- **Homebrew/WinGet ユーザー**: `CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE` による自動アップデートが利用可能になりました
- **エンタープライズ/チームユーザー**: OAuth リフレッシュ競合修正とサーバー管理設定ポリシーの適用修正により安定性が向上します
- **Gateway 利用者**: モデルディスカバリーがオプトイン方式に変更されたため、利用する場合は環境変数の設定が必要です
- **全ユーザー**: プロンプトキャッシュ TTL 修正によりキャッシュ効率が改善し、コスト削減が期待されます

### 必要なアクション

以下のコマンドで最新バージョンに更新できます。

```bash
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# Homebrew でのアップデート
brew upgrade claude-code

# 現在のバージョン確認
claude --version
```

**プラグインマニフェストで `themes` や `monitors` をトップレベルに宣言している場合は、`experimental` 配下に移動してください。**

### 移行ガイド (該当する場合)

- **プラグインマニフェストの更新**: `themes` と `monitors` を `"experimental": { ... }` 配下に移動する必要があります。トップレベル宣言は引き続き動作しますが、`claude plugin validate` で警告が表示されます
- **Gateway モデルディスカバリーの有効化**: v2.1.126-2.1.128 で自動的に有効だった Gateway モデルディスカバリーを引き続き使用する場合は、`CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1` を設定する必要があります
- **Ctrl+R 履歴検索の変更**: デフォルトが全プロジェクト横断検索に戻りました。プロジェクト限定検索が必要な場合は Ctrl+S で切り替えてください

## コード例

```bash
# URL からプラグインを読み込み
claude --plugin-url https://example.com/plugins/my-plugin.zip

# 同期出力を強制有効化 (Emacs eat ターミナル向け)
export CLAUDE_CODE_FORCE_SYNC_OUTPUT=1
claude

# 自動アップデートを有効化 (Homebrew 環境)
export CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE=1
claude

# Gateway モデルディスカバリーを有効化
export CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1
claude

# プラグインマニフェストの移行例 (plugin.json)
# 変更前:
# {
#   "name": "my-plugin",
#   "themes": { ... },
#   "monitors": { ... }
# }
#
# 変更後:
# {
#   "name": "my-plugin",
#   "experimental": {
#     "themes": { ... },
#     "monitors": { ... }
#   }
# }

# skillOverrides の設定例 (.claude/settings.json)
# {
#   "skillOverrides": {
#     "my-skill": "off",
#     "internal-only-skill": "user-invocable-only",
#     "verbose-skill": "name-only"
#   }
# }

# Ctrl+R 履歴検索
# Ctrl+R: 全プロジェクト横断検索 (デフォルト)
# Ctrl+S: 現在のプロジェクト/セッションに絞り込み
```

## まとめ

Claude Code v2.1.129 は、新機能・改善 10 件、バグ修正 17 件を含むリリースです。

主なハイライトは以下の通りです。

- **プラグイン配布の拡張**: `--plugin-url` フラグにより URL からプラグインを直接読み込み可能になり、チーム間でのプラグイン共有がさらに容易に
- **自動アップデート機能**: `CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE` により、Homebrew/WinGet 環境でバックグラウンドでのアップグレードと再起動プロンプトが実現
- **プロンプトキャッシュ TTL の修正**: 1 時間の TTL が 5 分にダウングレードされる問題の修正により、キャッシュ効率とコスト効率が大幅に改善
- **OAuth リフレッシュ競合の解決**: スリープ復帰後の全セッションログアウト問題が修正され、エンタープライズ環境での安定性が向上
- **Ctrl+R 履歴検索の復帰**: 全プロジェクト横断検索がデフォルトに戻り、v2.1.124 以前の使い慣れた動作に一致
- **Gateway モデルディスカバリーのオプトイン化**: 予期しない動作を防止するため、明示的な環境変数設定が必要に変更

全般的に、プラグインエコシステムの拡充、キャッシュ効率の改善、エンタープライズ環境での安定性向上が特徴的なアップデートとなっています。
