## 概要

Claude Code v2.1.183 では、自動モード (auto mode) における安全性が大幅に強化され、破壊的な Git コマンドやインフラ破棄コマンドの実行が適切にブロックされるようになった。また、設定管理の UX 改善、非推奨モデルの警告表示、多数のバグ修正が含まれている。本リリースは、エージェントが意図せずユーザーの作業を破壊するリスクを低減することに重点を置いたアップデートである。

## 詳細

### 背景

Claude Code の自動モードでは、エージェントがユーザーの承認なしにコマンドを実行できるため、破壊的な操作が誤って実行されるリスクが存在していた。特に `git reset --hard` や `git checkout -- .` といったコマンドは、ローカルの変更を不可逆的に削除する可能性があり、開発者にとって重大なデータ損失につながりかねない。今回のリリースでは、こうした安全上の懸念に対処するガードレールが導入された。

### 主な変更点

#### 安全性の強化

1. **破壊的 Git コマンドのブロック**: ユーザーがローカルの変更を破棄するよう明示的に指示していない場合、以下のコマンドが自動モードでブロックされるようになった。
   - `git reset --hard`
   - `git checkout -- .`
   - `git clean -fd`
   - `git stash drop`

2. **`git commit --amend` の制限**: 現在のセッションでエージェント自身が作成したコミット以外に対する `--amend` がブロックされるようになった。これにより、過去のコミットが意図せず上書きされるリスクが排除される。

3. **インフラ破棄コマンドの制御**: `terraform destroy`、`pulumi destroy`、`cdk destroy` は、ユーザーが対象スタックの破棄を明示的に要求した場合にのみ実行が許可される。

#### 設定・UX の改善

4. **非推奨モデルの警告表示**: リクエストされたモデルが非推奨 (deprecated) または自動的に新しいモデルへ更新された場合に警告が表示されるようになった。print モード (`-p`) では stderr に出力され、エージェントフロントマターで設定されたモデルも対象となる。

5. **`attribution.sessionUrl` 設定の追加**: Web および Remote Control セッションにおいて、コミットや PR から claude.ai のセッションリンクを省略するための設定が追加された。

6. **`/config --help` の追加**: `/config key=value` で使用可能なすべてのショートハンドキーを一覧表示できるようになった。

7. **`/config` トグル動作の変更**: Enter キーと Space キーの両方で設定値を変更でき、Esc キーは元に戻すのではなく保存して閉じる動作に変更された。

8. **起動時の "setup issues" 行の削除**: ロゴ下に表示されていた設定問題の通知行が削除され、設定問題の確認には `/doctor` コマンドまたは `--debug` フラグを使用する形式に変更された。

#### バグ修正

9. **`thinking.disabled.display` の 400 エラー修正**: サブエージェントの生成時やセッションタイトル生成時に発生していた 400 エラーが修正された。

10. **WebSearch のサブエージェント内空結果の修正**: サブエージェント内で WebSearch が空の結果を返す問題が修正された。

11. **vim モード時のカーソル位置修正**: ネイティブカーソルが有効な状態で履歴をナビゲートした後、ターミナルカーソルがプロンプトの上に取り残される問題が修正された。

12. **Windows Terminal でのフルスクリーン TUI 破損修正**: 高負荷のネストされたサブエージェント環境下で、ステータスラインの位置ずれ、スピナー行の重複、テキストの結合が発生する問題が修正された。

13. **thinking ブロックのみ返却時の無応答修正**: モデルが thinking ブロックのみを返した場合にターンが無言で完了する問題が修正され、Claude が 1 回再プロンプトを行うようになった。

14. **スキルの重複表示修正**: 複数のプラグインが有効な場合にユーザーレベルのスキルがスラッシュコマンドの自動補完で複数回表示される問題が修正された。

15. **MCP サーバーの認証スタブ修正**: 認証が必要な MCP サーバーが、ヘッドレス/SDK モードで auth-stub ツールをモデルに公開してしまう問題が修正された。

16. **tmux チームメイトペインの起動修正**: シェルの rc ファイル初期化が遅い場合にペインの起動が失敗する問題と、エージェント起動中に入力されたキーストロークがリーダープロンプトではなく新しい tmux ペインに漏れる問題が修正された。

17. **チームメイト終了時のバックグラウンドタスク修正**: チームメイトがターンを終了した際に、そのチームメイトが開始したバックグラウンドタスクが kill される問題が修正された。

18. **スケジュールタスクとウェブフックトリガーの分類修正**: これらの配信がキーボード入力として扱われていた問題が修正され、タスク通知として正しく分類されるようになった。自動モードではペンディングアクションの承認やセッションタイトルの設定ができなくなった。

19. **フォーカスモードの不要な表示修正**: レスポンスの下に "Ran N PostToolUse hooks" のタイミング行が表示される問題が修正された。

### 技術的な詳細

自動モードの安全性強化は、コマンド実行前のインテント検証レイヤーとして実装されている。エージェントが破壊的コマンドを実行しようとした際、以下のロジックで判定が行われる。

- **Git コマンド**: ユーザーの直近のプロンプトに「破棄」「リセット」「クリーン」等の意図が含まれているかを検証
- **`--amend`**: セッション内のコミット履歴を追跡し、エージェント自身が作成したコミットかどうかを判定
- **インフラ破棄**: ユーザーが対象スタック名を明示的に指定しているかを検証

これらのガードレールは、許可設定やフック設定とは独立して動作し、自動モードでのみ適用される。

## 開発者への影響

### 対象

- Claude Code を日常的に使用するすべての開発者
- 自動モードを活用してワークフローを高速化している開発者
- チーム開発で tmux チームメイト機能を使用している開発者
- CI/CD パイプラインで Claude Code をヘッドレスモードで運用しているチーム

### 必要なアクション

1. **アップデートの実施**: 以下のコマンドで最新版に更新する。
   ```bash
   claude update
   ```

2. **`/config` の動作変更への適応**: Esc キーが「元に戻す」から「保存して閉じる」に変更されたため、設定変更をキャンセルしたい場合は値を手動で戻す必要がある。

3. **`attribution.sessionUrl` の確認**: コミットメッセージから claude.ai のセッション URL を省略したい場合は、この新設定を有効にする。

4. **起動時の設定問題確認**: 起動時に自動表示されなくなったため、設定に問題がないか定期的に `/doctor` を実行して確認することを推奨する。

### 移行ガイド (該当する場合)

本リリースでは破壊的な変更はないため、特別な移行作業は不要である。ただし、以下の動作変更に注意が必要。

- 自動モードで破壊的コマンドを意図的に実行する場合は、プロンプトで明示的に指示する必要がある (例: 「ローカルの変更をすべて破棄してください」)
- `/config` の Esc キー動作が変更されたため、既存のワークフローに影響がある場合は操作方法を調整する

## まとめ

Claude Code v2.1.183 は、自動モードの安全性を根本的に強化するリリースである。破壊的な Git コマンドやインフラ破棄コマンドに対するガードレールの導入により、エージェントが意図せずユーザーの作業を破壊するリスクが大幅に低減された。また、設定管理の UX 改善や多数のバグ修正により、日常的な開発体験も向上している。特に tmux チームメイト機能やフォーカスモードを使用している開発者にとっては、安定性の向上を実感できるアップデートとなるだろう。すべての Claude Code ユーザーに対して、速やかなアップデートを推奨する。
