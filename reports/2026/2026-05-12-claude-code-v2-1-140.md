## 概要

Claude Code v2.1.140 がリリースされた。本バージョンでは、Agent ツールの `subagent_type` マッチングの改善や、エージェントのカラーパレット更新に加え、10 件のバグ修正とプラグインシステムの改善が含まれている。特にバックグラウンドサービスの信頼性向上、Windows 環境でのイベントループ停止問題の修正、設定ホットリロードのリグレッション修正が注目すべき変更点である。

## 詳細

### 背景

Claude Code は Anthropic が提供する CLI ベースの AI 開発支援ツールであり、頻繁なアップデートにより安定性と開発者体験の向上が継続的に行われている。v2.1.140 では、前バージョンで報告されたいくつかの問題への対応と、Agent ツールの使い勝手の改善が中心となっている。

### 主な変更点

#### 機能改善 (2 件)

1. **Agent ツールの `subagent_type` マッチング改善**: 大文字小文字やセパレータの違いを吸収するようになった。例えば `"Code Reviewer"` と指定しても `code-reviewer` として正しく解決される
2. **エージェントカラーパレットの更新**: エージェントの視覚的な識別性が向上

#### バグ修正 (10 件)

1. **`/goal` コマンドのハング修正**: `disableAllHooks` または `allowManagedHooksOnly` が設定されている場合にサイレントにハングしていた問題を修正。明確なメッセージが表示されるようになった
2. **設定ホットリロードのリグレッション修正**: シンボリックリンクされた設定ファイルが誤った変更イベントの帰属と不要な `ConfigChange` フックの発火を引き起こしていた問題を修正
3. **`claude --bg` の接続切断修正**: バックグラウンドサービスがアイドル終了する直前に "connection dropped mid-request" エラーが発生していた問題を修正
4. **バックグラウンドサービスの起動修正**: エンタープライズエンドポイントセキュリティが導入されたマシンでの起動失敗を、タイムアウトの延長により解決
5. **リモートマネージド設定のリトライ修正**: 401 エラー時にリトライされなかった問題を修正。トークンを強制リフレッシュした上で 1 回リトライするようになった
6. **マーケットプレイス自動更新ポリシーの永続化修正**: `extraKnownMarketplaces` の自動更新ポリシーが `known_marketplaces.json` に保存されなかった問題を修正
7. **`/loop` の冗長なウェイクアップ修正**: 完了通知機能を持つバックグラウンドタスクに対して不要なポーリングが行われていた問題を修正
8. **Windows イベントループ停止の修正**: 存在しない実行ファイル (例: `gh`) がチェックのたびに同期的な `where.exe` の再スポーンをトリガーし、イベントループが停止していた問題を修正
9. **`Read` ツールのバリデーション修正**: `offset` パラメータが空白パディングされた文字列や `+` プレフィックス付き文字列として渡された場合にバリデーションが失敗していた問題を修正
10. **ネイティブターミナルカーソルの修正**: ターミナルがフォーカスを失った際にカーソルが入力キャレットの位置に留まらなかった問題を修正

#### プラグインシステム改善 (1 件)

1. **デフォルトコンポーネントフォルダの警告追加**: `plugin.json` で対応するキーが設定されている場合にデフォルトコンポーネントフォルダ (例: `commands/`) がサイレントに無視される際、警告が表示されるようになった。`/doctor`、`claude plugin list`、`/plugin` で確認可能

### 技術的な詳細

- **`subagent_type` マッチング**: 入力値を正規化し、大文字小文字の区別とセパレータ (スペース、ハイフン、アンダースコアなど) の違いを吸収する柔軟なマッチングロジックが実装された
- **バックグラウンドサービスのアイドル終了対策**: サービスがアイドル終了する際のグレースフルシャットダウン処理が改善され、進行中のリクエストが適切に完了するようになった
- **Windows `where.exe` 問題**: 実行ファイルの存在チェックが非同期化され、同期的な `where.exe` のスポーンによるイベントループのブロッキングが解消された
- **設定ホットリロード**: シンボリックリンクの解決処理が改善され、実際のファイル変更のみが検出されるようになった

## 開発者への影響

### 対象

- Claude Code を日常的に使用している開発者
- バックグラウンドモード (`--bg`) を活用しているユーザー
- Windows 環境で Claude Code を使用しているユーザー
- プラグインを開発・利用しているユーザー
- エンタープライズ環境で Claude Code を運用しているチーム

### 必要なアクション

1. Claude Code を最新バージョンに更新する。

```bash
npm install -g @anthropic-ai/claude-code@latest
```

2. Windows 環境で動作の遅延を感じていたユーザーは、更新後にパフォーマンスの改善を確認する
3. `--bg` モードで接続エラーが発生していたユーザーは、更新後に問題が解消されたことを確認する
4. プラグイン開発者は `/doctor` コマンドで新しい警告メッセージを確認する

### 移行ガイド (該当する場合)

本バージョンは後方互換性を維持しており、特別な移行作業は不要である。通常のアップデート手順でバージョンを更新するだけで全ての修正と改善が適用される。

Claude Code v2.1.140 は、安定性と開発者体験の向上に重点を置いたリリースである。特にバックグラウンドサービスの信頼性向上 (接続切断修正、エンタープライズ環境対応)、Windows 環境でのパフォーマンス改善、設定ホットリロードのリグレッション修正が重要な変更点である。また、Agent ツールの `subagent_type` マッチングが柔軟になったことで、サブエージェントの指定がより直感的になった。プラグイン開発者にとっては、デフォルトコンポーネントフォルダが無視される際の警告表示により、設定ミスの早期発見が容易になる。全ユーザーに対してアップデートを推奨する。
