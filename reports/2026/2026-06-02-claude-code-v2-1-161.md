## 概要

Claude Code v2.1.161 は、OpenTelemetry メトリクスのカスタムディメンション対応、並列ツール呼び出しの安定性向上、多数のバグ修正を含むメンテナンスリリースである。OTEL ユーザーにとっては `OTEL_RESOURCE_ATTRIBUTES` の値がメトリクスデータポイントのラベルとして反映されるようになり、チームやリポジトリ単位での使用量分析が可能になった。並列ツール呼び出しでは、失敗した Bash コマンドが同じバッチ内の他のツール呼び出しをキャンセルしなくなり、各ツールが独立して結果を返すようになった。また、Linux クリップボード対応、Windows フックの修正、バックグラウンドセッションの安定化など 15 件以上のバグ修正が含まれている。

## 詳細

### 背景

v2.1.160 でのセキュリティ強化とバックグラウンドエージェントの安定化に続き、v2.1.161 ではオブザーバビリティと並列実行の信頼性に焦点が当てられている。企業環境では OpenTelemetry を使用して Claude Code の利用状況を監視するケースが増えており、チームやリポジトリごとにメトリクスをスライスしたいというニーズに応える改善が行われた。また、エージェントワークフローで複数のツールを並列に呼び出す際に、1 つの失敗が他の正常なツール呼び出しを巻き込んでキャンセルしてしまう問題は、ワークフローの信頼性を損なう重要な課題であった。

### 主な変更点

#### Changed (機能変更)

1. **OTEL メトリクスのカスタムディメンション対応**: `OTEL_RESOURCE_ATTRIBUTES` の値がメトリクスデータポイントのラベルとして含まれるようになった。これにより、チームやリポジトリなどのカスタムディメンションで使用量メトリクスをスライス可能になった
2. **`claude agents` の進捗表示改善**: ワーク分散時に詳細の前に `done/total` を表示し、peek で最長実行アイテムを表示するようになった
3. **`/mcp` の未使用コネクタ表示改善**: サインインしたことのない claude.ai コネクタを「Show unused connectors」行の下に折りたたむようになった
4. **並列ツール呼び出しの独立性確保**: 失敗した Bash コマンドが同じバッチ内の他のツール呼び出しをキャンセルしなくなった。各ツールが独立して結果を返すようになり、エージェントワークフローの信頼性が向上した

#### Fixed (バグ修正)

##### プラットフォーム固有の修正

### 技術的な詳細

#### OTEL メトリクスのカスタムディメンション

`OTEL_RESOURCE_ATTRIBUTES` 環境変数に設定された key-value ペアが、メトリクスデータポイントのラベルとして自動的に付与されるようになった。これにより、既存の OpenTelemetry バックエンド (Datadog, Grafana, New Relic など) でチーム別やリポジトリ別の使用量ダッシュボードを構築可能になる。

```bash
# 使用例: チームとリポジトリをカスタムディメンションとして設定
export OTEL_RESOURCE_ATTRIBUTES="team=platform,repo=my-service"
claude
# -> メトリクスに team=platform, repo=my-service ラベルが付与される
```

#### 並列ツール呼び出しの独立性

従来、並列ツール呼び出しバッチ内で 1 つの Bash コマンドが失敗すると、同じバッチ内の他のツール呼び出しもキャンセルされていた。v2.1.161 では各ツールが独立して実行・完了するようになり、1 つの失敗が他のツールに波及しなくなった。

**変更前:**

```
[並列バッチ]
  Tool A: bash "make test"   -> 失敗
  Tool B: bash "make lint"   -> キャンセル (Tool A の失敗により)
  Tool C: Read "src/main.py" -> キャンセル (Tool A の失敗により)
```

**変更後:**

```
[並列バッチ]
  Tool A: bash "make test"   -> 失敗 (結果を返す)
  Tool B: bash "make lint"   -> 成功 (独立して完了)
  Tool C: Read "src/main.py" -> 成功 (独立して完了)
```

## 開発者への影響

### 対象

- Claude Code CLI ユーザー全般
- OpenTelemetry を使用してメトリクスを収集しているチーム
- バックグラウンドセッション / エージェントを活用するユーザー
- Windows ユーザー
- Linux フルスクリーンモードユーザー
- `claude -p` をパイプラインで使用するユーザー

### 必要なアクション

1. **アップデートの実行**: `npm update -g @anthropic-ai/claude-code` で v2.1.161 に更新
2. **OTEL ユーザー**: `OTEL_RESOURCE_ATTRIBUTES` 環境変数を設定してカスタムディメンションを活用する

   ```bash
   export OTEL_RESOURCE_ATTRIBUTES="team=my-team,repo=my-repo,env=production"
   ```

3. **Windows フックユーザー**: bash を明示的に呼び出すフック定義が正常に動作するようになったため、ワークアラウンドを使用していた場合は元の定義に戻すことを検討
4. **MCP シークレットの確認**: `claude mcp` コマンドの出力にシークレットが含まれていた可能性があるため、ターミナル履歴やログの確認を推奨

### 移行ガイド (該当する場合)

本リリースには破壊的変更は含まれていないため、特別な移行作業は不要である。アップデートのみで全ての改善が適用される。

## コード例

```bash
# v2.1.161 へのアップデート
npm update -g @anthropic-ai/claude-code

# OTEL カスタムディメンションの設定例
export OTEL_RESOURCE_ATTRIBUTES="team=backend,repo=api-service,environment=staging"
claude

# 設定確認 (メトリクスエンドポイントにラベルが付与されていることを確認)
# Datadog: team:backend, repo:api-service でフィルタリング可能
# Grafana: {team="backend"} でクエリ可能

# 並列ツール呼び出しの動作確認
# エージェントが複数のコマンドを同時実行する際、
# 1 つが失敗しても他は正常に完了する
claude -p "src/ 内の全テストを実行し、同時に lint チェックも行ってください"
```
