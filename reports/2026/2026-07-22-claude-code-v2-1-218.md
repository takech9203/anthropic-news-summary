## 概要

Claude Code v2.1.218 がリリースされた。本リリースの最大の変更点は `/code-review` がバックグラウンドサブエージェントとして実行されるようになったことである。これにより、コードレビューの実行中も会話が占有されることなく作業を継続できる。また、`--ax-screen-reader` モードでの削除テキスト読み上げ対応、MCP サーバー接続失敗時の HTTP ステータス表示、auto モードの権限ダイアログ削減など、アクセシビリティと開発者体験の両面で大幅な改善が行われている。さらに、Windows パスの CJK 文字化け問題や左矢印キーによる会話消失バグなど、重大なバグ修正も多数含まれている。

## 詳細

### 背景

Claude Code のコードレビュー機能は開発ワークフローの中核を担う重要な機能であるが、従来は `/code-review` の実行中に会話全体がレビュー作業で埋め尽くされ、他の作業ができなくなる問題があった。v2.1.217 でサブエージェントの同時実行管理が強化されたことを受け、v2.1.218 ではコードレビュー自体をバックグラウンドサブエージェントとして実行する方式に移行し、マルチタスク性を大幅に向上させた。また、auto モードの改善が継続的に進められており、権限ダイアログの不要な表示を減らすことで開発フローの中断を最小限に抑える方向性が強化されている。

### 主な変更点

#### 新機能

- **`/code-review` のバックグラウンド実行**: コードレビューがバックグラウンドサブエージェントとして実行されるようになった。レビュー作業が会話を占有しなくなり、スタックされたスラッシュコマンドがレビュー対象として維持される
- **スクリーンリーダーでの削除テキスト読み上げ**: `--ax-screen-reader` モードで、単語削除 (`Option+Delete`、`Ctrl+W`) および行削除 (`Cmd+Backspace`、`Ctrl+U`、`Ctrl+K`) 時に削除されたテキストが読み上げられるようになった
- **MCP 接続エラーの詳細表示**: `claude mcp list` および `/mcp` コマンドで、接続失敗時に HTTP ステータスコードとエラーテキストが表示されるようになった。また、MCP 設定値に隠れた先頭/末尾空白がある場合の警告も追加
- **モデル切替時のファストモード通知**: `/config model=<x>` または Remote Control でモデルを切り替えた際に、ファストモードの変更がアナウンスされるようになった
- **スキル/プラグインのブール値拡張**: `yes`/`no`/`on`/`off`/`1`/`0` (大文字小文字不問) がスキルおよびプラグインのフロントマターブール値として受け付けられるようになった

#### 動作変更

- **`/deep-research` の手動起動限定**: `/deep-research` は手動で呼び出した場合のみ開始されるようになった。Claude が自発的に起動することはなくなった
- **plan モードでの auto 分類器活用**: plan モードと auto の併用時、静的解析で読み取り専用と証明できない Bash コマンドについて権限プロンプトを表示せず、auto モード分類器が判断するようになった
- **サーバー管理設定の承認プロンプト削減**: 無害な機能トグルやコストトグルについて、設定承認プロンプトが表示されなくなった
- **エージェント名のコロン禁止**: エージェント Markdown ファイルで `:` を含むエージェント名が拒否されるようになった。コロンはプラグインの名前空間区切りとして予約されている
- **`context: fork` スキルのバックグラウンド実行**: `context: fork` を持つスキルはデフォルトでバックグラウンド実行されるようになった。スキルごとに `background: false` でオプトアウト可能

#### auto モード改善

- **権限ダイアログの削減**: `dangerous-rm`、バックグラウンド `&`、Windows パス疑惑チェックが権限ダイアログを開かなくなり、auto モード分類器が判定する
- **`/ultrareview` エラーフィードバックの改善**: 無効な引数に対して Claude がそのまま再試行せず、修正できるようになった
- **IDE 操作のサンドボックス制限改善**: IDE インタラクション向けのサンドボックスコマンド制限が改善された
- **信頼ダイアログの改善**: リポジトリルートの名前を表示し、信頼付与の対象範囲を明確化

#### バグ修正

- **Windows パスの CJK 文字化け修正**: `\u` プレフィックスを含むパスセグメント (例: `C:\Users\unicorn`) がツール入力で CJK 文字に変換されてしまう問題を修正。該当ファイルにアクセスできなくなっていた
- **左矢印キーによる会話消失の修正**: 編集直後の左矢印キー押下で会話が取り消し不能な状態で破棄される問題を修正。確認が求められるようになり、エージェントビューで Esc を押すとバックグラウンドにした会話に戻れるようになった
- **複数行ペーストの修正**: `Ctrl+J` として改行をエンコードするターミナルで、複数行ペーストが改行の代わりに `j` を含む 1 行に圧縮される問題を修正
- **深くネストされた監視ディレクトリの削除時クラッシュ修正**: 深くネストされた監視ディレクトリツリーが削除または移動された際、および深くネストされた UI ツリーのレンダリング時に発生する最大コールスタック超過クラッシュを修正
- **エージェントフロントマターフックのセキュリティ修正**: 信頼されていないフォルダからエージェントフロントマターフックが実行される問題を修正。フックはエージェントファイル自身のフォルダがワークスペース信頼を承認済みであることを要求するようになった
- **リモートセッションのハートビート修正**: ワーカーが置き換えられた後もリモートセッションがハートビートを送信し続ける問題を修正。デスクトップおよび IDE プロセスが拒否されたリクエストを数秒ごとに永久に再試行し続けていた

#### その他のバグ修正

- `/context` がコンパクト後に古いトークン使用量を報告する問題を修正
- `/ultrareview` が "review my auth changes" のような記述的引数で失敗する問題を修正
- `/code-review ultra` が非インタラクティブセッションでローカルレビューを無言で実行する問題を修正
- Bedrock の application-inference-profile ARN の課金レート計算を修正
- 長い IDE 選択が絵文字の途中で切り詰められた際の文字化けを修正
- ツール中断後の偽 "[Request interrupted by user]" メッセージを修正
- VoiceOver がスペース入力時に "new line" と読み上げる問題を修正
- プラグイン/設定パネルでスクリーンリーダーがフォーカス行を追跡できない問題を修正
- PR イベントがセッション終了直後に失われる問題を修正
- Bedrock セットアップウィザードのプロファイル検証失敗を修正
- プロンプト履歴エントリの欠落や重複を修正
- コンテキストオーバーフロー後のリトライループを修正

### 技術的な詳細

#### `/code-review` のバックグラウンドサブエージェント化

従来の `/code-review` は会話のメインスレッドで実行されていたため、レビュー中はユーザーが他のコマンドを入力しても処理が後回しになっていた。v2.1.218 では、v2.1.217 で導入されたサブエージェント管理基盤を活用し、コードレビューを独立したバックグラウンドサブエージェントとして実行する。これにより以下の利点が得られる。

1. **会話の非占有**: レビュー結果はバックグラウンドで生成され、完了時に通知される
2. **スタックコマンドの保持**: 連続して入力されたスラッシュコマンドがレビュー対象のコンテキストとして維持される
3. **並列作業**: レビュー中に他のコーディング作業や質問を継続可能

#### auto モード分類器の権限判定拡張

v2.1.218 では、静的解析で安全性を証明できないコマンドの権限判定が auto モード分類器に委譲されるケースが拡大された。具体的には以下の 3 種類のチェックが対象となる。

1. **dangerous-rm チェック**: `rm -rf` などの危険な削除コマンド
2. **バックグラウンド `&` チェック**: バックグラウンド実行を含むコマンド
3. **Windows パス疑惑チェック**: Windows パス形式と判定されたコマンド

これらは以前は無条件で権限ダイアログを表示していたが、auto モード分類器がコンテキストを考慮して安全性を判断するため、不要な中断が削減される。

#### Windows パスの文字化け問題

Windows パスの `\u` プレフィックス (例: `C:\Users`) が JavaScript の Unicode エスケープシーケンスとして解釈され、後続の文字列と組み合わせて CJK 文字に変換されていた。例えば `C:\Users\unicorn` の `\u` + `nico` 部分が 1 つの Unicode 文字として解釈される。修正により、ツール入力のパス文字列処理でバックスラッシュが適切にエスケープされるようになった。

## 開発者への影響

### 対象

- Claude Code を日常的に使用するすべての開発者
- `/code-review` を頻繁に使用するチーム
- Windows 環境で Claude Code を使用する開発者 (パス文字化け修正)
- アクセシビリティ支援技術 (スクリーンリーダー) を使用する開発者
- MCP サーバーを設定・管理する開発者
- auto モードで効率的に作業する開発者
- カスタムスキルやプラグインを作成する開発者

### 必要なアクション

1. **Claude Code のアップデート**: v2.1.218 以上にアップデートすることを推奨。特に Windows ユーザーはパス文字化け修正の恩恵を受けるため優先度が高い
2. **`/code-review` ワークフローの確認**: バックグラウンド実行に変更されたため、レビュー結果の確認方法が変わる可能性がある。エージェントビューで完了通知を確認する
3. **`/deep-research` の使用方法確認**: Claude が自発的に起動しなくなったため、必要な場合は明示的に `/deep-research` を呼び出す
4. **エージェント名の確認**: `:` を含むエージェント名を使用している場合は名前の変更が必要
5. **スキル設定の確認**: `context: fork` を持つスキルがバックグラウンドで実行されるようになったため、フォアグラウンド実行が必要な場合は `background: false` を追加

### 移行ガイド (該当する場合)

`context: fork` スキルのバックグラウンド実行をオプトアウトする場合は、スキルのフロントマターに `background: false` を追加する。

```yaml
---
context: fork
background: false
---
```

エージェント名にコロンを使用している場合は、別の区切り文字に変更する必要がある。

```yaml
# 変更前 (エラーになる)
---
name: my:agent
---

# 変更後
---
name: my-agent
---
```

## コード例

```yaml
# スキルのフロントマターでブール値の新しい記法を使用
---
context: fork
background: yes      # true/false に加えて yes/no/on/off/1/0 が使用可能
enabled: on
verbose: 0
---
```

```bash
# /code-review はバックグラウンドで実行される
# レビュー中も会話を継続可能
/code-review

# レビューが完了するとエージェントビューに通知される
# 結果はバックグラウンドセッションとして確認可能

# /deep-research は手動呼び出しのみ
/deep-research "Claude API の認証フロー"
```

```bash
# MCP サーバーの接続状態を確認 (HTTP ステータス表示あり)
claude mcp list

# 出力例:
# ✓ github-server    connected
# ✗ custom-server    failed (HTTP 502: Bad Gateway)
# ⚠ local-server     config has leading whitespace in URL
```

## まとめ

Claude Code v2.1.218 は、コードレビューのバックグラウンド実行、アクセシビリティ強化、auto モードの権限判定改善、重要なバグ修正の 4 つの柱で構成されたリリースである。最も影響の大きい変更は `/code-review` のバックグラウンドサブエージェント化であり、レビュー実行中の会話継続が可能になったことで開発者の生産性が向上する。アクセシビリティ面では、スクリーンリーダーでの削除テキスト読み上げやフォーカス追跡の修正により、支援技術を使用する開発者の体験が大幅に改善された。auto モードでは、静的解析で証明できないコマンドの権限判定が分類器に委譲されるケースが拡大し、不要な権限ダイアログによる作業中断が削減された。バグ修正では、Windows パスの CJK 文字化けや左矢印キーによる会話消失など、データ損失に直結する重大な問題が解消されている。
