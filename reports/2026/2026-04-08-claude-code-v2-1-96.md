# Claude Code v2.1.96 リリース: Bedrock 認証の緊急修正 (ホットフィックス)

## 概要

Claude Code v2.1.96 が 2026 年 4 月 8 日にリリースされました。本リリースは、前日リリースされた v2.1.94 で導入されたリグレッションに対するホットフィックスです。`AWS_BEARER_TOKEN_BEDROCK` または `CLAUDE_CODE_SKIP_BEDROCK_AUTH` を使用している環境で Bedrock リクエストが `403 "Authorization header is missing"` エラーで失敗する問題が修正されました。v2.1.95 はスキップされており、Changelog にエントリはありません。

## 詳細

### 背景

2026 年 4 月 7 日にリリースされた Claude Code v2.1.94 は、新機能 5 件、改善 4 件、バグ修正 17 件を含む大型アップデートでした。その中でも注目すべき変更の 1 つが「Amazon Bedrock powered by Mantle 対応」(`CLAUDE_CODE_USE_MANTLE=1`) の追加です。

しかし、この大型リリースにより、Bedrock 認証の処理にリグレッションが発生しました。具体的には、`AWS_BEARER_TOKEN_BEDROCK` 環境変数によるベアラートークン認証、または `CLAUDE_CODE_SKIP_BEDROCK_AUTH` 環境変数による認証スキップ設定を使用しているユーザーが、Bedrock へのリクエスト時に `403 "Authorization header is missing"` エラーを受け取るようになりました。

この問題は Bedrock 経由で Claude Code を利用している企業ユーザーに直接影響するため、翌日の 4 月 8 日にホットフィックスとして v2.1.96 が緊急リリースされました。

### 主な変更点

#### バグ修正 (Fixed)

- **Bedrock 認証の修正**: `AWS_BEARER_TOKEN_BEDROCK` または `CLAUDE_CODE_SKIP_BEDROCK_AUTH` を使用している環境で、Bedrock リクエストが `403 "Authorization header is missing"` エラーで失敗する問題を修正しました (v2.1.94 でのリグレッション)

### 技術的な詳細

#### リグレッションの原因

v2.1.94 で追加された Mantle 対応の変更が、既存の Bedrock 認証フローに影響を与えました。`AWS_BEARER_TOKEN_BEDROCK` を使用したベアラートークン認証や、`CLAUDE_CODE_SKIP_BEDROCK_AUTH` を使用した認証スキップの処理パスにおいて、Authorization ヘッダーが正しくリクエストに付与されなくなっていました。

#### 影響を受けた環境変数

| 環境変数 | 用途 | 影響 |
|----------|------|------|
| `AWS_BEARER_TOKEN_BEDROCK` | Bedrock へのベアラートークン認証 | 403 エラーで全リクエスト失敗 |
| `CLAUDE_CODE_SKIP_BEDROCK_AUTH` | Bedrock 認証のスキップ設定 | 403 エラーで全リクエスト失敗 |

#### バージョン番号について

v2.1.95 は Changelog にエントリがなく、スキップされています。これは内部的なビルド番号の問題、または公開前に取り下げられたバージョンと考えられます。

## 開発者への影響

### 対象

- `AWS_BEARER_TOKEN_BEDROCK` を使用して Bedrock に接続しているユーザー
- `CLAUDE_CODE_SKIP_BEDROCK_AUTH` を設定して認証をスキップしているユーザー
- Amazon Bedrock 経由で Claude Code を利用している全てのエンタープライズユーザー

### 必要なアクション

v2.1.94 を使用しており、上記の環境変数を利用している場合は、直ちに v2.1.96 にアップデートしてください。

```bash
# npm でのアップデート
npm update -g @anthropic-ai/claude-code

# Homebrew でのアップデート
brew upgrade claude-code

# 現在のバージョン確認
claude --version
```

**確認が推奨される項目:**

- **Bedrock ユーザー**: v2.1.94 にアップデート後に `403 "Authorization header is missing"` エラーが発生していた場合、v2.1.96 で解消されます
- **標準の IAM 認証ユーザー**: `AWS_BEARER_TOKEN_BEDROCK` や `CLAUDE_CODE_SKIP_BEDROCK_AUTH` を使用していない場合、本リグレッションの影響は受けていない可能性があります
- **v2.1.92 以前のユーザー**: v2.1.94 をスキップして直接 v2.1.96 にアップデートすることで、大型アップデートの恩恵を受けつつリグレッションを回避できます

## コード例

### Bedrock ベアラートークン認証の設定

```bash
# AWS_BEARER_TOKEN_BEDROCK を使用した Bedrock 認証
export AWS_BEARER_TOKEN_BEDROCK="your-bearer-token"
export ANTHROPIC_MODEL="us.anthropic.claude-sonnet-4-6-20260220-v1:0"

# Claude Code を起動 (v2.1.96 で正常に動作)
claude
```

### Bedrock 認証スキップの設定

```bash
# カスタム認証プロキシを使用する場合など
export CLAUDE_CODE_SKIP_BEDROCK_AUTH=1
export ANTHROPIC_BASE_URL="https://your-proxy.example.com"

# Claude Code を起動
claude
```

### バージョン確認とアップデート

```bash
# 現在のバージョンが 2.1.94 かどうか確認
claude --version

# 2.1.94 の場合はアップデート
npm update -g @anthropic-ai/claude-code

# アップデート後のバージョン確認
claude --version
# 期待される出力: 2.1.96
```

## まとめ

Claude Code v2.1.96 は、v2.1.94 で導入された Bedrock 認証のリグレッションを修正するためのホットフィックスリリースです。変更は 1 件のみですが、影響を受けるユーザーにとっては致命的な問題でした。

v2.1.94 では Mantle 対応をはじめとする多数の変更が行われましたが、その過程で `AWS_BEARER_TOKEN_BEDROCK` および `CLAUDE_CODE_SKIP_BEDROCK_AUTH` を使用した認証パスが破損しました。これらの環境変数を利用しているユーザーは Bedrock への全てのリクエストが `403 "Authorization header is missing"` エラーで失敗する状態でした。

該当する環境変数を使用している Bedrock ユーザーは、直ちに v2.1.96 へのアップデートを実施してください。v2.1.94 の豊富な新機能と改善を安全に利用するためにも、本ホットフィックスの適用が必須です。
