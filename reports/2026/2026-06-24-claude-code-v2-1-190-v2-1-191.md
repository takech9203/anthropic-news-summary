## 概要

Claude Code v2.1.190 および v2.1.191 が 2026 年 6 月 24 日にリリースされた。v2.1.190 はバグ修正と信頼性向上を含むマイナーリリースであり、v2.1.191 は `/rewind` コマンドの追加、CPU 使用量の約 37% 削減、MCP サーバーの信頼性向上、多数のバグ修正を含む大規模なアップデートとなっている。

## 詳細

### 背景

Claude Code は継続的に改善が進められており、v2.1.191 ではユーザー体験の向上、パフォーマンス最適化、エンタープライズ向けの管理機能強化が重点的に行われた。特にストリーミング応答時の CPU 負荷削減と MCP 接続の安定性向上は、長時間セッションを利用する開発者にとって重要な改善である。

### 主な変更点

#### 新機能

- **`/rewind` コマンド**: `/clear` 実行前の会話状態から再開できる機能が追加された。誤って会話をクリアしてしまった場合のリカバリーが可能になった

#### パフォーマンス改善

- **CPU 使用量を約 37% 削減**: ストリーミング応答中のテキスト更新を 100ms 間隔で統合することで実現
- **長時間セッションのメモリ増加を抑制**: ターミナル出力キャッシュからのメモリリークを修正

#### MCP (Model Context Protocol) 改善

- **MCP サーバーの信頼性向上**: `tools/list`、`prompts/list`、`resources/list` のケイパビリティ検出が一時的なネットワークエラー時にショートバックオフ付きリトライを実行するようになった
- **MCP OAuth の改善**: ディスカバリーとトークンリクエストが一時的なネットワークエラー後に 1 回リトライし、ヘッドレス環境ではブラウザポップアップをスキップして URL 貼り付けプロンプトに直接移行
- **MCP エラーメッセージの改善**: HTTP 404 エラーが URL を表示し MCP 設定を参照するようになった

#### セキュリティとポリシー管理

- **サンドボックスネットワーク許可の改善**: 「Yes」で許可したホストがセッション終了まで記憶され、接続ごとの再プロンプトが不要になった
- **管理設定の修正**: `forceRemoteSettingsRefresh` が MDM またはファイルポリシー経由で設定された場合に有効化され、フェッチ時に `Cache-Control: no-cache` を送信してプロキシによるステールレスポンスを防止

#### バグ修正

- **スクロール位置の修正**: ストリーミング応答中に以前の出力を読んでいる際にスクロールが最下部にジャンプする問題を修正
- **バックグラウンドエージェントの修正**: 停止後にエージェントが復活する問題を修正し、タスクパネルからの停止が恒久的になった
- **`/voice` コマンド**: 組織のポリシーで無効化されている場合に汎用的な「利用不可」メッセージではなく制限の説明を表示
- **`/login` URL**: Windows Terminal で行折り返し時に URL が切り詰められる問題を修正
- **フルスクリーンモード**: Ghostty over ssh/tmux での Cmd+click リンク動作を修正
- **`claude agents` コマンド**: `/usage` などのビルトインスラッシュコマンドをバックグラウンドセッションにプロンプトテキストとして送信する問題を修正
- **`claude agents` 画像表示**: 貼り付けた画像のフルパスが表示される代わりに `[Image #N]` プレースホルダーを使用するように修正
- **フックのマッチャー修正**: カンマ区切りのマッチャー (例: `"Bash,PowerShell"`) が発火しない問題を修正
- **`/permissions` 修正**: 最近拒否されたタブで承認した変更がクローズ時に保存されるように修正
- **エージェントパネル**: ロスターをオーバーフローキャップ超えてスクロールした際に 1 行ジャンプする問題を修正
- **ウェルカムスプラッシュ**: デフォルトの 80x24 macOS Terminal ウィンドウからオーバーフローする問題を修正

#### UX 改善

- **vim モード**: NORMAL `/` でのプロンプト履歴検索時にスラッシュコマンドへの到達方法をヒント表示

### 技術的な詳細

ストリーミング応答時の CPU 使用量削減は、テキスト更新の統合 (coalescing) によって実現されている。従来は各テキストチャンクの受信ごとに UI を更新していたが、v2.1.191 では 100ms 間隔でバッチ処理を行うことで、レンダリング回数を大幅に削減した。これにより約 37% の CPU 使用量削減を達成している。

MCP の信頼性向上では、ケイパビリティ検出時のリトライメカニズムにショートバックオフを採用し、一時的なネットワーク障害に対する耐性を強化した。OAuth フローの改善では、CI/CD パイプラインなどのヘッドレス環境での利用が容易になった。

## 開発者への影響

### 対象

- Claude Code を日常的に利用する全ての開発者
- MCP サーバーを利用した拡張機能を活用する開発者
- エンタープライズ環境で管理ポリシーを適用している組織
- バックグラウンドエージェントを使用する開発者

### 必要なアクション

1. Claude Code を最新バージョンに更新する。

```bash
claude update
```

2. MCP サーバーを利用している場合、リトライ機能の恩恵を受けるために追加の設定変更は不要である
3. フックでカンマ区切りマッチャーを使用している場合、更新後に正常に動作することを確認する
4. バックグラウンドエージェントの停止動作が改善されたため、タスク管理のワークフローを再確認する

### 移行ガイド (該当する場合)

本リリースに破壊的変更は含まれていない。アップデートのみで全ての改善が適用される。

## まとめ

Claude Code v2.1.191 は、パフォーマンス、信頼性、ユーザー体験の全方位で大幅な改善を実現した重要なリリースである。特にストリーミング応答時の CPU 使用量約 37% 削減は、長時間の開発セッションにおけるシステムリソースの効率的な利用に貢献する。`/rewind` コマンドの追加により会話管理の柔軟性が向上し、MCP の信頼性改善によりリモートツールとの連携がより安定した。エンタープライズ環境向けのポリシー管理修正も含まれており、組織での大規模展開にも適した内容となっている。
