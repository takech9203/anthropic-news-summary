## 概要

Claude Code v2.1.179 がリリースされた。本リリースは安定性とバグ修正に焦点を当てたメンテナンスリリースであり、接続中断時のレスポンス保全、WSL2 環境でのスクロール回帰修正、サンドボックスの大規模ディレクトリツリー処理問題の解消など、9 件の修正と 1 件のパフォーマンス改善が含まれている。新機能の追加はないが、日常的な開発ワークフローにおける信頼性と使い勝手が大幅に向上している。

## 詳細

### 背景

前バージョン v2.1.178 ではパラメータベース権限ルールやネスト `.claude/` ディレクトリサポートなど大きな機能追加が行われた。今回の v2.1.179 は、それらの機能追加後に報告された問題や、v2.1.172 で発生した回帰バグを含む安定性の問題を集中的に修正するリリースである。特に WSL2 環境のスクロール問題は v2.1.172 からの回帰であり、Windows 環境で Claude Code を利用する開発者にとって重要な修正となっている。

### 主な変更点

#### 接続・ストリーミングの修正

1. **ストリーム中断時のレスポンス保全**: 接続がストリーミング途中で切断された場合、生のエラーメッセージが表示される代わりに、それまでに受信した部分的なレスポンスが保持されるようになった。また、スピナーが「running tool」の状態で停止し続ける問題も同時に修正された。

#### サンドボックスの修正

2. **大規模ディレクトリツリーでの glob 処理**: Linux 環境において、サンドボックスの `denyRead`/`allowRead` に設定された glob パターンが大規模ディレクトリツリーに対してマッチングを行う際、Bash ツールの説明文が巨大になりセッションが使用不可能になる問題が修正された。

#### UI の修正

3. **WSL2 でのマウスホイールスクロール**: Windows Terminal および VS Code 上の WSL2 でマウスホイールによるスクロールが動作しない問題が修正された。これは v2.1.172 で発生した回帰バグである。

4. **フィードバックサーベイの誤キャプチャ**: ターン完了直後に 1 桁の入力がセッション評価として即座にキャプチャされてしまう問題が修正された。

5. **ウェルカム画面のプロモバナー重複**: ウェルカム画面で複数のプロモーションバナーが重なって表示される問題が修正され、1 セッションにつき最大 1 つのプロモーションのみが表示されるようになった。

6. **Ctrl+O でのサブエージェントトランスクリプト表示**: サブエージェントの表示中に Ctrl+O を押してもサブエージェントのトランスクリプトが表示されない問題が修正された。

7. **プロンプト入力のフォーカス復帰**: プロンプト入力エリアをクリックしてもサブエージェント/フッターパネルからフォーカスが戻らない問題が修正された。

#### リモートセッションの改善

8. **バックグラウンドタスクのステータス表示**: リモートセッションでバックグラウンドタスクがターン間で「still running」と表示され続ける問題が修正された。

9. **プラグイン読み込みパフォーマンスの向上**: リモートセッションにおけるプラグインの読み込みパフォーマンスが改善された。

### 技術的な詳細

#### ストリーム中断時のレスポンス保全メカニズム

従来の実装では、接続が途中で切断された場合にエラーハンドラが発火し、受信済みの部分レスポンスが破棄されて生のエラーメッセージがユーザーに表示されていた。v2.1.179 では、ストリーミングバッファに蓄積された部分レスポンスを保持するフォールバック処理が追加され、接続エラー時でもそれまでの出力が表示される。これにより、長時間の生成中に接続が不安定になっても作業内容が失われない。

#### サンドボックス glob 処理の最適化

`denyRead`/`allowRead` の glob パターンが大規模ディレクトリツリーに対して展開される際、マッチしたパス全てが Bash ツールの説明文に含まれていた。これにより、数千のファイルを含むディレクトリでは説明文が膨大になり、トークン制限を超えてセッションが使用不可能になるケースがあった。修正では、glob のマッチ結果を直接説明文に含めるのではなく、効率的な内部表現に変換することでこの問題を解消している。

#### WSL2 スクロール回帰の原因

v2.1.172 でターミナルの入力処理に変更が加えられた際、WSL2 特有のマウスイベントエスケープシーケンスの処理に不具合が発生した。Windows Terminal と VS Code はそれぞれ異なるマウスイベントプロトコルを使用しているが、WSL2 経由で受信されるこれらのイベントが正しくパースされなくなっていた。今回の修正でマウスイベントのパーサーが WSL2 環境を適切に処理するようになった。

## 開発者への影響

### 対象

- Claude Code を利用するすべての開発者
- 特に WSL2 環境 (Windows Terminal / VS Code) で Claude Code を使用している開発者
- Linux 環境でサンドボックスの読み取り制限 (`denyRead`/`allowRead`) を設定しているユーザー
- リモートセッションを頻繁に利用するチーム
- 不安定なネットワーク環境で Claude Code を使用している開発者

### 必要なアクション

Claude Code を最新バージョンに更新するだけで、全ての修正が自動的に適用される。

```bash
# Claude Code の更新
claude update
```

特に以下の問題を経験していたユーザーは即座のアップデートを推奨する。

- WSL2 でスクロールが効かなくなっていた場合
- Linux でサンドボックス設定後にセッションが使用不可能になっていた場合
- ストリーミング中に接続が切れてレスポンスが失われていた場合

### 移行ガイド

本リリースはバグ修正のみのメンテナンスリリースのため、移行作業は一切不要である。設定ファイルの変更やワークフローの修正なしにシームレスにアップデートできる。

## まとめ

Claude Code v2.1.179 は、新機能の追加ではなく既存機能の安定性向上に集中したメンテナンスリリースである。接続中断時のレスポンス保全はネットワークが不安定な環境での作業信頼性を大きく向上させ、WSL2 スクロール回帰の修正は Windows 環境の開発者にとって重要な改善となっている。サンドボックスの大規模ディレクトリ処理問題の解消により Linux 環境でのセキュリティ設定がより実用的になり、リモートセッションのプラグイン読み込みパフォーマンス改善も含め、日常的な開発体験の品質が着実に向上している。前バージョン v2.1.178 の大きな機能追加を安定した基盤の上で運用できるよう支える、堅実なリリースである。
